1. GtDemo data sampler module operation

When many equilibrium states are generated by e.g. using a Process simulator or just the System dialog, it may not be  productive to print out all the results, pick up and plot required data "by hand". This operation - a crucial performance step in the interpretation of  modeling results - can be accelerated by creating GtDemo data samplers and plotters that work according to specifications in GtDemo records stored in  the user's project database extension files. GtDemo can sample virtually any numerical data by reading records of any type from a selected sequence of record keys. What data to extract and how to recalculate them, the user can specify in a math script text in the gExpr  record field. 

As in the case of  pgExpr script in the Process simulator, the GtDemo gExpr math script will be automatically translated when the GtDemo record is calculated and will be run every time when a database record from the list is loaded. From each sampled record (indexed with jR), the math script will pick up the required data, re-scale them or calculate some derived parameters, and store into a GtDemo abscissa vector x0 and table y0 (up to 10 columns). When scanning of database records is finished, these data objects can be displayed on a GtDemo window page, printed into a text file for subsequent import to worksheets or scientific graphing software for high-quality presentations, or plotted in a Graph Dialog window. The Graphs can be customized, zoomed, saved and printed. As an option, a sequence of experimental points (entered by the user into xE and yE fields) can be shown as scattered symbols on the graph over the sampled (or calculated) data.One of several GtDemo records can be created for sampling the same set of GEMS database records.


1.2. Writing  data sampling math scritps

To make it easier, we recommend usage of the template math script library, accessible via the "Help" "Script" menu command (or F3 hot key). You will see a list of available scripts (record keys with template math scripts begin with "mscript..."); select an appropriate one and press "Ok". In the appearing SDref module window, the template script can be found on Page 2. Read the comments, then mark the script text using the mouse drag, copy it (Ctrl-C), switch to Page 1 of the GtDemo window, click on the gExpr field and paste the text there (Ctrl-V). Now, you can edit the script according to your own needs of the data sampling. To see the data fields exactly, you can open some submodule windows from the "Single Thermodynamic System", where you will also find the data object labels and names of compositions, species, and phases. After you have edited the script (but before calculations), save the GtDemo record to database.

How the GtDemo math scripts operate? On Page 2 of GtDemo window, you can see two special data objects: next  contains 1 if the next data record is to be sampled, or 0 if the sequence is finished; and jR - the index of (currently processed) data record from the GD_rkl list visible on Page 3. The jR index together with  x0, y0 arrays must be used as variables in the GtDemo Math Script expressions  that define what data to sample and assign to x0[jR]  and y0[jR][0], y0[jR][1], .... cells. The jR index is incremented automatically when GtSemo sampler is running. The typical GtDemo script looks like this:

$ Abscissa
x0[jR] =: pH;
$ Ordinates
y0[jR][0] =: lga[{CO2@}];
y0[jR][1] =: lga[{HCO3-}] + 0.4343*lnGam[{HCO3-}];
y0[jR][2] =: lga[{CO3-2}] + 0.4343*lnGam[{CO3-2}];
y0[jR][3] =: lgm_t[{C}];
$ End
This script collect data for inorganic carbon aqueous species from each SysEq record read in a sequence indexed by jR. The first, third and the last line are comments. The second line picks up the calculated pH and stores it in the abscissa vector x0. The rest of the script collects logarithms of molalities of aqueous carbonate species and total dissolved carbon into four columns of the y0 table (obviously, four columns should have been ordered during create/remake).  There is an infinite number of possible math scripts, of course. Some examples are given below.

Data Sampler for constructing a Lippmann diagram

This example assumes that a series of SysEq records has been generated to describe solubility of a binary (Ca,Mn)CO3 solid solution in water at different Ca/Mn ratios in the bulk composition. Two GtDemo records must be created: one sampler would collect data for the solidus line, and another - for the solutus line on the Lippmann diagram. Accordingly, two scripts are necessary. 

The full diagram can be plotted in a commercial worksheet after printing both GtDemo samplers into a text file and importing data in the worksheet program (only x0[jR] and y0[jR][0] columns).

$ Sampler 1: Lippmann diagram - solidus
   x0[jR] =: Wxx[{rhodochrosite}];
   y0[jR][0] =: lg( my[{CO3-2}]*gamma[{CO3-2}] * ( my[{Ca+2}]*gamma[{Ca+2}] +
                my[{Mn+2}]*gamma[{Mn+2}] ));
$ Total dissolved Ca and Mn vs mole fraction in solid (optional)
  y0[jR][1] =: lgm_t[{Ca}];
  y0[jR][2] =: lgm_t[{Mn}];
$ end

Please, note that the first ordinate y0[jR][0] is the same in both scripts, but abscissas are different.

$ Sampler 2: Lippmann diagram - solutus
  x0[jR] =: my[{Mn+2}]*gamma[{Mn+2}] / ( my[{Ca+2}]*gamma[{Ca+2}] +
                 my[{Mn+2}] * gamma[{Mn+2}] );
  y0[jR][0] =: lg( my[{CO3-2}]*gamma[{CO3-2}] * ( my[{Ca+2}]*gamma[{Ca+2}] +
                 my[{Mn+2}]*gamma[{Mn+2}] ));
$ Ionic Ca and Mn vs mole fraction in aqueous phase  (optional)
  y0[jR][1] =: lg( my[{Ca+2}] );
  y0[jR][2] =: lg( my[{Mn+2}] );
$ end

 Alternatively,  the whole diagram can be plotted in a single GtDemo sampler in a transposed form using the script below: 

$ Log of the total solubility product
x0[jR] =: lg( my[{CO3-2}]*gamma[{CO3-2}] * ( my[{Ca+2}]*gamma[{Ca+2}] +
          my[{Mn+2}]*gamma[{Mn+2}] )); 
$ Solidus (mole fraction in solid)
y0[jR][0] =: Wxx[{rhodochrosite}];
$ Solutus (cationic mole fraction in solution)
y0[jR][1] =: my[{Mn+2}] * gamma[{Mn+2}] / ( my[{Ca+2}]*gamma[{Ca+2}] +
             my[{Mn+2}] * gamma[{Mn+2}] );
$ End of script

More examples can be found in the GtDemo template math script library.



1.3. Running GtDemo data samplers

After editing the GtDemo data fields on Page 1, press the "Calculate" button or execute "Record" "Calculate" menu command to run the data sampler. If the math script contains syntax errors, you may be asked to fix them before the program can proceed. Calculations may take some time, especially if many large SysEq records are sampled because each such record is expanded into the full GEM IPM data structure and thermodynamic data are calculated if temperature or pressure differs.You may see a small progress window during GtDemo calculations. When all database records are sampled, a message "GtDemo calculation finished OK" will appear in the status line of the "GtDemo" window. Shift to "Page 2"  button and proceed with analyzing, plotting and printing the results. If you want also to plot experimental data then this is the right time to type this data into the xE (abscissa) and yE fields.  


1.4. Plotting and exporting results

Click on the "Plot data..." toolbar button (or execute the "Record" "Plot" menu command) to display the Graph dialog with plotted results and empirical data points, if provided. Now, the graph colors, lines, scales, symbols etc. can be customized (through the "Customize..." button) and later on saved in this GtDemo record.  The plot names from the legend can be dragged and dropped on the plot lines in the graphic image by using the left mouse button. Note that if you close the graph dialog and open it again (or customize it), you will have to drag-and-drop the legend names again. The picture can be saved into a file (png, jpeg or bitmap) by clicking a "Save Image..." button, or printed on a system printer ("Print..." button; may not work properly on some Linux printers and currently in Win32 version). We recommend saving the plots into .png files. 

The sampled (with or without scripts or with/without empirical data) can also be printed into an ASCII file using the "Print" toolbar button on the "GtDemo" window and selecting an appropriate printing script from the appearing list.
 
The GtDemo data sampler can always be re-run after the gExpr script has been modified or some sampled data records have been recalculated. Accordingly, the plot on the Graph dialog can be re-customized.



2. GtDemo Create/Remake Wizard


To create/modify a GtDemo record, press a "Data Sampling and Plotting (GtDemo)" button in the "Gibbs Energy Minimization Modelling Project" dialog. In the "GtDemo" window, select a "Record" "Create" menu command and then type in or select a GtDemo record key.  After you press "Ok", a selection dialog entitled "Please, choose the record type for the data sampling" will appear. There you have to select a database chain from where multiple records are to be sampled and press "Ok" (note that "Probe" will be implemented in forthcoming versions of GEMS). 

A "GtDemo Remake dialog" will appear (you can also get there directly by executing a "Record" "Remake" menu command for the currently loaded GtDemo record). As usual, it is good to type in there the GDname and GDcom fields. The only necessary field to be entered is gDimXY[0,1] - the number of columns for sampled data in the x0 table (default is 2). Do not enter anything into the gDimXY[0,0] cell (number of lines in the x0 and y0 sampler arrays) as it will be automatically set later upon selection of records to be sampled. If you want to plot experimental data over sampled lines then turn on the flag GDps[0,0] and set the number of points and values in the gDimEf[0,0] and gDimEf[0,1] fields, respectively. Optionally (to facilitate writing the gExpr math script), you can set non-zero dimensions of a vector-line gd_wc and/or number of columns in the gQp table. Press "Ok" when ready.

If "Process" record type has been chosen for sampling, the data from all SysEq records generated by an existing process simulator will actually be sampled using the SysEq record list in the Process record; if this is the case, after the "GtDemo" Remake dailog, the program will ask you to select one Process record from the appearing list entitled "Process: Please, select a record key:". After that, a "Please, mark some record keys for data sampling" list of SysEq record keys will appear; press the "Select All" button or mark (using the left mouse button) the records you want to sample and press Ok.

If any other record type than "Process" has been selected for sampling then after the "GtDemo Remake" dialog, the program will ask you to set a record key filter to make the selection list shorter (simply press "Ok" to select from all available records). Then mark the requred database records in the "Please, mark some record keys for data sampling" list and press "Ok".

Now, Page 1 of the GtDemo  window appears. Fill out the GDname and GDcom lines (if not done yet) and also type  into gX_Nam and gY_Nam fields the names of abscissa and ordinate (for the plot legend), as well as the column (plot line) identifiers in the gLnam field. If you want to plot experimental data then also type the identifier(s) in gLnamE field, go to Page 3 of GtDemo window and type the data into xE vector (abscissa) and yE columns (ordinates); this typing can also be done later. Now, what remains is to write in the gExpr field a math script that will tell the program what data to  sample, how to recalculate and put it into the x0 vector (abscissa) and y0 columns (ordinates) of the GtDemo sampler.

4. GtDemo Data Entries

 
GDname - Comment line for the full name of this GtDemo sampler definition
    
This string (up to 70 characters) should be entered by the user for identification of this GtDemo record, best during creation or remake. It will appear on the plot and in the text printouts. 
  
   
GDcom - Comment line for optional enhancement of the GdName line
 
This string (up to 70 characters) can also be entered to enhance identification of the data sampler. 
   
  
gX_Nam  - Abscissa name (for the graph), up to 8 characters
     
Here, the user should type a short name for the abscissa axis that will be shown on the graph. Typical examples: "pH", "log M", "Mol.Fr.%". Ideally, the abscissa name should include units of measurement.
   
 
gY_Nam - Ordinate name (for the graph), up to 8 characters
   
Here, the user should type a short name (up to 8 characters) for the ordinate axis to be shown on the graph. Typical examples: "log M", "Molal", "log a". Ideally, the ordinate name should define units of measurement because names of the plot lines can be given separately in the gLnam list. 
 

gLnam  - List of names of columns in the y0 array, up to 6 characters
   
Here, the user should enter names of columns (plot lines) corresponding to columns in the y0 (sampled data) table. They will be displayed on the graph legend together with coloured symbols and plot lines. From there, you can drag-and-drop the names onto the graph field (see also the "Plotting and exporting results" chapter). Number of elements in this list is defined in the gDimXY[1] counter. 
 
   
 
gLnamE -  List of names of columns in the yE array of empirical data
  
Enter here the name(s) of column(s) in the yE array of empirical data. Similar to names in the gLnam list, these will also be displayed on the graph legend. Number of elements in this list is defined in the gDimEf[1] counter. 
   
     
gExpr  - Math script expressions for data sampling
 
Equations in this math script text provide you with power and flexibility in extracting any input or output numerical data and perform various calculations before plotting results or exporting them for use in other programs. To facilitate writing of data sampling math scripts, a template library is provided (see also "Writing process data sampling math scripts" chapter).
 
The minimum possible set of  gExpr  math script expressions for data sampling has the following form:
x0[jR] =: <expression>;
y0[jR][0] =: <expression>;
. . .
y0[jR][n] =: <expression>;
here,  n stands for the last valid index of columns in the data sampling array y0;  <expression> must contain some labels of data objects visible on the GEM-Selektor window pages. Arbitrary expressions for intermediate calculations can also be specified; in this case, usage of the optional arrays gd_wc and/or gQp is recommended.  
  
Lists of (hidden) data objects that can be used in GtDemo math scripts are provided in a separate file.
  
gExprE - Math script expressions for empirical data (optional)
  
This text field can be used for generation of large arrays of empirical data xE, yE by specifying some functional equations as math script expressions. The math script should have the following nested-loop structure:
jE=: 0;
do begin
iE =: 0;
do begin
xE[iE] =: <expression>;
........
yE[iE][jE] =: <expression>;
........
iE =: <expression>;
end
jE =: <expression>;
end
Expressions in this text are executed only once before the sampling cycle begins.
 
If empirical data are supposed to be entered by hand then this text (hidden by default) contains a "do nothing" operator
next =: next;
   
x0 -  Vector-column of abscissa x
   
This vector-column (of dimension gDimXY[0] set automatically) will be used as an abscissa (x axis) for plotting the sampled data. Contents of this vector must be filled out in the  gExpr  math script, to be executed during the GtDemo sampling run. 
   
   
y0 - Data table of ordinate(s)  y
 
Each column of the y0 sampled data table will be plotted as a separate curve.  Number of columns in the y0 table can be set in the gDimXY[1] counter upon remake/create of the GtDemo record. Number of rows is set automatically from gDimXY[0] counter.  
   
 
xE - Vector-column of abscissa for empirical data 
 
In this vector-column, the user can enter empirical (experimental) data that will be used as an abscissa for plotting empirical (scattered) points on the graph. The number of elements in the xE vector can be given in the gDimEf[0] counter on remake/create of the GtDemo record.
 
    
yE - Vector-column (or table) of ordinate(s) for empirical data 
 
Here, the user can enter empirical (experimental) data to be used as ordinate(s) for plotting empirical (scattered) points on the graph. The number of lines (ordinates) can be defined in the gDimEf[1] counter and that of points - in the gDimEf[0] counter upon remake/create of the GtDemo record.

Optionally, data in this array can be generated using the math script provided in the gExprE field. 
    
 
gd_wc - Vector-row of work cells for GtDemo math scripts (optional)
   
You can type some initial constants here or use the cells as work variables in the gExpr or gExprE math scripts. 

By default, this array is not allocated; it can always be allocated upon remake by setting a positive number of cells (e.g. 10) in the GDnwcq[0] counter.
 
   
gQp - Table of work cells - a row per sampled record (optional)
  
This optional array can be used in the gExpr or gExprE math scripts for storing intermediate data during execution of GtDemo sampler, e.g. as gQp[jR][0] =: <expression>; or y0[jR][1] =:  gQp[jR][0] + gQp[jR][1]; 

By default, this array is not allocated, but it can always be allocated upon remake by setting a positive number of columns (e.g. 3) in the GDnwcq[1] counter. The number of rows will be automatically taken from the gDimXY[0] counter. 

Attention: Contents of gQp array will be lost after calculations because they are not saved in GtDemo records.
  
 
jR - Index of currently sampled database record
  
This index is automatically incremented from 0 to gDimXY[0]-1 when database records are read and sampled consecutively according to the GD_rkl list. 
 
This index should always be used for correct assignments of data to demo arrays x0, y0 in the gExpr math script. It is not recommended to change this index in the math script operators because this may result in memory corruption or the program may crash. 
  
   
next - A flag to continue (1) or finish (0) the data sampling steps
 
This flag is automatically set to 1 at the start of the GtDemo sampling and to 0 when all records have been sampled or a certain condition in the math script has been fulfilled. 
   
GSDref - List of SDref keys to data sources
 
This list can contain one or more SDref record keys (in packed format) referencing to bibliography or your comments to this data sampler definition (e.g. source of the experimental data). 
Allocated if the  GDnlrk[2] counter is set to a non-zero positive number upon remake or create.  
 

GSDval - List of short comments to SDref  record keys in the GSDref list.
 
This list of comments is allocated together with GSDref  if the GDnlrk[2] counter is set to a positive number. 
  
   
i0 - Internal column index for the use in gExpr math scripts
 
    
iE - Internal column index for the use in gExprE math scripts
   
    
j0  - Internal row index for the use in gExpr math scripts
 
  
jE - Internal row index for the use in gExprE math scripts
   
 


4.1. Data Objects Accessible through Create/Remake Wizard Only


GDps[0] - Flag for using the empirical data (create xE, yE arrays) { + - }
 
This flag tells the GtDemo sampler to allocate the xE and yE data objects where the user can enter some experimental data to be displayed as scattered symbols on the graph. 
Possible values:  '+' on ;  '-' off (default). 

Attention! This flag will be automatically switched on after the user has entered positive values into both gDimEf[0] and gDimEf[1] fields; otherwise (i.e. if at least one of those fields contains 0), this flag will be switched off.

This flag tells the GtDemo sampler to collect data from IComp records listed in the GD_rkl list.
Possible values:  '+' on (default);  '-' off. 

This one and the following 8 flags are usually set upon create/remake by choosing the record type to sample from the list entitled "Please, choose the record type for the data sampling". 
   
 
GDps[2] - Flag to sample data from Compos records {+ -}
 
This flag tells the GtDemo sampler to collect data from Compos records listed in the GD_rkl list.
Possible values:  '+' on  '-' off (default).
 
   
GDps[3] - Flag to sample data from DComp records {+ -}
 
This flag tells the GtDemo sampler to collect data from DComp records listed in the GD_rkl list.
Possible values:  '+' on  '-' off (default).
   
  
GDps[4] - Flag to sample data from ReacDC records {+ -}

Possible values:  '+' on  '-' off (default).

This flag tells the GtDemo sampler to collect data from RtParam records listed in the GD_rkl list.

GDps[6] - Flag to sample data from Phase records {+ -}
 
This flag tells the GtDemo sampler to collect data from Phase records listed in the GD_rkl list.
Possible values:  '+' on  '-' off (default).

Possible values:  '+' on  '-' off (default).

This flag tells the GtDemo sampler to collect data from SysEq records generated by a selected Process simulator. 
Possible values:  '+' on  '-' off (default). If this flag is set on, the GDps[6] flag will also be set on.

GDnlrk[0] - Total number of database records to be sampled
 
This counter also defines the number of rows in the GD_rkl list, as well as in x0, y0 and gQp arrays. 
It is set automatically after selection of database records  from the list upon remake/create.
  
 
GDnlrk[1] - Database chain index
 
This cell keeps the GEMS database record type from where the data will be sampled (integer values from 4 to 11).
    
  
GDnlrk[2] - Number of rows in the GSDref and GSDval lists
 
This counter defines how many data source references will be allocated.  Default is 0 (GSDref and GSDval lists will not be allocated and seen on Page 1 of the GtDemo window). 
   
 
GDnlrk[3] -  Reserved
 
 
GDnwcq[0] - Number of cells in the gd_wc work vector

If the user has entered a positive integer value here, the gd_wc array will be allocated. 
Default value: 0 (gd_wc array will not be allocated).
 
  
GDnwcq[1] - Number of columns in the gQp work array

If the user has entered a positive integer value here, the gQp array will be allocated. 
Default value: 0 (gQp array will not be allocated).
 
 
gDimXY - Dimensions of data sampler arrays x0, y0. 
    
The first cell gDimXY[0] contains number of rows N in x0, y0 (automatically copied from GDnlrk[0] counter).
 
The second cell gDimXY[1] contains number of columns M in the y0 table. This value must be entered by the user during remake/create of the GtDemo record. Default value is 2; maximum number is 16.
 
There must be a strict correspondence between the number of columns in the y0 table and references to it in the gExpr math script operators, where the second index at y0 must not exceed M-1 and be no less than 0, otherwise it may result in a memory corruption or a program crash. 
  
   

gDimEf - Dimensions of arrays of empirical data xE, yE
     
The first cell gDimEf[0] contains number of rows N in xE, yE arrays. 
 
The second cell gDimEf[1] contains number of columns M in the yE array. 

Both values must be entered by the user during remake/create of the GtDemo record. Default values are 0. The empirical data arrays xE,yE  will be allocated only if both counters contain positive non-zero values.
  
    
GDproc - Process record key (to sample SysEq records generated by this process simulator)
   
This field contains a Process record key (in compact form) from where the list of generated SysEq records will be taken and a selection of records to sample will be made. The Process record key can be chosen from the list or typed in during create/remake. 

 
GD_rkl - List of record keys of database record to be sampled
  
This list is used for loading database records for sampling data according to the jR index. The list can be modified only upon remake of theGtDemo sampler. Record keys in the list are kept in the compressed format. The database chain to read records from is defined by a database chain index in the GDnlrk[1] cell.  
 
When SysEq records are processed (chain index 10), each record will be expanded into full GEM IPM data structure (see gems_bcc.html, gems_scm.html,gems_ipm.html and gems_sys.html manuals). If necessary, thermodynamic data will be corrected for temperature and pressure automatically.

W09GDrem: No record keys matching a template! Repeat?
 
On remake/create of this GtDemo record, the specified template does not match any of the available record keys. 
 
Try a general template (asterisks in all record key fields) to see all available records, or make sure that proper database files are linked in (do it in Database Mintenance mode). 
 
 
E91MSTran: Error in translation of GtDemo math script
 
There are syntax or pre-processing errors in the gExpr math script. The remaining (not yet processed) part of the script will be printed after the error message in order to help you localize the error.
 
To correct the script, press "Ok" to close the error message, examine the math script syntax in the gExpr field, fix the error (or data object label, or phase/species name), save the GtDemo record to database and calculate the data sampler again.
 
 
E92MSTran: Error in GtDemo math script for empirical data:
 
There are syntax errors in the gExprE math script. 
Edit the script to fix the problem, then save the record to database and try again to re-calculate it.
