1. Thermochemical Data Format


DComp records contain the standard state thermodynamic properties of species (Dependent Components, DC) in a "thermochemical" format  -  molar properties of formation from chemical elements at standard states (Independent Components, IC), rather than properties of reaction with other species. Thermodynamic data for reactions are kept in GEMS database in a special ReacDC format records.
 

Minimal information about a species in a DComp record includes:

  • phase (standard and reference) state;
  • name (identifier), also short name in the DComp record key;
  • stoichiometry (chemical formula, expressed in IC symbols);
  • Default type of species;
  • standard molar (partial molal) Gibbs energy of formation G0Tr,Pr  from IC at reference Tr, Pr  ;
  • standard molar (partial molal) volume, entropy and heat capacity at Tr, Pr .


DComp records may contain several optional data vectors holding coefficients for various T,P dependencies of thermodynamic properties or equations of state, used in the different methods of temperature - pressure extrapolation.
 

DComp database module (as any other in GEM-Selektor database) consists of several files which can be anytime linked or unlinked by the user through the "Files" menu entry in the "DComp" window. Each file belongs either to the "kernel database" or to the user's profile extension, and normally contains a number of DComp records comprizing a subset of the whole "internally consistent" thermodynamic dataset. In this version of GEM-Selektor kernel database, two published datasets are incorporated: the SUPCRT98 database for aqueous species and gases [1997SHO/SAS and references therein], and the USGS thermochemical database for minerals [1995ROB/HEM]. This does not imply that thermodynamic data are always consistent between different datasets! The user should be careful in using different datasets together and, in general, should avoid mixing of the data from the alternative datasets (e.g., [1988BER] and [1990HOL/POW] in Phase, Profile and SysEq definitions, wherever possible.
 

Clicking on a "Dependent Components (DComp)" button in the "Thermodynamic Database Management" dialog opens a "DComp" window with two pages. Page 1 provides a screen form access to standard state thermodynamic data and stoichiometry of the species, whereas Page 2 shows configuration flags and optional arrays with empirical coefficients. Icon buttons and menu commands in "DComp" window are the same as in other module windows (see g_miscel.html and gt_howto.html). They provide all functionality necessary for maintaining, cleaning and extending the database.
 

Available DComp records can be browsed ("Record List" "Browse" menu or "Forward" and "Backward" icon buttons in the toolbar) until one is selected. You can also "Load", i.e. directly select one record from the list of available record keys, by pressing the "Show" icon button or "Record" "Show" item. The data fields (each displays a tooltip when pointed with mouse) can then be edited immediately (just click with left mouse button in the data field).
 

Please, note that 0 (zero) is a meaningful number in GEM-Selektor thermodynamic database. If data is not known then an "Empty" value has to be entered (back apostrophe ` in this version). The PrTr field contains reference pressure and temperature (1 bar and 25 oC by default). One or more bibliographic references to sources of data are usually specified in dSDref list; clicking on any of these and pressing F2 opens a "SDref" window with full contents of the reference - SDref record.
 

Two optional vector-columns are usually present on  Page 2  of the DComp window. The aiCpT data object contains up to 11 empirical coefficients of temperature dependence of heat capacity function Cp0  = f  (T ), and the  TCint  field shows temperature range (in centigrade) of applicability of  aiCpT  coefficients. The first three non-zero coefficients correspond to Majer-Kelly equation,  the first five non-zero coefficients - to Haas-Fisher equation, together covering most of published databases [see 1994NOR/MUN]. More than one equation can be specified for one mineral having phase transitions; then on remake of DComp record, one has to provide a required number of columns to be created in the aiCpT andTCint data objects. Compressibility / expandability data can also be entered, either directly into  BetAlp  fields, or as optional coefficients of  V0  = f  (T, P  )  equation (ai_Vtp data object).
 

What optional data must be entered and how they will be used in thermodynamic calculations, is defined by the TPcMod codes (see Page 2  of the "DComp" window) set up during remake/create of a DComp record.  For instance, the "CSC" method codes (most common for minerals) tell that the standard integration over temperature interval Tr   to  T   using Cp0  = f  (T) equation coefficients will be performed while keeping standard molar volume constant. This way, the standard molar properties of the mineral (G0, H0, S0,  and Cp0  ) are automatically computed by GEM-Selektor for any given  T, P  of interest within the interval of temperature defined in TCint  . Calculations will be also performed for temperatures above the upper limit in TCint, although one must keep in mind that, in most cases, empirical coeffcients of Cp0  = f (T) equation are not suitable for extrapolation.
 

The same thermochemical format is used for gases. There are some differences, however. First, molar volume of an ideal gas changes strongly with temperature and pressure according to PV  = nRT  equation. Second, at pressures greater than a few bars, most gases do not behave ideally and at elevated T, P  do not mix ideally (see [1994NOR/MUN] for more detail).  In Selektor codes, a reasonable compromise is achieved by applying the "Correspondent States Theory" to calculate fugacities of individual pure gases a  T,P   of interest and then consider the gas phase as an ideal mixture of real gases. For that, an optional  CritPg  vector can be specified for keeping critical parameters for individual gases, needed for the correspondent-state theory calculations (under construction). This version of GEM-Selektor treats gas phase at low temperatures and pressures as an ideal mixture of ideal gases; the respective phase definition is created automatically upon opening the system profile.
 

A third form of DComp data format is used for aqueous ions and complexes from SUPCRT92-98 datasets [1992JOH/OEL; 1997SHO/SAS]. The increments to standard-reference partial molal properties for aqueous species are calculated from Tr, Pr    to elevated T  < 1000 oC and P  < 5000 bar using revised HKF EoS (Helgeson-Kirkham-Flowers equation of state, [1981HEL/KIR;1997SHO/SAS; 1992BOR/SHV] ). All the necessary subroutines from the published SUPCRT92 source code [1992JOH/OEL] were translated from fortran to C/C++ and built into the Selektor-A and then the GEM-Selektor code. In addition to input standard-state properties G0,H0 and S0 at Tr, Pr ,  calculations with the HKF EoS require as input seven empirical coefficients kept in the ai_HKF optional vector (allocated when the first three of  TPcMod codes are set to "HKF" on remake, see Page 2 of the "DComp" window).
 

As an example, pick up a DComp record keyed "a:WS+6:SO4-2:sa0:" which keeps the standard partial molal thermodynamic properties of sulfate ion. On Page 2 of "DComp" window, the ai_HKF vector contains empirical parameters of HKF EoS. SUPCRT92 data, taken as kernel database in GEM-Selektor, are consistent with the three-term Debye-Hueckel activity model of aqueous electrolyte [more in 1997LAN; 1994NOR/MUN]. This model uses either common or individual (Kielland) ion size parameter(s) of aqueous species, which can be entered ino the ab[0] field. Calculation of aqueous activity coefficients with Debye-Hueckel equation with a common third parameter is built into GEM-Selektor code (see Phase definition); in this case, definition of aqueous phase is created automatically upon opening of the system profile. Other calculations (e.g. Davies, SIT, Pitzer models) should be set explicitly in Phase records for aqueous electrolyte phase.
 
In all cases, the stoichiometry of a dependent component is critically important and must be entered into the DCform field on Page 1 of "DComp" window according to chemical formula syntax.

 

1.1. DComp module calculations

When the Calculate button (or menu command, or F9 hot key) is clicked from the "DComp" window, GEM-Selektor will parse and check the DC chemical formula , calculate gram-formula mass and charge, perform cross-checking of standard molar properties, and (optionally) calculate heat capacity and molar volume at reference temperature and pressure. If any problems are encountered, various diagnostic messages will be displayed and, after fixing the problem, recalculation of the DComp record must be repeated until all the input data becomes consistent.
 
 

2. Description of Input Data Objects and Fields


DCname - Full name of Dependent Component (species)
 
This is a string up to 70 characters long, entered by the user. It serves only for identification and commenting purposes on the DComp window page. Examples:
 
Ca+2 ion
Portlandite (crystalline)

  
DCform - Chemical formula (stoichiometry) of Dependent Component (species)
 
This is the most important input data field on the DComp screen forms. Enter here a chemical formula of the species, expressed in symbols of Independent Components (IComp) at present available in the database. The formula completely defines stoichiometry and charge of the Dependent Component. Formulae are parsed and analyzed in many modules of GEM-Selektor to be converted into lines of the stoichiometry matrix, for calculation of molar masses, phase/species compatibility checks etc.  GEM-Selektor supports a simple and convenient formula syntax, very close to the conventional chemical notation.
Formula is a string of text, maximum 70 characters long. Only symbols of Independent Components (IC) from currently available IComp records can be used. Each IC symbol begins with an upper-case Latin letter, followed by zero to four lower-case Latin letters or _ (underscore) characters.
 
Explicit valence of IC can be specified within vertical bars ||immediately after the IC symbol, e.g. H2S|-2| (hydrogen sulfide).
 
IC symbols with stoichiometry coefficients can be grouped in brackets ([]) followed by a common stoichiometry coefficient, e.g. Th(OH)4. Up to 3 levels of brackets can be nested into each other. On translation of the formula, the stoichiometry coefficient after brackets will be applied to each of IC symbols inside. For instance, writing Eu2(CO3)3 is equivalent to Eu2C3O9. Stoichiometry coefficients (except charge) must be positive but not necessarily integer (decimal point notation is allowed, e.g. 2.67). No formula stoichiometry coefficient after the IC symbol means that this coefficient equals 1.
 
Atomic mass of an isotope must be specified in slashes // immediately before an IC symbol, e.g. /12/C. Charge (+, +2, ..., -, -2, ...,@) must always be provided in formulae of aqueous species and surface complexes.  Charge can also be non-integer and is always addressed to Zz  independent component.
 
More examples of species chemical formulae:
 
Na+, Cl-, NaCl@               aqueous species
SiO2, Fe3Si4O10(OH)2         quarts, minnesotaite
Fe|3|2O3                        hematite
/3/H2/18/O                       isotopic form of water
 
The DCform field is processed when you click on "Calculate" icon or "Record" "Calculate" menu item in the DComp window. You may get error messages if something is wrong with the formula stoichiometry or charge. Check carefully the formulae when you enter them - to avoid nasty errors in chemical system definitions and GEM IPM calculations later on. 
 
   
Zz - Formula charge of this species (Dependent Component)
 
Charge per unit formula  (can be non-integer) to be entered by the user. For aqueous and surface species it is set automatically (upon clicking on "Calculate" icon or "Record" "Calculate" menu item) using the default- or explicitly specified valences of independent components. The following default valences are used for some common multivalent elements (see the respective IComp records):
 
C  + 4, Cl  -1, S  +6, V  +5, Mn  +2, Fe   +2, Cu   +1, U   +4.
  
  
M0 - Molecular (gram-formula) mass, g/mol
  
Molecular mass of a compound is calculated automatically (upon clicking on "Calculate" icon or "Record" "Calculate" menu item) from its chemical formula  and atomic masses extracted from M0i fields in IComp records. There is no need to enter data into this field, but if you already know gram-formula mass of the species, to compare it with calculated M0 value would be a good test of correctness for the chemical formula that you have just typed into the DCform field. 
   
   
V0d[0]- Molar volume of Dependent Component at standard state (J/bar)
 
Standard-state molar (partial molal for aqueous and surface species) volume V0  at reference pressure Pr  and temperature  Tr   must be entered by the user (except for gases) in units of measurement defined by the code in DCset[2] cell.
 
There is no need to enter V0   value for gases  because  V0  =  RT   = 2478.93 J/bar = 24789.3 cm3/mol for any ideal gas at P= 1 bar and  T = 25  C  and  it will be set automatically. Moreover, even though  V0  value has been entered, it will be automatically replaced (upon clicking on "Calculate" icon or "Record" "Calculate" menu item) by a value calculated for any non-gaseous species whose properties are defined by an equation of state (e.g., HKF EoS).
   
  
V0d[1]- Uncertainty of standard molar (partial molal) volume
 
Absolute uncertainty of V0  value (see V0d[0] cell),  if known, must be entered by user in the same units of measurement as  V0  . If unknown, put an EMPTY value ("---").
 
In this version of GEMS, entering data into this field is desirable for data accumulation, to be used for assesments of uncertainty propagation that will be available in the next version.
 
  
G0d[0] - Standard molar (partial molal) Gibbs energy of formation of DC from IC at Tr and Pr.
 
The Gibbs energy  G0  at reference pressure  Pr  and temperature  Tr   must always be entered by the user in units of measurement defined by the code in DCset[1] cell, if one of the two related values - either  H0   or  S0   -  is unknown (i.e. the H0d[0] or S0d[0] field will contain "Empty value"). Remember that 0 (zero) is regarded as a valid (non-empty) value for thermodynamic properties!
 
If both H0   and  S0 values are known (non-empty) and you entered  "Empty value" into this (G0d[0]) field then (upon clicking on "Calculate" icon or "Record" "Calculate" menu item) the value of G0  will be calculated automatically from H0   and  S0  using the  GHS_DC()  algorithm.
 
If all three valid numbers  G0  H0  and S0  were entered then (upon clicking on "Calculate" icon), the enthalpy value H*  will be found from  G0  and S0  using the  GHS_DC()  algorithm and compared to H0 . You will get a warning message that three values are inconsistent, if absolute difference  ¦H* -  H0  ¦  exceeds  40  J/mol. If this happens, check all three entered values - maybe one of them is misprinted, and check also chemical formula (DCform field) for stoichiometry coefficients, valences (default or explicit), and unit formula charge (Zz field).
  
 
G0d[1] - Uncertainty of standard molar (partial molal) Gibbs energy of formation
 
Absolute uncertainty of G0  value (see G0d[0] cell),  if known, must be entered by the user in the same units of measurement as  G0  . If unknown, put an "Empty value" (`) there.
 
In this version of GEM-Selektor, entering data into this field is desirable for data accumulation, to be used for assesments of uncertainty propagation that will be available in the next version.
  
 
H0d[0]- Molar (partial molal) enthalpy of formation of DC from IC at standard state and Tr, Pr
 
The enthalpy  H0  at reference pressure  Pr  and temperature  Tr   must always be entered by the user in units of measurement defined by the code in DCset[1] cell (e.g., J/mol), if one of the two related values - either  G0   or  S0   -  is unknown (i.e. theG0d[0] or S0d[0] field will contain "Empty value"). Remember that 0 (zero) is regarded as a valid (non-empty) value for thermodynamic properties!
 
If both G0   and  S0 values are known (non-empty) and you entered  "Empty value" into this (H0d[0]) field then (upon clicking on "Calculate" icon) the value of  H0  will be calculated automatically from G0   and S0 using the  GHS_DC()  algorithm.
 
If all three non-empty numbers  G0  H0  and S0  have been entered then (upon clicking on "Calculate" icon), the enthalpy value H*  will be found from G0  and S0  using the GHS_DC()  algorithm and compared to H0 . You will get a warning message that three values are inconsistent, if absolute difference  ¦H* - H0  ¦  exceeds  40  J/mol. If this happens, check all three entered values - maybe one of them is misprinted, and check also chemical formula (DCform field) for stoichiometry coefficients, valences (default or explicit), and unit formula charge (Zz field).
 
 
H0d[1] - Uncertainty of standard molar (partial molal) enthalpy of formation
 
Absolute uncertainty of H0  value (see H0d[0] cell),  if known, must be entered by the user in the same units of measurement as  H0  . If unknown, put the EMPTY value ("`" or "---") here.
 
In this version of GEM-Selektor, entering data into this field is desirable for data accumulation, to be used for assesments of uncertainty propagation that will be available in the next version.
 
 
S0d[0] - Absolute molar (partial molal) entropy of Dependent Component at the standard state
 
Third-law entropy  S0  at reference pressure  Pr  and temperature  Tr   must always be entered by the user in units of measurement defined by the code in DCset[1] cell (e.g., J/K/mol), if one of the two related values - either  G0   or  H0   -  is unknown (i.e. theG0d[0] or H0d[0] field will contain "Empty value"). Remember that 0 (zero) is regarded as a valid (non-empty) value for thermodynamic properties!
 
Important: Absolute entropy S0  of a compound is different from entropy of its formation from chemical elements  S0F  - please make sure that you enter  S0  value into this field!
 
If both G0   and  H0 values are known (non-empty) and you entered  "Empty value" into this (S0d[0]) field then (upon clicking on "Calculate" icon) the value of  S0  will be calculated automatically from G0   and H0 using the  GHS_DC()  algorithm.
 
If all three non-empty numbers  G0  H0  and S0  have been entered then (upon clicking on "Calculate" icon), the enthalpy value H*  will be found from G0  and S0  using the GHS_DC()  algorithm and compared to H0 . You will get a warning message that three values are inconsistent, if absolute difference  ¦H* - H0  ¦  exceeds  40  J/mol. If this happens, check all three entered values - maybe one of them is misprinted, and check also chemical formula (DCform field) for stoichiometry coefficients, valences (default or explicit), and unit formula charge (Zz field).
   
  
S0d[1]- Uncertainty of standard molar (partial molal) absolute entropy of formation
 
Absolute uncertainty of S0  value (see S0d[0] cell),  if known, must be entered by the user in the same units of measurement as  S0  . If unknown, put an EMPTY value ("`" or "---") here.
 
In this version of GEM-Selektor, entering data into this field is desirable for data accumulation, to be used for assesments of uncertainty propagation that will be available in the next version.
 
 
Cp0d[0] - Absolute molar (partial molal) isobaric heat capacity Cp0  of DC at standard state and reference Tr, Pr.
 
The  Cp0  value must be entered into this cell only in cases when no coefficients for Cp = f(T)  equation or equation of state are allocated. Otherwise, Cp0  value at Tr , Pr  will be calculated automatically upon clicking on "Calculate" icon from the  Cp = f(T)  coefficients.
 
Units of measurement: the same as for S0  ( given in DCset[1] cell).
 
 
Cp0d[1] - Uncertainty of   Cp0   at reference Tr, Pr .
 
Can be entered by user in the same units of measurement as  Cp0 .
 
To be used for assesments of uncertainty propagation in the next versions of GEM-Selektor.
 
 
PrTr[0] - Reference pressure Pr   (bar)
 
This cell must contain a value of reference pressure Pr   to define standard state molar (partial molal) thermodynamic propertiesV0,  G0,  H0,  S0,  Cp0  of a Dependent Component. When a new DComp record is created, Pr   is set to a default value 1 bar.
 
Units of measurement: defined by a code in DCset[3].
  
  
PrTr[1] - Reference temperature Tr  (oC)
 
This cell must contain a value of reference pressure Tr   to define standard state molar (partial molal) thermodynamic propertiesV0,  G0,  H0,  S0,  Cp0  of a Dependent Component. When a new DComp record is created, Tr   is set to a default value 25 oC (298.15 K).
 
Units of measurement: defined by a code in DCset[4].
   
  
BetAlp[0] - Coefficient of isothermal compressibility b (or bulk modulus at 298 K)
 
Isothermal compressibility  b (beta) of a mineral compound at Tr  can be entered by the user into this field if a method of calculation of  V0   increment from Pr   to  P   is specified ( 'E' in  TPcMod[2] field).
Note that this version calculates compressibility for minerals using the Birch-Murnaghan EoS according to [Holland and Powell, 1998]; in this case, this cell must contain a non-zero value of the bulk modulus k at 298 K (in kbar). 
 
Units of measurement: (1/kbar) or kbar
 
 
BetAlp[1] - Coefficient of isobaric thermal expansion a
 
Isobaric expansibility a (alpha) at Pr  of a mineral compound can be entered by the user if a method of calculation of  V0   increment from Tr   to  T  and Pr   to  P   is specified ( 'E' in  TPcMod[2] field).
  
For calculations according to [Holland and Powell, 1998], this cell must contain a value of high-temperature limit thermal expansion coefficient a (in 1/K).
 
Units of measurement: (1/K).
 
 
ab[0]- Ion-size parameter ao  (Debye-Hueckel equation) for aqueous species; optional Vmax parameter for solids
 
This cell contains (user-supplied) value of Kielland ion-size parameter ao  for Debye-Hueckel family of equations for calculation of activity coefficients of aqueous species. Units of measurement: Angstroems (A).
 
This value will copied into a dc_cf array of non-ideality parameters in the aqueous phase definition (Phase record) and may be edited there if aqueous electrolyte phase is explicitly defined. The ao  value from this cell will be automatically used in calculations of chemical equilibria, if the default aqueous phase is specified in the modelling project.
 
If absent from the Kielland (1936) table (see [1994NOR/MUN] or [1997LAN]), the following default values can be used:
 
Charge        0      -1    -2;  +1  -3; -4; ...    +2; +3   +3; +4
 ao value         0     3.5      4.5      5.0      6.0     9.0
 
Negative values b > -1 or positive values b < +1 can be entered here for neutral aqueous species. If the built-in Debye-Hueckel equation is used then the activity coefficients will be calculated for such j-th species as 
lnGam[j] = 2.303 * I * b., where I  is the effective molal ionic strength.  

If b = 0  then  lnGam[j] = 2.303 * I * bg wherebg is a common background electrolyte parameter (default 0.064 as for NaCl, [1981HEL/KIR]). In the explicit definition of aqueous phase, other values of this parameter can be set in the ph_cf[5] cell in the Phase window.  
 
For any (charged or neutral) aqueous species, setting of this cell to -1 will result in lnGam[j] = 0 if the built-in Debye-Hueckel calculation has been chosen.
  
For minerals, this field may optionally contain the Vmax parameter (j/bar) used in lambda-transition calculations according to Holland and Powell (1998).

 
ab[1] - Individual parameter b (optional)
 
Individual parameter b can be used in a linear term of the Truesdell-Jones modification or 3-term Debye- Hueckel equation (see [1994NOR/MUN] or [1997LAN]).
 
To be used, requires an explicit definition of aqueous electrolyte phase. If unknown, set "Empty value" here.
 

LamST[0] - Entropy of lambda transition (J/mol/K)

The value of Smax can be entered and employed in thermodynamic calculations for condensed substances undergoing phase transitions, based on Landau theory as implemented in calculation scheme by Holland and Powell (1998).
 

LamST[1] - Temperature of lambda transition (C)
 
The critical temperature of lambda transition Tcrit can be enetered here. The temperature must be entered in centigrade. Note that the corresponding critical volume Vmax of lambda transition (required to calculate the pressure dependence of the phase transition) needs to be entered into the ab[0] field.
   
 
TCint[0] - Lower temperature validity limit(s) for  Cp  = f (T  )   equation
 
Each column in this line must contain a lower temperature boundary (in centigrade) of the respective  Cp  = f (T  )  equation coefficients.
 
If two or more  Cp  = f (T  )   equations are specified then the upper boundary of the previous temperature interval must be the same as the lower boundary of the next interval.
 
Units of measurement: defined by the code in  DCset[4].
 
If, during thermodynamic calculations,  current temperature T  goes outside all of the temperature intervals specified in the  TCint  object, then this Dependent Component will be marked  "e" (extrapolated) in TPmark  column of the "MTparm" system profile window. Properties of this DC will be calculated from the  Cp  = f (T  )  coefficients for the closest temperature interval. However, this may include large error because, in general, coefficients of such Cp  = f (T  )  equations are just fitting parameters and do not allow extrapolations.
 
 
TCint[1] - Upper temperature validity limit(s) for  Cp  = f (T  )   equation
 
Each column in this line must contain an upper temperature boundary (in centigrade) of the respective  Cp  = f (T  )  equation coefficients.
 
If two or more  Cp  = f (T  )   equations are specified then the upper boundary of the previous temperature interval must be the same as the lower boundary of the next interval.
 
Units of measurement: defined by the code in  DCset[4].
 
 
 
aiCpT - Array of empirical coefficients of  Cp0 = f (T )  equations
 
Each column of this data object has 11 cells for input coefficients of temperature power terms in the isobaric heat capacity Cp0  = f (T  )  equation of the following general form:
 
Cp0 = a0  + a1T  +  a2 T ^-2  + a3 T ^-0.5  +  a4 T ^2  + a5 T ^3 +
         + a6 T^4  + a7 T ^-3  + a8 T ^-1  + a9 T ^0.5  .
 
The last cell is not used (reserved). This equation utilizes most of published thermodynamic data. The first three coefficients correspond to Majer-Kelley equation, the first 5 coefficients - to Haas-Fisher equation (cf. [1994NOR/MUN]).  Applicable for solids (from Tr  to T  at Pr ), gases (from Tr  to T  at Pr ), and aqueous species (from Tr  to T  at PSAT ) for calculation of their standard molar (molal) thermodynamic properties using Schwarzman-Temkin functions.
 
Unspecified coefficients must be set to zeros.
Units of measurement for Cp0 : defined by code in DCset[1]. Real values must be entered (i.e. without assumed scale multipliers).
 
Value of Cp0  at Tr  = 25 C  is computed from the above equation coefficients every time when this DComp record is recalculated, and the result placed into  Cp0d[0] field. This is a good test for correctness of input - if calculated number appearing in   Cp0d[0]  is not withing the typical range (a few hundred J/K/mol, positive for minerals and gases) then some of the Cp coeffcients have been entered incorrectly and need to be fixed.
 
aiCpT[0] - Free term coefficient a0  of general Cp  = f (T  )  equation
 
Don't forget to set value of this coefficient if  Cp0  is assumed constant over the whole temperature interval - this value will be taken into  Cp0d[0] field upon recalculation of this DComp record.
 
aiCpT[1] -
Coefficient a1  of the  a1T  term  of  general Cp  = f (T  )  equation
aiCpT[2]
Coefficient a2  of the  a2T^-2  term  of  general Cp  = f (T  )  equation
aiCpT[3]
Coefficient a3  of the  a3T^-0.5  term  of  general Cp  = f (T  )  equation
aiCpT[4]
Coefficient a4  of the  a4T^2  term  of  general Cp  = f(T  )  equation
aiCpT[5]
Coefficient a5  of the  a5T^3  term  of  general Cp  = f (T  )  equation
aiCpT[6]
Coefficient a6  of the  a6T^4  term  of  general Cp  = f (T  )  equation
aiCpT[7]
Coefficient a7  of the  a7T^-3  term  of  general Cp  = f (T  )  equation
aiCpT[8]
Coefficient a8  of the  a8T^-1  term  of  general Cp  = f (T  )  equation
aiCpT[9]
Coefficient a9  of the  a9T^0.5  term  of  general Cp  = f (T  )  equation

Coefficient a10  of  general Cp  = f (T  )  equation  (reserved for future use)

This optional vector contains empirical parameters of revised Helgeson- Kirkham- Flowers equation of state for calculation of standard partial molal properties of aqueous species up to 1000 oC and 5000 bar [1981HEL/KIR; 1997SHO/SAS]. aiNEs - empirical coefficients describing hydration properties of aqueous nonelectrolyte species
The thermodynamic properties of aqueous nonelectrolyte species can be conveniently described through the properties of hydration, which are added to the properties of the species in the ideal gas state. This concept is used is several recently published equations of state for aqueous nonelectrolytes, such as dissolved gases (but in principle it is also applicable to electrolytes). Currently, the model proposed by Akinfiev and Diamond (2003) is implemented in GEM-Selektor, which requires 3 species-dependent coefficients (xi, a and b). In the future, other models such as that by Plyasunov et al. (2001a; 2001b) might be added.

aiNEs[0]
Nonelectrolyte parameter xi to be used for the Akinfiev-Diamond model. This parameter describes how different the volumetric behavior of the aqueous nonelectrolyte species is from the pure water solvent (Akinfiev and Diamond 2003).
 
aiNEs[1]
Nonelectrolyte parameter a to be used for the Akinfiev-Diamond model. This parameter is the constant term in the empirical function that describes the difference in second virial coefficient between the aqueous nonelectrolyte species and the water solvent.

aiNEs[2]
Nonelectrolyte parameter a to be used for the Akinfiev-Diamond model. his parameter is the temperature dependent term in the empirical function that describes the difference in second virial coefficient between the aqueous nonelectrolyte species and the water solvent.

 3. Creating/Remaking DComp records


To construct a DComp record, close your modelling project and switch to the "Thermodynamic Database" mode of GEM-Selektor. There, open the "Dependent Components (DComp)" module window and execute the "Record" "Create" menu command. In the appearing "DComp: Insert a new record key" dialog, make a new DComp record key (at best selecting one of existing keys "From List" and modifying it), and press "Ok". This record key denotes a dependent component to be defined in this DComp record.  A small "DComp: Remake Dialog ..." will appear. You can read the tooltips on all the configuration fields and flags, or click on any of them and press F1 to read help pages (available below in this manual). Change necessary fields and press "Ok" to proceed.  

Now, you should see Page 1 of the DComp module window. The third record key field (species name) appears in the DCname field; you can enhance this comment line (recommended). The second line (DCform) is still empty, but it is obligatory to enter there the chemical formula of the new species. 

Now, you must set up the standard molar properties in V0d[0]; G0d[0];S0d[0] or  H0d[0] cells. Zeros are valid quantities here, and any cell with unknown data must be set as an EMPTY value ("---" or "`").  You can enter values into  G0d[0]  and either S0r[0] or H0r[0] (another must be set EMPTY and will be restored in the calculation). In any case, at least two of three parameters must be given non-EMPTY values. If all three are given then the program will check their consistensy and report if the values are inconsistent. 

In most cases (by default), DComp records are set to use one set of the Cp=f(T) equation coefficients and constant molar volume for temperature extrapolations. Alternatively, you can define more than one sets and phase transitions, if you have the necessary data. In any case, you have to enter the temperature interval of validity of the temperature extrapolation (in the TC_int data field). This is not necessary if HKF EoS coefficients are to be specified as the respective intervals of temperature and pressure are inherent to built-in SUPCRT92 calculation subroutines. In the latter case, there is no need to enter values into V0d[0] and Cp0d[0] cells because they will be automatically calculated from HKF EoS data.

This method code (one upper-case letter) defines the technique to be used for calculation of standard molar (molal) thermodynamic properties of this Dependent Component at temperature T different from Tr.
Possible codes: see User's Reference.

This method code defines the specific technique and routine to be used in  accounting for the temperature dependence of standard molar (molal) thermodynamic properties of this DC at T not equal Tr.
Possible codes: see User's Reference.

It is quite important to set DC class code correctly because the program uses it to select equations applied for calculation of concentrations and chemical potentials of this component during GEM computations. You can override this default code in Phase definition  (PhDCC column).

Flags controlling allocation of dynamic data in DCOMP record

These flags are used by the program to allocate or deallocate optional dynamic arrays where the user can put some empirical coefficients or parameters used in various recalculations of the thermodynamic data at T and P different from reference Tr, Pr. Possible values:
 

*        allocate and initialize;
+        keep and do not initialize;
-        delete if exists.


The dynamic data control flags can be changed only in a configuration dialog at remake/create of the DComp record. In other situations, the flags are not editable to prevent the program memory corruption.
 

 
DCset[5] - Flag of Cp=f(T) coefficient array allocation(* + -)

If allocated, DC_cnt[0] value must be non-zero - do not forget to set it to number of heat capacity equations that you wish to provide for this DC! These equations will be used in calculation of molar properties at elevated temperatures. 
 
 
DCset[6] - Are there phase transitions (+ -)?

W08DCrun: Inconsistent values of H0, S0 or G0 ->  <value>
 
This message appears after calculation of the DComp record if all three non-empty values have been entered into G0d[0],H0d[0] and S0d[0] cells, but these are inconsistent. The -> <value> suggests a consistent G0 value assuming that H0 and S0 values are correct. If you agree with that then press "yes" and the suggested G0 value will be inserted into the G0d[0] cell, previous value of which will be lost. 
 
However, most frequently the H0 values are inconsistent, or the S0 value can be misprinted. In this case, press "no", then put an EMPTY "---" value into either H0d[0] and S0d[0] cell, and calculate DComp again. The correct value will automatically replace the EMPTY value.
