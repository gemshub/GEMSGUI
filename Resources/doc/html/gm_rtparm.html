1. Introduction

When thermodynamic data are entered into DComp records, or new ReacDC definitions are constructed, misprints and errors can be introduced. Some of them are automatically detected in DComp / ReacDC subroutines, i.e. such errors as incomplete or inconsistent standard state data, wrong formula stoichiometry, incorrect species charge; the corresponding diagnostic messages are then issued. Some other errors can be detected only when a series of thermodynamic data for varying T,P  is computed and plotted on graphs. Thus, the RTParm screen forms and graphics are provided for facilitating the process of "debugging"  thermodynamic data and their T,P dependencies for individual species. The RTParm module takes the input data from DComp or ReacDC record having a key with the same first four fields as in the RTParm record key. Note that the override mechanism does not apply to RTParm module operation.

RTParm module workflow consists of three basic steps.

Create/remake for a given DComp or ReacDC record. RTParm Wizard appears to configure the calculations. After the wizard is finished, RTParm arrays will be allocated, and the 'Scripts' page appears. There, the user can fill out the RTname and RTcom comment lines; edit the math script in tExpr field, if necessary. This script will collect data from the objects visible on the TPwork page and puts it into xT, xP and yF array elements.

Calculation. Clicking on 'Calculate' toolbar button runs the RTParm tabulator. This calculator iterates through T,P pairs as geven in xT and xP vectors. For each T,P, thermodynamic properties are calculated and put into fields seen in TPWork page, from where they are sampled by tExpr script into the xF table accessible in the 'Tables' page. On that page, the user can edit there the names of axis and plots, and click on the 'Plot data' button to see a customizable plot.

Plotting experimental data over calculated results is an option. To do this, the necessary number of experimental points and parameters per point must be set in RTParm wizard in beforehand. The user can type or paste the values into xTE, xPE and ytE arrays (appearing on Setup page), and even optionally re-scale them upon RTParm calculation by entering the script into tExprE field. open the plot again, and customise it, if necessary.

The RTParm record can always be re-configured by clicking on Remake toolbar button, or cloned for the same or different DComp or ReacDC record.



2. RTParm Create/Remake Wizard

To create a new RTParm record, first make sure that the source DComp or ReacDC record has already been created, calculated and saved in the database. Then switch to the "RTParm" window and select the "Record" "Create" menu command.

A small dialog will appear asking you to choose a source data record from either DComp or ReacDC records. Press "DComp" or "ReacDC" buttons (depending on what a source record you actually have). This will display a list of all available record keys, where you have to mark one and press the "Ok" button.

Next, you will see a "RTParm: Please, set a new record key" dialog where you have to enter only the last field (shown as the asterisk), usually a calculation variant because more then one RTParm record can be created for the same source DComp or ReacDC record. Enter "1", "2", or any small integer you like, and press "Ok" when ready. This will bring you into the RTParm Wizard (see below).

2.2. RTParm Wizard Step 2 - Selection of items to sample

This step provides a convenient way of selecting results of RTParm calculations that should be sampled by the RTParm script into the yF table (and possibly plotted), along with desired unit conversions and re-scaling. The sampling script will be executed for each row of yF (xT, xP) tables; the rows have index jTP that automatically will run from 0 to nTP-1, where nTP is the number of rows. Therefore, the script contains of assignments to elements of the yF arrays, e.g. yf[jTP][0], ... . The values of thermodynamic properties at T,P for a given calculation step jTP are accessible in data objects visible in TPwork page of RTParm window.

The 'Item Selection' list contains entries that correspond to data objects in the TPwork page. Each entry shows a tooltip by hovering the cursor over it. Left click on the entry results in its addition to the script (with incremented column idex), clicking again removes the item operator from the script.

By right-clicking on the item, a small menu appears, where "Calculator" can be selected. This opens the GEMS calculator, in which one can select the operation and the second operand. For instance, to convert Joules to kiloJoules, select '/' (divide), type 1000 in the operand field, and click Ok. Typical result of such manipulations is shown in a screenshot below.




It is not recommended to select all items for one RTParm record; you can always clone two or three such records for the same ReacDC or DComp. For instance, either the molar properties for the reaction-defined component (as shown above), or the properties of the reaction should be selected.

When ready, click "Next >" to proceed. The script can be edited in the Scripts page later on.


2.3. RTParm Wizard Step 3 - Important dimensions

This wizard page provides an additional setup of dimensions for the data tabulation arrays. This group has three spinboxes (from top to bottom).

(1)  Number of columns in the yF table (plot curves), 1 to 10. Default: 1. The user should check this number, which should not be less than the number of operators assigning yF[jTP][0], ... values in the RTParm script. Add more columns if some such operators will be added later directly by editing the script. The number of rows (T-P pairs) in yF, xP, xT arrays will be set automatically.

(2) Optional: Number of rows in xTE, xPE, ytE arrays for experimental data. Default: 0 (these arrays are not provided). Set here (if any) the number of  data points that should be plotted over the calculated curves. If this dimension is set to a positive number then another script editor tExprE will appear, where an optional script for re-scaling experimental values in ytE table can be entered. 

(3) Optional: Number of columns in the ytE array for experimental data. Default: 1. Change this if more than one number per experimental data point will be entered into the ytE table.


2.4. RTParm Wizard Step 4 - Workflow suggestions

Here the user can set the (optional) number of links to SDref bibliography records (default: 0), for instance, to the source of experimental data.

This wizard page also contains a brief description of RTParm workflow:

(1) RTParm arrays will be allocated; 'Scripts' page appears. Fill out RTname and RTcom comment lines. Edit the math script in tExpr; field, if necessary. This script collects data from the objects visible on the TPwork page and puts it into xT, xP and yF array elements.

'(2) Click on 'Calculate' toolbar button to run the tabulator (in case of error message, fix the script and try calculation again). Check sampled results on the 'Tables' page. Enter names of axis and plots, and click on the 'Plot data' button to see a customizable plot.

(3) To plot experimental data over calculated results, close the graph dialog, type or paste the values into xTE, xPE and ytE arrays, open the plot again, and customise it, if necessary.


3. RTParm Module Window

RTParm is a tool for tabulating and plotting thermodynamic data against temperature T and/or pressure P. The input is taken from a DComp or ReacDC record specified in this RTParm record key. Upon calculation, the results will be tabulated in yF table on the 'Tables' page, as specified in 'tExpr' math script. Results can be plotted, exported into text files, or copy-pasted to other programs. Simple 'tExpr' scripts that extract calculated results into the yF table can be edited on "Scripts" page.


3.1. RTParm Window 'Scripts' Page

This page provides access to comment lines that identify this RTParm calculation; the pressure and temperature iterators (usually set in the RTParm wizard); and the tExpr text editor field, where the RTParm script to sample calculated data can be modified or enhanced.


3.2. RTParm Window 'Tables' Page

This page contains a composite table made of xT, xP and yF data objects. After RTParm calculation, the table will contain T,P values and thermodynamic properties, arranged in a row per T,P pair. The tXName and tLnam fields contain table headers (also shown on the plots), these can be edited at any time. The contents of yF table can be plotted against the selected abscissa (xT or xP) upon clicking on the "Plot Data..." toolbar button. These contents can also be selected, copied to clipboard, or pasted from the clipboard using the right-click menu.


3.3. RTParm Window 'TPwork' Page

This page gives access to contents of the TPwork data structure where the thermodynamic properties calculation results are placed at every step (i.e. each T,P pair) and from where they are picked up by the tExpr script. Not all fields are used when calculations are done for the DComp record, then data fields for reaction properties (tdSr, tdGr, twK, tlogK, tdHr, tdCpr, tdVr) are not updated. The uncertainty properties (tdevS, ...) are also not updated in this version of GEM-Selektor. 


3.4. RTParm Window 'Setup' Page

This page shows the summary of RTParm record setup. Optionally, it also provides access to xTE, xPE, ytE arrays with empirical (experimental) thermodynamic properties that can be plotted over the calculated results from yF table. The yTE values can also be calculated (rescaled) if a valid script is entered in tExprE text editor field.


4. RTParm module operation 

Several RTParm records can be created for one DComp or one ReacDC record. The RTParm function, taken together with flexibility of DComp and ReacDC definitions implemented in the GEMS code, provides a substantial help for matching thermodynamic data and extending the project thermodynamic database. Since the same DCthermo() and RCthermo() routines are used in both RTParm and System Project calculations, the tests done by RTParm will make you confident that nothing unexpected can happen with thermodynamic data during calculation of equilibria at non-ambient temperatures and pressures.


4.1. Setting up the RTParm iterators

On the "Scripts" page of RTParm window, you can see two iterators (three-cell columns) rpPi and rpTi. There you can specify minimum, maximum and increment of pressure and temperature, if different from default values (0 to 375 oC in 15 oC increments at PSAT). The rpMode parameter defines in which order the P  and T  values will be iterated, and the rpNpT fields define how many pressure and temperature points will be generated, together with the total number of T,P  points that depends on rpMode value. The numbers of  P  and T  points must be consistent with the respective iterator values.


4.2. Writing RTparm data sampling math scritps

The data calculated at each P,T  point are put by the program into data fields visible in the "TPwork" page of the RTParm window. A math script (tExpr field on the "Scripts" page) is needed to copy (or re-calculate) the necessary values and store them into xT, xP  and   yF  arrays (visible on the "Tables" page). The template scripts can be found in the SDref database (e.g.  "mscript:0001:rtparm:" record key) after you have pressed the F3 hot key. Such a script must use an internal  jTP index variable - the current index of the processed P,T  point; the program will run the script automatically, iterating it through the whole sequence of T,P  points after the "Record" "Calculate" menu command is executed or the "Calculate" toolbar button is pressed.


4.3. Running RTParm calculations

Calculations are performed  with "Record" "Calculate" command or "Calculate" toolbar button. It is advisable to save the RTParm record to database when the program will ask for that. The results will be collected in xT, xP and yF arrays visible in Tables window.


4.4. Plotting and exporting results

When calculations are finished, the results can be displayed graphically upon execution of the "Record" "Plot" menu command (or toolbar button) and can be customized in the same way as in the Process or GtDemo record.

The calculated data can also be printed into a text file using a "Record" "Print"command using a default ("pscript:0000:rtparm") or used-defined (as SDref record beginning with "pscript:...") printing script.  The user can also copy-paste the data into another program using the right-click menu.


4.5. Special cases of  RTParm calculations

If you have a few experimental (or otherwise derived empirical) data points, you can plot them over the calculated curve(s). To do it, first remake this RTParm record; in the RTParm wizard, select number of experimental points and number of parameters per points. Then switch to the Setup page of the RTParm window. There you will see new data objects xTE, xPE (abscissas) and yTE (ordinate(s)). Enter there your experimental data and plot the results again, as described above. You can customize the xTE-yTE plot by setting its line size 0 and symbol size 8 or 10, to represent experimental data as large scattered symbols. 

Now, you can perform simple trial-and-error fitting of temperature (or pressure) correction parameters against experimental data by changing them in the "parent" DComp (or ReacDC) record and then repeating the RTParm calculation and plotting.


5. Description of Data Entries

RTname - Comment line for the full name of this RTParm data sampler definition
   
This string (up to 70 characters) should be entered by the user for identification of this RTParm record, best during creation or remake. It will appear on the plot and in the text printouts. 
   
 
RTcom - Comment line for optional enhancement of the RTname string
  
This string (up to 70 characters) can also be entered to enhance identification of the RTParm sampler.
 
    
rtSDrf - List of SDref record keys referencing literature, as well as template math and print scripts.
  
This list can contain one or more SDref record keys (in packed format) referencing to bibliography or your comments to this RTParm sampler. Allocated if the  RpSize[0] counter is set to a positive number upon remake or create.  
 
   
rtSDvl - User's comments to SDref record keys in the rtSDrf list.
 
This list of comments is allocated if the RpSize[0] counter is set to a positive number of lines in the rtSDrf list.
   
  
rpPi[0] - Pressure P, bar: start value of the pressure iterator
 
This iterator defines how the current pressure (twP cell) will be changed in RTParm calculations. This cell contains start pressure (0 or positive real number in bars). 
 
If you wish to keep the pressure P constant then set  rpPi[0] = P; rpPi[1] = P; rpPi[2] = 0.
 
At the beginning of RTParm calculation, twP is automatically set equal to rpPi[0].
   
 
rpPi[1]  - Pressure P, bar: end value of the iterator
    
This real non-negative value defines the end point of RTParm calculation, which continues as long as twP <= rpPi[1] and jTP < rpNpT[2].   
    
    
rpPi[2] - Pressure P, bar: step value of the iterator
  
This real value defines how pressure will be (linearly) incremented in the process simulation. Default is 0 (constant pressure); otherwise the program will change the twP cell according to a given non-zero value. 

If  rpPi[1] < rpPi[0] then rpPi[2] must be negative (decreasing pressure in the process simulation). Note that negative values of current pressure twP are not allowed and will cause error messages.

This increment value must be zero in order to keep current pressure twP constant. 
 
 
twP - Current pressure of interest P (bar).
 
Current pressure P in RTParm calculation. Automatic Initialization: twP =: rpPi[0]. Can be incremented explicitly in the tExpr math script expressions; if this is not done not and rpPi[2] <> 0 then twP will be automatically incremented as  twP = rpPi[0] + jTP*rpPi[2] .
 
    
rpTi[0] - Temperature TC, centigrade: start value of the temperature iterator
  
This iterator defines how the current temperature twTC (twTK) will be changed in RTParm calculations. This cell contains the start (real non-negative) value of temperature. 

If you wish to keep temperature TC constant then set rpTi[0] =: TC; rpTi[1] =: TC; rpTi[2] =: 0.
    
At the beginning of RTParm calculation, twTC is automatically set equal to rpTi[0] and twTK is set to twTC + 273.15. 
 
   
rpTi[1] - Temperature TC, end value of the iterator
  
This real non-negative value defines the end point of RTParm calculation, which continues as long as twTC <= rpTi[1] and jTP < rpNpT[2].   
 
   
rpTi[2] - Temperature TC, step value of the iterator
   
This real value defines how temperature will be (linearly) incremented in the process simulation. Default is 0 (constant temperature); otherwise the program will change the twTC and twTK cells according to a given non-zero value. 

If  rpTi[1] < rpTi[0] then rpTi[2] must be negative (decreasing temperature in the process simulation). Note that negative values of current temperature twTC are not allowed and will cause error messages.

This increment value must be zero in order to keep current temperature twTC (twTK) constant. 
    
 
twTC - TC - current temperature, centigrade
   
Current pressure TC in RTParm calculation. Automatic initialization: twTC =: rpTi[0]. Can be incremented explicitly in the tExpr math script expressions; if this is not done not and rpTi[2] <> 0 then twTC will be automatically incremented as  twTC = rpTi[0] + jTP*rpTi[2] .
 
   
twTK - T - current temperature, Kelvins
  
This work cell is automatically calculated as  twTK =: twTC + 273.15  at each RTParm calculation step in order to facilitate writing of tExpr  math scripts.
  
  
tXName  - Abscissa name (to be shown on the graph legend)
 
 Here, the user should type a short name (up to 8 characters) for the abscissa axis that will be shown on the graph. Typical example: "T,C; P,bar".
   
 
tYName - Ordinate name (to be shown on the graph legend)
  
  Here, the user should type a short name (up to 8 characters) for the ordinate axis to be shown on the graph. Typical examples: "kJ/mol", "cal/K/mol", "log K". Ideally, the ordinate name should define units of measurement because names of the plot lines can be given separately in the tLnam list. 
   
 
tLnam -  Names of the yF columns (and plot lines)
  
Here, the user should enter names of columns (plot lines) corresponding to columns in the yF (sampled data) table. The names must not exceed 6 characters. They will be displayed on the graph legend together with coloured symbols and plot lines. From there, you can drag-and-drop the names onto the graph field (see also the "Plotting and exporting results" chapter). Number of elements in this list is defined in the rDimXY[1] counter. 
  

tLnamE -  Names of the ytE columns for empirical data
  
Enter here the name(s) of column(s) in the ytE array of empirical data. Similar to names in the tLnam list, these will also be displayed on the graph legend. Number of elements in this list is defined in the rDimEf[1] counter. 
 
  
tExpr - RTParm data sampling math script
 
Math script text to sample data to yF (optionally xT, xP) arrays from the data objects (twTK, twG etc.) shown on the TPwork page of the RTparm window. 
 
The scripts must use an internal index jTP for the indexation of  yF, xT,xP  elements. The jTP index is automatically incremented by the program at each T,P  point during RTParm calculations.
  
To facilitate writing of math script equations, a template math script library is provided (see "Writing Process Simulation Math Scripts" chapter).
 
The minimum possible set of  tExpr  math script expressions for data sampling has the following form:
yF[jTP][0] =: <expression>;
. . .
yF[jTP][n] =: <expression>;
here,  n stands for the last valid index of columns in the data sampling array yF;  <expression> must contain some labels of data objects visible on the TPwork or Tables pages of the RTParm window. Arbitrary expressions for intermediate calculations can also be specified. 
   
 
tExprE - RTParm math script for empirical data (optional)
 
Math script text to calculate data to ytE (optionally xTE, xPE) empirical data arrays. The scripts can use internal indices  tiE, tjE  for indexation of  ytE, xTE, xPE elements.
  
 
xT - T vector-column of temperature, usually an abscissa on the plots
  
This vector of temperatures is generated or entered by the user depending on the value of rpMode parameter.
  
  
xP - P vector-column of pressures, optionally an abscissa on the plots
  
This vector of pressures is generated or entered by the user depending on the value of rpMode parameter.
  
  
yF -  RTParm table of sampled data 
 
Table of thermodynamic properties calculated according to T,P from xT and xP vectors, and sampled by the math script (tExpr data field). Number of columns must be specified by the user in the rDimXY[1] cell.
  
  
  xTE - T vector (temperatures), abscissa on the plots for empirical data (optional)
  
This vector of temperatures for empirical data to display on the plot is usually entered by the user
  
  
xPE - P vector (pressures), abscissa on the plots for empirical data (optional)
  
This vector of pressures for empirical data is usually entered by the user.
  
  
ytE - RTParm input table of empirical data (optional) 
   
Table of empirical (independently known) thermodynamic properties to plot together with calculated properties. Usually entered by the user. Numbers of rows and columns must be specified in the rDimEf[0] and rDimEf[1] cells.
  
 
rtSize - Setup values for plots and graph axii.
  
 Internal object, values are automatically modified when the graph dialog legend is customized by the user.

This one-character code defines where the program can look for the input standard molar properties and T,P correction parameters. The program automatically collects thermodynamic data from the database for a species marked with "d" (DComp) or "r" (ReacDC) codes.

rDimXY[0]- Total number of T,P points (i.e. rows in the  xT, xP,yF  arrays).
 
This number needs not to be specified on remake - it will be copied automatically from rpNpT[2]. 
  
  
rDimXY[1] - Input number of columns in the yF table (plot lines) - must be entered by the user!
  
For plotting, 1 or 2 columns are best to specify. For instance, S0 and Cp0 values can be shown on the same plot because they are expressed in the same scale (likewise, G0 and H0). logK (for ReacDC records) is usually shown as a single line on the plot.

Must be entered if the empirical data will be plotted (rpUn[5] flag is switched on)

rpNpT[0] - Input number of tabulation points along pressure P
  
This number must be set by the user in agreement with the rpMode parameter.
  
 
rpNpT[1] - Input number of tabulation points along temperature T
 
This number must be set by the user in accord with the rpMode parameter.
  

rpNpT[2] - Total number of temperature-pressure T,P tabulation points
 
There is no need to enter this number - it will be automatically calculated from rpNpT[0] and rpNpT[1] values  according to the rpMode parameter.
  
 
rpMode - Input mode of indexation of T,P tabulation points.
  
This parameter defines the mode of indexation of temperatures and pressures during the RTParm calculation. In modes 1,2 and 3, pairs of pressures and temperatures are automatically generated from intervals and increments in rpTi and rpPi  iterator fields and placed in xT and xP vectors.
The mode 0 permits to enter manually any desired pairs of T and P into xT and xP vectors, respectively.
 
Possible codes:  
0 - the user must enter all P and T values in xP and xT vectors. rpNpT[2] = rpNpT[1] = rpNpT[0].
1- increments in cycle on P nested into cycle on T. rpNpT[2] = rpNpT[1] * rpNpT[0].
2- increments in cycle on T nested into cycle on P (default). rpNpT[2] = rpNpT[1] * rpNpT[0]. 
3- increment of T and P in one cycle. rpNpT[2] = rpNpT[1] = rpNpT[0]. 
 

5.2. Data Objects on TPwork page

These data objects are mainly work cells internally used by GEM-Selektor to keep thermodynamic data calculated in DCthermo() or RCthermo() subroutines at P,T of interest for a given Dependent Component (described either in DComp or in ReacDC record). Almost all data from this TPwork structure can (and should) be used in the tExpr data sampling math script.

DRkey  - DComp or ReacDC key of record with source data (obsolete?)
 
Key of the currently processed DComp (or ReacDC) record (may become obsolete in the next versions).
  

twUn[0] - Source data origin { d r i n f }

Upon calculation of RTParm, inconsistency between the P and T iterators, tabulation mode, and dimensions of data tables has been found.

Action: Remake the record, check settings in the RTParm wizard, and try calculation again.  

 
E08RTrem: Invalid mode of calculation - bailing out...

Confirmation that the user has pressed "No" or "cancel in responce to warning W07RTrem.

  Check the script, fix misprints and try calculation again.

Check the script, fix it and try calculation again. If no calculation of experimental data are necessary, make sure that the Empty value "---" is present as the text of the script.
