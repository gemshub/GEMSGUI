1. Introduction 

This chain of GEMS database records contains (optional) definitions of  Predefined Composition Objects (PCO) such as rocks, ores, waters, fluids etc. All PCO input data are automatically converted into a single stoichiometry vector containing number of moles of all involved independent components (IComp).

PCO are accessible through the list of  Compos record keys in the "System" "Compos" window, very convenient for setting up the bulk elemental composition of a complex chemical system. Since each PCO is referred to as a single entity, you can specify any desired number of grams, moles etc. of it to be added to the bulk chemical composition vector  b  - the main input data in GEM calculation of equilibria in chemical systems.. If complex ingredients are involved, such as rocks or sea-salt, this possibility may save you a lot of  time.

Addressing Compos names in the Process Simulator math scripts will also permit you to simulate various mixing processes in batch calculations in a quite simple and efficient way, as well as to define the "process extent" models with changing rock/water or rock/fluid ratios, evaporation, weathering etc.

Each PCO has a unique name (identifier) in the first field of the record key and, regardless of the particular specifications, contains numbers of moles of independent components. Upon creation of Compos records, the GEMS code offers several flexible and efficient methods for specification of GCO compositions via molar, mass, volume fractions, molality or molarity units, and absolute masses or number of moles of IC, DC or user-defined stoichiometries. Virtually any published composition can be represented close to its original appearance by creating a Compos record.

As usual in GEMS, the easiest way to create a new PCO (in addition to some provided in the Default database) is to "clone" existing Compos records with "Record" "Create" or "Record" "Save As" menu commands, then remake and re-calculate the new records whenever appropriate. The new Compos record keys will be automatically added to the System Profile when you open it next time.


1.1. Compos Module Calculations

Let us consider an example of creating a new Compos record for the stoichiometry of kaolinite. First, press "Show" button to get the  "Kaolinite:MIN:Al2Si2O5(OH)4:" record. You will see  a very simple screen where composition of kaolinite is expressed in moles of chemical elements in the  CIvector. The  MasVol[0]  cell shows kaolinite molar mass in kg, and the  MasVol[6] cell displays the total number of moles of elements per mole of kaolinite.

Now, we will try to express the same composition in mole percents of petrogenic oxides Al2O3, SiO2 and H2O. Execute the "Record" "Create" command; in the appearing record key dialog, modify the previous record key as follows:

Kaolinite1
MIN
Moles_of_oxides
and press OK. Next, in the "Remake" dialog, press the "Remake" button. In the appearing "Compos: Remake parameters" dialog, turn the third switch on ('+'). In the numeric fields to the right, put  3 (the number of required formulae for petrogenic oxides) into the third one, and press the "OK" button. In the IComp selection list which appears next, mark the chemical elements present in formula units (in this example, Al, H, O and Si), and press the "Ok" button.

Now, the "Compos" screen form appears, with an empty  list of user-defined formulae "formU". Type "SiO2" into the first line; "Al2O3" into the second, and "H2O" into the third line. Now,  specify composition of kaolinite in the  CA  vector (to the right) by entering 2 against "SiO2" formula, 1 against "Al2O3", and 2 against "H2O". Check that in the CI vector (to the left), only zeros are present (it is also possible to have PCOs defined using both CA and CI vectors, but we do not use this possibility in this example). When you press the "Calculate" icon or menu item, these numbers will be interpreted depending on the units of concentration coded in the  AUcl  vector (in between), then converted into numbers of moles and added to the bulk PCO composition vector  PCO. By default,  the 'M' codes (mole amounts) appear in the  AUcl column.

Note that, during calculation of Compos data, the  MasVol[0] and  MasVol[6]  cells are used for normalizing the specified composition to certain mass or total number of IC moles, respectively.  MasVol[0] contains total mass of this predefined composition object in kilograms.  If 0 or EMPTY value  '---'  were specified by the user then the mass fractions cannot be used in the input data, and the PCO mass will be automatically calculated from the provided moles or molar (molal) concentrations. A positive value should be entered in the  MasVol[0] cell if at least one value in either of the  CI, CD  or  CA  vectors is specified as mass percent  '%'  in the CIcl, CDcl or  AUcl  columns.

Back to the kaolinite example, type zero into MasVol[6] and 0.1 into MasVol[0] cells.  Now, press the "Calculate"  icon (or menu command) to calculate 100 g of kaolinite stoichiometry (you will see new numbers in the leftmost PCO  vector on Page 1), and "Save" button to save the new record to the database file in your profile directory. Do not forget to delete this exercise record before you open your modelling project next time.

It is good to know that each element of the  CI, CD  or  CA   (and their equivalents in the System window dialogs) can be set up separately with its own  units of concentration code. This  gives the user an unmatched freedom in creating bulk compositions or modifying them. When specifying different units, one should take care of their "base quantities". For example, when setting molality units, the mass of water-solvent must always be set in beforehand.

Read carefully all the tooltips and help pages to the  MasVol  cells for instructions how to set base quantities and units of concentration correctly. Examination of the available Compos records and some experimentation will give you the right feeling how  to work with PCO records. Negative numbers can be used in the  CI,  CD  or  CA  vectors, as long as they do not produce negative numbers of moles in the total  PCO  composition vector.

In a way similar to that described  above,  you can create another PCO definition for kaolinite composition, but using formulae from DComp records for corundum, quartz and water instead of making formula units (an exercise).

4. Compos Window Data Entries

 
BCname - Input full name of this Predefined Composition Object.
 
This comment field may contain a string up to 70 characters long. You may enter this string to identify the PCO definition on the Compos  screen form. Not used in any calculations.
 
 
BCnote - Input comment to BCname (optional).
 
This is an optional comment to the PCO name (max 70 characters). Not used in any calculations.
 


Mass (kg) and volume (L) data for this PCO:

MasVol[0]  - Ms  (input normalization mass, output total PCO mass).
 
This field contains total mass of this PCO in kilograms. There are two modes of using this data field: the output and the input mode.
 
(1) Output mode. If 0 or EMPTY value is entered by the user before calculation of the PCO vector, then the resulting Ms  value (PCO total mass) will be automatically placed into this cell, calculated from IC moles in the PCO  vector and their molar masses collected from the respective IComp records. In this case, mass fraction codes cannot be used in any of the columns CIcl,CDcl  and  AUcl.
 
(2) Input mode. If any positive value Ms  is entered into this cell then the PCO composition vector will be normalized to this Ms  value during calculation of Compos record data.
 
Normalization occurs as follows: (1) PCO vector elements (moles) are calculated as specified in other data fields of the Compos window; (2) the total mass Mp  of  the PCO is computed by multiplying each of the PCO vector elements to molar mass of the respective IC; (3) Each PCO vector element is multipled by a normalization factor Ms/Mp.
 
If any positive value Ms  is entered then, in addition, some values in CI, CD or  CA vectors can be specified in mass percent ('%' in classifiers  CIcl, CDcl  and  AUcl) or other mass fraction concentration units.  In the process of PCO calculation, these mass concentrations will be scaled into numbers of moles of IC using stoichiometries (where needed) and the provided Ms  value.
 
 
MasVol[1]  - Vs  (input normalisation volume,  for specifying volume concentrations)
 
If any positive Vs  value (in L) is entered then it will be used in PCO calculations to scale all values in CI, CD or  CA vectors that were specified as volume fractions ('V' code in classifiers CIcl, CDcl  and  AUcl) into moles of  independent components..
 
The Vs  value is not used if volume concentrations are not specified anywhere in this PCO definition. Default value of  Vs  is 0. If  Vs  = 0 or EMPTY then volume concentrations must not be specified.
 
 
MasVol[2]  - Mw   (input mass of water-solvent, for specifying molalities)
 
Positive value of water-solvent mass Mw   (in kg)  must be entered, if at least one of the elements of  CI, CD or  CA vectors is given in molal units (codes 'm', 'i', 'p' in classifiers  CIcl, CDcl  and  AUcl). Upon the PCO vector calculation, molal concentrations will be automatically converted into moles of IC using stoichiometries (where needed) and the provided Mw  value.
 
The Mw  value is not used if molality units are not specified anywhere in this PCO definition. Default value of  Ms   is 0. If  Ms  = 0 or EMPTY then molal concentrations must not be specified.
 
 
 MasVol[3]  - Maq  (input mass of aqueous solution for dissolved mass concentrations)
 
Non-empty positive value of mass of aqueous solution Maq   (in kg) must be entered, if at least one element of  CI, CD or  CA  vectors will be set as concentration per mass of aqueous phase (codes 'C',' c', 'a'  in CIcl, CDcl  and  AUcl  classifiers). During the PCO calculation, these  mass concentrations will be automatically converted into IC moles using  Maq  value and  chemical formulae where necessary.
 
Maq  is not used by the program if aqueous mass concentrations are not specified.
 
 
MasVol[4]  - Vaq  (input volume of aqueous solution for molarities)

Non-empty positive value of volume of aqueous solution Vaq  (in L) must be entered into this field, if at least one element of  CI, CD  or CA  vectors is supposed to be given in molarity units (codes 'L',' j', 'q'  in either of  CIcl, CDcl  and  AUcl  classifiers). Molarities will be automatically converted into IC moles using Vaq   and chemical formulae.

Vaq  is not used by the program if molarity concentrations are not specified.
 

 
MasVol[5]  - Pg  (input pressure in gas, for partial pressures)
 
This field is reserved for future use.
 
 
MasVol[6]  - Ns   (input total number of IC moles in this PCO)

A non-zero positive Ns   value (moles) must be entered, if at least one element of vectors  CI,CD or  CA  will be specified in mole-fraction units (codes 'n', 'N', 'f'  in classifiers  CIcl, CDcl  and  AUcl). These mole fractions will be automatically converted into IC moles using Ns  value and then added to the PCO vector.

If no mole fractions are specified and non-zero positive Ns   value is given then upon calculation, the PCO vector will be normalized such that sum of all its elements equals  Ns.

If no mole fractions are specified and this cell contains 0 or EMPTY value then, after PCO calculation, it will be replaced by the sum of all elements of the PCO  vector.
 
 
MasVol[7]  - Mf  (input mass fraction (%) of a compound specified by  Ctext  relative to the whole PCO mass)
 
This field must contain 0 if   Ctext is not specified.
 
If   Ctext is specified and Ms  = 0 then, during PCO calculation,  Mf  is interpreted as number of moles of the compound with stoichiometry defined in   Ctext formula. In this case,  Ms  = 0 is considered as Ms  = 1 (mol).
 
If   Ctext is specified and Ms   > 0 then Mf   is interpreted as a mass fraction of Ms   assigned to   Ctext.  Negative Mf  values are allowed as long as they do not produce negative numbers of moles in the  PCO  vector.
 
 
PCO - Output vector of bulk numbers of moles of IC in this Predefined Composition Object.
 
This dynamic vector must always present in Compos record. After the PCO calculation, it holds a bulk composition of this predefined composition object, expressed in positive numbers of moles of Independent Components. All elements with zero- or negative numbers of moles will be eliminated from the PCO vector.
 
The PCO vector is calculated by adding all quantities converted from numbers specified in CI, CD and CA  vectors and MasVol numbers  (scaled according to codes in CIcl,CDcl  and  AUcl  classifiers).
 
Elements of the PCO vector ca be regarded as stoichiometry coefficients of the "chemical formula" of this object.
 
   
unPCO - Input uncertainty of   PCO  elements in moles
 
(optional, for use in future versions).
 
 
Ctext - Text-formula for incrementing the PCO vector.
 
 
symIC - List of IC symbols corresponding to elements of  PCO  vector
 
This list of IC symbols is created automatically when resulting bulk PCO is compressed. The list actually refers to IComp record keys.
 
This list is used internally when System Profile lists are generated and bulk composition of the system is computed in SysEq records.
  
 
formU - List of arbitrary formula units
 
This list of user-defined formulae permits a convenient specification of PCO composition using wet chemistry analyses or other kinds of regular geochemical data.
 
Examples uf user-defined formulae: SiO2, Al2O3, and any correct chemical formulae up to 23 character long, typed by user in lines of the formU list. No empty lines are allowed!
 
The list formU is created as the stage of creation/reorganization of Compos  record, when user has typed nAU > 0 at the remake prompt. Then formulae can be entered or edited directly in Compos  screen form.
 
More about writing chemical formulae: see ... .

 
symDC - List of DC record keys
 
This list of DC record keys is created automatically after the user has marked DComp and/or ReacDC keys at remake/create of Compos record.
  
 
CIcl - Units of IC quantity/concentration
 
This classifier list contains units of measurements for IC named in the list symIC and quantities/ concentrations specified in CI vector. The user may set up different codes for different IC, according to certain rules.
 
Default code:  M (moles).
 
 
CDcl - Units of DC quantity/concentration
 
This list codes units of measurements for DC stoichiometries named in the list symDC and quantities/concentrations specified in CD vector. The user may set up different codes for different DC, according to certain rules.
Default code:  M (moles).
 
 
AUcl - Units of formulae quantity/concentration
 
This list codes units of measurements for user-defined formulae (UDF), listed in the formU object, with quantities/concentrations specified in CA vector. The User may set up different codes for different UDF, according to certain rules.
Default code:  M (moles).
 
 
CmDCS  - Source of DC data: d-DComp  r-ReacDC { d r }
 
Possible values:
d     look for DComp  record referred to with the line in symDC list;
r      look for ReacDC record.
 
This classifier is created automatically on remake.
 
 
CI - Quantity/concentration of IC C(IC)
 
This optional dynamic vector contains quantities or concentrations of IC used for specification of the GCO. Corresponding IC symbols are listed in symIC vector.
 
Each element of CI vector is scaled according to the code in CIcl classifier and then added to respective element of Cm vector. Zero and negative values can also be specified as long as they do not produce negative numbers in the Cm vector.
 
 
CD - Quantity/concentration of DC C(DC)
 
This optional dynamic vector contains quantities/concentrations of DC formulae used in definition of GCO. The DC formulae are automatically extracted by the program from DCOMP of REACDC records referred to by leys in symDC list.
 
During calculation of the bulk Cm vector, each element of CD vector is scaled as prescribed by the class code in CDcl classifier. Then the DC formula is used in conversion to numbers IC moles finally added to the elements of Cm vector.
 
 
CA - Quantity/concentration of user-defined formula units  C(AC)
 
This optional dynamic vector contains quantities or concentrations of the formulae units (UDF) which user has defined in the formU list.
 
The vector is created together with formU and AUcl objects at re-organization of the COMPOS record. CA vector can contain zeros, positive and negative numbers.
 
When composition of PCO is computed, the UDF are scanned sequentially and corresponding element of the CA vector is normalized as prescribed by the code in AUcl classifier, then converted into (positive or negative) number of IC moles and added to Cm vector of the bulk PCO composition.
 
 
bSDref - List of SDref keys to data sources
  
  
bSDval - List of comments to data sources
 
 

4.1. Data entries accessible for editing through Create/Remake wizard only

Setup Flags (BC_set)

BC_set[0] - Use quantities of Independent Components in this PCO definition?
 
Possible values:
+    keep  CI and CIcl vectors (if present) but do not fill with default values;
*    create/keep  CI and CIcl vectors and fill them out with the default values (0 and 'M', respectively);
-     delete  CI and CIcl  vectors (previously contained data will be lost).
 
If  CI vector is present then the non-zero quantities defined in this vector will be converted to numbers of moles according to codes given in the CIcl vector, and added to the bulk  PCO  composition vector. In this process, the molar masses if independent components will be taken from the IComp records referred to in the  symIC  list of record keys selected by the user during remake of this Compos definition.
 
 
BC_set[1] - Use stoichiometries of Dependent Components in this PCO definition?
 
Possible values:
+   keep CD , CDcl and  symDC vectors (if present) but do not fill them with default values;
*   create/keep CD , CDcl and  symDC  vectors and fill them out with default values (0, 'M', and  EMPTY, respectively);
-   delete CD, CDcl  and  symDC vectors (previously contained data will be lost).
 
If CD vector is present then the non-zero quantities defined in this vector  will be converted to numbers of moles according to the codes given in the CDcl vector, and added to the bulk PCO composition vector. In this process, the chemical formulae of compounds will be taken from the respective  DComp  or  ReacDC  records, referred to in the symDC  list of record keys selected by the user during remake of this Compos record.
 
 
BC_set[2] - Create user-defined formulae (UDF)  in this PCO definition?

 
Possible values:
+   keep CA, AUcl and formU vectors (if present) but do not fill them with default values;
*   create/keep CA, AUcl and formU vectors and fill them out with default values (0, 'M', and  EMPTY, respectively);
-   delete CA, AUcl and formU vectors (previously contained data will be lost).
If CA vector is present then nonzero quantities/concentrations defined in this vector  will be converted to numbers of moles according to the codes given in the AUcl vector, and added to bulk PCO composition vector, by analyzing the user-defined chemical formulae from the  formU  list.
 
 
BC_set[3] - Create a large formula text for this PCO definition?
 
Possible values:
+   or  *     keep/create formula text field ;
-    delete  the formula text field.
 
PCO definition can be enhanced using a large chemical formula which you can type later into Ctext  field, created on remake of this Compos record.
 
If the non-EMPTY formula text is present and it is written according to GEMS chemical formula syntax, then this formula is parsed, converted to the number of moles of IC depending on the scaling factor  MasVol[7], and added to the bulk PCO composition vector.
 
 
BC_set[4] - Create the  unPCO vector of uncertainties for this PCO definition?
 
Possible values:
+    keep the unPCO vector but do not fill it with default values;
*    create/keep the unPCO vector and fill it with default values (EMPTY);
-    delete the unPCO  vector.
 
If this vector is present, the user can enter there the absolute uncertainties of the respective PCO  vector elements in moles.
 
For future use in the uncertainty propagation calculations in GEM-Selektor v.3.x-PSI.
 
 
BC_set[5]
Reserved for future use.
 
 

Setup Dimensions  to specify sizes of optional vectors and lists

 
BC_dim[0] - Number of Independent Components used in defining this PCO.
 
Automatically changed by the program to set dimensions of PCO, CIcl and  CI  vectors when the Compos record is created "from scratch" (in "Clear All" mode). The user may be asked to set this number upon remake if only CI vector will be used for defining this PCO. Do not edit this field if not asked to do it!
 
 
BC_dim[1] - Total number of Dependent Component (DComp+ReacdDC) stoichiometies to define this PCO.
 
Automatically calculated by the program when the user selects DComp or ReacDC record keys during remake stage. Defines dimensions of the symDC, CDcl and CD vectors. For information only.
 
 
BC_dim[2] - Number of formula units created in this PCO definition.
 
This number is entered by the user the on remake of the Compos record. Defines dimensionality of the formU, AUcl  and  CA  vectors. For information only.
 
 
BC_dim[3] - Number of references to data sources.
 
This number is entered by the user on remake of the Compos record. Defines dimensionality of  bSDref  and  bSDval  objects. For information only.
 
 
BC_dim[4] - Total number of IC records currently available in the IComp database chain.
 
For information only.
 
 
BC_dim[5] - Final number of Independent Components used in PCO definition.
 
Internally calculated by the program. Defines final dimensions of the PCO, CIcl and  CI  vectors. For information only.
 
 
bsICm - Complete work list of IC record keys currently available in the IComp chain.
 
For information only.
 
 
ICmm - Work vector of atomic (molar) masses of independent components.
 
For information only.
 
 
BCtprn
Internal object for printing script.
 Reserved for debugging purposes.
