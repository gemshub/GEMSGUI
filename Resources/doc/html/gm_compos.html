1. Introduction 

This chain of GEMS database records contains definitions of  Predefined Composition Objects (PCO) such as rocks, ores, waters, fluids etc. Compos records are very versatile in the choice of forms in which the compositions can be entered. Upon calculation of Compos record, all input data will be automatically converted into a single PCO vector containing number of moles of all involved Independent Components (IComp).

PCOs are accessible through the list of  Compos record keys in the "System" "Compos" window, very convenient for setting up the bulk elemental composition of a complex chemical system. Since each PCO is referred to as a single entity, the user can specify any desired number of grams, moles etc. of it in the system recipe, to be added to the bulk chemical composition vector  n(b)  - the main input data in GEM calculation of equilibria in chemical systems. If complex ingredients are involved, such as rocks or sea-salt, this functionality may save users a lot of  time.

The default database of GEM-Selektor contains many simple Compos records that will be automatically copied into a newly-created are in most cases sufficient for assembling complex recipes of geochemical systems. Any missing PCO can be created by the user in project database after switching to the Database mode of GEM-Selektor.

Addressing Compos names in the Process Simulator scripts will also let the user to simulate various mixing processes in batch calculations in a quite simple and efficient way, as well as to define the "process extent" reaction-path models with changing rock/water or rock/fluid ratios, evaporation, weathering etc.

Each PCO has a unique name (identifier) in the first field of the record key and, regardless of the particular specification, contains numbers of moles of Independent Components. Upon creation of Compos records, the GEM-Selektor code offers several flexible and efficient methods for specification of GCO compositions via molar, mass, volume fractions, molality or molarity units, and absolute masses or number of moles of IC, DC or user-defined stoichiometries. Virtually any published composition can be represented close to its original appearance by creating a Compos record.

As usual in GEM-Selektor, the easiest way to create a new PCO (in addition to some provided in the Default database) is to "clone" existing Compos records with "Record" "Create" or "Record" "Save As" menu commands, then remake and re-calculate the new records whenever appropriate. The new Compos record keys will be automatically added to the System Profile when you open it next time.

The PCO composition is expressed in moles, and normalized to 1 kg total mass (or 49.8067 total moles). Now the record can be saved to project database; it will be added to the Compos list in the chemical system definition when the modeling project is open next time. One mole of this PCO means exactly one kilogram mass added to the system composition.

It is very easy to re-normalize the PCO to any given mass. For example, to re-normalize the haplogranite composition to 2 kg, enter 2 into the MasVol[0] cell; 0 into the MasVol[6] cell; and click the "Calculate" toolbar button. Note that, if at least one code of mass fraction is used somewhere in the Compos record then the MasVol[0] cell cannot contain 0.

In general, there are two modes of Compos calculations, depending on what was entered into the MasVol[0] data field.

(1) Output mode. If 0 or EMPTY value is entered by the user before calculation of the PCO vector, then the resulting Ms value (PCO total mass) will be automatically placed into this cell, calculated from IC mole amounts in the PCO vector and their molar masses collected from the respective IComp records. In this case, mass fraction codes cannot be used in any of the columns CIcl, CDcl  and  AUcl.

(2) Input mode. If any positive value Ms is entered into this cell then the PCO composition vector will be normalized to this Ms value during the calculation of Compos record data.

Normalization occurs as follows: (1) PCO vector elements (moles) are calculated, as specified in other data fields of the Compos window; (2) the total mass Mp  of  the PCO is computed by multiplying each of the PCO vector elements to molar mass of the respective IC; (3) Each PCO vector element is multiplied by a normalization factor Ms/Mp.

If any positive value Ms  is entered then, in addition, some values in CI, CD or  CA vectors can be specified in mass percent ('%' in classifiers  CIcl, CDcl  and  AUcl) or other mass fraction concentration units.  In the process of PCO calculation, these mass concentrations will be scaled into numbers of moles of IC using stoichiometries (where needed) and the provided Ms  value.

It is good to know that each element of the  CI, CD  or  CA  vectors can be set up separately with its own  units of concentration code. This  gives the user an unmatched freedom in creating bulk compositions or modifying them. When specifying different units, one should take care of their "base quantities". For example, when setting molality units, the mass of water-solvent must always be set in beforehand.

Read carefully all the tooltips and help pages to the  MasVol  cells for instructions how to set base quantities and units of concentration correctly. Examination of the available Compos records and some experimentation will give you the right feeling how to work with PCOs. Negative numbers can be used in the  CI,  CD  or  CA  vectors, as long as they do not produce negative numbers of moles in the total  PCO  composition vector.

Coming back to the haplogranite example, there are many alternative ways to express the same PCO. For instance, the record can be re-configured such that 1 kg of large formula text is entered into the Ctext field:

(SiO2)10.998(Al2O3)2.0829(K2O)0.5834(Na2O)1.00002(H2O)0.54959

Alternatively, the PCO vector can be simply copy-pasted into the CI vector (always created automatically, even if not ordered in Compos Wizard). For fluid PCOs, it may be convenient to compose them using formulae of aqueous ions, complexes, or dissolved minerals from DComp/ReacDC records (this gives an additional opportunity to check the charge balance).

2.1. Compos Wizard Step 1 - Predefined Composition Object (PCO) configuration

Compos record contains data for a Predefined Composition Object (PCO), which describes a salt, the air, a mineral, a rock, a natural water, etc., treated  as a single named entity in the chemical system recipe.

PCO can be configured according to the source data: as IC or DC amounts/concentrations; as user-defined formula (UDF) units; or as a large user-defined formula entered into a text editor field. All possible methods of input can be used simultaneously. Independent Components that define this PCO must always be specified.

Upon re-calculation, all entered quantities will be added (after unit conversion) into a single IC mole amounts PCO vector, optionally scaled to a given total mass MasVol[0] (in kg) or total number of moles MasVol[6]. This vector is visible on the PCO page of Compos window.

This wizard page contains two control element groups.

Group 1: PCO input data configuration

Checkbox: Use amounts of Independent Components (IComp) in this PCO definition (default)? Allows entering quantities of Independent Components into  the CI column in units as specified in the CIcl column on the PCO page of Compos window. The ICs must be selected from the list appearing after completion of this wizard.

Checkbox: Use formulae of Dependent Components (from DComp/ReacDC records) in this PCO? If turned on, allows entering quantities of Dependent Components (formulae) into  the CD column in units as specified in the CDcl column on the Settings page of Compos window. The ICs must be selected from the list appearing after completion of this wizard.

Spinbox: Set number of user-defined formula units for this PCO definition (0 by default). If turned on, allows entering of UDF formulae into the formU list, the amounts of UDFs into the CA column in units as given in the AUcl column on the Settings page of Compos window.

Group 2: Optional: Input user-defined formula (UDF) text

Checkbox: Use a user-defined formula text input field for this PCO definition? If turned on, allows the user entering a large formula into the Ctext field on the PCO page of Compos window. The amount of this complex (e.g. rock-forming mineral) stoichiometry can be entered below together with units of measurement.

Spinbox: Select units of measurement for this UDF quantity  (default: M)

Editor field: Enter here the UDF quantity or amount in selected units (default: 1). This amount will appear in the MasVol[7] cell on the PCO page, where it can be edited later on.


2.2. Compos Wizard Step 2 - Additional settings and next actions

'Options' group:

Spinbox: Set here the number of links to SDref bibliography records (default 0). If set to 1 or more, this will allocate bSDref and bSDval lists on the Settings page, where the user can enter the key(s) of SDref containing source-of-data references and what is taken from which source.

Checkbox: Use a vector of data uncertainties in this PCO definition? (not used for equilibria calculations in this version of GEM-Selektor).

The wizard page also reminds briefly, what will happen after the user clicks "Finish".

(1) For a PCO definition using IComp amounts, a list of available IComp records will be shown, asking you to mark the desired ones.

(2) In a PCO definition using DC formulae, a list of available ReacDC/DComp records will be shown, asking you to mark the desired ones.

(3) Page 1 of the 'Compos' window appears. Fill out the BCname comment and (optionally) BCnote line. Then enter data and formulae wherever needed; check units of amount/concentration.

(4) Click on 'Calculate' toolbar button to compute the PCO vector. Check or enter the normalization values in MasVol[0] and MasVol[6] fields and calculate again, if needed. Setting both fields to zero disables the normalization of PCO to the total mass or total amount of moles, respectively.

(5) When done, save the Compos record to the project database. It will be taken into chemical system definition when the project is open next time.


3. Compos Module Window

This window consists of two pages: PCO and Settings. In the PCO page, the final PCO composition can be seen, as well as parts of it can be specified as quantities of Independent Components and of a large user-defined formula. In the Settings page, parts of PCO can be specified via quantities of user-defined formula units (e.g. petrogenic oxides) and optionally as quantities of formulae extracted from selected DComp/ReacDC records.


3.1. Compos Window 'PCO' Page

This page always shows the BCname and BCnote comment lines, it is recommended to type there what this PCO is for later reference.

Below, a MasVol row of 8 real numbers is visible, where the normalization factors can be specified for calculation of the PCO, and an optional amount of the user-defined formula can be entered.

The final PCO composition (used in calculations of bulk composition of chemical system) can be found in a table consisting of three columns: symIC, PCO, and (optional) unPCO.

Other data objects appear if allocated in the Compos Wizard. They form a table made of columns symIC, CIcl and CI, where quantities of ICs can be entered. 


3.2. Compos Window 'Settings' Page

This page always shows the BC_set object with configuration flags, and BC_dim row with dimensions of dynamic data arrays in this Compos record. Other data objects appear only when they were allocated in the Compos Wizard.

The table composed of formU, AUcl, and CA columns lets the user to enter formula units and their quantities, to be added to the PCO vector upon re-calculation.

The table composed of CmDCS, symDC, CDcl, and CD columns lets the user to enter quantities of DC formulae extracted from selected DComp or ReacDC records.


4. Compos Window Data Entries

 
BCname - Input full name of this Predefined Composition Object.

This comment field may contain a string up to 70 characters long. Please, enter this string to identify the PCO definition on the Compos  screen form. Not used in any calculations.

 
BCnote - Input comment to BCname (optional).

This is an optional comment to the PCO name (max 70 characters). Not used in any calculations.


Mass (in kg), volume (in dm3) and amount (in moles) data for this PCO:

MasVol[0]  - Ms  (input normalization mass, output total PCO mass).

This field contains total mass of this PCO in kilograms. There are two modes of using this data field: the output and the input mode.

(1) Output mode. If 0 or EMPTY value is entered by the user before calculation of the PCO vector, then the resulting Ms  value (PCO total mass) will be automatically placed into this cell, calculated from IC mole amounts in the PCO  vector and their molar masses collected from the respective IComp records. In this case, mass fraction codes cannot be used in any of the columns CIcl, CDcl  and  AUcl.

(2) Input mode. If any positive value Ms  is entered into this cell then the PCO composition vector will be normalized to this Ms  value during calculation of Compos record data.

Normalization occurs as follows: (1) PCO vector elements (moles) are calculated as specified in other data fields of the Compos window; (2) the total mass Mp  of  the PCO is computed by multiplying each of the PCO vector elements to molar mass of the respective IC; (3) Each PCO vector element is multiplied by a normalization factor Ms/Mp.

If any positive value Ms  is entered then, in addition, some values in CI, CD or  CA vectors can be specified in mass percent ('%' in classifiers  CIcl, CDcl  and  AUcl) or other mass fraction concentration units.  In the process of PCO calculation, these mass concentrations will be scaled into numbers of moles of IC using stoichiometries (where needed) and the provided Ms  value.

 
MasVol[1]  - Vs  (input normalization volume,  for specifying volume concentrations)

If any positive Vs  value (in L) is entered then it will be used in PCO calculations to scale all values in CI, CD or  CA vectors that were specified as volume fractions ('V' code in classifiers CIcl, CDcl  and  AUcl) into moles of  independent components..

The Vs  value is not used if volume concentrations are not specified anywhere in this PCO definition. Default value of  Vs  is 0. If  Vs  = 0 or EMPTY then volume concentration codes must not be specified.

 
MasVol[2]  - Mw   (input mass of water-solvent, for specifying molalities)

Positive value of water-solvent mass Mw   (in kg)  must be entered, if at least one of the elements of  CI, CD or  CA vectors is given in molal units (codes 'm', 'i', 'p' in classifiers  CIcl, CDcl  and  AUcl). Upon the PCO vector calculation, molal concentrations will be automatically converted into moles of IC using stoichiometries (where needed) and the provided Mw  value.

The Mw  value is not used if molality units are not specified anywhere in this PCO definition. Default value of  Ms   is 0. If  Ms  = 0 or EMPTY then molal concentrations must not be specified.

 
 MasVol[3]  - Maq  (input mass of aqueous solution for dissolved mass concentrations)

Non-empty positive value of mass of aqueous solution Maq   (in kg) must be entered, if at least one element of  CI, CD or  CA  vectors will be set as concentration per mass of aqueous phase (codes 'C',' c', 'a'  in CIcl, CDcl  and  AUcl  classifiers). During the PCO calculation, these  mass concentrations will be automatically converted into IC moles using  Maq  value and  chemical formulae where necessary.

Maq  is not used by the program if aqueous mass concentrations are not specified.

 
MasVol[4]  - Vaq  (input volume of aqueous solution for molarities)

Non-empty positive value of volume of aqueous solution Vaq  (in L) must be entered into this field, if at least one element of  CI, CD  or CA  vectors is supposed to be given in molarity units (codes 'L',' j', 'q'  in either of  CIcl, CDcl  and  AUcl  classifiers). Molarities will be automatically converted into IC moles using Vaq   and chemical formulae.

Vaq  is not used by the program if molarity concentrations are not specified.
 
MasVol[5]  - Pg  (input pressure in gas, for partial pressures)

This field is reserved for future use.

 
MasVol[6]  - Ns   (input total number of IC moles in this PCO)

A non-zero positive Ns   value (moles) must be entered, if at least one element of vectors  CI,CD or  CA  will be specified in mole-fraction units (codes 'n', 'N', 'f'  in classifiers  CIcl, CDcl  and  AUcl). These mole fractions will be automatically converted into IC moles using Ns  value and then added to the PCO vector.

If no mole fractions are specified and non-zero positive Ns   value is given then upon calculation, the PCO vector will be normalized such that sum of all its elements equals  Ns.

If no mole fractions are specified and this cell contains 0 or EMPTY value then, after PCO calculation, it will be replaced by the sum of all elements of the PCO  vector.

 
MasVol[7]  - Mf  (input mass fraction (%) of a compound specified by  Ctext  relative to the whole PCO mass)

This field must contain 0 if   Ctext is not specified.

If   Ctext is specified and Ms  = 0 then, during PCO calculation,  Mf  is interpreted as number of moles of the compound with stoichiometry defined in   Ctext formula. In this case,  Ms  = 0 is considered as Ms  = 1 (mol).

If   Ctext is specified and Ms   > 0 then Mf   is interpreted as a mass fraction of Ms   assigned to   Ctext.  Negative Mf  values are allowed as long as they do not produce negative numbers of moles in the  PCO  vector.

 
PCO - Output vector of bulk numbers of moles of IC in this Predefined Composition Object.

This dynamic vector is always present in the Compos record. After the PCO calculation, it holds a bulk composition of this predefined composition object, expressed in positive numbers of moles of Independent Components. All elements with zero- or negative numbers of moles will be eliminated from the PCO vector.

The PCO vector is calculated by adding all quantities converted from numbers specified in CI, CD and CA  vectors and MasVol numbers  (scaled according to codes in CIcl, CDcl  and  AUcl  classifiers).

Elements of the PCO vector ca be regarded as stoichiometry coefficients of the "chemical formula" of this object.

   
unPCO - Input uncertainty of   PCO  elements in moles

(optional, for use in future versions).

 
Ctext - Text-formula for incrementing the PCO vector.

In this text editor field, any valid chemical formula can be entered that contains IC symbols from the symIC list.  This formula will be automatically parsed upon calculation of the Compos record.

If no formula is available then the Empty value "---" must be entered.  

 
symIC - List of IC symbols corresponding to elements of  PCO  vector

This list of IC symbols is created automatically when resulting bulk PCO is compressed. The list actually refers to IComp record keys.

This list is used internally when System Profile lists are generated and bulk composition of the system is computed in SysEq records.

 
formU - List of arbitrary formula units

This list of user-defined formulae permits a convenient specification of PCO composition using wet chemistry analyses or other kinds of regular geochemical data.

Examples of user-defined formulae: SiO2, Al2O3, and any valid chemical formulae up to 20 character long, typed by the user in rows of the formU list. No empty rows are allowed!

The list formU is created as the stage of creation/reorganization of Compos  record, when user has typed nAU > 0 in the Compos Wizard. Then formulae can be entered or edited directly in the Compos window page, before calculation of PCO.

 
symDC - List of DC record keys

This list of DC record keys is created automatically after the user has marked DComp and/or ReacDC keys at remake/create of Compos record.

 
CIcl - Units of IC quantity/concentration

This classifier list contains codes of units of measurement for IC named in the list symIC and quantities/ concentrations specified in CI vector. The user may set up different codes for different IC, according to certain rules.

Default code:  M (moles).

 
CDcl - Units of DC quantity/concentration

This list contains codes of units of measurement for DC stoichiometries named in the list symDC and quantities/concentrations specified in CD vector. The user may set up different codes for different DC, according to certain rules.

Default code:  M (moles).
 
 
AUcl - Units of formulae quantity/concentration
 
This list contains codes of units of measurement for user-defined formulae (UDF), listed in the formU object, with quantities/concentrations specified in CA vector. The user may set up different codes for different UDF, according to certain rules.
Default code:  M (moles).
 
 
CmDCS  - Source of DC data: d-DComp  r-ReacDC { d r }

Possible values:

d     look for DComp  record referred to with the line in symDC list;
r      look for ReacDC record.

 This classifier is created automatically upon create/remake of the Compos record.

 
CI - Quantity/concentration of IC C(IC)

This optional dynamic vector contains quantities or concentrations of IC used for specification of the PCO. Corresponding IC symbols are listed in symIC vector.

Each element of CI vector is scaled according to the code in CIcl classifier and then added to respective element of PCO vector. Zero and negative values can also be specified as long as they do not produce negative numbers in the PCO vector.

 
CD - Quantity/concentration of DC C(DC)
 
This optional dynamic vector contains quantities/concentrations of DC formulae used in definition of PCO. The DC formulae are automatically extracted by the program from DComp of ReacDC records referred to by keys in the symDC list.
 
During calculation of the bulk PCO vector, each element of CD vector is scaled as prescribed by the class code in CDcl classifier. Then the DC formula is used in conversion to numbers IC moles finally added to the elements of PCO vector.
 
 
CA - Quantity/concentration of user-defined formula units  C(AC)

This optional dynamic vector contains quantities or concentrations of the formulae units (UDF) which user has defined in the formU list.

The vector is created together with formU and AUcl objects at re-organization of the COMPOS record. The CA vector can contain zeros, positive and negative numbers.

When composition of PCO is computed, the UDFs are scanned sequentially and corresponding element of the CA vector is normalized as prescribed by the code in AUcl classifier, then converted into (positive or negative) number of IC moles and added to PCO vector of the bulk PCO composition.

 
bSDref - List of SDref keys to data sources
  
  
bSDval - List of comments to data sources
 
 

4.1. Data entries accessible for editing through Create/Remake wizard only

Setup Flags (BC_set)

BC_set[0] - Use quantities of Independent Components in this PCO definition?
 
Possible values:
+    keep  CI and CIcl vectors (if present) but do not fill with default values;
*    create/keep  CI and CIcl vectors and fill them out with the default values (0 and 'M', respectively);
-     delete  CI and CIcl  vectors (previously contained data will be lost).
 
If  CI vector is present then the non-zero quantities defined in this vector will be converted to numbers of moles according to codes given in the CIcl vector, and added to the bulk  PCO  composition vector. In this process, the molar masses if independent components will be taken from the IComp records referred to in the  symIC  list of record keys selected by the user during remake of this Compos definition.
 
 
BC_set[1] - Use stoichiometries of Dependent Components in this PCO definition?
 
Possible values:
+   keep CD , CDcl and  symDC vectors (if present) but do not fill them with default values;
*   create/keep CD , CDcl and  symDC  vectors and fill them out with default values (0, 'M', and  EMPTY, respectively);
-   delete CD, CDcl  and  symDC vectors (previously contained data will be lost).
 
If CD vector is present then the non-zero quantities defined in this vector  will be converted to numbers of moles according to the codes given in the CDcl vector, and added to the bulk PCO composition vector. In this process, the chemical formulae of compounds will be taken from the respective  DComp  or  ReacDC  records, referred to in the symDC  list of record keys selected by the user during remake of this Compos record.
 
 
BC_set[2] - Create user-defined formulae (UDF)  in this PCO definition?

 
Possible values:
+   keep CA, AUcl and formU vectors (if present) but do not fill them with default values;
*   create/keep CA, AUcl and formU vectors and fill them out with default values (0, 'M', and  EMPTY, respectively);
-   delete CA, AUcl and formU vectors (previously contained data will be lost).
If CA vector is present then nonzero quantities/concentrations defined in this vector  will be converted to numbers of moles according to the codes given in the AUcl vector, and added to bulk PCO composition vector, by analyzing the user-defined chemical formulae from the  formU  list.
 
 
BC_set[3] - Create a large formula text for this PCO definition?
 
Possible values:
+   or  *     keep/create formula text field ;
-    delete  the formula text field.
 
PCO definition can be enhanced using a large chemical formula which you can type later into Ctext  field, created on remake of this Compos record.
 
If the non-EMPTY formula text is present and it is written according to GEMS chemical formula syntax, then this formula is parsed, converted to the number of moles of IC depending on the scaling factor  MasVol[7], and added to the bulk PCO composition vector.
 
 
BC_set[4] - Create the  unPCO vector of uncertainties for this PCO definition?
 
Possible values:
+    keep the unPCO vector but do not fill it with default values;
*    create/keep the unPCO vector and fill it with default values (EMPTY);
-    delete the unPCO  vector.
 
If this vector is present, the user can enter there the absolute uncertainties of the respective PCO  vector elements in moles.
 
For future use in the uncertainty propagation calculations in GEM-Selektor v.3.x-PSI.
 
 
BC_set[5]
Reserved for future use.

5. Warnings and Error Messages 

E00BCrem: Attempt to access corrupt dynamic memory.
E01BCrem: Attempt to free corrupt dynamic memory.
E02BCrem: Attempt to allocate corrupt dynamic memory.
E04BCrem: Dynamic memory corruption in Compos data structure.

One of these error messages indicates a dynamic memory corruption.   

Action: Restart GEMS and try the same work again. If the message appears, send a bug report. 

  
E06BCrem: Compos record configuration cancelled by the user.

  This message appears when Compos Wizard failed or was cancelled by the user.

 
W03BCrem: No Independent Components selected (bc[q].N,Nmax < 1)
W05BCrun: IComp data record keys are not selected
 (maybe, some PDB files are not linked)

These warnings may appear if no IC was selected for setting the PCO, or when Compos record was calculated, but some IC symbols could not be found in the IComp database.


W07BCrem: < 1 IComp keys marked for PCO definition
  Repeat marking, Proceed to defining PCO in different ways, or  Cancel assembling PCO definition?

A reminder that no IComp keys were selected from the IComp record key list. Will make no problems when PCO is defined via formulae or DComp/ReacDC.

 
E08BCrem:  < 1 IComp keys selected for PCO definition...

  This message appears if 'Cancel' was answered for the W07BCrem warning.

 
E09BCrem:  < 1 ReacDC&DComp records selected for PCO definition.

  This message appears if the DC formula mode was activated in Compo Wizard, but no ReacDC and/or DComp records were marked in the record key list.

E10BCrun:  Wrong index of Independent Component

Typically, occurs upon Compos calculation if one of IComp symbols parsed from the formula was not found in the IComp database. Possibly, not all necessary IComp files are linked to the record chain.
