1. The Project Chemical System (CSD) Submodule

In "Equilibria Calculation" mode, this part of the GEM-Selektor UI (user interface) provides an overview (through the "Input:..." tab of the Single System Dialog) and a detailed access (through the "System" and "MTparm" windows) to the basis of the currently open modeling project.

Here, the user can look through phases and their components;  decide which phases and components should be excluded from the current calculation of equilibrium state; set bulk chemical composition of the system for this calculation (much facilitated by the "Recipe Dialog") and set optional metastability constraints; check thermodynamic data at temperature/pressure of interest; and do "fine-tuning" of the system definition, in other various ways.

The effect of any changed setting can be immediately seen in the computed speciation and other GEM calculation results in the "Results:..." tab of the Single System Dialog, as well as in detail in "EqDemo" window pages (and in the "Multi" window pages for advanced users).


1.1. The 'Input' Tab of the Single System Dialog

This dialog, commonly used in the "Equilibria Calculation" mode of GEMS, gives a convenient list view on the setup of chemical system and its equilibrium state. The dialog consists of two tabs: one called "Input: System Definition" described in this section, and another "Results: Equilibrium State" described here....

The "Input..." tab contains a hierarchical list of phases available in the modeling project system, with names visible in the "Phases/species" column (get more help on any column by selecting any data element in the column and pressing F1) . By clicking on a small square with '+' to the left of the Phase name expands the list showing subordinate entries for phase components. Input data in other columns are treated differently for a phase or for a Dependent Component (see below).

To the right of Phase name, a number of Dependent Components (species, end members) included into this phase is shown. The solution (multi-component) phases are listed first, and pure (single-component) phases last in the list. Next-to-right, the "Type" column shows the phase aggregate state against the Phase name, and DC class code against the component name. The next column "On/off" contains '+' against any phase or Dependent Component to be taken into calculation of equilibrium in this SysEq record, or '-' if this component or phase has to be excluded (turned off). To turn phases or components on/off, double-click on '+' or '-' and change the flag using the appearing drop-down list. Note that turning the phase off also turns off all its components; turning the phase on also turns all its components on.  However, in a phase that has been turned off, switching a component on does not turn the phase automatically on.

The value in "Add to BC" column for a phase corresponds to the xp_ vector of quantities of phase bulk elemental composition, or to the xd_ vector of quantities of DC stoichiometry to be added to the system bulk composition B_ . Elements of this column can also be accessed through the Input Recipe dialog, or the "Phase" and "DComp" pages of the "System" window. Interpretation of numbers in this column depends on the codes of unit of measurement that can be set in the "UC" column immediately to the left. To change the default code to different one, double-click on the code and select from a drop-down list.

The "G0 corr." column contains optional increments to molar Gibbs energy function at temperature and pressure of interest that will be applied only in calculation of this equilibrium state (and SysEq records directly cloned from this one). For a phase species (Dependent Component), this value (according to units of measurement code in the "UG" column) will be applied to the respective gTP value. Note that any units code selected here will be used for immediate conversion into Joules per mol (code J) before the quantity will be written into the gEx_ vector (the units code itself is not stored and will be converted to 'J'). The other possible codes are:

For Phase as the whole, the "G0 corr" cell may optionally contain a specific Gibbs energy increment  in Joules per gram of the phase. This column is also accessible in the "System" "Phase" window page as Yof_ data object. Default value is 0. The increment value for the phase can either be set directly, or via specific surface area Aalp_ and specific interface Gibbs energy Sigm_ (both are initially copied from the respective Phase record and can be modified here). If the Yof_ increment for a phase is set to zero but Aalp_ and Sigm_ are given non-zero numbers then the Yof_ will be calculated from Freundlich-Ostwald equation as Yof_ = 2./3.*Aalp_*Sigm_  upon clicking the "Calculate BCC" toolbar button or menu command. During calculation of equilibrium state, the value of Yof_ will be converted in normalized J/mol/RT units and incremented to gTP values for all components of the phase (in sorption phases, to sorbent components only). 

Other columns of the "Input:... " tab (on the right-hand side of "G0 corr.") are only applicable to Dependent Components; they can be used for setting up optional metastability / kinetic constraints on the sought-for amount of any component.

The "Lower_KC" column contains metastability (kinetic) constraints from below on mole amount of Dependent Component in GEM primal solution. The values in this column mirror the contents of the dll_ data object, which is accessible in the "System" "DComp" window page. The dll_ values can also be set in the Input Recipe dialog. Trivial constraint is 0; maximum possible non-negative value must be compatible with the bulk composition of the system. Such metastability constraints are used e.g. to simulate mineral precipitation kinetics in the over-saturated aquatic system.   

The "Upper_KC" column contains metastability (kinetic) constraints from above on mole amount of Dependent Component in GEM primal solution. The values in this column mirror the contents of the dul_ data object, which is accessible in the "System" "DComp" window page. The dul_ values can also be set in the Input Recipe dialog. Trivial constraint is 1e6 moles; minimum possible value is 0. Such metastability constraints are used e.g. to simulate mineral dissolution kinetics in the under-saturated aquatic system.  

The "UK" column contains codes of units of measurement (mirrored to the RSC_ data vector accessible in the "System" "DComp" window page) for the respective dll_ and dul_ metastability constraints. Default code is 'M' (moles). Other codes should not be used in this version of GEMS and will be available in future versions. 

The "KC type" column contains the type code of metastability constraints (mirrored to the RLC_ data vector accessible in the "System" "DComp" window page) that can be set separately for each dependent component. Possible codes:

Code
 Description
O
 Constraints are disabled
L
 Only constraint from below will be used
U
 Only constraint from above will be used
B (default)
 Two-side constraints will be used

Note that both "Kin.lower" and "Kin.upper" constraints can be set to non-trivial values (0 and 1e6 moles, respectively) for the same Dependent Component simultaneously, either equal to each other, or such that the dll_ amount is less than the dul_ amount (more...). Any such constraint will influence the results of calculation of equilibrium state only if the respective DC is not switched off in this system definition.The dul_ constraint (from above) can be set to 0 for all components of a solid phase to exclude it from the mass balance, but still be able to calculate its stability (saturation) index and activities of all its components.

The information shown in the "Input:" tab can be written into a text file using the "Print" "Chemical System Definition" menu command. It can also be copy-pasted into Microsoft Excel or another electronic spreadsheet program using the right-click menu (below) that appears upon right-clicking with the cursor over any data entry:


 

In this menu, the "Help" command (or F1 hot key) shows the help window with specific information for this data item. The "Select ..." commands highlight respective parts of the dialog tab. The "Copy" command takes selected data into clipboard, and "Copy with names" takes data with column headings into the clipboard. After that, the data can be pasted into a third-party spreadsheet or into a text editor window.

Detailed CSD, BCC and thermodynamic data can be printed into text files as formatted table-like text using the first two commands in the "Print" menu that uses print scripts supplied within GEMS installer (these scripts are copied into the modeling project as SDRef records and can be modified by the user).

 

2. The System Input Recipe Dialog

The purpose of "Input Recipe" dialog is to facilitate creating or modifying the input bulk composition and optional constraints by keeping all selected items in a single, well observable "Recipe Input" table. Selection occurs with help of "Property" list and context-dependent lists that appear in the "Selection" area.

In the upper part, the dialog also shows the "tname" comment field; it is strongly recommended to write there something to identify this particular system and recipe setup. 

"Property" list area

Clicking on an entry in this area opens the respective list of compositions, components, phases, or other inputs in the "Selection" area. Only one entry at a time can be selected (highlighted in gray or blue background); below this list, a short tooltip about this entry will be shown. At present, the entries are:

Selecting the "Compos (xa_)" entry shows the list of all Compos names (extracted from the first field of the Compos record key) currently available in the project database. Clicking on a Compos name adds the respective row (with xa_ property and the Compos name) to the "Recipe Input" table. In this row, the user must set a non-zero quantity and (if necessary) select the unit of measurement.  

Selecting the "DComp (xd_)" entry shows the list of all DC names (extracted from the third field of the DComp or ReacDC record key) available in the project system definition. Clicking then on a DC name adds the respective row with xd_ property to the "Recipe Input" table, for the user to enter the quantity (and optionally select the "Units").  Note that this quantity will be added to B_ vector only if the respective DC is not switched off in this particular chemical system definition (this SysEq record).

Selecting the "IComp (xd_)" entry shows the list of all Independent Component names (extracted from the first field of the IComp record key) available in the project system definition. Clicking then on a name adds the respective row with bi_ property to the "Recipe Input" table, for the user to enter the quantity (and optionally select the "Units").

Selecting the "Phase (xp_)" entry shows the list of all Phase names (extracted from the third field of the Phase record key) available in the project system definition. Clicking then on a Phase name adds the respective row with xp_ property to the "Recipe Input" table, for the user to enter the quantity (and optionally select the "Units"). Note that the bulk composition of this phase in given quantity will be added to B_ vector only if (i) the respective phase is not switched off in this system definition, and (ii) its amount is not zero in the just-computed equilibrium state (or in another SysEq record referred to from this SysEq record, see details here...).

Selecting the "Kin.lower (dll_)" entry shows the list of all DC names (extracted from the third field of the DComp or ReacDC record key) available in the project system definition. Clicking then on a DC name adds the respective row with dll_  property to the "Recipe Input" table, for the user to enter a positive number of moles (consistent with the B_ vector and DC formula) such that the actual amount of this DC in the computed GEM primal solution (vector x) would be greater than or equal to. Typically, such a constraint is used to prevent a mineral phase from dissolution and keep it in partial equilibrium state.    

Selecting the "Kin.upper (dul_)" entry shows the list of all DC names (extracted from the third field of the DComp or ReacDC record key) available in the project system definition. Clicking then on a DC name adds the respective row with dul_ property to the "Recipe Input" table, for the user to enter a zero or greater number of moles (less than default 1e6) such that the actual amount of this DC in the computed GEM primal solution (vector x) would be less than or equal to. Typically, such a constraint is used to prevent a mineral phase from precipitation and keep it in partial equilibrium state; zero value excludes the DC from the mass balance while allowing to obtain its activity or stability (saturation) index.

Note that both "Kin.lower" and "Kin.upper" constraints can be set to non-trivial values (0 and 1e6 moles, respectively) for the same Dependent Component simultaneously, either equal or such that the dll_ amount is less than the dul_ amount (more...). These constraints will influence the results of calculation of equilibrium only if the respective DC is not switched off in this system definition.

Selecting the "G0 shift (gEx_)" entry shows the list of all DC names (extracted from the third field of the DComp or ReacDC record key) available in the project system definition. Clicking then on a DC name adds the respective row with gEx_ property to the "Recipe Input" table, for the user to enter the number of Joules per mol (of this DC). If this DC is not switched off in this system definition, the program will add this number to the respective gTP value for this DC that will influence results of calculation of equilibrium state only for this particular system (SysEq record). This is a convenient tool for adjusting thermodynamic properties of some chemical species without making changes in DComp or ReacDC records that would influence all systems created in the project.

In this area, the names of compositions, Dependent Components, and phases available in the project system are shown, with already selected names (for a chosen property) highlighted in gray or blue background. "Recipe Input" table 

This table has four columns: "Property", "Name", "Quantity" and "Units". It must have at least one row for one of xa_, xd_, xp_ and bi_ properties and non-sero quantity to set a valid bulk chemical composition of the system. As many rows as necessary can be added by performing selections in the "Property" and "Selection" areas. Any row can be removed (regardless of Quantity and Units) by deselecting the respective item in the respective property; this restores the default value of the respective data object field (usually 0; in some cases 1.0 as given here...). The whole table can anytime be selected (by right-clicking its top-left corner) and copy-pasted to Excel or another spreadsheet program.

The "Property" column shows the data object label where the "Quantity" for the respective row will be stored (can also be viewed and accessed in "System" window pages). The entered quantities will not be lost after closing the Input Recipe dialog (by clicking "Ok"), and will appear again upon opening the dialog. However, any row added by selecting "Property" and "Name" that contains default quantity (usually 0, in a few cases 1) will not appear after closing - opening the dialog.

The "Units" column shows in each row separately the code of unit of measurement in which "Quantity" is given.  To change (default) code, double-click in the respective cell and select another code from the drop-down list. The codes are also accessible in Compos, DComp, Phase and IComp pages of the System window, where a code also applies for each row separately.

To check Quantities and Units in the System window pages, close the Input Recipe dialog (by clicking "Ok") and in the Equilibria Calculation mode, click on the "BCC" toolbar button (next-to-the-right of the "Recipe Wizard" button) or use "Data" "Detailed bulk composition..." menu command. Any non-default quantities entered there into the xa_, xd_, xp_, bi_, dll_, dul_ and other fields will appear as rows in the Input Recipe dialog upon opening it again. The same is true for any changes in codes of units of measurement. Note that some of these codes depend on preset values of other inputs, also accessible on the Compos page of the System window.

 

3. How to Setup a Chemical System

In order to compute an equilibrium state within a modeling project, a so-called "recipe" must be created that defines the total bulk composition of the chemical system and optional metastability constraints. Next, this bulk composition (vector B_) must be checked for consistency with the list of phases and their components; possible conflicts can be resolved either by adding missing amounts of Independent Components to vector B_ or turning off incompatible Dependent Components and/or phases.

When there is no conflict between the bulk composition and phases/components, the whole project system setup including thermodynamic data for components and phases will be condensed into a "GEM Task", i.e. an internal data structure delivered to the GEM IPM algorithm for calculation of primal (equilibrium speciation) and dual (chemical potentials of independent components) solutions.

After the GEM IPM algorithm has converged, both primal and dual solutions can be saved together with the "expanded" setup and recipe into a SysEq record. Detailed output (molalities, activities, pH, Eh, amounts, masses and volumes of phases, etc.) is automatically computed out of the primal and dual GEM solutions and displayed in the Single System Dialog and in more detail in the EqDemo window.  

This typical workflow for computing equilibria in a single system is reflected by a sequence of upper toolbar buttons visible in the "Equilibria Calculation" mode of GEMS, as shown in the fragment below:

       From left to right, the buttons are (hover the cursor over buttons to see tooltips):


Clone a new record from this one This is the most typical way to create new SysEq records by inheriting all settings and recipe from the current system definition. It is possible to change temperature and/or pressure and/or recipe number and/or constraints variant number in the new record key, as well as alter some parts of the recipe and/or metastability constraints using the Recipe dialog. 

Open Recipe dialog to set BCC Opens the Recipe dialog (more...) allowing to change some elements used in calculation of BCC (bulk chemical composition) and/or some metastability constraints.

Open System window (for detailed input) Provides detailed access through System window pages to configuration options, including those not directly accessible through the Recipe dialog.

During GEM calculation, a Progress dialog pops up, which can also be run in Stepwise mode (more...). When IPM has converged, the dialog shows some main information about equilibrium state. Clicking "Accept" button saves the new SysEq record to project database (this can be avoided by clicking a "Dismiss" button). Then the "Results:" tab is shown in the main Single System window.

Open EqDemo window (for detailed results) Click this button for exploring detailed GEM IPM output not seen in the "Results:" tab, especially contents of EqIC (dual solution, total dissolved molalities) and Phase (volumes, masses, stability criteria, compositions of phases) pages (more...).  

There are many ways to put together the Recipe of the system; in most cases, the Recipe dialog is the most convenient one.

When you create the first system in a new modeling project, or create a new system from scratch, it will be essential to check the Recipe and the calculated bulk chemical composition (BCC) for consistency with the stoichiometry of Dependent Components (more...).


3.1. Checking Bulk Chemical Composition of the System for Conflicts

In GEM IPM problem setup, the bulk chemical composition (BCC) of the system must contain positive total mole amounts of all Independent Components (IC) except electrical charge ("Zz") whose amount is usually set to 0 but may also be positive or negative. Furthermore, Dependent Components are not allowed to be composed of ICs that are not present in currently valid BCC (vector b); any DC with formula involving a different IC must be excluded. Finally, phases with zero number of components (DCs) as well as solution phases with less than two DCs are not allowed and must be excluded too.

When a new chemical system definition (CSD) is created, it's BCC is not known in advance; after its ingredients are set using Recipe dialog and perhaps the System window, the B_ vector must be calculated, and all DCs and Phases checked for consistency with this BCC. This can be done as test before actual calculation of equilibrium state by clicking the "Calculate BCC and check it for conflicts" button (or "Calculate" "BCC Check" menu command). If no conflicts are found, the user can proceed to computing the equilibrium state. If there is a conflict, the "Missing ICs Conflict Warning" dialog will appear for user's response (more...).  
 

3.2. The 'Missing ICs' Conflict Warning Dialog

This dialog cannot be accessed directly; it appears when a possible conflict is detected (either on "Calculate BCC" or "Calculate Equilibrium" command), and typically looks like this:




In this situation, the user's reaction is necessary because the program does not know whether the user (i) forgot to give amounts of some ICs in the recipe, or (ii) did it for purpose.

In the case (i), check elements for which you forgot to provide quantities in the system recipe (or click "RETAIN ALL"), and then click "CHECK/Ok". This will bring you back to the Single System dialog, where you can start Recipe dialog and re-consider the composition setup. Optionally, the default amount can be edited in the lowermost field (to immediately proceed with calculation of equilibrium, do not set the default amount less than 1e-10 mol). This loop can be repeated as many times as necessary. "Cancel" (click twice) leaves the dialog without turning off inconsistent ICs and DCs, also letting you to re-consider the recipe and the configuration of phases and components.

In the case (ii), click "EXCLUDE ALL" and then click "CHECK/Ok". Back in the "Singe System" dialog, on the "Input" tab you can see indeed that all components containing "missing" ICs have been turned off. This is a very efficient time-saving method of shrinking the system definition in an extensive project, especially if this project is directly linked to DB.default files. You may wish to turn some DCs or phases in addition.

If, at a later stage, you need to create a system with more ICs, proceed as follows to save time on manual turning DCs and Phases on. Clone any suitable system and select "Record" "Remake" menu command to open the System Setup wizard (more...). There, proceed to Step 2, check in the middle a box "Would you like to activate (turn on) all IComp, DComp/ReacDC and Phase entries?", and click "Finish". Now you can extend the recipe with necessary additions for more elements; irrelevant ones can be turned off again using this (i.e. "Missing ICs") dialog.


4. Project System Submodule Window

This window provides a detailed access to the basis of the currently open modeling project and settings for the current chemical system definition (CSD) to be saved in a SysEq record after calculation of equilibrium state. Here, the user can, for instance look through phases and their components;  decide which phases and components should be excluded from the current CSD; set bulk chemical composition (BCC) of the system for this calculation, and set optional metastability constraints.

This window can be opened using "Data" "Detailed bulk composition" menu command, as well as the "Open System window..." upper toolbar button. The window can be opened and closed many times without loss of data; the data will be saved in project database in SysEq record after the equilibrium state is computed; for the existing SysEq record, data can be saved at any time using "Save..." toolbar button or "Record" "Save" menu command.

Any data field or table on every page can be copied into clipboard (and pasted into third-party spreadsheet or text editor) by right-clicking on any data field and selecting from the appearing menu.


4.1. System Window Compos Page

This page provides access to name and note (comment) of this SysEq record, and to the CCvp list of Compos record keys in the project database, combined with the xa_ data object for composing the system recipe (xa_ cell contents can also be set using Recipe dialog and are frequently addressed in Process simulation scripts). Units of measurement for Compos quantities can be set as XAun_ codes to the left of the xa_ column (default 'g' - grams). Negative values in xa_ are allowed, as long as they do not result in negative values in the BCC vector (B_).

To see contents of any Compos appearing in Compos window,  click on a record key in CCvp list and press F7 on the keyboard (or right-click and select "Show record" menu item). Any such record can even be edited, re-calculated, and saved to project database, if necessary.

Any composition object with the respective xa_ quantity equal to 0 will have no effect on the calculated bulk chemical composition (BCC) of the system. If a non-zero xa_ quantity is given then the resulting increment to the BCC (B_ vector) depends on the unit of measurement and on whether (i) all Independent Components in the Compos records co-inside with ICs turned on in this system definition, or (ii) not. These two cases are indicated in the  AC_on column to the left of the CCvp Compos record key list. The case (i) is indicated with '+'; the case (ii) - with '*' flag. In the case (ii), only part of Compos stoichiometry that fits with turned-on ICs in the system definition will be used in calculation of the BCC.

For instance, given the BaCO3 amount of 1 mole in the recipe of the system where Ba and O are turned on and C is turned off, the BCC calculation will add 1 mole of Ba and 3 moles of O to the B_ vector.  If 1 kilogram of the BaCO3 Compos is specified instead, then 5.4 mol Ba and 16.2 mol O will be added (i.e. corresponding to 1 kg of BaO3 and not 1 kg of BaCO3). This example shows that using in the recipe any Compos marked as '*' in the AC_on column must be avoided or done very carefully. 

In this version of GEM-Selektor, even if an entry in AC_on has been edited to '-' (turn off), it will be reset to '+' or '*' upon the BCC calculation. The natural way to disable Compos from influencing the system recipe is to set its quantity to zero in the xa_ vector.

The Masses and Volums data objects visible in this page (some also can be changed via the Recipe dialog) provide basis for BCC calculations if quantities like xa_, xd_, xp_ and bi_ are given in certain units of measurement (more...). Two numbers - total mass of the system in Masses[4] and total number of moles in BCC in Masses[5] - are automatically computed by BCC calculation and are shown for checking purposes.

The Procs data object in this version of GEMS is reserved for future usage by Process simulators.

4.2. System Window DComp Page

This page provides access to the list of Dependent Components in all phases available in the project database, as well as to DC metastability constraints. It consists of two tables (to see the second table, scroll down the window page).

The upper table starts (from left to right) with the DC_on column where a DC that is turned on is indicated with '+' and that turned off - with '-'. Any DC can be turned on/off separately by double-click and select at the respective cell.  The next, DCSvp column shows where the data for this DC is taken: from DComp record ('d') or ReacDC record (r); the record key is shown in DCvp list next to the right. Click on a record key and press F7 on the keyboard (or right-click and select "Show record" menu item) to see record contents appearing in DComp or ReacDC window.

Chemical formula (stoichiometry) of any DC that is turned on can be used in setting part of the recipe of the system by entering a non-zero amount into the respective cell of the xd_ column (also accessible in Recipe dialog). Negative values are allowed, as long as they do not result in negative values in the BCC vector (B_). Units of measurement for DC formula quantities can be set as XDun_ codes to the left of the xd_ column (default 'M' - moles).

In the lnGmf column, usually containing zeros, optional corrections to the stability of DCs can be set in the ln (activity coefficient) scale; positive values suppress and negative values enhance the stability of DC. These stability corrections are used only at AIA (automatic initial approximation) step of the IPM algorithm; the corrections are useful in rare cases when the AIA obtained directly is not good otherwise. 

The lower table shows again the DCvp list of DComp/ReacDC record keys; to the left of it, the mDCC column shows the DC (species) class codes, used by GEM IPM to select proper equations for primal chemical potentials, activity coefficients, etc. (these codes are collected from Phase records when the project is opened). If necessary, any class code can be changed by double-clicking and selecting another code from drop-down list. These changes remain valid until the project is closed; for steady change, it must be done in the respective Phase record.

The remaining four columns can be used for setting some non-trivial metastability constraints (this can also be done from the Recipe dialog, see more...).

The RLC_ column contains a code for using the DC metastability constraints given in dll_ and dul_: default is 'B' (both constraints to be used).

The dll_  column contains metastability (kinetic) constraints from below on mole amount of DC in GEM primal solution: trivial (default) value is 0.

The dul_  column contains metastability (kinetic) constraint from above on mole amount of DC in GEM primal solution: trivial (default) value is 1e6 moles.

The RSC_ column contains codes of concentration/quantity scale for DC metastability constraints in respective dll_ and dul_ cells; default is 'M' (moles).

In calculation of equilibria, metastability constraints do not apply to Dependent Components that have been turned off. 

4.3. System Window Phase Page

This page provides access to the list of Phases available in the project system that can be included in calculation of equilibrium state for the current SysEq definition (to be stored as project database record).

The PC_on column shows, which phase will be included in calculation (turned on, '+') or not included (turned off, '-'). To change the status, double-click on the cell and select '+' or '-'. This can also be done on the "Input:" tab of the Single System dialog. Note that switching the phase off will turn all DCs off that are assigned to this phase. However, unlike the behavior in the "Input:" tab, turning the phase on will not turn its components on. Instead, at least one of phase components must be turned on in the DComp page of the System window, then the phase will be turned on automatically.

The PHvp column contains a list of Phase record keys with names of phases in the third record key field. Click on a record key and press F7 on the keyboard (or right-click and select "Show record" menu item) to see contents of this record appearing in the Phase window. It is possible to edit and save Phase records, however for changes to take effect, it may be necessary to re-open the project or re-load the SysEq record.

The xp_ column (if visible) may contain non-zero quantities of phase bulk elemental composition to be added to BCC (the B_vector) according to units of measurement given in the XPun_ column to the left. These quantities are also accessible via the Recipe dialog or in the "Input:" tab of Single System dialog. Note that the bulk composition of a phase in given quantity will be added to B_ vector only if (i) the respective phase is not turned off in this system definition, and (ii) its amount is not zero in the just-computed equilibrium state (or in another SysEq record referred to in the tPhEQ field above). in principle, a negative quantity is acceptable, as long as it does not produce negative amounts in the B_ vector (except charge "Zz").

The remaining four columns on the right-hand side (Yof_,  Aalp_, Sigm_[0] and Sigm_[1]) can be optionally used for defining specific Gibbs energy increment (in J/g) for the phase, in most cases related to its specific surface area and specific Gibbs energy of the surface in contact with water or gas.

The Aalp_ column contains specific surface area (in m2/g) for phases (default value 0), which must be set to positive number for any sorption or dispersed phase.

The Sigm_ columns contain specific Gibbs energy of phase surface - aqueous (column [0]) or phase surface - gas (column [1]) interface in J/m2 (default 0). A non-zero value can be used e.g. for modeling the impact of dispersity on phase stability, Ostwald ripening, etc. Note that the positive value usually corresponds to particles, and negative value to capillary surfaces of a condensed phase. This part of the code is still under construction, so the meaning of this field may change in near future.

dul_ - Input upper metastability restrictions for DC quantities  x  in equilibrium state
   
Here, positive non-trivial metastability restrictions from above can be set to some sought-for mole quantities of Dependent Components at equilibrium state. The GEM IPM algorithm will keep these quantities less than or equal to the prescribed restrictions. Default (trivial) value is 1e6 (106) moles. These restrictions will be applied if the respective codes in the RLC_ column are set to 'B' or 'U'  values. In the case of two-side restrictions ('B' code), both dll_ and dul_ values for a given DC can be set to the same number.
The upper metastability restrictions provide a good tool to simulate precipitation kinetics and related processes as a sequence of partial equilibrium states. Too high upper restrictions may do no harm (if greater than 1e6, will be truncated to 1e6 mol). However,  inadequate (too low) restrictions for some DCs that are going to appear in major concentrations in complete equilibrium (for example, onto H2O-solvent if H2O vapour is not included in the gas phase) may lead to an inconsistent GEM task and result in a convergence fault of the IPM algorithm.
 For any Dependent Component, setting its upper metastability restriction to 0 will exclude this component from the mass balance, while retaining it in thermodynamic part of the GEM task. In this case, all kinds of activity values for this DC will still be present in GEM IPM output. For example, the fugacity of a gas can be monitored without letting this gas actually appear in positive quantity in equilibrium state. Another practically interesting case is to monitor the saturation index of a pure solid phase that is not allowed to precipitate. In GEMS, this is the only way to construct a system which is oversaturated with respect to some condensed phase.  
 
  
dll_ -Input lower metastability restrictions for DC quantities  x  in equilibrium state
 
Here, positive non-trivial metastability restrictions from below can be set to some sought-for mole quantities of Dependent Components in partial equilibrium state. The GEM IPM algorithm will keep these quantities greater than or equal to the prescribed restrictions. Default (trivial) restriction is 0 moles. These restrictions will be applied if the respective codes in the RLC_ column are set to "B" or "L"  values. 
The lower metastability restrictions provide a good tool to simulate dissolution kinetics and related processes as a sequence of partial equilibrium states. All Dependent Components (with trivial and non-trivial restrictions) are subject to the same mass balance for total amounts of Independent Components as given in the B_ (b) vector. Therefore, inadequate (too high) lower restrictions may violate the mole balance for some ICs and thus will make the whole CSD and GEM task intractable and impossible to converge for mass balance, causing a fault in GEM IPM algorithm. In general, setting a lower metastability restriction is the only way in GEMS to construct a system which is undersaturated to some condensed phase present in positive amount in the mass balance.
Attention! Any upper restriction cannot exceed the total mole amount of the respective IC in the B_ vector. For example: suppose that 0.06 mol of Fe is given in the B_ vector. Thus, any lower restriction to hematite Fe2O3 must be less than 0.03 mol, for siderite FeCO3 - 0.06 mol, and for magnetite Fe3O4 - 0.02 mol. If all three restrictions are applied at the same time then their total (with due account for DC stoichiometry) must not result in total  B_(Fe) >= 0.06 mol. In future versions of GEM-Selektor, automatic consistensy checks will be implemented.
  
 
gEx_  - Input optional additions to input g(T,P) values for Dependent Components (J/mol)

You can "fine tune" input thermodynamic data provided in the gTP vector by adding positive or negative numbers (in J/mol) entered in the corresponding cells of this (gEx_) vector. Entering +5708 corresponds to 1 logK unit decrease of stability of this component at 25 C; -5708 J/mole leads to 1 logK unit increase. Any proportional numbers can be applied. Default value is zero.
Assigning additions greater than +100000 J/mole will make DCs effectively unstable and eliminate them from the equilibrium state, although such species remain turned on in the CSD.
Attention! The gEx_ additions apply only to DC that are turned on in the CSD, and will be ignored for any excluded (turned off) Dependent Component. The gEx_increments affect only this particular system (SysEq record) and have no effect on the contents of the respective DComp or ReacDC records in the project database. 

      
Yof_ - Input Phase metastability parameter, J/g
  
This optional correction to input thermodynamic data can be entered in joules per 1 g of phase mass. Upon BCC calculation, it will be multiplied by the phase molar mass (variable for multicomponent phases) and added to gTP values of all Dependent Components in this phase. Defaut value iz zero.
The phase metastability parameter Yof_ is automatically computed upon creation or remake of the SysEq record for dispersed and sorption  phases from specific surface area and surface free energy in pure water, originally provided in the respective Phase record.

 
lnGmf_ - Fictive DC activity coefficients to be used only in AIA LP simplex initial approximation (in ln scale) 
  
This vector (length n(L)) contains natural logartihms of fictive DC activity coefficients to be used in the AIA LP simplex initial approximation only, in order to force small amounts of some DCs in the initial approximation.
These corrections are necessary in the systems containing solid solution phases with miscibility gaps, or when the feasible initial approximation cannot be obtained otherwise.
Default value is 0.  
For surface species and for end-members of solution phases marked in Phase definitions as "J" ("junior"), the respective cells in the lnGmf_ vector will be automatically set to 6.907755 (or ln of the other value specified in numerical setting Pa_GAN[1] of the GEM IPM algorithm).

7. System Setup (EqStat Remake) Wizard

Chemical System Definition (CSD) contains the 'recipe' to set bulk elemental composition, the lists of phases and their components to be considered in the calculation of equilibrium states, and optional metastability constraints and/or parameters of surfaces and surface species. All this information is used in creation of the GEM task data structure that is used directly in GEM IPM calculation of an equilibrium state.

The CSD information (in compressed form) is kept in the modeling project data base as part of a SysEq record and may change from one such record to another. Another part og the SysEq record keeps the calculated primal and dual GEM solution, as well as the activity coefficients (also in condensed form); the whole GEM task is automatically  restored in Equilibria Calculation mode either by reading a SysEq record or by calculating equilibrium state using GEM IPM algorithm.

In this wizard, it is possible to (re)configure some optional CSD features needed for certain classes of GEM tasks (in most cases, e.g. by 'cloning' SysEq records, this is not necessary).  It is also recommended to fill out System name and Comment fields (if not yet done) - this will help identify this CSD and recipe in the future.


7.1. System Setup Wizard Step 1

This wizard page allows to control optional CSD objects that are important for certain classes of chemical systems.

In particular, the first four checkboxes must be turned on if sorption phases with electrostatic SCMs and Langmuir-type surface activity coefficient terms are included in the system.

The last checkbox (about using the electroneutrality equation), activated by default, must be deactivated if no aqueous, sorption, plasma and charged melt components will be used in the system.  

Some checkboxes are reserved for future versions. 

It is also possible to enter here CSD identification (visible in tname field in System window) and comment to it (tnote field).

 

7.2. System Setup Wizard Step 2

In this wizard page, additional configurations can be performed, particularly important for setting up a parent system for sequential reactor process simulations. 

There are two checkboxes: "PbPH" and "TurnOn". 

Checkbox PbPH: Use in this CSD recipe the bulk compositions of phases from a previously computed equilibrium state  (default state: On). 

Unchecking this box makes  the xp_ or MbXs data objects unusable for calculation of this system bulk chemical composition.

If this box is checked on, compositions of phase(s) and total solids can be obtained as specified in the field below entitled "Key of SysEq record to be used in getting phase compositions for the xp_ and MbXs recipe data objects:".

If this field contains "empty" value (---) then, immediately upon calculation of equilibrium in this SysEq record, the equilibrium compositions of phases (present in positive amounts) from the GEM task data structure will be available for the next (cloned) new SysEq record. 

If this field contains a key of SysEq record that exists in this project then BCC calculation for the currently re-configured SysEq record will use phase compositions from another SysEq record pointed by record key in this field. If nothing or '*' is given there, you will be asked to select one such record from the list. If you cancel selection from the list, the '---' ('empty' value) will be inserted.

Checkbox TurnOn: Would you like to turn on all IComp, DComp/ReacDC and Phase entries?

Default state is Off. Setting this checkbox On will (after finishing this wizard) turn all ICs, DCs and Phases on in this system definition. This is very helpful if more Independent Components will be used in an otherwise retained recipe taken over from the previous SysEq records. Perhaps, some phases or DCs will have to be turned off in addition to automatic BCC calculation action (more...).
