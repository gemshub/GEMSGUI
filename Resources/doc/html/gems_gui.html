1. Introduction

The GEM-Selektor GUI consists of dialogs, (sub)module windows, and the Helpviewer browser. All these elements (based on the Qt GUI toolkit) are designed in Windows style; their appearance and control elements (mouse, cursors, toolbars, menus, text editing) are practically the same as in any Windows (or Linux/KDE) application.

Dialogs control the program operation (together with toolbars and menus) and display some results in graphical form (e.g. Graph Dialog, System Dialog). 

Module windows provide access for the user to view and edit input and output data. There are several data types (for phases, species, systems, compositions, etc.) in GEM-Selektor code. Each data type (1) must be stored separately as records of a certain format in the database, (2) has its own collection of subroutines for processing/calculation, and (3) has its own format of screen presentation. Taken together, these three items comprise a module - a major building block of the GEM-Selektor code and database. A submodule differs only in that it has no separate format of database records - the input/output data for a submodule are distributed between other modules.

Data Object Descriptors (DOD) form a backbone of data manipulation in the GEM-Selektor GUI and database. The user can only display, change or store those data that have a valid DOD that keeps runtime information about type and dimensions (number of rows and columns) and memory location of the data item. Each DOD has a unique label (up to 6 ASCII characters) that is seen on module window pages, references this data in scripts, is used to store data in a database record, and points to a HTML documentation page. All DODs (more than 1000) form an array that is automatically maintained by the program. A GEMS database record format is defined simply as several consecutive DODs in the DOD array (not all data objects, however, have to be stored in the database).



2. GEM-Selektor Dialogs

When you see an appearing dialog, you are supposed to: (1) decide about what the program has to do; (2) select a record from the module database with data to be processed; or (3) react to a warning or an error message.

A good example of the first kind is the GEM-Selektor Main Dialog appearing when you start the program that will wait until you decide what to do: to exit the program, to read this (Getting Started) manual or the About information, to change some settings and preferences, or (finally!) to perform one of two major jobs of the code: calculate chemical equilibria, or maintain thermodynamic (and other) databases.

The second kind of a dialog you will see every time when you press the "Display"  toolbar icon on any module window (for instance, DComp). This will show a "Select data record key" dialog with a list of available record keys. The size of the list can be reduced using a "Filter..." button where you can set a template to "filter" the record key list, e.g., to show only aqueous species of calcium (the filter template: "a:Ca*:*:*:"). Anyway, you have to select one record and press "Ok" to proceed. In some other dialogs, multiple list entries can be selected.

Several "critical" dialogs have their own "Help" buttons to display relevant information in the Helpviewer browser. Note that in this case, you will have to close the browser in order to proceed working with the dialog. 

Third kind of dialogs appear when an error has occurred during program operation. In this case, an error message will be displayed and you will have to press one of the buttons to tell the program what to do next. Error messages with explanations are usually collected at the end of  module manual HTML files.

3. Module and Submodule Windows

Every module of GEM-Selektor displays data in its own window which has one or several pages, selectable by mouse-clicking on tabs in the upper-left corner. Each page, in fact, is a screen form composed of the data object fields, arranged into single cells (skalars), vectors (rows or columns), tables and string lists, according to structure and type of the data object. Label of the data object usually appears near its first cell and (upon clicking on a cell) with indexes - in the bottom left corner of the window. The clicked cell on one data object field is active (contains a "focus"), i.e. can be edited or asked for its help page.

In this version of GEM-Selektor, the cell rims on the data object fields are highlighted in different colors to facilitate location of input, output, supporting, auxiliary, or temporary data. Input cells (where you are required to enter data) are highlighted in green; output data cells - in blue or magenta colors. Lists of species and phases with "Show Record" function are highlighted in yellow, and "inactive" lists and data - in gray. Multiple selection, copy and paste functionality will be supported in future versions of the code.

4. Menu and Toolbar Controls


The picture above shows a small and simple module window, which (nevertheless) has all standard menu and toolbar controls (toolbars show tooltips). All toolbar icons except "Help for module"  have their menu command counterparts.

The "Record" menu entry contains all commands that create, display, save, remake and calculate a single module data instance that is kept in the GEMS database as a separate record (with record key and modification date/time seen below the toolbar).

The "Record List" menu entry contains commands for processing multiple records (selected from the list of record keys), including export, import, rename, delete, copy and move.

The "Database Files" menu entries operate with database files containing records for this module; several files can be linked (the small "F" button shows which files are linked at the moment), added/created, and compressed.

The "Help" menu entry gives access to HTML documentation and scripts that can be executed by this module.
 

5. Mouse Controls and Calculator

On module window pages, left-mouse click selects a data object cell as "active" for editing and shows its label (with indexes) in the bottom-left window corner . A right-mouse click on the active cell and brings you a small menu with some entries and hot keys, usually "Help (F1)", "Calculator (F8)" and, in some cases, "Bibliography (F2)" or "Show Record (F7)" for certain entries in record key lists. The hot keys can be used right away after setting focus on a data cell, without right-mouse clicking. 

The built-in calculator can perform operations on a single (active) cell, as well as on a block of cells (in vector or matrix object). Here you can find more about the calculator functionality.

6. Editing Text and Data Fields

Editing active cells is very simple - just type numbers or characters; note that some data objects may be protected from editing. You can move focus with mouse pointer and click into another cell in a different data object field. Within a vector or matrix field, you can also press  Tab  to go to the next cell. Editing text fields (e.g. math scripts) is supported by standard mark-copy-paste functionality using the system clipboard (in Windows or KDE desktop, the clipboard is accessible to all applications, for example, to other text editors). Here you can find a list of text-editing commands (not all of them are supported in single-line editors like active numeric cells).

7. Access to HTML Runtime and Online Documentation

If tool tips are not sufficient, or if you wish to learn more systematically about the program, then HTML documentation files can be opened in a context-sensitive way in the built-in "Helpviewer" browser window of GEM-Selektor. This simple browser is capable of showing HTML files with lists, tables and images, following hyperlinks, keeping history, back-and forth steps, and bookmarks; the browser is not capable, however, to show frames, work with CSS (cascade stype sheets), or interprete any scripts and forms. Alternatively, you can use separately any standard browser (e.g. Mozilla, Netscape, Internet Explorer or Opera) for a systematic reading or printing the GEM-Selektor documentation.

7.1. Browsing Help Files for Modules and Dialogs

Many GEM-Selektor dialogs and all module windows (e.g. DComp) have their counterpart HTML documentation files (e.g. gm_dcomp.html). You can view such a help file in the Helpviewer window using the "Help"   toolbar icon. The  "Help" menu contains three commands: "Help Contents" (opens the Table of Contents with hyperlinks to all GEM-Selektor documentation files); "Scripts..." to access SDref records with scripts that can be executed in this module; and the "About" command that shows the g_about.html file of GEM-Selektor.

All the module manuals (such as gm_dcomp.html) have a similar structure. The title is followed by a list of contents  (hyperlinked with all headings in the text), then the "general description" part about the module operation, a list of data objects with explanations, and (not in all files) a commented list of error messages. Click on a line in the list of contents to jump directly to the required section. The explanations typically contain hyperlinks to some common documentation files, for instance, the User's Reference (g_useref.html) or Glossary (glossary.html).
 

7.2. Getting Help Pages on Data Object Fields

Input and output data in GEM-Selektor can be viewed or edited through the data object fields on the module window pages. For example, from GEMS Main Dialog, select "Thermodynamic Database" and on the appearing dialog, click on the "Dependent Components (DComp)" button. You will see a DComp:: window. Click on the "Display"  icon in the toolbar and select any record from the appearing list. Now, the numbers and text appear in  white data object cells, most of which have labels on top or to the left (on gray background).

These are data object labels, used in the program to access help pages and data from within the print/export, import and math scripts. Even if the labels are not shown for some fields, you still can see them upon a left mouse click on the field, together with the tooltip contents. For example, click on the first field from top on Page 1; you should see a DCname[0,0] label in the left lowermost corner of the window, followed by a tooltip string.

Now, press F1 hot key; in the upper part of the appearing Helpviewer browser window, you should see a DCname entry with the tooltip, followed by an explanation text. You can now browse the help file, follow hyperlinks, etc. until you read enough about the subject. There is no need to close the Helpviewer every time - GEM-Selektor calculations proceed independently of the browser; you can always step back and read some pages again, or simply follow instructions from this manual right away and see what happens Just change back to DComp window, click and press F1 on another data object (e.g. G0d); the help page will change to that object.

Sometimes, even different cells of the same (vector) data object have separate help entries in the documentation files. To check that, click on "Page 2" button to switch to the second page of the DComp window. There you should see a vector-column of temperature correction coefficients; click and press F1 on different cells of this column to see the respective help entries. 

An alternative way of accessing help page entries is to right-mouse click on the data object cell and select "Help" in the appearing cursor menu. From this pop-up menu, you can also start the built-in calculator and (in xSDref lists) access SDref records with bibliographic references, library scripts, or your own comments. 

7.3. Getting Bibliographic References

To display a bibliographic record, come back to Page 1 of the DComp window, do the left mouse click on any line in the dSDref list (in the lower left corner) and then press F2 hot key (or right-mouse click and select "SDref-record F2" from the appearing pop-up menu). You will get contents of this record in the SDref window. In principle, you can edit it now and save as a new SDref database record.  

A new bibliographic record can also be created by first typing a new compressed SDref key in a  xSDref list line and then pressing F2. In the appearing SDref window, the contents of the prefiously displayed record may be visible; just edit the fields putting in the requred new information (authors, title, journal, year etc.). When finished, press the "Save"     icon to save the record to the database. 

If you know that the required SDref record already exists then put an asterisk "*" into an  xSDref list entry, press F2, and select the required SDref record key from the appearing list.

7.4. Setup of Access to Help Files

Start GEM-Selektor and click on the "Preferences" button (on Main Dialog) that opens a Settings and Preferences dialog. Find a "Documentation Base" radiobutton group in the upper part that controls where the Helpwindow browser looks for HTML documentation files. If the "Local directory" button is selected (default), the files will be read using the specified directory path on your PC.

However, if you switch to the "Remote URL"  button, the program will access directly the most recent version of GEMS documentation posted on the specified web site (make sure that your internet connection is active!). We recommend it for users having a LAN or DSL network connection, as the documentation will be constantly corrected, improved and extended, and direct internet access is the only way to get it before the next version of GEMS package will be officially available for downloading.
