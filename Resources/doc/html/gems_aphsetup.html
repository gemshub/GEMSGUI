Automatic Setup of Aqueous and Gas/Fluid Phases

This dialog facilitates definition of the aqueous electrolyte phase and/or the gas/fluid mixture phase that are automatically assembled upon opening or creating a GEMS3 modelling project. Alternatively, a Phase record already available in the project database can be selected, or the aqueous phase (or gas/fluid mixture) can be excluded from the template chemical system definition in this modeling project.

The first tab "Select Aqueous Electrolyte Model" lets you define a built-in function for calculating activity coefficients of individual aqueous species. These can be selected by clicking on radio buttons on the left-hand side. The two last choices should be used when none of the built-in functions is applicable, or no aqueous phase should be included into the chemical system definition. In the "Parameters for the aqueous model" frame, you can change some default parameters used in some of the built-in aqueous activity functions that are described below.

Most aqueous activity coefficient models that are currently supported by the GEMS3 code belong to extended Debye-Hückel (EDH) models (see document "/Resources/doc/pdf/Activity-Coeffs.pdf" for details and references). In these models, the extended term (b_gamma) can be a function of pressure and temperature or a constant value. In the drop-down list "b_gamma(P,T) mode", the operation mode of the b_gamma function can be selected. If the mode "constant" is chosen, the value of b_gamma at reference state conditions (25 oC, 1 bar) needs to be selected in the "b_gamma(1,298) value" drop-down list, or another value be entered there.

Alternatively, for calculations at hydrothermal conditions, the b_gamma term can be a function of temperature and pressure. In this case, one of four built-in b_gamma functions for different so-called background electrolytes (NaCl, KCl, NaOH, KOH) can be selected. The concept of the background electrolyte means an assumption that the aqueous solution is dominantly composed of that particular electrolyte, so the model should be chosen accordingly. For instance, if in your project you mainly deal with NaCl-rich fluids (the most common case) or with seawater, choose the NaCl variant; for cementitious systems, the KOH variant can be chosen.

Note that for internal thermodynamic consistency, T-P-dependent b_gamma functions should be used in conjunction with the extended Debye-Hückel model after Helgeson and coworkers, i.e. the second model in this tab, which has the code "H". If a T-P dependent b_gamma model is selected, there is no need to enter a number into the "b_gamma(1,298) value" field because this value is automatically calculated by a built-in routine. However, it is still necessary to set the appropriate value of the ion size parameter a0 for each of the four background electrolytes. The value of a0 (also called "distance of closest approach") is selected from the drop-down list "Common a0 value" (or entered there). The recommended a0 values for the four background electrolytes are:

Background electrolyte

       NaCl

       KCl

       NaOH

       KOH

a0

       3.72

      4.08

      3.31

      3.67
 

From the Debye-Hueckel models described above, the activity coefficients of aqueous electrolyte species (charged ions and complexes) will be calculated. For uncharged species (neutral ion pairs and complexes, neutral dissolved gas molecules such as CO2, H2S etc.) and for water solvent, different activity coefficient model options can be used. Four combinations exist, which can be specified using the "Gamma (neutral species)" and "Gamma (water solvent)" drop-down lists. In the former, the activity coefficient of any neutral aqueous species can be either set to unity or calculated as b_gamma * IS; in the latter, the activity coefficient of water-solvent can be set to unity or computed from the osmotic coefficient by the built-in routine.

Note that the basic model should be selected (by clicking the appropriate radio button on the left side of the tab) before setting up details of calculation modes and parameters (T-P-dependence of parameters, calculation of activity coefficients for neutral species and water solvent) on the right side of the tab. Because of thermodynamic consistency, options that explicitly calculate the activity coefficient of water solvent should only be used with the extended Debye-Hückel model after Helgeson and coworkers.

Clicking on the "Check" button displays a record key of the Phase record to be automatically generated; the chosen model and its parameters will be identified in the PhName and PhNote fields of the record key. You can check them later when you open the "Phase" page of the "System" window from the "Single Thermodynamic System" dialog, mark the aqueous phase record key, and press F7 keyboard key. Note that, before the same modelling project is opened next time, this Phase definition record will be automatically deleted and generated again without displaying this setup dialog.

The second tab "Select Gas/Fluid Mixture Model" defines the automatically generated Phase record for the gas or fluid mixture. It functions in a way similar to that for the aqueous phase. The generic models that are currently supported are:

  1. ideal gas mixture of ideal gases (I);
  2. non-ideal gas (fluid) mixture calculated with the Churakov-Gottschalk (CG) model (F);
  3. non-ideal gas (fluid) mixture calculated with the Peng-Robinson (PR) model (7); 
  4. non-ideal gas (fluid) mixture calculated with the Peng-Robinson-Stryjek-Vera (PRSV) model (P);
  5. non-ideal gas (fluid) mixture calculated with the Soave-Redlich-Kwong (SRK) model (E).

See document "/Resources/doc/pdf/Activity-Coeffs.pdf" for details and references. Other automatic models may be added in future versions. Note that interaction parameters of mixing between fluid components cannot be entered into the atomatic fluid phase; instead, a user-prepared Phase record must be selected.

The use of automatically generated definition of an ideal mixture of ideal gases ('I', default) implies that:

  • DComp records for gases with keys starting with 'g' (gaseous) phase state code are available in the project database;
  • All such records will be taken into one automatically generated Phase record with key beginning with 'g'.

The use of the automatically generated definition of a non-ideal gas/fluid mixture implies that:

  • consistent parameters such as CritPg vector are provided in DComp records available in the project database;
  • these DComp record keys start with the 'f' (fluid) phase state code instead of 'g' (gaseous) phase state code;
  • All such records will be taken into one automatically generated Phase record with key beginning with 'f'.

No additional options for the gas/fluid mixture (right-hand side of the wizard tab) are available in this version of GEM-Selektor.

Any "specific interaction" non-ideal model of aqueous electrolyte or gas (fluid) mixture cannot be composed in the automatic mode and must be previosly created (prepared by the user) in GEMS3 Database Mode as a specific Phase definition record. Such a definition typically contains a list of included chemical species, and an array of interaction parameter coefficients entered into the ph_cf table.

One or more available Phase record can be selected by choosing the option "Do not generate; select a user-defined Phase record from database, U" (present on both tabs). After selecting this option, a list of suitable Phase records will be displayed, from which one record must be selected (one or more for the fluid/gas phases). If only one suitable record is available in the project database, the list will not appear, and this record will be taken automatically. By clicking the "Check" button, you can see which record has been chosen in the "Phase record key" field. 

The setup of automatically generated or selected Phase records for aqueous and gas phases can be changed any time only upon opening the modelling project. To change, deactivate the "Retain setup of aqueous (and gas) phases" option in the "Modelling Projects" dialog before selecting an existing project to be opened.

An easy way to prepare a Phase definition e.g. for non-ideal fluid with specific interaction coefficients is described in the Phase help file.
