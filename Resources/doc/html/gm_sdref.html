1. About SDref Module

The SDref record type serves for keeping a library of print, export, import scripts and template math scripts, as well as bibliographic references to data sources. You can also create SDref records to describe your modelling project and definitions of chemical systems or process simulators, especially if you intend to share your project directory with other people later on.

If you work with (template) scripts from the default script library or create your own scripts:

  •     Go to Print/Export scripts manual
  •     Go to Import scripts manual
  •     Go to Math Script reference manual

It is important for scientific applications to keep track of bibliographic sources of input data. The GEM-Selektor code can provide access to one or more records with bibliographic references and comments to them (stored in  SDref database module) from DComp, ReacDC, RTParm, Compos,Phase, SysEq, Process and GtDemo module windows. References to SDref records (record keys) are stored in the database together with numerical input data.

 SDref record key has three fields and looks (in packed form) nearly like a usual reference to the literature, for instance "Karpov_ea:1997:pap:" . The last field, in most cases, identifies whether the data source is a paper, book, report, computer file, reference material, or just an internal comment. In the case of scripts, the first record key field contains name of the script and the last field - name of the module that can use this script. 

The SDref records are displayed in the "SDref" window, which has two pages (selectable in the top-left corner). Page 1 contains text fields with basic bibliographic data, while Page 2 may contain a comment, an abstract (or script), or reference(s) to other SDref record(s).

When remaking/creating a database record for another module (e.g. ReacDC), you can specify how many references to SDref records are needed. Later on, you can enter the module input data, and at the same time put reference(s) to data source(s) into line(s) of the  xSDref list ( x  stands for  d,r,p ...), either by selecting one of the already present SDref record keys, or typing in a new record key. The respective contents of the new SDref record can be entered and saved immediately (encouraged), or at a later time (not recommended). Users can write very short comments on which parameters have been taken from the data source in the corresponding lines of the xSDval list, usually visible to the right of  xSDref.

2. How to Create and Use SDref Records

Contents of SDref records are displayed in a separate  "SDref"  window every time when you focus (by doing left-mouse click) on any line in any  xSDref  list and then press F2. Alternatively, you can perform a right-mouse click and select an "SDreference" item of the appearing small menu.

To replace a SDref link with another one present in the database, you can edit the respective line of xSDref list. Remove previous characters and type an asterisk "*" (value ANY) at the first position, then press  F2 key. In the appearing small "SDref: Get Key" dialog, press a "Get List" button and select the required record key out of the list of available ones, then press "Ok" . The selected key (in packed form) should become visible in the xSDref list and contents of SDref record should be displayed in the SDref window on your desktop.

A new bibliographic record can be created by first typing a new (packed) SDref key in a  xSDref list line and then pressing  F2. This will open the SDref module window, in which usually the contents of previously read record are visible. Just fill out the required fields (authors, title, journal, year etc.) and press the "Save" icon   or execute the "Record" "Save" menu command, when finished.
 
Alternative way of creating new bibliographic records: from the GEM-Selektor Main Dialog, click on the "Thermodynamic database" button and in the appearing Thermodynamic Database Management Dialog, press the "Other Data" button. This will open a Database Management (2) Dialog. Click on the "Bibliography and Scripts (SDref)" button. You will see the SDref window, the same as if you would press F2 on any xSDref field.

Select "Create" entry in menu item "Record". In the appearing "Insert new record key" dialog, type in three fields of the new record key, then press "Ok". In the appearing "Remake" dialog, you can optionally set the number of references to other SDref  records (default is 0), or simply press "Ok".

Now, the SDref window will appear again with the key of a newly-created record visible below the toolbar. Just fill out all the fields in the screen forms on Pages 1 and 2 and press "Save" icon  or select a "Record" "Save" menu item to save the contents into your current project database file (you will be asked where to save if more than one file is linked to SDref chain).

It is possible to access SDref records containing scripts from any module window. Just click on the "Help" menu item, and then select "Scripts...". You will see a list of all SDref records with scripts that can be executed by this module of GEM-Selektor; select one record key and press "Ok" to get it in the SDref window. Now, you can also create your own script for this module just by "cloning" one of the existing SDref records and then editing some of the data fields.

3. Input Data Fields of SDref Module Window

 SDauth - Script name or Author(s) name(s)
Here, the author(s) names, best in the usual bibliographic reference style, e.g. Robie R.A. and Hemingway B.S. 
If there are more than three authors, or list of the authors is longer than 70 characters, type the first three names followed by "et al."
 
  
SDtitl - Title of the publication, script or comments, complete
 
 Type here the full title (arbitrary length). Example:
Thermodynamic properties of minerals and related substances at 298.15 K and 1 Bar(10^5 Pascals) pressure and at higher temperatures
 
  
SDedit - Edition (journal), editors, city,...
 
 Type here:
    journal name for a paper;
    book title, editors and publishers for a chapter;
    publisher for a book;
    institution name and place for a thesis or report; and so on.
 
     
SDvoly - Year, volume, issue
 
 Type here:
    year, volume and issue for a journal paper;
    year for a chapter, monograph, report, thesis or unpublished data.
   
   
SDpage - Pages, other data
 
 Type here page numbers or other appropriate information.
 
 
SDabst - Abstract
 
 This field may contain (optional) abstract up to 16 kbytes long, or "Empty value" ( `)  if no abstract is provided. 
In the case of script, this field will contain a collection of template math scripts or the text of executable print/export or import script. 
You can also enter here a description of your modelling project, test problems etc.
 
   
SDnote - Comments
 
 Type here a very short comment  (up to 70 characters)
   
   
SD_kwd - List of keywords (reserved)
   
Not used in this version of GEM-Selektor.
   
  
SDnRef - Number of references to other SDref records in the SDrefs list.
   
   
SDrefs - List of references (SDref record keys) to related data sources.
 
Mouse-click and press F2 to see a referenced SDref record.
 
 

4. Default Script Libraries 

GEM-Selektor comes with a collection of SDref records containing print/export, import, and template math scripts that should  help users to perform data import/export operations and to write math scripts. This default script library is located in the sdref.template.scripts.*  files in /program/DB.default directory. The files are automatically linked to every user-created modelling project to make the script libraries accessible. 

From any GEM-Selektor module window, a print script record can be selected when the "Print" icon is clicked or the "Record" "Print..." menu command activated. In most cases, this will show a list of script records available for this module, from which you will have to select a desired script and press "Ok". Then a File Selection dialog appears, there you can select a directory and enter the file name to where the data should be printed.

In the same way, export and import scripts can be executed. To export data from one or more database module records, use "Record List" "Export..." menu command and mark records to be exported; to import data into one or more records, use the "Record List" "Import..." menu command.

Template math scripts cannot be executed immediately; but they can help you writing your own scripts, as described in the respective manual.

Below, the script records available in the sdref.template.scripts.*  database files are listed for each module, with brief explanations. Check these SDref records for more specific information. More scripts are likely to be added to the default script library in future versions of the default GEM-Selektor database.


4.1. Print- and Export Scripts

SDref  module

pscript-script:0000:sdref:
A simple script to print contents of a SDref record.
 

IComp module

pscript-full:0000:icomp:
This script prints the full contents of an IComp record (including unused data fields).

pscript-short:0001:icomp:
A script for printing only those IComp data that are used in the present version of GEM-Selektor.

DComp module

escript-table-min:0000:dcomp:
This is an export script for tabulating data for minerals from many DComp records.

pscript-mineral:0000:dcomp:
This script prints data from one DComp record (minerals, gases).

pscript-aqHKF:0001:dcomp:
A script for printing data from one DComp record with HKF EoS parameters for aqueous species.

ReacDC module

pscript-3term:0000:reacdc:
This script prints ReacDC data for 1-, 2- and 3-term temperature extrapolations of reaction properties.

RTParm module

pscript-noempd:0000:rtparm:
A script for printing RTParm tabulated data (no experimental values).

pscript-empdat:0001:rtparm:
This script prints tabulated data from one RTParm record, together with experimental values.

Phase module

pscript-ideal:0000:phase:
This script prints a Phase definition for the ideal solution or single-component phase.
 
pscript-sorption:0001:phase:
A script for printing Phase definition of a sorption phase.

pscript-solution:0003:phase: 
This script prints a Phase record that defines a non-ideal solution model for a multi-component phase.

Compos module

pscript-CIandCA:0000:compos:
A script for printing a Compos record where the PCO is defined using both the quantities of IC (CI vector) and the quantities of formula units (CA vector).

pscript-onlyCI:0001:compos:
This script prints a Compos record where the PCO is defined using the quantities of IC (CI vector) only.

pscript-onlyCA:0002:compos:
 This script prints a Compos record where the PCO is defined using  the quantities of formula units (CA vector) only.

System (Project) submodule, SysEq records

pscript-g-gx-v:0000:mtparm:
This script prints a thermodynamic data table for pressure P and temperature T of interest for all Dependent Components in the project system definition list (P and T are defined in the current SysEq record key).

pscript-normal:0000:system:
This script prints tables for Independent Components, Dependent Components, Phases, and Predefined Composition Objects for the chemical thermodynamic system defined in the currently loaded SysEq record. In these tables, lines will be printed only for those components, phases and PCO, some parameters or flags for which have been changed relative to the default values in the project system definition. 

pscript-seqreac:0001:system:
This script prints tables for Independent Components, Dependent Components, Phases, and Predefined Composition Objects for the chemical thermodynamic system defined in the currently loaded SysEq record. In addition, quantities of phases to be added from previously calculated equilibrium state (Sequential Reactor Compartment) will be printed.
 
pscript-full:0002:system:
This script prints tables for Independent Components, Dependent Components, Phases, and Predefined Composition Objects for the chemical thermodynamic system defined in the currently loaded SysEq record. All data from the project system definition (including that for unchanged species, PCO or phases) will be printed.
 
pscript-sorption:0004:system:
This script prints tables for Independent Components, Dependent Components, Phases, and Predefined Composition Objects for the chemical thermodynamic system defined in the currently loaded SysEq record in the same way as the pscript-normal:0000:system: does. In addition, the input parameters for surface types, surface species of sorption phases, and EDL models will be printed. 

System (IPM) submodule, SysEq records

pscript-eqst-aquat:0000:multi:
A script for printing results of GEM IPM calculation of chemical equilibrium state (aqueous speciation, phase assemblage etc.) in the aquatic system.
  
pscript-eqst-noaqa:0001:multi:
This script prints the results of GEM IPM calculation of chemical equilibrium state (phase assemblage and composition, chemical potentials etc.) in the system without aqueous electrolyte phase.

pscript-eqst-aqsorp:0002:multi:
This script prints the results of GEM IPM calculation of chemical equilibrium state (aqueous and surface speciation, surface type charges and potentials, surface activity terms etc.) in the system with aqueous electrolyte and sorption phase(s).
 

Process module

pscript-noplot:0000:proces:
This script prints a Process simulator definition without data sampler scripts, arrays and plotting.

pscript-plot-empd:0001:proces:
A script for printing a Process simulator definition with data sampler scripts, arrays, plotting, and empirical (experimental) data arrays for fitting.

pscript-plot-noempd:0002:proces:
This script prints a Process simulator definition with data sampler scripts, arrays, plotting, but without empirical (experimental) data arrays.

GtDemo module

pscript-noempdat:0000:gtdemo:
A script for printing a GtDemo sampler definition without empirical (experimental) data arrays.

pscript-empdat:0001:gtdemo:
This script prints a GtDemo sampler definition with empirical (experimental) data arrays.


4.2. Import Scripts

DComp module

iscript-SUPCRT-aqs:0000:dcomp:
A script for importing thermodynamic data for aqueous ions and complexes from an ASCII text file in SPRONS format to the DComp records for aqueous species with HKF EoS coefficients.

iscript-SUPCRT-gas:0000:dcomp: 
This script imports thermodynamic data for gases from an ASCII text file in SPRONS format to the DComp records with one column of Cp=f(T) coefficients.
 
iscript-SUPCRT-min1:0001:dcomp: 
This script imports thermodynamic data for minerals from an ASCII text file in SPRONS format to the DComp records with one column of Cp=f(T) coefficients.

iscript-SUPCRT-min2:0002:dcomp: 
This script imports thermodynamic data for minerals from an ASCII text file in SPRONS format to the DComp records with two columns of Cp=f(T) coefficients.
 
iscript-SUPCRT-min3:0003:dcomp: 
This script imports thermodynamic data for minerals from an ASCII text file in SPRONS format to the DComp records with three columns of Cp=f(T) coefficients.

iscript-Gottschalk:0011:dcomp: 
This script imports thermodynamic data for minerals from an ASCII text file in Gottschalk (1997, 2002)  format to the DComp records with one column of Cp=f(T) coefficients and a set of Birch-Murnaghan V=f(T,P) coefficients (under construction).


4.3. Template Math Scripts

RTParm module

mscript-dcompG-extr:0000:rtparm:
Scripts for RTParm tabulation of thermodynamic data calculated using a DComp input record.

mscript-reac2t-extr:0001:rtparm:
Scripts for RTParm tabulation of thermodynamic data calculated using a ReacDC input record (with 2-term temperature approximation of reaction properties).

Phase module

mscript-2-Margules:0001:phase: 
These scripts can be used for setting up a Phase definition with a regular or subregular binary Margules solid solution mixing model.

mscript-2-Gugg:0002:phase: 
Scripts for setting up a Phase definition with a regular or subregular binary Guggenheim solid solution mixing model.

Process module (simulator, P_expr field)

mscript-titr-lgincr:0000:proces:
Scripts for setting up a Process simulator performing a direct titration with logarithmic titrant increments.

mscript-titr-increm:0001:proces:
Scripts for setting up a Process simulator performing a direct titration with linear titrant increments.

mscript-titr-direct:0002:proces:
Scripts for setting up a Process simulator performing a direct titration with arbitrary increments entered in the input modC array.

mscript-titr-invers:0003:proces:
Scripts for a Process simulator to create a sequence of inverse titration points for the linear pH increments using the built-in golden section algorithm. The necessary additions of acid or base are collected in the modC vector (for further conversion of the inverse-titration Process simulator into an ordinary direct titration with increments from the modC vector). This sort of Process simulations is slow - up to 16 GEM IPM calculations will be performed per each pH point.

Process module (sampler, pgExpr field)

mscript-plot-solgas:0000:proces:
These scripts show how to sample and plot results during the Process simulation in the non-aquatic system.
 
mscript-plot-aqueos:0001:proces:
Scripts to sample and plot results (total dissolved metals, aqueous speciation, pH etc.) during the Process simulation in the aquatic system.
 
mscript-plot-sorpt:0002:proces:
Scripts show how to sample and plot results (surface speciation, densities, charges and potentials) during the Process simulation in the aquatic system with sorption phase(s).
 

GtDemo module

mscript-solgas:0000:gtdemo:
These GtDemo math scripts show how to sample and plot results from a sequence of calculated equilibria in the non-aquatic system.

mscript-aqueous:0001:gtdemo:
Scripts to sample and plot results from a sequence of calculated equilibria in the aquatic system.

mscript-sorption:0002:gtdemo:
Template GtDemo scripts to sample and plot results from a sequence of calculated equilibria in the aquatic system with sorption phase(s).

mscript-miscel:0003:gtdemo:
A collection of various useful GtDemo sampler math scripts.
