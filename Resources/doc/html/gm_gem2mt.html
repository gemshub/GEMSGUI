2. GEM2MT Create/Remake Wizard

The GEM2MT simulator is a tool for automation of simple reactive mass transport models coupled with GEM calculation of equilibrium states in spatially distributed nodes (boxes, volumes) over time steps.

It is possible to perform 1D advection-diffusion numerical finite-difference modeling, as well as the box-flux (reservoir) dynamics.

Changes of species concentrations or amounts of phases through the node profile with time can be graphically monitored and logged to disk files.

The GEM2MT simulator can be configured by choosing its mode of operation and setting up the number of nodes (boxes, volumes).

You may also have to type math scripts that define how to distribute initial system compositions over nodes, and which data to sample from nodes for graphic presentation or export to text files.

This wizard helps you configure and set-up the RMT simulation step-by-step.


2.1. GEM2MT Wizard Step 1:  Select the mode of simulation

There are three groups of GEM2MT tasks:
  • Box-flux (S,F,B) simulations, where the tranport is defined in terms of fluxes of mobile phase groups between boxes / reactors.
  • Numerical finite-difference (A,D) or random-walk (W) 1D reactive transport simulations.
  • Preparation (T) of initial chemical systems and exporting them to GEMS3K input files for external coupled RMT codes.

2.2. GEM2MT Wizard Step 2: Dimensions for transport model and initial chemical systems

Set here the number of local-equilibrium nodes (boxes, volumes) for the transport model (except T mode), and the number of system definitions for initialization of nodes or for export to GEMS3K input files. You can set also some options related to usage of math scripts and file output of simulation results.

nC defines how many spatially distributed nodes (boxes, volumes) will be considered in the mass transport model. Typically, 10 to 1000 nodes can be used. Exception: in  T mode, nC = nIV must be set.

ntM  is the maximum number of time steps (at least one, default 1000, negative- unlimited). GEM2MT simulation will stop when this number is reached or upon the user's cancel command. The "real" time scale can be defined later in GEM2MT window.

nIV is the number of initial chemical system definitions are needed for the initial distribution into all nodes to define the time-zero state of the reactive mass transport problem. This distribution is set in the DiCp vector, which can be filled out upon execution of the math script for node initialization (in many cases created by this wizard, perhaps modified by the user before GEM2MT execution).

Lb is the number of optional user-defined formula units for setting composition recipes of initial chemical system definitions (on Page 1 of GEM2MT window). If Lb is set to 0 then the tick below should be set on to use quantities of Independent Components instead (both ways can be used together). Typical formula units are oxides ("H2O", "CO2", ... ) or other compounds, or more complex stoichiometries like bulk rock compositions. in GEM2MT module, the length of formula unit is limited to 40 characters.

If options for math scripts are not selected then the respective wizard pages will not be accessible, and the text editor fields on GEM2MT window pages will not be visible. However, in some simulation modes the scripts are mandatory and will be created irrespective of user's selection.


2.3. GEM2MT Wizard Step 3: Settings for GEMS3K data exchange interface

These settings are important for the T mode (preparation of GEMS3K .dch, .ipm and .dbr input files for external coupled codes), and may be useful for certain built-in reactive transport modes. Settings are arranged in three groups.

(1) GEMS3K memory and file data exchange

Select ICs, DCs and phases to be exchanged via DATABR (take all, if unchecked): At present, leaving this option unchecked is recommended, which will take all components and phases into RMT data exchange.

Write IPM, DCH and DBR files in binary mode (in text mode, if unchecked): Text mode (unchecked) is default and recommended for keeping the files human-readable.

Do not write data items that contain only default values (text mode only): If this option is checked, only the data entries that contain non-default values will be written into data exchange files, while those containing only default values will be skipped. This produces compact files that are easy to read and modify in text editor (especially DBR files), but may potentially become incompatible with future GEMS3K versions if some default values will change.

Write .dch, .ipm, .dbr files with comments for all data entries (text mode only): This is the "verbose" mode for education purposes. If this option is checked, a comment line will appear before each data entry, explaining what data it contains (similar to GEM-Selektor tooltips on data objects).


(2) Automatic setup of GEMS3K lookup arrays with thermodynamic data

This setup is used for creating lookup array with regular T (temperature) and P (pressure) grid. Default is one pressure value (1 bar) and one temperature value (25 C), in which case no interpolation of thermodynamic data is performed. Another single value of T or P can be set by entering equal number in "Minimal" and "Maximal" fields, while leaving 0 in "Step" field. RMT calculations will be performed if the actual T and P values in nodes deviate from grid T,P values less than respective "Tolerance".

To perform RMT simulations with variable T and/or P, the lookup arrays must be created by setting "Maximal" T or P greater than "Minimal" and setting "Step" to 5 - 10 C for temperature and 10 - 50 bar for pressure. Leaving one of "Step" values zero means that simulations will be performed at constant value of this parameter and variable values of another. For a given node, interpolation of thermodynamic data will be done if its actual T,P values differ from the closest grid pair by more than "Tolerance".


(3) T and P grid values for lookup arrays

Enter pressure and temperature grid values instead of automatic setup. If this option is activated, the automatic setup of T,P grid is disabled, and irregular T and P sequences for the lookup grid must be entered manually or by copy-pasting from a spreadsheet. For this purpose, set numbers of T and P grid values in spin boxes.

Disable data interpolation between grid T values (or P values), use data for TP pairs instead. Activation of this option disables data interpolation over TP grid. In this case, equal number of T and P points must be entered, and the T,P values must be entered pairwise. RMT calculations can then only be performed when T,P values for a given node correspond to one of the given T,P pairs (within "Tolerance").

The T and P values can be entered in the appearing dialog.

2.4. GEM2MT Wizard Step 4: Settings for mass transport fluxes in reactor / box models (S, F, B modes)

This wizard page appears in S, F and B modes only and completes the setup of reactor/box RMT simulation.

The number of boxes nC has already been defined at Step 2. nC tells how many equilibrating reactors (boxes) will be considered in the megasystem.

At each time step, the transport of matter between boxes is performed by fluxes of mobile phase groups (MPG). There are fluxes of three types: normal, source, and sink.

A normal flux is defined by the index of its source box, where the flux begins; the index of its destination box, where the flux ends; the index of MPG definition; the flux order (0, 1, -1, 2, -2), and the flux rate constant. 

A source flux is defined by its bulk composition (amounts of independent components), which can be entered in GEM2MT window if nSFD > 0 is set below on this wizard page; the index of its destination box; the flux order (0, -1, -2); and the flux rate constant.

A sink flux is defined by the index of its source box; the index of MPG definition; the flux order (0, 1, 2); and the flux rate constant.

All nFD fluxes should be set in the list accessible on Fluxes page of GEM2MT module window. nSFD bulk compositions of source fluxes should be provided in the table for source fluxes at the same page. All this has to be done by the user in the B mode. However, in S and B modes, nearly the whole setup of fluxes will be done automatically. The user must only select which phases to include in a single MPG (using checkboxes below). Optionally, the user can set nSFD to 1 which will define a source flux into the first reactor (and enter its bulk composition later on), and then proceed to the next wizard step.

The settings are  organized in two groups.

(1) Dimensions of MPG fluxes:

nPG - Number of mobile phase groups (MPG): set it only for the B mode (will be set automatically to 1 in S and F modes)

nFD - Total number of MPG flux definitions: set it only for the B mode (will be set automatically in S and F modes)

nSFD - Number of source flux definitions: optional to set in B, S and F modes. In S and F modes, can be 0 or 1; if set to 0 then the first reactor (index 0) will be treated as constant source of MPG transported into the second reactor; otherwise, the composition of the source flux at a given rate will be added to bulk composition of 0-th reactor at every time step.


(2) Only S and F mode (sequential and flow-through reactors):

Move aqueous (&aq) phase: this is default for F and S - mode reactor setups, the moving MPG will consist of aqueous phase only.

Move gas/fluid/plasma (&gas) phase(s): Composition of all gaseous phases together can be moved from one reactor to the next, also as an addition to aqueous phase composition.

Move all &solids together ('dialysis bag'): Alternatively to two above options, the bulk composition of all solid phases will be moved from reactor to reactor.

In S and F modes, the outgoing flux from the last reactor is automatically defined as a sink flux.


2.5. GEM2MT Wizard Step 5: Create a script for nodes / boxes initialization and time change ('Next >' to keep old script)

This page configures the automatic setup of initial properties of nodes / boxes in the megasystem by execution of the automatically created script that appears on the right side.  In the GEM2MT window, the initial script can be modified and extended; just click 'Next >' to keep the old script with user's modifications.

The script has two sections, the first block (before 'else') for the initialization of nodes / boxes will be executed once at time step 0, while the second block (if uncommented) will be executed at every subsequent time step. The initialization script can save a lot of time, especially when there are many nodes / boxes to be created.

Checking/unchecking any option will result in an insertion/removal of respective operator(s) into the script for node initialization. The exact form of the operators, as well as the availability of some options, depends on the chosen RMT simulation type. Options 'Pressure' and 'Temperature' are normally selected by default in all RMT cases, and set for all nodes a constant P and T pair taken from Step 3 of this wizard. Options 'Initial system', 'Node type' and 'MPG flux' will be automatically activated in S or F modes; the respective parts of the initialization script will be set automtatically to create sequential unidirectional transport of MPG from node to node.

2.6. GEM2MT Wizard Step 6: Create a script for graphical monitoring of simulation ('Next >'  to keep old script)

To monitor some changes in the nodes / boxes profile upon time steps, the script can be automatically created after setting the dimensions and selecting the node properties, components and phases. Optionally, some experimental data can be plotted over the node profile(s).

This is independent of the VTK flie output that can be configured at the next wizard step.

The graph dialog will be updated for every time step. The simulation can be stopped any time by closing the Graph dialog and clicking "Cancel" on appearing small dialog. In some modes, the simulation can be resumed later.

To retain the old monitoring script, just click 'Next >' on this page.

Number of plots (columns in the yt table, from 0 to 20) to sample data from all nodes:  we recommend to set this to 1, 2 or 3 in order to keep the plot easily understandable (all results can be sent to VTK files and any profile can then be shown using Paraview or similar software).
Note: the number of rows in the yt table will be set to nC (total number of nodes / boxes).

Number of rows (points) in the xEt, yEt arrays for plotting experimental data (optional)

Number of columns (plots) in the  yEt array for plotting experimental data (optional)

2.7. GEM2MT Wizard Step 7: Configure output of results to VTK-format files

The graphical monitoring (set at the previous step) can only show the node profiles for selected properties at current time step, but not a property profile for a selected node over different time steps.

Complete output of RMT simulation results can be done in VTK text format files (see http://www.vtk.org) written into a folder with the same name as this GEM2MT record. Each file in that folder corresponds to a time step and contains the values of all selected properties arranged in data vectors corresponding to profiles over all nodes / boxes.

The generated VTK files can be retained and visualized later on by using Paraview or similar external software.

Option:

Select data fields to VTK output (if unchecked, use all DATABR data fields)

If this option is off (default) then all scalars and data for all components and phases available in the last instance of the DATABR node array in GEM2MT will be written into VTK files. Otherwise, you have to do selections in the lists appearing below.

To retain selections done earlier, just click 'Next >' .


2.8. GEM2MT Wizard Step 8: Other optional settings

At present, most of these settings are reserved for future development.

After you click "Finish" (to be enhanced):

(1)  Page 1 of the GEM2MT module window will appear. Enter simulator name / comment.

(2) There, set up the compositions and ID names of initial chemical systems.

(3) Check the initialization script and settings (for all modes); if in T mode, click 'Calculate'

(4) Check node properties table and (for S, F, B modes) the MPG fluxes list; check monitoring scripts

(5) Click 'Calculate', and watch graphical monitoring of the node profiles

During the first run, do not use the graphic window. After the simulation has finished, you can see the sampled results on the Graph page; click on the "Plot data" button to see customizable graphs. If you wish to plot some experimental values of one of calculated and sampled data, then close the graph dialog, find on the "Graph" page arrays xEp and yEp, and type the empirical data in there. When ready, open the plot again and customise it.
