1. UnSpace Module Operation

In (geo)chemical thermodynamic modelling, the input parameters are usually treated in a deterministic way, as fixed certain values. However, in reality, neither thermodynamic data (stability constants, activity coefficients) nor variables of state (bulk composition of the system, temperature, pressure) can be exactly known. By ignoring the impact of these inherent uncertainties onto calculated results, the modeller risks wrong interpretations or conclusions.

Reasonable uncertainty intervals can be obtained from analytical error estimates or from a critical compilation of thermodynamic constants. In this case, the sensitivity and robustness of a particular (geo)chemical model can be assessed by Monte Carlo-type simulations that, from the variation of uncertain input parameters, generate many “sample variants” and treat them statistically.

However, the main problem lies not in generation, but in the interpretation of sampled variants because, at a large uncertainty of many input parameters, the a priori correct result may not be the most frequent one among the sampled variants. A new method to solve this problem is a combination of: (i)  sampling of chemical equilibria calculations in the multi-dimensional “uncertainty space” over a uniform grid using the GEM IPM algorithm; and (ii) advanced analysis combining “dual thermodynamics” with the “decision-making” statistical criteria projected from the game theory. This method can be especially efficient when some independent information can be used for “filtering” the sampled variants after application of (quantile) decision-making criteria, which allows for a stepwise reduction of the initially applied uncertainty intervals. The UnSpace module of GEM-Selektor comprises an implementation of the approach to sensitivity studies outlined above. Details about the approach and the terminology used can be found in <TM-44-04-01.pdf>.

The UnSpace module performs the uniform multi-dimensional grid sampling in the normalized “uncertainty space”; convolution of sample GEM calculations into the dual-solution based “payoff matrix” (with optional filtering using available experimental data); and application of the Laplace- and other decision-making statistical criteria to find "optimally consistent" variant(s) of input data. This is normally done according to the following schematic algorithm:

  1. Formulation of the problem and the modelling scenario

  2. Preparation of the "parent" deterministic CSD and calculation of the equilibrium state

  3. Creation and confiduration of a new UnSpace record (sample size nQ,  dimensionality of uncertainty space etc.)

  4. Specification of uncertainty intervals, input parameter groups and selection of the payoff function/matrix type

  5. Incorporation of available experimental constraints into "filters" on GEM output parameters

  6. GEM calculations of sample equilibria - "probing" the Uncertianty Space, with storing the newly generated SysEq records to database

  7. Calculation of the "payoff" matrix containing values of "dual-thermodynamic distance" between variants of input and output GEM data

  8. Statistical and decision-making ranking of sampled variants (and, optionally, phase groups) using the Laplace and other criteria

  9. Filtering of sampled variants (optional)

  10. Choice of a dominant phase group (optional)

  11. Correction of boundaries of uncertainty intervals and of location of their centre

  12. If the initial uncertainty intervals have been significantly decreased then go to item (6), otherwise go to the next item  (13)

  13. Refined statistical analysis of the sampled GEM solutions (eventually, DualTh calculations)

  14. Incorporation of additional restrictions, if necessary (in this case, go to item 6)

  15. Final choice and acceptance of an "optimally consistent" variant with improved input parameters and smaller uncertainty intervals

4. Description of Data Entries

Tu[0] Adapted value of temperature T (C)
This cell contains the adapted value of temperature (in Centigrade). If no adaptive loops are specified in the flag unPafl[6]  then the value in this cell will be automatically set equal to that in the cell  Tu[1].
 
Tu[1] Input initial (deterministic) value of temperature T (C)
This input cell contains the initial deterministic value of temperature (in Centigrade). If the initial half-interval (Tu[3]) is set to zero then all sample equilibria will be calculated with this (deterministic) temperature. Default value is 25.
 
Tu[2] Adapted uncertainty half-interval for temperature T (C) or 0 (default)

The adapted uncertainty half-interval for temperature (in Centigrade). If no adaptive loops are specified in the flag unPafl[6]  then the value in this cell will be automatically set equal to that in the cell  Tu[3].

Tu[3] Input initial uncertainty half-interval for temperature T (C) or 0 (default)

The input uncertainty half-interval for temperature (in Centigrade). If this cell is set to zero then all sample equilibria will be calculated with the constant (deterministic) temperature given in Tu[1]. Default value is 0.


Tu[4] Current value of temperature (during generation of sample variants)

This cell contains the current value of temperature during UnSpace generation of sampled variants. Can be used for troubleshooting purposes.
 

Pu[0] Adapted value of pressure P (bar)

This cell contains the adapted value of pressure (in bars). If no adaptive loops are specified in the flag unPafl[6]  then the value in this cell will be automatically set equal to that in the cell  Pu[1].

Pu[1] Input initial (deterministic) value of pressure P (bar)

This input cell contains the initial (deterministic) value of pressure (in bars). If the half-interval (Pu[3]) is set to zero then all sample equilibria will be calculated with this pressure. Default value is 1.


Pu[2] Adapted uncertainty half-interval for pressure P (bar)

The adapted uncertainty half-interval for pressure (in bars). If no adaptive loops are specified in the flag unPafl[6]  then the value in this cell will be automatically set equal to that in the cell  Pu[3].

Pu[3] Input initial uncertainty half-interval for pressure P (bar) or 0 (default)

The input uncertainty half-interval for pressure (in bars). If this cell is set to zero then all sample equilibria will be calculated with the constant (deterministic) pressure given in Pu[1]. Default value is 0.


Pu[4] Current value of pressure (during generation of sample variants)

This cell contains the current value of pressure during UnSpace generation of sampled variants. Can be used for troubleshooting purposes.


Vu[0] Adapted value of constrained volume V of the system (in  cm^3, reserved)

Reserved for future versions; default value 0.


Vu[1] Input initial (deterministic) value of constrained volume V of the system (in  cm^3, reserved)

Reserved for future versions; default value 0.

Vu[2] Adapted uncertainty half-interval for the constrained volume of the system (in cm^3, reserved)

Reserved for future versions; default 0.

Vu[3] Input initial uncertainty half-interval for volume of the system (in cm^3, reserved)

Reserved for future versions; default 0.


Vu[4] Current value of constrained volume V of the system  (reserved)

Reserved for future versions; default 0.


pHf[0] Input lower filter limit for pH (default 0)

This filter can be adjusted if independent experimental pH values are available for this system. During filtering, the calculated pH values in UnSpace sample variants will be compared with the lower and upper limits and the variants fitting within the limits declared as passed through the filter.

Default value: 0.
 

pHf[1] Input upper filter limit for pH (default 14)

This filter can be adjusted if independent experimental pH values are available for this system. During filtering, the calculated pH values in UnSpace sample variants will be compared with the lower and upper limits, and the variants fitting within the limits declared as passed through the filter.

Default value: 14.


Ehf[0] Input lower filter limit for Eh (Volts, default -1)

This filter can be adjusted if independent experimental Eh values are available for this system. During filtering, the calculated Eh values in UnSpace sample variants will be compared with the lower and upper limits, and the variants fitting within the limits declared as passed through the filter.

Default value: -1 V.


Ehf[1] Input upper filter limit for Eh (Volts, default 1)

This filter can be adjusted if independent experimental Eh values are available for this system. During filtering, the calculated Eh values in UnSpace sample variants will be compared with the lower and upper limits, and the variants fitting within the limits declared as passed through the filter.

Default value: +1 V.


ICf[0] Input lower filter limit for ionic strength (molal, default: 0)

This filter can be adjusted if independent experimental values of effective ionic strength I are available for this system. During filtering, the calculated I values in UnSpace sample variants will be compared with the lower and upper limits, and the variants fitting within the limits declared as passed through the filter.
 
Default value: 0.
 

ICf[1] Input upper filter limit for ionic strength (molal, default: 3)

This filter can be adjusted if independent experimental values of effective ionic strength I are available for this system. During filtering, the calculated I values in UnSpace sample variants will be compared with the lower and upper limits, and the variants fitting within the limits declared as passed through the filter.
 
Default value: 3.


qLev  Input: quantile level (default 0.05)

The standard criteria (Laplace etc.) select a single sample variant as "optimal". An option is to select a fraction of variants (defined by this quantile level) such that fall closest to the "optimal" value of criterion or criteria. The "quantile criterion" value and its coordinates can then be found by averaging.

Default value: 0.05 (or 5%).

unGCv  Input: [nG]  generation codes for UnSpace groups: 0- Belov; 1-uniform; 2-normal

This input vector of  unG[0]  elements contains codes that control the generation of sample variant coordinates along each of UnSpace dimensions (groups). Possible values:
 
0:      Sample ccordinates generated using Belov's regular uniform grid algorithm:;
1:      Sample coordinates generated by Monte-Carlo method with uniform distribution;
2:      Sample coordinates generated by Monte-Carlo method with normal distribution.

Default value: 0.


unGvG  Input:  0-deterministic DC; value >0 is an index of a group of varying G0 to which this DC is assigned

This input vector assigns which dependent component (DC) in the system will be taken as deterministic, i.e. with certain constant molar Gibbs energy function G0 value, and which will be considered as uncertain and allocated to an UnSpace group. The respective cell content for the uncertain DC is the index of that UnSpace group ( > 0  and <= unG[0] ).  All deterministic DCs must be given the group index 0 (the so-called deterministic group)

Default value: 0  (deterministic).


unGvP  Input: [Ls] 0-deterministic DC; value >0 is an index of a group varying gamma params (reserved)

Reserved for future versions. In this version must be filled with zeros.


unGvS  Input: [L] 0-deterministic DC; value >0 is an index of a group varying S (reserved)

Reserved for future versions. In this version must be filled with zeros.


unGvV  Input:  0-deterministic DC; value >0 is an index of a group varying V

This input vector assigns which dependent component (DC) in the system will be taken as deterministic, i.e. with certain constant molar volumes V0, and which will be considered as uncertain and allocated to an UnSpace group. The respective cell content for the uncertain DC is the index of that UnSpace group ( > 0  and <= unG[0] ).  All deterministic DCs must be given the group index 0 (the so-called deterministic group)

Default value: 0  (deterministic).


unGvB  Input:  0-deterministic IC; value >0 is an index of a group varying b, to which this IC is assigned

This input vector assigns which independent component (IC) in the system will be taken as deterministic, i.e. with certain constant amount in the bulk composition vector b, and which will be considered as uncertain and allocated to an UnSpace group. The respective cell content for the uncertain IC is the index of that UnSpace group ( > 0  and <= unG[0] ).  All deterministic ICs must be given the group index 0 (the so-called deterministic group)

Default value: 0  (deterministic).


PhAf Input phase indexes of known equilibrium phase assemblage for a primary filter 

This input vector of dimensionality unSiz[0]  is allocated when the flag  unPafl[0] is turned on. Enter in the PhAf vector indices of phases (according to indexation of the system definition) belonging to a priori known equilibrium phase assemblage. The remaining cells should contain negative or EMPTY values "---". The aqueous phase usually has index 0, the gas phase (in presence of aqeuous) has index 1, and so on.

When provided, the phase indexes in PhAf will be used as a primary filter  to screen the calculated sample variants, discarding all that have different phase assemblages. If other kinds of filters are used then they will be applied according to the "logical AND" rule.

The scale depends on the  unPsfl[0] code (% of deterministic value or absolute).

The corrected half-intervals can be found in the intGc[0] column after each adaptive (re)calculation.

Appearance of some single-component phases present in very small amounts (< 10-9 mol) may be influenced by the selected value of a numerical threshold  Pa_DS (see "Project:: Numerical Configuration and Settings" dialog window). For UnSpace calculations, it is recommended that this threshold is set to 1e-9.

unStat[4] UnSpace generation status:

0  indefinite;
1  on-going generation run;
2  done generation run;
3 generation run error (code)

This flag shows the status of UnSpace sampling process which consists in scanning the multidimensional "uncertainty space" using the Belov algorithm. For each point of the Belov's grid, a q-th combination of input parameter values is generated, the corresponding SysEq record is created, and the equilibrium state is calculated and saved in the project database. Before this process starts, the status flag has vaule of 0 or 2 (if the whole sampling process has been completed at least once). During sampling, the flag is set to 1. If an error occurred, the flag is turned to 3 and error message with the respective error code is dispalyed.
