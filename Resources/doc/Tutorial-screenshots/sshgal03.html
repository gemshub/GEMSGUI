(6) After giving a name to a new project, you will see an ElementsDialog:

By clicking toggle buttons with element symbols, you can conveniently set up the stoichiometry basis of all chemical systems to be created within your modelling project.  Selected elements (Independent Components)  are shown as pressed and highlighted buttons.

After you have made a selection (as shown on the screenshot above) and pressed "Ok", the GEMS will search the default "kernel" chemical thermodynamic database for all suitable species (Dependent Components), definitions of phases, and pre-defined compositions, and copy all database records found into your new project's database. During this process, some Phase and Compos records may be discarded; if needed, the list of those can be found in the "DiscardedRecords.txt" debugging file located in the GEMS2 installation directory.

Now, GEMS will ask you to select a built-in model for aqueous activity coefficients:

 

Several commonly-used models are built into GEMS code and listed in the above radio-button list; upon selection, some common parameters (shown on the right side) can be adjusted. Clicking the "Check" button will update contents of the dialog, for instance, the record key of the automatically created definition for the aqueous phase.

 Details about built-in calculations of acitivity coefficients in the latest GEMS version can be found in the  Activity-coeffs.pdf  manual on this site. Other, user-defined aqueous activity models (e.g. Pitzer, SIT) must be provided in advance in aqueous phase definition records in the database.
The second page of this dialog can be used for selecting the gas/fluid mixture model (not needed in our present example, where the default ideal gas mixture will be used).

For now, just click the "Ok" button; the aqueous phase definition "aq_gen" using the Davies equation will be created automatically (all parameters on the right side will be ignored in this case). This operation may take some time (as indicated by a small "Loading Modeling Project..." progress dialog shown below).
