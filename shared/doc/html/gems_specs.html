GEM-Selektor version 3

 

Technical data and specifications

Known to be compiled and work on RedHat Linux 6.1/Intel, FreeBSD 3.3/Intel, Solaris 7 on Intel and Sparc
The work is being done to complete Win32 version - excpeted to end in Febrary 2000.

GEMS is compiled using EGCS-1.1.2 (or with GCC-2.95.2 with some patches) for Unix platforms and Borland C++ 5.4 compiler (C++ Builder 4) for Win32 (need some patches).
For Unices all one have to do is modify Makefile.common to correct paths to QT include and libs, and probably set compiler and linker names. For BorlandC++/Win32 special set of makefiles used as 'make' here differs a lot.

GEMS sources are about 45000 line of code written in C++.
GUI part depends on QT library from Troll Tech (www.troll.no); GEMS core depends on string and auto_ptr class from C++ STL library and uses templates.

GEMS occupies about 7MB of RAM
With initial database GEMS requiers about 7MB on hard drive
Pentium 133 or better should be fine for Intels.
We hope you have at least 256 color in your X Window System. :-)

GEMS RPM installs gems executable in /usr/bin and support and DB files in /usr/lib/gems.
