SyPhEQ  - Key of SysEq record to supply phase compositions for this BCC.

This line may contain a SysEq record key with results of a previous calculation of equilibrium state for another CSD variant of a system with the same name (third field of SysEq record key). This field is used only in a "sequential reactor" mode of irreversible mass transfer simulations; in other modes, contents of this data field are ignored and should be set to EMPTY value "---".  

 The "sequential reactor" mode is activated upon create/remake of the project system (and SysEq record) by setting the SyPvec[3] flag to "+". After the remake is finished, you will be asked to select a SysEq record key from the appearing list, and it will be automatically copied into the SyPhEQ field. At the same time, the xp_ vector will be allocated (see "Phases" page of the "System" window), where quantities of phases to borrow from selected equilibrium state should be entered.  

During the BCC calculation, the bulk composition of multicomponent phases from the equilibrium calculation specified by the SysEq record key in the SyPhEQ filed will be used for the addition to the bulk composition of this system according to quantities given in the xp_ vector. Usually, the composition of equilibrated aqueous or gas phase is transferred in this way. Such "sequential reactor" models are helpful in simulating infiltration metasomatosis or weathering, where the fluid composition changes upon equilibriation while moving through the rock column. 

When "cloning" subsequent SysEq records and CSD from this one, simply edit the SysEq record key displayed in SyPhEQ field (to point to the previously calculated equilibrium); no remake is required anymore.


SYname - Comment line for full identification of this CSD 

Here the user has to enter a comment string that identifies this chemical system. The string will also be printed at the beginning of the CSD or EqDemo printouts (if selected in the "Print" menu of the System Dialog).


SYnote - Extension to the CSD identification field. 
 
This comment field can be used for (optional) enhancement of CSD identification. 

 
Masses[0] -  reserved

Molality of reference electrolyte. Reserved for future use.
 
 
Masses[1] -  Anticipated mass (kg) of water-solvent (Mwat ).

Specifying this entry is necessary for usage of molality concentration units (coded in the XAun_,  XDun_,XPun_ or  BCun_ vectors)  for some cells of the  xa_, xd_, xp_ and  bi_  vectors, respecitvely. 

Default value: 0 (addition of molalities will have no effect to bulk composition of the system). Positive value will cause to convert molal concentrations (positive or negative) into mole quantities of Independent Components and add them to B_ vector.
 

Masses[2]  -  Anticipated total mass of the system Msys  (kg).
  
Positive Msys  value must be entered if if at least one cell in the bi_, xd_,xp_ or xa_ vectors is specified as mass percent concentrations ( coded in BCun_, XDun_, XPun, or XAun_ vectors). In this case, the mass fractions will be recalculated to mole quantities of Independent Components using stoichiometry and Msys value, then added to the B_ vector.
  
Default value: 0 (addition of mass fractions will have no effect to bulk composition of the system).
  
 
Masses[3]  - Anticipated mass of aqueous phase Maq  (kg).
  
Positive Maq  value must be entered if if at least one cell in the bi_, xd_, xp_ or xa_ vectors is specified as concentrations per 1 kg of aqueous solution ( coded in BCun_, XDun_,XPun, or XAun_ vectors). In this case, the concentrations will be recalculated into mole quantities of Independent Components using stoichiometry and Maq  value, then added to the B_ vector.
  
Default value: 0 (addition of concentration per mass of aqueous solution will have no effect to bulk composition of the system).
 
 
Masses[4] - Final total mass of the system MBX (kg).
  
This field contains total mass of the system MBX  in kg, automatically calculated from the B_ vector of input  bulk chemical composition upon "Calculate BCC" user's command (also done when "Calculate IPM" command is executed). Can be used for checking input composition data, or refining the Msys  input value.
   
  
  
 Masses[5] -  Total number of IC moles in the system NMS  (for mole %% calculations).
If a default value 0 or EMPTY was specified then the NMS value will be automatically calculated from the B_ vector of input  bulk chemical composition upon "Calculate BCC" user's command (also done when "Calculate IPM" command is executed).

If NMS > 0 was entered by the user it will be interpreted as total number of moles to normalize the  input mole percents or mole fraction concentrations ( coded in BCun_, XDun_,XPun,orXAun_ vectors) in some cells of the bi_, xd_, xp_ or xa_ vectors.

 
 
Volums[0]  - Anticipated volume Vsys  of the system (L)

Positive Vsys  value must be entered here if if at least one cell in the  bi_,xd_, xp_ or xa_ vectors is specified as volume fraction concentrations ( coded in BCun_, XDun_, XPun,orXAun_ vectors). The volume fractions will be recalculated into mole quantities of Independent Components using stoichiometry and Vsys  value, then added to the B_ vector.
  
Default value: 0 (addition of volume fractions will have no effect to bulk composition of the system).
 
 

Volums[1] - Anticipated volume Vaq  of aqueous phase (L) for molarities

Positive Vaq  value must be entered here if if at least one cell in the  bi_,xd_, xp_ or xa_ vectors is specified as molarity  concentrations ( coded in  BCun_, XDun_, XPun,orXAun_ vectors). Molarities will be recalculated into mole quantities of Independent Components using stoichiometry and Vaq  value, then added to the B_ vector.
  
Default value: 0 (addition of molarities will have no effect to bulk composition of the system).
  
 
 
L1_ - Number of Dependent Components selected within each phase
 
This vector defines how many species (Dependent Components) were included within each phase in this CSD and will be used in IPM calculation of the equilibrium state. Good for debugging purposes: for instance, a potentially dangerous situation that only one component was selected in the non-ideal multi-component or adsorption phase can be seen before the IPM calculation.  
   

RLC_ - Type codes for DC metastability restrictions given in the  dll_  and/or  dul_  vectors
  
This code, for each line of the dll_ and dul_ column vectors (or their counterparts in the "Input:..." listview page of the "Single thermodynamic Systen" dialog), defines how the sought-for mole quantity x of the respective Dependent Component in the prime solution will be restricted by the IPM algorithm. 
 
Possible codes:  
O - trivial or no restrictions (even if non-trivial numbers have been entered in dll_ or dul_ cells)
L  - respective value in x vector cell will be kept greater or equal to that entered in the dll_ cell.   
U  - respective value in x vector cell will be kept lower or equal to that entered in the dul_ cell
B  - (default), the value in x vector cell will be kept within the interval set in dll_ and dul_ cells.
 
 
 
RSC_ - Codes of concentration/quantity for DC metastability restrictions in  dll_  and dul_  vectors
 
These codes define quantity/concentration units in which the restrictions in dll_ and dul_ vectors are specified. 
Attention! Only "M" code (moles) can be used in this version of GEM-Selektor, all other codes are reserved for implementation in future versions!  
   
 
IC_on -  Selector of Independent Components into CSD { +  - }
 
Using switches in this selection vector, you can define the Independent Components (IC) that will form  the stoichiometry basis for the bulk composition of chemical system. Possible values: 
+  IC with the same index will be taken into this CSD (default); 
-   the respective IC will be excluded from this CSD. 
Some IC will be automatically excluded after calculation of BCC (bulk chemical composition) if zero total mole quantities in B_ vector have been calculated for these IC; before excluding them, the program displays a "Missing IC warning" dialog. The BCC calculation subroutine checks that all Dependent Components whose stoichiometry includes an excluded IC will also be turned off in the DC_on selector.

Attention! The program will not automatically turn on an excluded IC and its DCs if non-zero quantity of this IC were entered in either bi_ or xa_ vectors. Unless you turn on the IC and respective DC, the contents of the respective cells in the bi_, xa_ and xd_ vectors will be ignored in the BCC calculation.
  
 

DC_on - Selector of Dependent Components into CSD { +  - }
  
Using switches in this selection vector, you can define which Dependent Components (DC, species)  will be taken into this variant of chemical system and IPM calculation of equilibrium state. Switches in this vector can also be accessed conveniently from the "Input:..." listview page of the "Single Thermodynamic System" dialog. Possible values: 
+  DC with the same index will be taken into this CSD (default); 
-   the respective DC will be excluded from this CSD. 
Some DC can be automatically excluded after calculation of BCC (bulk chemical composition) if their formulae contain at least one excluded IC. If, later on, this IC will be manually included then its DC can also be manually turned on.

Attention! Unless an excluded DC will be turned on again, the contents of the respective cell in the xd_ vector will be ignored in BCC calculation. To include the previously excluded DC, first turn on the necessary IC (in IC_on selector) and Phases (in PC_on selector). 
 

PC_on - Selector of Phases to be included into CSD { +  - }
  
Using switches in this selection vector, you can define which Phases will be taken into this variant of chemical system and appear in GEM IPM calculation of equilibrium state. Switches in this vector can also be accessed conveniently from the "Input:..." listview page of the "Single Thermodynamic System" dialog. Possible values: 
+  Phase with the same index will be taken into this CSD (default); 
-   the respective Phase will be excluded from this CSD. 
Some Phases can be automatically excluded after calculation of BCC (bulk chemical composition) all their DC are going to be excluded. Later on, you can turn on this phase but its DC (species) will not be automatically included; you must check the DC_on selector part for this phase and manually turn the required species on. This is easier in the hierarchical list view on the "Input:" page of the "Single Thermodynamic System" dialog.

Attention! Unless an excluded Phase will be turned on again, the contents of the respective cell in the xp_ vector (present in "sequential reactors" mode) will be ignored in the BCC calculation.  
 

AC_on - Selector of PCO (Compos) to be used in the BCC calculation { + - * }
  
Using switches in this selector, you can define which PCO stoichiometries will be actually used in calculation of the bulk composition vector B_. Possible values: 
+  PCO with the same index is fully taken into this BCC calculation (default); 
-   the respective PCO will be excluded from this BCC calculation;
*  only part of the PCO stoichiometry has been used (automatically set after the BCC calculation).
Normally, all the PCO are turned on because the program first checks the contents of the xa_ vector cells: if a cell contains zero then it does not contribute to the bulk composition and its switch is not affected. But you can turn the switch off if the respective cell in xa_ vector is not zero in order to deactivate this increment in BCC calculation but preserve the value for later times. 
   
 
BCun_ - Codes for quantities/concentrations of Independent Components given in the bi_ vector

You can set a separate code of units of measurements  for each cell of the bi_ vector. 
Default code is "M" (moles). 
Attention! Check the rules of application when setting concentrations in the bi_ vector..

 

XDun_ - Codes for quantities/concentrations of Dependent Components given in the xd_ vector

You can set a separate code of units of measurements  for each cell of the xd_ vector. 
Default code is "M" (moles). 
Attention! Check the rules of application when setting concentrations in the xd_ vector.
   

XAun_  - Codes for quantities/concentrations of PCO given in the xa_ vector

You can set a separate code of units of measurements  for each cell of the xa_ vector. 
Default code is "g" (grams). 
Attention! Check the rules of application when setting concentrations in the xa_ vector.
   

XPun_ - Codes for quantities/concentrations of Phases given in the xp_ vector
 
The XPun_ and xp_ vectors are used only in "sequential reactors" mode, in which the bulk compositions of phases will be taken from equilibrium speciation stored in another SysEq record referenced in the SyPhEQ field.
You can set a separate code of units of measurements  for each cell of the xp_ vector. 
Default code is "g" (grams). 
Attention! Check the rules of application when setting concentrations in the xp_ vector.
  
  

B_ - Input bulk composition of the system (moles of IC) obtained from BCC calculation
 
This vector is calculated from the contents of xa_,xd_, xp_ and bi_ vectors where the user should have entered data to define the system bulk composition. 

All Independent Components for which zeros were calculated in the B_ vector cells will be (after the warning dialog) excluded from CSD together with their Dependent Components. Default values for the B_ vector cells are zeros. All non-zero elements of the B_ vector will be copied into the b vector as input for the GEM IPM calculation of equilibrium state.  

Cells of the B_ vector cannot be edited directly. However, to change them, you can e.g. enter some mole quantities of IC in the adjacent bi_ vector, and they will be added when "Calculate BCC" command is executed next time (this command is automatically when "Calculate IPM" command is activated).
  
 
   
bi_ - Input quantities or concentrations of Independent Components contributing to B_ vector
  
Here, you can enter IC quantities or concentrations to formulate (part of) bulk composition of the system. Default value is zero.
 
The entered numbers will be interpreted according to quantity/concentration codes that you can set in the respective cells of the BCun_ selector. The actual calculation occurs on "Calculate BCC" command. 

Negative values can also be specified in bi_ vector cells, as long as they do not produce zero or negative values in the B_ vector. 
  
 

xa_ - Input quantities or concentrations of PCO contributing to B_ vector
  
Enter here the PCO quantities or concentrations to formulate (part of) bulk composition of the system. Default value is zero. This is the most convenient place to set up bulk composition of the chemical system - a main input data for calculation of equilibria by GEM IPM algorithm. 
 
The entered numbers will be interpreted according to quantity/concentration codes that you can set in the respective cells of the XAun_ selector. The actual calculation occurs on "Calculate BCC" command. 

Negative values can also be specified in xa_ vector cells, as long as they do not produce zero or negative values in the B_ vector. 
   

xd_  - Input quantities or concentrations of Dependent Components contributing to B_ vector
 
Here, you can enter quantities or concentrations of DC  stoichiometries to formulate (part of) bulk composition of the system. Default value is zero.

Attention! An entered positive quantity of DC in this vector does not mean that this species will appear at equilibrium state in the same quantity, if at all. Only stoichiometry (formula) from the respective DComp (DCform field) or ReacDC (RDform field) record will be extracted and used in incrementing the B_ vector according to quantity/concentration codes that you can set in the respective cells of the XDun_ selector. The actual calculation occurs on "Calculate BCC" command. 

Negative values can also be specified in the xd_ vector cells, as long as they do not produce zero or negative values in the B_ vector.
  
 
 
xp_  - Input quantities or concentrations of Phases (from another SysEq) contributing to B_ vector
   
Here, you can enter quantities or concentrations of DC  stoichiometries to formulate (part of) bulk composition of the system. Default value is zero.

Attention! This xp_ vector is accessible only in "sequential reactors" mode. An entered positive quantity of a Phase in this vector does not mean that this phase will appear at equilibrium state in the same quantity, if at all, or will have the same composition. Only the bulk elemental composition of this Phase at another equilibrium state, taken from a SysEq record specified in the  SyPhEQ  field,  will be extracted and used in incrementing the B_ vector according to quantity/concentration codes that you can set in the respective cells of the XPun_ selector. The actual calculation occurs upon the "Calculate BCC" command. 

Negative values can also be specified in the xp_ vector cells, as long as they do not produce zeros or negative values in the calculated  B_ vector.
 
  
dul_ - Upper metastability restrictions for DC mole quantities  x  at equilibrium state
   
In this input column, you can set positive non-trivial metastability restrictions from above to the sought-for mole quantities of Dependent Components at equilibrium state. The GEM IPM algorithm will keep these quantities lower or equal to the prescribed restrictions. Default (trivial) value is 1e6 moles. These restrictions will be applied if the respective codes in the RLC_ column are set to "B" or "U"  values. 

This vector is a good tool to simulate precipitation kinetics and related processes. Keep in mind, however, that inadequate (too low) restrictions to DC that are going to appear in major concentrations at complete equilibrium (for example, onto H2O-solvent if H2O vapour is not included in the gas phase) may lead to an inconsistent CSD that may cause wrong results,  convergence faults, or crash in the IPM module. 
  
   
dll_ - Lower metastability restrictions for DC mole quantities  x  at equilibrium state
 
Here you can set positive non-trivial metastability restrictions from below to the sought-for mole quantities of Dependent Components at equilibrium state. The GEM IPM algorithm will keep these quantities greater or equal to the prescribed restrictions. Default (trivial) value is 0 moles. These restrictions will be applied if the respective codes in the RLC_ column are set to "B" or "L"  values. 

This vector is a good tool to simulate dissolution kinetics and related processes. Keep in mind, however, that inadequate (too high) restrictions will violate the mole balance constraints for some IC (stoichiometry units) and thus will make the whole CSD  inconsistent. In turn, this may cause wrong results, bad mass balance precision, convergence faults, or crash in the IPM module. 
 
In any case, an upper restriction cannot exceed the total mole quantity of the respective IC in the B_ vector. Example: 0.06 mol of Fe is given in the B_ vector. Thus, a lower restriction to hematite Fe2O3 must be less than 0.03 mol, for siderite FeCO3 - 0.06 mol, and for magnetite Fe3O4 - 0.02 mol. If all three restrictions are applied at the same time then their total (with due account for DC stoichiometry) must not result in total  b (Fe) >= 0.06 mol. In future versions of GEM-Selektor, automatic consistensy checks will be implemented.

  
 
gEx_  - Optional additions to input g(T,P) values for Dependent Components (J/mol)

You can "fine tune" input thermodynamic data provided in the gTP vector by adding positive or negative numbers (in J/mol) entered in the corresponding cells of this (gEx_) vector. Entering +5708 corresponds to 1 logK unit decrease of stability of this component at 25 C; -5708 J/mole leads to 1 logK unit increase. Any proportional numbers can be applied. 
Default value is zero.
 
Assigning increments greater than +100000 J/mole will effectively eliminate the species from the equilibrium state, although this species will remain turned on in the CSD. Doing this will not change dimensionalities in the GEM IPM problem.
    
Attention! The gEx_ additions apply only to DC that are turned on in the CSD and will be ignored for any excluded (turned off) Dependent Component. The gEx_increments have no effect on the contents of the respective DComp or ReacDC records in the project thermodynamic database.   
      
Yof_ - Input Phase metastability parameter, J/g
  
This optional correction to input thermodynamic data can be entered in joules per 1 g of phase mass. Upon BCC calculation, it will be multiplied by the phase molar mass (check! variable for multicomponent phases) and added to gTP values of all Dependent Components in this phase.  
Defaut value iz zero.
 
The phase metastability parameter Yof_ is automatically computed upon creation or remake of the SysEq record for dispersed and sorption  phases from specific surface area and surface free energy in pure water provided in the respective Phase records.

  
 
lnGmf_ - DC activity coefficients for Simplex initial approximation (in ln scale)  
  
This optional input vector contains natural logarithms of DC activity coefficients that will be applied only in calculation of the Simplex initial approximation (IPM module). These coefficients may be inavoidable for solid solution phases with miscibility gaps, or when an initial approximation can not be obtained otherwise (for some sorption phases). 
Default value is zero.
 
For end-members marked in Phase definitions as "J" ("junior"), the respective cells in the lnGmf_ vector will be automatically set to 6.907755 (or ln of the value specified in numerical settings of the GEM IPM module). 
  
 
sSDref
List of SDref keys to sources of data or comments 
 
 
sSDval
Comments to SDref lines

6.2. Counters of Components and Phases in CSD

The counters listed below cannot be edited; they are automatically updated after the "Calculate BCC" command execution. They define dimensionalities of the "Multi" work structure for GEM IPM calculation of equilibrium states. 

N_ - N, Number of Independent Components included into CSD
   
This number is usually less than the total number of IC in the project system because some ICs may be excluded from the bulk composition. 
  
 
L_[0] - L, Number of Dependent Components included into CSD
 
This number is usually less than the total number of DC in the project system because some phases with their species or some individual DCs may be excluded from the system definition.. 
   

L_[1] - Number of Dependent Components included into  multicomponent phases (Ls)
   
   
L_[2] - Number of species selected into the aqueous phase (Lw, including solvent)
 
  
L_[3] - Number of species selected into sorption phase(s) (Lsor)
 
  
L_[4] - Number of species selected into gas phase (Lg)
 
  
L_[5] - Number of species selected into the liquid hydrocarbon phase (Lhc, reserved)
 
  
Fi_[0] - Number of phases taken into this CSD (Fi)
