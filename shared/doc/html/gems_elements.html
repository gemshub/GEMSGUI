1. Introduction to Project Elements Selection Dialog

The  ElementsDialog appears when an independent modelling project is created or extended. The aim of this dialog is to facilitate collection of necessary data from GEMS default database and copying it  into the user's project database, when and where appropriate. The main basis for the data selection is the system stoichiometry, i.e. the list of selected chemical elements (and additional independent components). 

The ElementsDialog consists of a "Periodic Table" composed of buttons with chemical element symbols. The program automatically checks all IComp records accessible at the moment of opening of the dialog, and makes the buttons of all available elements clickable and their symbols dark. The same happens with additional Independent Components, which (if present) will be shown on buttons in the "Other" column on the right side.

Before clicking on the elements buttons to select them into the stoichiometry basis of the project system, it may be necessary to change some of the "filter" settings at the bottom of the ElementsDialog. The "Set Filters" button opens a separate SetFiltersDialog.

To the right of "Set Filters" button, a series of checkboxes in the upper row called "Phases:" configures which types of phases will be selected into the project system:

Checkbox name
 Default value
 Explanation
Gas
On
 Defines whether gaseous phase will be present in the project database.
Aqueous
 On
 Defines whether aqueous electrolyte phase will be present in the project database
Solids
 On
 Defines whether single-component solid (or liquid) phases will be copied into the project database
Solutions
 Off
 Defines whether non-electrolyte liquid- or solid-solution phases will be copied into the project database
Sorption
 Off
 Defines whether sorption phases will be copied into the project database

If some of single-component solid phases are not needed in the project, they can be deleted later (in "Thermodynamic Database" mode).

A "data selection" row of checkboxes (called "Data:") defines in which parts of the default database the program should look for the available data to build the modelling project.

Checkbox name
 Default value
 Explanation
Kernel (Nagra/PSI)
 On
 If on, all files containing a keyword "kernel" as an element of the file name will be linked in; if off, all such files will be unlinked (i.e. will become "invisible" for setting up the project). In In this version of GEMS package, the kernel part of Default database contains data imported from Nagra/PSI data base, version 01/01.
Complem. (Slop98)
 Off
 If on, all files containing a keyword "complem" in the file name will be linked in; otherwise, they all will be unlinked. In this version of GEMS package, the complementary part of Default database contains data imported from Slop98.dat.
Organic
 Off
 If on, all files containing a keyword "organic" in the file name will be linked in; otherwise, they all will be unlinked.
Specific
 Off
 If on, all files containing a keyword "specific" in the file name will be linked in; otherwise, they all will be unlinked.

This is a simple but efficient way of maintaining a "layered" structure of the default thermodynamic database.  Contents of the respective layers will be described in the database documentation; see also a "thermodata" section of GEMS-PSI home page (http://les.web.psi.ch/Software/GEMS-PSI/thermodata.html).

Important! If any of checkboxes in the "Data:" group were changed or the "Set Filters" dialog was accessed, then selection of buttons for chemical elements and additional IC will be reset. For this reason, first (if necessary) change filtering settings, and only then select Independent Components to be taken into the project system. 

After clicking toggle buttons for all necessary elements, press "Ok" to proceed. The process of construction of the project definition structure (and copying records from the default into the project database) will start immediately.
