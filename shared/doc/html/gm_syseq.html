1. SysEq: Packed CSD and GEM results for equilibrium state

The main purpose of SysEq module to keep in SysEq database records only that part of information about CSD and GEM IPM calculated equilibrium speciation, which differs from that kept in the lists and vectors of the Project System data structure (stored in a Project record). By default, all species, phases and compositions involved in the project system are considered as "switched on". The default vector of bulk composition contains only zeros, as most of other numeric vectors, including those where the equilibrium  speciation and chemical potentials are to be placed after the GEM IPM calculation.

For a particular chemical system definition to be stored in a SysEq record, one has to specify its label (name), bulk composition variant number , volume constraint V (optional), temperature T, pressure P, and additional constraints defining which species and  phases will be excluded or assigned g0 increments or kinetic restrictions. The label, variant number of bulk composition, V, T, P and variant number of set of additional constraints appear in SysEq record key and should be set when a new SysEq record is created from the "Single Thermodynamic System" dialog (see more in gems_sys.html manual). 

When the system definition is done and the equilibrium state is computed, another task of SysEq record is to store the results in a maximally economic form. This is especially important for kinetic-dependent processes or coupled chemical-equilibria - mass transport problems, or inverse problems, which may generate hundreds or even thousands of SysEq records. When you press "Accept" button on Progress Dialog window, a special algorithm is involved which again puts in the fields of SysEq record only the values  (with index) different from the "default" profile numbers (usually zeros for numerical data). This results in most cases in very compact SysEq records (0.5-4 Kb) then stored in your database extension file. When you select this record next time, a special subroutine unpacks it into the full Profile structure and you can examine how the system has been defined and which equilibrium speciation was obtained. The system definition can be updated and re-calculated many times, or used as a template for "cloning" new SysEq records (this is automatically done by the Process simulator).

3.4.5. Configuring Lists of Species and Phases

 After you have pressed the "Clear All" button in "Reorganization of data fields" dialog,  you will get into a "SysEq: Remake Parameters" window. This window contains many switches and flags, and also displays a help line for every switch where you do a mouse click. Please, examine all switches and read hepl lines for them. The default configuration of switches is usually sufficient. For the first time, simply click on the OK button. You will get into a "System::" definition window. Fill in the two comment lines with a few words sufficient for that you could recognize what is this system definition after one year. Then type in some numbers into the xa_ column to define bulk composition of the system. If the GCO list which you see on this screen is insufficient for specifying all the bulk composition, press "Dcomp" button and put appropriate numbers into xd_ column there (take care about units of measurement). Press  "Phase" button and check out phases which you will definitely not consider in your modeling (don't check out most of solid phases as you may loose the advantages of GEM algorithm in selecting stable phases). Then press "Icomp" button and complete defining bulk composition of the system by putting numbers into the bi_ column, if necessary.


Now, select the SysEq dialog window and press a big  "BCC" button (Bulk Composition Calculation) in the upper left corner. In most cases, you will get a warning like this:

This warning is very important. DB is a numerical threshold of the IPM algorithm defining a minimum possible number of moles of an independent component in bulk composition of the system. So, the first line of this warning means that there are some chemical elements for which calculation of the bulk composition results in zero, too small positive, or negative value.

One possibility is that you  do not have data for these elements and do not want to consider them in your calculations. If so, press a "Switch off" button - this calls a very mighty subroutine which will turn off all independent components that have not been specified, all species which include them in stoichiometry, and each phase in which all species were switched off. As this powerful operation may drastically change dimensions of the GEM IPM problem, we strongly advice you to press the "BCC" button once more (if everything is OK, the above message should not appear again). Examine the first column in either of System:: Icomp, Phase and Dcomp screen forms to see what exactly has been switched off.

Another possibility is that you did something wrong when specifying the bulk composition (entered too small value, or missed to put in correct code of units of measurements, forgot to type some numbers somewhere, etc.). In this doubtful situation, press the "Fill" button and examine the vector  B_  on  System:: Icomp screen form. The trace values are normally 1e-7 or 1e-9 depending on the DB parameter value. Then check all screen forms in System :: window, find the problem, fix it and press the "BCC" button again. Proceed with calculations only after that happy moment when the worning above does not appear anymore. Remember that it is much easier and safer to switch off some species and phases than to turn scores of them on after they have been automatically excluded by mistake.

Sometimes on pressing the "BCC" or the "GEM IPM" buttons, the program issues another warning that "Charge imbalance in calculated bulk composition. Zero it off?".  This may happen if composition of the system is specified through stoichiometry of some charged species (e.g., aqueous ions) or one of Compos records is in error. Press "Yes" button to set total charge (Zz) to zero in the B_ vector and proceed with calculations, or "No"  to see how big is charge imbalance, find and correct the bug, and re-calculate the bulk composition.

Now you a ready to proceed with calculation of equilibrium in the newly-defined system; close the "System" window to speed the program up. Press "Save" button (optionally).
