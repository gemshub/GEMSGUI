1. Thermochemical Data Format

DComp records contain the standard state thermodynamic properties of species (Dependent Components, DC) in a "thermochemical" format, that is,  the standard properties per mole of DC, rather than properties of its reaction with other species. Thermodynamic data for reactions are kept in GEMS database in a specialized ReacDC format records.  

Minimal information about a chemical species (compound) in a DComp record includes:

  • standard state;
  • name (identifier), also a short name in the DComp record key;
  • stoichiometry (chemical formula, expressed in IC symbols);
  • Default class code of species (compound);
  • standard molar (partial molal) Gibbs energy of formation G0Tr,Pr  from IC at reference Tr, Pr  (usually Tr = 298.15 K; Pr = 1 bar);
  • standard molar (partial molal) volume, absolute entropy, and heat capacity at Tr, Pr.

DComp records may contain several optional data vectors holding coefficients for various T,P dependencies of thermodynamic properties or equations of state, used in various methods of temperature - pressure extrapolation.

The DComp database chain (as any other chain in GEMS database) consists of several files which can be anytime linked or unlinked by the user through the "Database Files" menu entry in the "DComp" window. Each file belongs either to the "kernel database" or to the user's project database, and normally contains a number of DComp records comprizing a subset of the whole "internally consistent" thermodynamic dataset. In this version of GEM-Selektor default database, two published datasets are incorporated:

  • "kernel" Nagra-PSI database in GEMS adaptartion (Thoenen and Kulik, 2003);
  • "complementary" SUPCRT98 database for aqueous species and gases [1997SHO/SAS and references therein].

Third-party "specific" databases in GEMS format are available from their vendor web sites (see links in http://gems.web.psi.ch). Thermodynamic data is not always consistent between different datasets! The user, in general, should avoid mixing of thermodynamic data from the alternative datasets, wherever possible, and check such data for consistency otherwise.

In the Database mode of GEMS3, available DComp records can be looked for in the record key list (on the left side) until one is found, selected and opened to see its contents in the DComp module window. The contents of different records can be browsed one-by-one using "Display next record" and "Display previous record" toolbar buttons. The data fields (each displays a tooltip when pointed with mouse) can then be edited immediately.

What optional data must be entered and how it will be used in thermodynamic calculations, is defined by the TPcMod codes (see Page 2  of the "DComp" window) set up during remake/create of a DComp record using a DComp Wizard.  For instance, the "CSC" method codes (most common for minerals) tell that the standard integration over temperature interval Tr   to  T   using Cp0  = f (T) equation coefficients will be performed while keeping standard molar volume constant. This way, the standard molar properties of the mineral (G0, H0, S0,  and Cp0  ) are automatically computed by GEM-Selektor for any given  T, P  of interest within the specified interval. 

The same thermochemical format is used for gases. There are some differences, however. First, molar volume of an ideal gas changes strongly with temperature and pressure according to the PV  = nRT  equation. Second, at pressures greater than a few bars, most gases do not behave ideally and at elevated T, P  do not mix ideally. To account for this, the models of non-ideal fluids must be used, several of which are implemented in the TSolMod library of phase models. Most such models are based on the "Correspondent States Theory" to calculate fugacities of individual pure gases a  T,P   of interest and then consider the gas phase as an ideal or non-ideal mixture of real gases. To implement such models, an optional  CritPg  vector can be specified in DComp records for keeping critical parameters for individual gases. This version of GEM-Selektor treats gas phase at low temperatures and pressures as an ideal mixture of ideal gases; the respective phase definition is created automatically upon opening the modeling project.

A third form of DComp data format is used for aqueous ions and complexes from SUPCRT92 and younger datasets [Johnson et al., 1992; Shock et al., 1997]. The increments to standard-reference partial molal properties for aqueous species are calculated from Tr, Pr    to elevated T  < 1000 oC and P  < 5000 bar using the revised HKF EoS (Helgeson-Kirkham-Flowers equation of state, [Helgeson et al., 1981; Shock et al., 1997; Borisov & Shvarov, 1992; Anderson, 2005 ). All the necessary subroutines from the published SUPCRT92 source code [Johnson et al., 1992] were translated from fortran to C/C++ and built into the Selektor-A and then the GEM-Selektor code. In addition to input standard-state properties G0,H0 and S0 at Tr, Pr ,  calculations with the HKF EoS require as input up to seven empirical coefficients kept in the ai_HKF optional vector.

To reduce on typing errors, many text files with thermodynamic data can be imported into GEMS DComp database chain using import scripts that can be customized or written by the users. For instance, SUPCRT data were imported in this way. Multiple DComp records can be also exported to text files using export scripts, also editable by the user (e.g. for data tabulation purposes, or conversion to other database formats).


1.1. DComp module calculations

When the "Re-Calculate" button (or menu command, or F9 hot key) is clicked from the DComp window, GEM-Selektor will parse and check the DC chemical formula, calculate gram-formula mass and charge, perform cross-checking of standard molar properties, and (optionally) calculate heat capacity and molar volume at reference temperature and pressure. If any problems are encountered, various diagnostic messages will be displayed and, after fixing the problem, recalculation of the DComp record must be repeated until all the input data becomes consistent.

When entering thermodynamic data, it is important to realize that  out of three main properties (G0,H0 and S0) of the Dependent Component, only two are sufficient, because the third one can be calculated out of them in a consistent way. In some literature sources, H0 and S0 are provided, in others G0 and S0.  The molar Cp0 values are usually automatically calculated from entered Cp0  = f (T) coefficients or from the HKF EoS coefficients (in the latter case, also standard partial molal volume V0 will be calculated).

2.3. DComp Wizard Step 3

Optional:  in the spinbox, set the number of links to bibliography SDref records (default 0). Setting at least one link is recommended for the data traceability (the respective fields dSDref and dSDval will appear on Page 1 of the DComp window).

What will happen after you click "Finish":

(1) Page 1 of the "DComp" window appears on the desktop. Fill out the DCname comment field and (essential!) enter the DC formula into the DCform field.

(2) Enter the standard molar properties of the substance into V0d[0], G0d[0], H0d[0], S0d[0], Cp0d[0] cells. If some of them are unknown (e.g. H0), enter the 'empty' value ('---').  Any two of three values G0, H0 and S0 should be entered - the third one (if given as 'empty') will be retrieved upon re-calculation. Note that 0 (zero) is a non-empty value. If all three non-empty values are given then the program will check them for consistensy and issue a warning, if necessary.

(3) Switch to Page 2  and enter values into TCint, ai_Cp or aiHKF arrays, whatever appropriate. In some cases, fluid EoS coefficients, critical parameters, or V(T,P) coefficients may need to be entered. Unknown coefficients should be given as zeros (default).

(4) Go to Page 1 and click on 'Calculate' toolbar button. For unknown DC properties (e.g. H0), non-empty values will appear. If Cp=f(T) (or HKF) coefficients were entered, the Cp0 (and V0) values will also be calculated. Save the record to database. To test the T,P corrections, create a RTParm tabulator for this DComp record.


3. DComp Module Window

This window consists of two pages.  Page 1 provides a screen form access to standard state thermodynamic data and stoichiometry of the species, whereas Page 2 shows configuration flags and optional arrays with empirical coefficients.

Please, note that 0 (zero) is a meaningful number in GEM-Selektor thermodynamic database. If the data item is not known then an Empty value  '---' has to be entered.


3.1. DComp Window Page 1

This simple page starts with a DCname comment line where a detailed name of this DC should be typed, i.e. "CaSO4 neutral aqueous complex".

The stoichiometry of any Dependent Component is its main property, which must be entered into the DCform field according to the chemical formula syntax. For instance, as "CaSO4@" for the CaSO4 neutral aqueous complex.

In the next row, the molar mass M0 and formula charge Zz of this DC are visible (calculated from the chemical formula).  In the ab field, optional parameters for aqueous complexes can be entered.

Below, the V0d, G0d, H0d, S0d, Cp0d  data fields are seen that contain the standard molar (or partial molal) properties of this DC at reference temperature and pressure. Each field consists of two cells. The first one (e.g. G0d[0]) contains the property, and the second (e.g. G0d[1]) may contain the uncertainty of this property or Empty value '---' as default.

The PrTr field below contains reference pressure and temperature (1 bar and 25 oC by default).

The LamST and BetAlp data fields may contain non-empty values for certain mineral compounds or surface species. Compressibility / expansibility for condensed substances can also be entered, either directly into  BetAlp fields, or as optional coefficients of  Vo = f (T,P)  equation (ai_Vtp data object on Page 2).

One or more bibliographic references to sources of data are usually specified in dSDref list; clicking on any of these and pressing F2 opens a SDref window with full contents of the reference located in SDrefrecord. If SDref record with this key (in dSDref field) does not exist, the user can enter the bibliographic data (and optionally the abstract) and save this SDref data to project database.


3.2. DComp Window Page 2

In the upper part of this page, the TPcMod data object shows the selected method codes for calculation of T,P corrections of thermodynamic data, along with configuration flags, the date of record creation, and dimensions of optional data vectors. All this can only be changed within the DComp Wizard by re-making the DComp record. 

For DCs such as gases, fluids, minerals, and non-electrolyte liquids, two data objects are usually present on  Page 2  of the DComp window. The aiCpT data object contains up to 11 polynomial coefficients of temperature dependence of the heat capacity function Cpo  = f (T), and the  TCint  field shows the temperature range (in oC) of applicability of those  aiCpT coefficients. The first three non-zero coefficients correspond to Majer-Kelly equation,  the first five non-zero coefficients - to Haas-Fisher equation, together covering most of the published databases [Nordstrom and Munoz, 1994].

More than one equation can be specified for one compound having e.g. polymorph phase transitions. If this is the case then in DComp remake wizard, one has to set a required number of columns to be created in the aiCpT and TCint data objects. In the latter, the temperature intervals must be set in the ascending order without gaps, as shown on the screenshot below for Hem (hematite) data from SUPCRT database. In this screenshot, the first phase transition (see FT_p[0] column) occurs at T= 676.85 oC, the second - at 776.85 oC. The whole range of applicability of Cp = f(T) coefficients extends from 0 to 1526.85 oC.




For (gaseous) components of non-ideal fluids, in addition to TCint and aiCpT data objects, the critical parameters should be entered into the optional CritPg vector which will be automatically allocated if 'R', 'T', '7' or '8' method code for P (pressure) corrections of thermodynamic properties has been selected in DComp wizard.

For aqueous ions and complexes from SUPCRT database, only the ai_HKF optional vector HKF EoS requires as input up to seven empirical coefficients. This vector is allocated if the first three of  TPcMod codes have been set to "HKF" in DComp wizard, as illustrated below.

4. Description of Input Data Objects and Fields

DCname - Full name of Dependent Component (species)
This is a string up to 70 characters long, entered by the user. It serves only for identification and commenting purposes on the DComp window page. Examples: 
       Ca+2 ion
       Portlandite (crystalline)
  
DCform - Chemical formula (stoichiometry) of Dependent Component (species)
This is the most important input data field on the DComp screen forms. Enter here a chemical formula of the species, expressed in symbols of Independent Components (IComp) currently available in the project database. The formula completely defines stoichiometry and charge of the Dependent Component. Formulae are parsed and analyzed in many modules of GEM-Selektor to be converted into lines of the stoichiometry matrix, for calculation of molar masses, phase/species compatibility checks etc. 
GEM-Selektor supports a simple and convenient formula syntax, very similar to the conventional chemical notation. Formula is a string of text, maximum 70 characters long. Only symbols of Independent Components (IC) from currently available IComp records can be used. Each IC symbol begins with an upper-case Latin letter, followed by zero to four lower-case Latin letters or _ (underscore) characters.
Explicit valence of IC can be specified within vertical bars ||immediately after the IC symbol, e.g. H2S|-2| (hydrogen sulfide).
IC symbols with stoichiometry coefficients can be grouped in brackets ([]) followed by a common stoichiometry coefficient, e.g. Th(OH)4. Up to 3 levels of brackets can be nested into each other. On translation of the formula, the stoichiometry coefficient after brackets will be applied to each of IC symbols inside. For instance, writing Eu2(CO3)3 is equivalent to Eu2C3O9. Stoichiometry coefficients (except charge) must be positive but not necessarily integer (decimal point notation is allowed, e.g. 2.67). No formula stoichiometry coefficient after the IC symbol means that this coefficient equals 1.
Atomic mass of an isotope must be specified in slashes // immediately before an IC symbol, e.g. /12/C.
The formula Charge (+, +2, ..., -, -2, ...,@) must always be provided in formulae of aqueous species and surface complexes.  Charge can also be non-integer and is always addressed to Zz  independent component.
For an end member of a multi-site (sublattice) solid solution, the information about moieties occupancies on structural sites can be given in the chemical formula. For example, the usual formulae of feldspar end members are KAlSi3O8  (microcline); NaAlSi3O8 (albite);  CaAl2Si2O8  (anorthite),  etc. However, such formulae do not contain information about sites and moieties that substitute on them.
This information can be incorporated by writing DComp formulae of Feldspar phase end members in the following way: 
{K}:{Al}{Si}3:O8    {Na}:{Al}{Si}3:O8     {Ca}:{Al}2{Si}2:O8
Essential is that each moiety (incl. vacancy Va) is taken in the {} braces; the colon ‘:’ separates sublattice sites which must follow the same order in all end members of the phase. The moiety site occupation numbers appear just as stoichiometry coefficients after the moieties. Upon parsing the formula, the number of sublattices is counted by encountered colons ‘:’ (two sublattices in the above example); and the moieties are collected in a list along with their site multiplicity numbers and the site index.
More examples of DC (species) chemical formulae:
Na+, Cl-, NaCl@                  aqueous species
SiO2, Fe3Si4O10(OH)2         quarts, minnesotaite
Fe|3|2O3                              hematite
/3/H2/18/O                              isotopic form of water
The DCform field is processed when you click on 'Re-Calculate' toolbar button, or "Record" "Calculate" menu item in the DComp window. You may get error messages if something is wrong with the formula stoichiometry or charge. Check carefully the formulae when you enter them in order to avoid nasty errors in chemical system definitions and GEM IPM3 calculations later on. 
   
Zz - Formula charge of this species (Dependent Component)
Charge per unit formula  (can be non-integer) to be entered by the user. For aqueous and surface species it is set automatically (upon clicking on "Calculate" icon or "Record" "Calculate" menu item) using the default- or explicitly specified valences of independent components. The following default valences are used for some common multivalent elements (see the respective IComp records): 
C  + 4, Cl  -1, S  +6, V  +5, Mn  +2, Fe   +2, Cu   +1, U   +4.
  
M0 - Molecular (gram-formula) mass, g/mol
Molecular mass of a compound is calculated automatically (upon clicking on "Calculate" icon or "Record" "Calculate" menu item) from its chemical formula  and atomic masses extracted from M0i fields in IComp records. There is no need to enter data into this field, but if you already know gram-formula mass of the species, to compare it with calculated M0 value would be a good test of correctness for the chemical formula that you have just typed into the DCform field.
   
V0d[0]- Molar volume of Dependent Component at standard state (J/bar)
Standard-state molar (partial molal for aqueous and surface species) volume V0  at reference pressure Pr  and temperature  Tr   must be entered by the user (except for gases) in units of measurement defined by the code in DCset[2] cell.
There is no need to enter V0   value for gases  because  V0  =  RT   = 2478.93 J/bar = 24789.3 cm3/mol for any ideal gas at P= 1 bar and  T = 25  C  and  it will be set automatically. Moreover, even though  V0  value has been entered, it will be automatically replaced (upon clicking on "Calculate" icon or "Record" "Calculate" menu item) by a value calculated for any non-gaseous species whose properties are defined by an equation of state (e.g., HKF EoS).  
  
V0d[1]- Uncertainty of standard molar (partial molal) volume
Absolute uncertainty of V0  value (see V0d[0] cell),  if known, must be entered by user in the same units of measurement as  V0  . If unknown, put an EMPTY value ("---").
In this version of GEM-Selektor, entering data into this field is desirable for data accumulation, to be used for assesments of uncertainty propagation that will be available in the next version.
  
G0d[0] - Standard molar (partial molal) Gibbs energy of formation of DC from IC at Tr and Pr.
The Gibbs energy  G0  at reference pressure  Pr  and temperature  Tr   must always be entered by the user in units of measurement defined by the code in DCset[1] cell, if one of the two related values - either  H0   or  S0   -  is unknown (i.e. the H0d[0] or S0d[0] field will contain "Empty value"). Remember that 0 (zero) is regarded as a valid (non-empty) value for thermodynamic properties! 
If both H0   and  S0 values are known (non-empty) and you entered  "Empty value" into this (G0d[0]) field then (upon clicking on "Calculate" icon or "Record" "Calculate" menu item) the value of G0  will be calculated automatically from H0   and  S0  using the  GHS_DC()  algorithm.
If all three valid numbers  G0  H0  and S0  were entered then (upon clicking on "Calculate" icon), the enthalpy value H*  will be found from  G0  and S0  using the  GHS_DC()  algorithm and compared to H0 . You will get a warning message that three values are inconsistent, if absolute difference  ¦H* -  H0  ¦  exceeds  40  J/mol. If this happens, check all three entered values - maybe one of them is misprinted, and check also chemical formula (DCform field) for stoichiometry coefficients, valences (default or explicit), and unit formula charge (Zz field). 
 
G0d[1] - Uncertainty of standard molar (partial molal) Gibbs energy of formation
Absolute uncertainty of G0  value (see G0d[0] cell),  if known, must be entered by the user in the same units of measurement as  G0  . If unknown, put an EMPTY value ("---") there.
In this version of GEM-Selektor, entering data into this field is desirable for data accumulation, to be used for assesments of uncertainty propagation that will be available in the next version.
 
H0d[0]- Molar (partial molal) enthalpy of formation of DC from IC at standard state and Tr, Pr
The enthalpy  H0  at reference pressure  Pr  and temperature  Tr   must always be entered by the user in units of measurement defined by the code in DCset[1] cell (e.g., J/mol), if one of the two related values - either  G0   or  S0   -  is unknown (i.e. the G0d[0] or S0d[0] field will contain "Empty value"). Remember that 0 (zero) is regarded as a valid (non-empty) value for thermodynamic properties!
If both G0   and  S0 values are known (non-empty) and you entered  "Empty value" into this (H0d[0]) field then (upon clicking on "Calculate" icon) the value of  H0  will be calculated automatically from G0   and S0 using the  GHS_DC()  algorithm.
If all three non-empty numbers  G0  H0  and S0  have been entered then (upon clicking on "Calculate" icon), the enthalpy value H*  will be found from G0  and S0  using the GHS_DC()  algorithm and compared to H0 . You will get a warning message that three values are inconsistent, if absolute difference  ¦H* - H0  ¦  exceeds  40  J/mol. If this happens, check all three entered values - maybe one of them is misprinted, and check also chemical formula (DCform field) for stoichiometry coefficients, valences (default or explicit), and unit formula charge (Zz field).
 
H0d[1] - Uncertainty of standard molar (partial molal) enthalpy of formation
Absolute uncertainty of H0  value (see H0d[0] cell),  if known, must be entered by the user in the same units of measurement as  H0  . If unknown, put the EMPTY value ("`" or "---") here.
In this version of GEM-Selektor, entering data into this field is desirable for data accumulation, to be used for assesments of uncertainty propagation that will be available in the next version.
 
S0d[0] - Absolute molar (partial molal) entropy of Dependent Component at the standard state
Third-law entropy  S0  at reference pressure  Pr  and temperature  Tr   must always be entered by the user in units of measurement defined by the code in DCset[1] cell (e.g., J/K/mol), if one of the two related values - either  G0   or  H0   -  is unknown (i.e. the G0d[0] or H0d[0] field will contain "Empty value"). Remember that 0 (zero) is regarded as a valid (non-empty) value for thermodynamic properties!
Important: Absolute entropy S0  of a compound is different from entropy of its formation from chemical elements  S0F  - please make sure that you enter  S0  value into this field!
If both G0   and  H0 values are known (non-empty) and you entered  "Empty value" into this (S0d[0]) field then (upon clicking on "Calculate" icon) the value of  S0  will be calculated automatically from G0   and H0 using the  GHS_DC()  algorithm.
If all three non-empty numbers  G0  H0  and S0  have been entered then (upon clicking on "Calculate" icon), the enthalpy value H*  will be found from G0  and S0  using the GHS_DC()  algorithm and compared to H0 . You will get a warning message that three values are inconsistent, if absolute difference  ¦H* - H0  ¦  exceeds  40  J/mol. If this happens, check all three entered values - maybe one of them is misprinted, and check also chemical formula (DCform field) for stoichiometry coefficients, valences (default or explicit), and unit formula charge (Zz field). 
  
S0d[1]- Uncertainty of standard molar (partial molal) absolute entropy of formation

4.1. Empirical Coefficients of Cp=f(T) equations

 
aiCpT - Array of empirical coefficients of  Cp0 = f (T )  equations 
Each column of this data object has 11 cells for input coefficients of temperature power terms in the isobaric heat capacity Cp0  = f (T  )  equation of the following general form:
 
Cp0 = a0  + a1T  +  a2 T ^-2  + a3 T ^-0.5  +  a4 T ^2  + a5 T ^3 + a6 T^4  + a7 T ^-3  + a8 T ^-1  + a9 T ^0.5
 
The last cell is not used (reserved). This equation utilizes most of published thermodynamic data. The first three coefficients correspond to Majer-Kelley equation, the first 5 coefficients - to Haas-Fisher equation (cf. [Nordstrom&Munoz,1994]).  Applicable for solids (from Tr  to T  at Pr ), gases (from Tr  to T  at Pr ), and aqueous species (from Tr  to T  at PSAT ) for calculation of their standard molar (molal) thermodynamic properties using the Schwarzman-Temkin functions.
Unspecified coefficients must be set to zeros.
Units of measurement for Cp0 : defined by code in DCset[1]. Real values must be entered (i.e. without assumed scale multipliers).
Value of Cp0  at Tr  = 25 C  is computed from the above equation coefficients every time when this DComp record is recalculated, and the result placed into  Cp0d[0] field. This is a good test for correctness of input - if calculated number appearing in   Cp0d[0]  is not withing the typical range (a few hundred J/K/mol, positive for minerals and gases) then some of the Cp coeffcients have been entered incorrectly and need to be fixed.

One of the fields  TPcMod[0], TPcMod[1], or TPcMod[2] contains invalid or unsupported code of the method of temperature/pressure corrections.

W07DCrun: Please, check stoichiometry, charge or valences in the formula
 
In the formula of aqueous or plasma species, the explicitly given charge is different from that calculated using (default or explicit) valences  of independent components. 
  
Action: Check stoichiometry coefficients and enter explicit valences after some IC symbols, if necessary; then try again.
 
 
W08DCrun: Inconsistent values of H0, S0 or G0 ->  <value>
 
This message appears after calculation of the DComp record if all three non-empty values have been entered into G0d[0],H0d[0] and S0d[0] cells, but these are inconsistent. The -> <value> suggests a consistent G0 value assuming that H0 and S0 values are correct. If you agree with that then press "yes" and the suggested G0 value will be inserted into the G0d[0] cell, previous value of which will be lost. 
 
However, most frequently the H0 values are inconsistent, or the S0 value can be misprinted. In this case, press "no", then put an EMPTY "---" value into either H0d[0] and S0d[0] cell, and calculate DComp again. The correct value will automatically replace the EMPTY value.

 
E09DCrun: Inconsistent DC standard properties - bailing out...
 
This message confirms that the user has replied "No" on the W08DCrun warning. 
 
Action: Correct the input data and try the calculation again.
  
 
W10DCrun: One of values G0, H0, or S0 is missing!
 
This message appears after calculation of the DComp record if all if only one non-empty value have been entered into G0d[0], H0d[0] and S0d[0] cells.
 
Action: Enter one more non-empty value and try the calculation again.
 
 
W11DCrun: Insufficient data to recalculate uncertainties. Continue?
 
The message appears if less than two non-empty values have been entered into G0d[1], H0d[1] and S0d[1] cells.
 
Action: Press "Yes" to avoid matching calculations of uncertainties, or press "No", enter missing values and calculate the record again. 
 
 
E12DCrun: Insufficient data for uncertainties - bailing out...
 
This message confirms that the user has replied "No" on the W11DCrun warning. 
 
Action: Correct the input data and try the calculation again.
  
 
E13DCrun: Invalid method code!
E14DCrun: Invalid method code or input data!
 
One of the fields  TPcMod[0], TPcMod[1], or TPcMod[2] contains invalid or unsupported code of the method of temperature/pressure corrections, or the input data are insufficient for a chosen combination of methods
 
Action: Remake this record and reconsider method codes or provide sufficient input data.

 
E15DCrun: PARCOR error:  If Z!=0, either S or Rx must be given!
 
Internal error in the PARCOR prediction of HKF EoS parameters. 
 
 
E16DCrun: In database chain  <id> Data record not found, key <key>
 
This message can appear if the chemical formula parser could not find an IComp record for a symbol of IC. Maybe, a proper IComp database file is not linked. The message asks whether you would like to create a new IComp record for this IC (if so then press "Yes"), or check the DC formula and/or the file configuration in the IComp chain (press "No"). The latter sutiation is much more typical. 
 
 
E17DCrun: New record creation dismissed...
E18DCrun: Record creation dismissed...
 
These messages confirm the rejection answer ("No") to the E16DCrun message. 
 
 
E19DCrun: This record alredy exists!
 
 You have attempted to create a DComp record entering the new key that already exists in the current database chain. 
  
Action: Create the record again and give a different key (best, use the "From list..." button, select one of existing keys and modify it). 
 
 
W20DCrun: Remake of the new record finished Ok
 
This confirms that create/remake of a DComp record caused no problems. It is recommended to re-calculate the data before saving the record to database. 
 
 
W21DCrun: Calculation failed!
 
Generic warning that something was frong with calculation if the DComp data. 
 
Action: check data fields and try calculation again.
 
 
E22DCrun: Failure!!! Database chain <id>  is corrupt, data record key  <key>
  
This is a serious error pointing to a corruption of program memory or a database file. 

Action: First, re-start the GEMS and try again. If the problem persists, try to compress files in this database chain (you may lose corrupted records and will have to restore them from backup files). If this does not help, please, send a bug report with the database file(s) that cause the problem. 
  
 
E23DCrun: Undefined twp!
 
Internal error in SUPCRT92 interface (memory corruption). Please, send a bug report. 
 
 
E24DCrun: (TdegC > TMAX+TOL) || (Pbars > PMAX+TOL)!
 
In the SUPCRT92 calculation, temperature T or pressure P go beyond feasible intervals (0 to 1000 oC and 0 to 5 kbar, respectively)
  
Action: Fix the T or P input values (or Pr, Tr) and try again.
 
 
E25DCrun: gfun92()- error in HGK calculations
 
 Internal error in SUPCRT92 calculations of properties of H2O.
 
 
E26DCrun: This DComp record does not have HKF EoS coefficients!
   
 The method codes suggest SUPCRT92 calculation of temperature/pressure corrections, but the vector aiHKF of HKF EoS coefficients contains empty values, or it is not allocated. 
 
Action: remake this DComp record, supply HKF coefficients and try again. 

 
E27DCrun: TpworkList: Invalid number of TPWORK structures!
 
Internal error in temperature/pressure correction calculations. Please, send a bug report. 


5.1. Error messages from the Formula Parser

These error messages may appear during processing of chemical formulae in calculations of DComp, ReacDC, Compos records, as well as during creation or opening of modelling projects. If you work in "Thermodynamic Database" mode of GEM-Selektor, it is usually sufficient to edit the formula field (e.g. in DComp window page) and calculate the record again. If an error message appears when you work in "Calculation of Equilibria" mode, then you have to find out from which DComp, ReacDC or Compos record the formula has been taken, then switch to the "Thermodynamic Database" mode, open the record and edit the formula there.

E30FPrun: A symbol of element expected here!
 
Syntax error in writing an IC stoichiometry coefficient (in the formula fragment shown).

Action: Check stoichiometry coefficients after element symbols or brackets; 
            also, check explicit valences (in || vertical lines).
   
 
E31FPrun: Errors in scanning chemical formula
 
A generic message, usually follows another error message(s) from the formula parser. 

Action: Edit the formula and try again. 


E32FPrun: Null length of the formula string
 
No characters where a chemical formula was expected. 

Maybe, you forgot to type in the formula text. If not then the dynamic memory may be corrupt (restart GEMS and try again; if it does not help, please, submit a bug report).


E33FPrun: This is not a symbol of IComp!
 
The currently processed (shown) IC symbol is syntactically correct, but not found in the list of currently available IComp records.
  
Action: check if all necessary IComp files are linked in (open the IComp window), 
            or correct the misprinted IC symbol and try again.
 

W34FPrun: Charge disbalance in the formula: <text> calculated charge: <value>
 
In the formula of aqueous or surface species, the explicitly given charge is different from that calculated using (default or explicit) valences  of independent components. 
  
Action: Check stoichiometry coefficients and enter explicit valences after some IC symbols, if necessary; then try again.
 

E35FPrun: Error in the formula list!
 
This generic message may appear when a project formula list is processed during opening of the modeling project. 
 
 Action: Notify the DComp/ReacDC record that caused the problem; switch to "Thermodynamic Database" mode; edit the formula, calculate and save the record to database; switch back to "Calculation of Equilibria" mode; and try opening the project again.


E36FPrun: Invalid formula length!
 
 The length of formula exceeds 80 characters. May point to memory corruption (restart GEMS and try again; if it does not help, please, submit a bug report). . 


E37FPrun: Invalid  symbol of IComp <symbol> in formula in DComp/ReacDC record: <key>
 
This message may appear during opening of the modeling project, if the IComp <symbol> is not included in the project list. 

Action: Notify the DComp/ReacDC record that caused the problem; switch to "Thermodynamic Database" mode; edit the formula, calculate and save the record to database; switch back to "Calculation of Equilibria" mode; and try opening the project again. If the formula is Ok then the modelling project is corrupt or needs to be remade.
