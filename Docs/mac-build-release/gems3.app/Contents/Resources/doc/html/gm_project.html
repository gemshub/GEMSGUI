1. Introduction: Project Structure and Record

Modelling Project is actually a sub-folder in GEMS "/projects" folder, which contains GEMS database files that keep all settings and data pertinent to a modeling application. In particular, this sub-folder contains a "template" definition of the chemical system and numerical settings (Project file); the input thermodynamic and composition data records (IComp, DComp, ReacDC, Phase, Compos files); the records defining particular systems and their calculation results (SysEq), as well as Process simulator (Proces file), Graphic/tabulated data sampler (GtDemo file) records, and reactive transport problems (GEM2MT file), references and print/export scripts (SDref file).

Each project has one "project" record in the "project" database file with "template" chemical system definition, together with links to a collection of GEMS database files where the input data is kept and modelling results will be collected. As a rule, these database files are located in the same project subfolder.

There are two types of projects: (1) standalone and (2) linked. When a standalone project is created, the program copies all necessary data from the default database (supplied with the program) into the project database files and then disconnects the project from the default database. Such projects can be shared between many people or can be presented as test examples, regardless of the particular version or update of the default GEMS database. When a linked project is created, the program creates project database files only for storing the input data extensions and the modelling results, but  just links the default database files for the extraction of input thermodynamic data. Consequently, behavior of the models in such a project may change if they will be run within a different version of GEMS code and its default thermodynamic database. We recommend creating standalone projects whenever possible, even though it requires some more work for updating the copied ReacDC records. 

The largest part of the Project record consists of several interconnected lists of record keys:

  • IComp list ICvp (defines the stoichiometry basis of the system in terms of Independent Components)
  • DComp/ReacDC list DCvp (chemical species available in this stoichiometry basis)
  • Phase list PHvp (phases that potentially may appear at equilibrium states)
  • Compos list CCvp (PCO that facilitate setting up the total bulk composition of the system).

In addition, the Project record contains a MSfls list of 'nicknames' of all database files linked to this project, where the program can seek for the records with input data or to store the modelling results. All these lists are assembled automatically when the user creates a new project, by picking up all records available in all currently linked database files. Next time when you switch to "Calculation of Equilibria" mode of GEMS and open the project, its Project record will be read from the database; the program will link database files according to the MSfls configuration list; and then it will read the data from the database files and compare it to what has been previously stored in the project record lists. This is a quite efficient way to detect what has been changed, added or removed from the project database after the last modeling session, and to update the project itself and already calculated systems accordingly.  If you have added/removed some species or phases,  the program will display warnings (there is a probability that you did not really want it!). Just press "Ok" or "Ok to all" buttons if you are sure that these changes are correct. 

Thermodynamic data array is another important part of the project system framework. It is allocated in the program memory after the project has been successfully opened; but thermodynamic data arrays are never saved in the Project database record. Thermodynamic properties (by default, molar Gibbs energy gTP and molar volume vTP) of all species referenced in the  DComp/ReacDC list DCvp are automatically calculated every time when you specify different temperature and pressure in a SysEq record key. This is done by reading all the DComp or  ReacDC records in a sequence and applying necessary calculation subroutines separately to each record. Thus computed thermodynamic data for T,P of interest will be retained without further re-calculation during the whole modelling project session, or until different temperature of pressure will be specified for a currently loaded SysEq record (usually just by reading it from the database). In the Project Wizard, you can allocate some additional arrays (e.g. molar absolute entropy sTP, molar heat capacity CpTP, molar enthalpy hTP), if necessary.  

Location of the /projects folder (where all projects will be stored as subfolders) is defined by the  -u  command line parameter of gems3 executable file (you can see this location in the Preferences Dialog). For instance, "-u D:\GEMS3test" in the command line tells the program to look for existing projects and to store a newly-created project "MyProject" in "D:\GEMS3test\projects\MyProject" folder. This new project is as a collection of database files, all containing "MyProject" as a file name element. If the -u parameter is not given then the program will look for projects at the default location, "/Library/gems3/projects". In Windows and Linux variants, this location is at the same level as the location where the gems3 executable file resides (i.e. "/Gems3-app" in an upper-level folder under Windows or in user's home directory under Linux). In the Mac OS X variant, the default location of executable is "/Applications/Gems3.app/Contents/MacOS" and that of user's projects - "/Users/<user>/Library/gems3".

When creating a new project, the program uses the Default database files, located in the "/Resources/DB.default" directory. In Windows and Linux variants, the folder "/Resources" (which also contains this help database) is installed inside of the "/Gems3-app" folder at the same level as the gems3 executable file. In Mac OS X variant of GEMS, the folder "/Resources" is part of the Gems3 application, i.e. "/Applications/Gems3.app/Contents/Resources". A different location of the folder in which the executable file and "/Resources" are found (you can see this location in the Preferences Dialog) can be specified by the  -s command line parameter. For instance, "-s D:\GEMS3test\Gems3-app" in the command line tells that the default database is located in  "D:\GEMS3test\Gems3-app\Resources\DB.default" folder.

There are two main types of modelling projects (and, accordingly, two project setup modes) that can be selected in the Preferences Dialog. 

A "linked" type project links selected default database files to the project during its whole lifetime. When any equilibria are calculated, the input phase definitions and thermodynamic data are collected directly from the default database files; only records added by the user (or created by modeling calculations) will be stored in the project database files.

A "standalone" type project (default in GEM-Selektor v.3) links default database files to project only at the stage of its creation or upgrade. The default database is searched for all species and phases records conforming with the chosen list of Independent Components and other selection criteria in the ElementsDialog, and such records are immediately copied into the project database files. When creation of the project is finished, all default database files are disconnected,  and the project becomes standalone, i.e. independent of the default database. During the project lifetime, the user can change, delete or add any phase definition, composition, or thermodynamic data records at her/his discretion. Such projects will not be affected by any future changes or extensions of the Default database and can be exchanged between different users. Another advantage is that the system template in an independent project is relatively small and easier to observe because it usually involves only a small part of the default database.

1.1. Modes of Geochemical Modeling within a Project

At present, there are three levels of calculation of equilibrium states within a modeling project:

  • Single System
  • Process Simulation
  • Reactive Transport Simulation

They are described, respectively, in the gems_sys / gems_bcc; gm_proces; and gm_gem2mt help manuals.


1.2. Creating a New Modelling Project

To create a standalone project, get into the "Modelling projects" dialog and click the "New Project" button. This will bring you to a small "Enter a new record key" dialog. Type a short project name instead of "MyWork" (or click "From List" to pick up one of existing Project record keys and modify it); change the comment line (spaces are not allowed), and press "Ok". 

This will bring you to Step 1 of a wizard for checking selections of source database/subdatabase, Phase/DC filters, and then to a large Elements Dialog that looks like the Periodic Table. Select necessary elements (and other Independent Components) and press "Ok" when ready. Follow warnings and messages that will appear on the screen, as described in more detail in the GEMS projects HOWTO manual.

To create a new "linked" project is more complicated because in this case, you cannot use the ElementsDialog and you must know what is contained in various Default database files in order to make a proper selection. Proceed as described in more detail in the GEMS projects HOWTO manual.

Attention! For introducing the GEM-Selektor v.3.3.x (and respective v.3.3.x of the GEMS3K code) in April 2015, the Phase record and data definition structure, as well as the Phase remake wizard, were substantially extended in order to accommodate more flexible mixing models (TSolMod), built-in mineral-water reaction kinetics (TKinMet), and a new implementation of sorption and surface complexation models (TSorpMod; work in progress). 

For backward compatibility, all modelling projects created in GEM-Selektor v.3.2.x or earlier, will open and work normally in GEM-Selektor v.3.3.x; accordingly, GEMS3K I/O files exported from GEM-Selektor v.3.2.x are readable and will work in coupled codes and examples using GEMS3K v.3.3.x. A prerequisite of this is that no Phase record is created or remade in such a project with the "old" record format.

Extension of old modeling project in GEM-Selektor v.3.3.x with newly created Phase records, remaking existing Phase records, or via Project remake/extension wizard is also possible. Before doing that, the user needs only once to open the project; switch to Database Mode; open the Phase window; and remake one-by-one all Phase records available in the project (saving each record back after remake). This will ensure that new or modified phases with extended structure and functionality will work correctly.

Note that if some Phase records remain in the "old" format (i.e. not remade) then this will lead to errors or crashes during calculations of equilibria in systems from this project. So, it is essential that ALL Phase records go once through the Phase remake wizard! Opening and closing the modeling project even once in the GEM-Selektor v.3.3.x will render this project unusable in earlier versions of GEM-Selektor (3.2.x or less).  


2. Project Remake Wizard

Modelling project is a collection of GEMS database files for a specific range of modeling applications, located in an (automatically created) folder having the same name as the project. These files contain records of different types (components, phases, compositions, chemical system definitions (CSDs), results of equilibrium calculations, process and reactive mass transport simulations, etc.); taken together, they comprise a project database (more...).

Main part of the modeling project is a generic definition of chemical thermodynamic system, stored in a single Project record (having the same name as the project). This record includes lists of record keys for Independent Components, Dependent Components, Phases and compositions kept in the project database.  The Project record also contains links to database files where the input data is kept and the modelling results will be stored, as well as the numerical controls to be used in GEM calculations within this project. These contents are accessible through four pages of the Project module window (more...).

The project system image in computer memory also includes arrays of thermodynamic data of all kinds to be used in calculations of equilibria or in process or reactive transport simulations. These data arrays are accessible in "Equilibria Calculation" mode through "Data" "Thermodynamic Data" menu command; they are updated automatically when temperature or pressure has changed in the SysEq record key, or in the Process or GEM2MT module calculation.


2.1. Project Wizard Step 1

Step 1 - Project configuration (thermodynamic data and numerical settings)

At this step, some additional arrays of thermodynamic data for Dependent Components can be allocated, and pre-configured sets of GEM IPM3 algorithm numerical controls can be selected for the current project.

Within the group "Allocation flags for optional DC thermodynamic data arrays",  activation of one or several check-boxes will result in allocation and calculation of selected thermodynamic properties of Dependent Components. gTP (molar Gibbs energy function) and vTP (molar volume) are always allocated and used by default in calculations of equilibria by IPM3 algorithm and, for this reason, do not appear in this list. Arrays marked as 'reserved' are not yet implemented in this version of GEM-Selektor; checking their boxes on will have no effect. 

The drop-down list "Change pre-configured sets of GEM IPM3 numerical controls" contains several options as listed below:

Option

Comment

Retain previous controls (for old projects) or take default

This option (default) leaves numerical settings unchanged in the existing project, or takes the next option (normal standard) in the newly-created project.

Restore normal (standard) variant for fast convergence

Use it for projects where no strong non-ideal interactions are expected and no special mass balance precision is required.

Variant for aquatic systems with moderate non-ideality

The same as normal (standard) variant, but optimized for aquatic systems at moderate ionic strength.

Variant for aquatic systems with SCMs in sorption phases

This option has smoothing parameters necessary for computing surface complexation models with Langmuir-type or similar surface activity coefficient terms.

Variant for fluid-rock systems with highly non-ideal phases

Select this option if in your project systems there are complex non-ideal solid solutions with a chance to form miscibility gaps

Variant of strict settings for highest accuracy (may be slow)

This set of numerical controls should be used in reactive transport calculations (in GEM2MT module or exported in GEMS3K input files) where mass balance residuals must be minimal.

Variant of strict settings for reactive mass transport (with SCM)

The same as previous option, but with smoothing parameters set for surface complexation models.

After the project is opened and GEMS goes into "Equilibria Calculation" mode, the numerical controls can be checked and edited in the Project module window (more...).

Important note: when any modeling project created in older versions of GEMS (GEMS2 or GEMS3 before August 2011) is opened for the first time, it is mandatory to choose in this drop-down list any appropriate option except the first one ("Retain previous settings"...). This is because values of some old numerical controls are incompatible with the current version of GEMS3K kernel code.


2.2. Project Wizard Step 2

Step 2 - Project configuration (units of measurement)

There is no need to change settings in this wizard page.

After this wizard is finished:

The modeling project will be opened, the GEMS code goes into "Equilibria Calculation" mode.

Upon entering the "Equilibria Calculation" mode, the project configuration will be saved in a Projec record in the project folder.

Additional changes of settings can be done in Controls, Settings, and Config pages of the Project module window. This will affect newly created SysEq, Process, ... records; previously created records will remain intact. Save the Project record to keep changes for the next project session.


3. Project Module Window

This window provides 'fine-grained' access to project controls and settings through its four pages (screenforms) described below.

The Project window can be opened using "Modules" "Project" menu command or clicking on the "Project" left toolbar icon.


3.1. Project Window PLists (Project System Lists) Page

This (information) page displays (more details here):

MSfls - the list of 'nicknames' of files currently linked to the project (more...);

ICvp - the list of record keys of Independent Components comprising the stoichiometry basis of generic chemical system definition;

CCVp - the list of Compos record keys of Predefined Composition objects (PCO) used in constructing the recipes of chemical systems;

DCvp - the list of record keys of Dependent Components used in Phases included into generic chemical system definition; on the right-hand side, 'd' indicates DComp and 'r' indicates ReacDC record key; 

PHvp - the list of Phases comprising the generic chemical system definition; on the right-hand side, the number indicates how many Dependent Components from the DCvp list are grouped into this phase definition (in strict order, i.e. first all components of the first phase, then all components of the second phase, and so on).

Note that some ICs, DCs, and Phases may be switched off in specific SysEq records (i.e. they do not appear in equilibrium calculation results).  


3.2. Project Window Controls (GEM IPM Numerical Controls) Page

This window can be important in the case of numerical problems with calculation of equilibria (lack of convergence, failure in mass balance, divergence of dual solution etc.). If this happens, some adjustments of GEM IPM numerical controls may be necessary.

Typically, this page looks like shown below. The title field (Pa_SPP) indicates revisions of GEMS and the kernel code under which the set of controls has been reset last time. If this is outdated, it makes sense to close the project and open it again in the Project Setup Wizard mode (more...), then select a predefined set of controls as described here. Typically, this results in good numerical convergence. If this did not help, you may need to adjust some controls individually. In the Controls page, the respective fields are arranged beginning with the controls that are more frequently modified (for instance, Pa_DK, Pa_LLG, Pa_DHB).


If you modify some controls and save the Projec record then these modifications will be available in the next sessions for this project; otherwise, the modifications will be lost when the project is closed or another project is opened.

See also explanations of specific numerical controls of the GEM IPM3 algorithm in gems_ipm.html file.


3.3. Project Window Settings (GEMS Default Settings) Page

This page shows default values and flags that GEMS uses when creating records of different types.  Do not change them unless you know a very specific purpose.

See more details in gems_miscel.html file.


3.4. Project Window Config (Project Configuration) Page

This page shows (for troubleshooting purposes) the flags and dimensions of the modeling project definition (stored in the Projec record).

The data fields here are not editable and can be changed either automatically during creation or extension of the project, or through Project Wizard.

Flag codes:    '+'  allocate (on)    '-'  remove (off)    '*'  allocate and set default values.

This flag allocates a DCvp list of available DComp and ReacDC record keys for all species (dependent components) in all thermodynamic phases included into the GEM modelling system. Default code is on. This list cannot be switched off. 
   
   
MSpmv[3] - Allocation flag for the list of IComp records { + * }

4.2. Lists of phases, components  and compositions

L1vp - L1 vector, shows a number of DC included to each phase
 
This vector contains FI elements. Each element shows how many species (DC) are included in the respective phase. A rule in GEMS is that multi-component phases appear first in the list.
 
 
PHvp - List of Phase record keys
 
This list of Phase record keys links phases that can appear at equilibrium states in systems constracted from this project multisystem definition with the respective records in GEMS thermodynamic database. 
 
Third field of each Phase record key (phase name) is used by the Math Script preprocessor to convert phase names into indices in the project multisystem framework.
   
 
DCvp - List of DC record keys (of Dcomp or ReacDC records)
 
This list of DC record keys defines the generic list of species that can appear at equilibrium states in systems constructed from this project multisystem definition. Components are listed in the same order as they appear in  phases as listed in the PHvp Phase list, i.e. all species for the first (usually aqueous) phase, then all components for the second phase, etc., as counted in the L1vp vector. 

Third field of each DC record key (species name) is used by the by the Math Script preprocessor to convert species names into indices in the project multisystem framework.
 
 
CCvp - List of Compos record keys
 
This list of Compos record keys defines PCO that conform to the stoichiometry basis (ICvp list) of the project multisystem definition. Any of these PCO can be used in calculation of the bulk composition (b vector) in the CSD (see gems_bcc.html manual). 
 
First field of each Compos record key (PCO name) is used by the Math Script preprocessor to convert PCO names into indices in the project multisystem framework. 
  
 
ICvp - List of IComp record keys (stoichiometry basis)
 
This list of IC symbols (IComp record keys) defines the stoichiometry basis of all chemical systems which can be constructed using this project multisystem definition. 

First field of each IComp record key (IC symbol) is used by the Math Script preprocessor to convert IC symbols into indices in the project multisystem framework.
   
 
MSfls - List of database files currently linked to this modelling project
This list is made of the automatically constructed identifiers of database files, e.g.  "sdnagra-psi". 

Created automatically when the project is created or re-configured. 
   
DCSvp - Code of data source for DC: d-Dcomp r-ReacDC { d r }
This column (created automatically) defines the database chain from where the program reads the respective record pointed by the record key in the DCvp list.  
  
mPHC - Class codes of phases { a g p m s l x d h }
  
 This column (created automatically) contains the Phase state codes, internally used in GEM IPM procedures.   
 
mDCC - DC (species) class codes { T E S W G V C H N I J M [digit] X Y Z A B P Q R O }
 
This column (created automatically) contains the Species class codes, internally used in GEM IPM procedures to select correct equations for chemical potentials, activity coefficients etc. .
  
  
mICC - Type codes for IC { e o h a z v i [integer]}
 
This column (created automatically) contains the IC type codes, internally used in GEM IPM procedures.
 
  
For_DC - List of DC chemical formulae (comma separated)
 
This automatically created list of DC (species) chemical formulae is the most economical way to store the  stoichiometry matrix A of the project multisystem into a Project database record. Formulae go in the same order as species names in the DCvp list. Whenever necessary, the formulae are extracted and automatically converted into rows of the stoichiometry matrix. Comma must follow each formula, except the last one. Comments are not allowed in this list.
   
  
For_IC - List of IC chemical formulae (reserved)
  
 
ICDCv - List of IC-to-DC references (reserved)
  
 
Vale_ - Vector of default valencies of IC
   
This vector is used internally when converting formulae from the For_DC list into the rows of the stoichiometry matrix, in order to avoid reading of IComp records.

This vector is used internally to calculate molar masses of Dependent Components when converting formulae from the For_DC list into the rows of the stoichiometry matrix,  in order to avoid reading of DComp and IComp records.

 Reserved for use in future versions.

"HowTo" Manuals

6. How to Create, Open or Expand a GEM-Selektor Modeling Project

GEM calculations of chemical thermodynamic equilibria can only be performed in the context of a currently open modelling project (more...).

To access modeling projects, switch into "Computation of equilibria" mode by clicking the Equilibrium Mode   button (or select the "Modules" "Change Mode" menu command). This opens a "Modelling Projects" dialog, which usually shows a list of accessible existing projects. Each project (record) has a key that consists of two fields: Name and Comment. The Name is also the name of the project folder; the Comment  gives a hint on the modeling application contained in the project.

To open an existing modeling project, double-click on its key in the list, or click once on it and then click on the "Open Project" button. Below the list, there are several options, some of which affect how the project will be opened (more...). After the project is loaded, GEM-Selektor turns into the "Equilibria Calculation" mode with "Single Thermodynamic System" dialog visible (more...).

Attention! For using in GEM-Selektor v.3.3.x introduced in April 2015 the modelling projects created in GEM-Selektor v.3.2.x or earlier, please read carefully here (Section 1.3) and follow advices there.  

To create a new modeling project, click on a "New Project" button, enter the name and the comment into the appearing record key dialog, and click "Ok" (more...). In the appearing Step 1 page of the wizard, check the selection of Phase/component filters and the selection of source database/subdatabase. In the appearing next Step 2 page "Elements Dialog", select elements (Independent Components) to form the system (more...) and click "Ok". In the appearing "Setup of aqueous and gas phases" dialog, optionally change settings and click "Ok" (more...). This brings GEMS into the Equilibrium Calculations mode; the SysEq record key dialog appears for the first system in the new project. Click "Ok" to proceed with the default key, or edit the name, pressure, or temperature, and then click "Ok". In the appearing "Input Recipe" dialog, type a comment to this system definition in the 'tname' field, then make selections and enter numbers for the input of bulk composition (more...), and click "Ok". Now check, calculate, and save the first equilibrium state in the project (more...). Proceed with 'cloning' the next SysEq record.

To expand (clone) a project into a new one: activate the option "Create a new project using the selected one..."; select the project in the list; and click the "New Project" button. In the appearing record key dialog, enter a new project name and a comment, click "Ok"; in the Elements Wizard Step 1, if necessary, check selection of source database/subdatabase, then at Step 2 select additional elements, and proceed as by creating a brand new project (more...).  

To share a modeling project with other users of GEM-Selektor (or to back-up it): locate the user modelling projects directory (use "Help" "Preferences" menu command to open Settings and Preferences dialog and see this location path under "User's projects folder"). Close GEMS, locate the project and zip it as a whole subfolder. Place this zip file in backup folder or send it to somebody else. To add a project, unzip it's folder into your projects folder, then start GEMS; you should see it in the projects list. We recommend to backup the project folder that you currently work with at least one- or two times per day.

To remove a modeling project: exit GEMS; use file manager or terminal to locate your projects subfolder; select the project folder there; delete it or move it away completely. 


6.1. Main Settings to Create or Open a Project

To create a standalone project, get into the "Modelling projects" dialog and click the "New Project" button. This will bring you to a small "Enter a new record key" dialog. Type a short project name instead of "MyWork" (or click "From List" to pick up one of existing Project record keys and modify it); change the comment line (spaces are not allowed), and press "Ok". This will bring you to a large Elements Wizard that has two steps, one for checking the selection of source built-in database or sub-database and/or Phase/component filters, and another showing a dialog that looks like the Periodic Table; see the next section (also for the comments on how to create a "linked" project without using the Elements Dialog). 


6.2. Elements Dialog and Selections from the Default Database

In this dialog (Step 2 of the Elements Wizard), you can select elements (and other Independent Components) to form the stoichiometry basis of the modeling project. By default, the program will look through default (selected at Step 1) database/subdatabase files, find all dependent components, phases and compositions that are consistent with the chosen list of Independent Components, and copy these records into respective files of the project database. When finished, the built-in database files will be disconnected. The user can then modify and extend the project database, but this will have no effect on the default database; if any changes occurred in the built-in database after the program package upgrade, this will also have no effect on the project database. This kind of 'independent' modeling projects is typical, and it is safe for most applications.   

However, it may be necessary to work with projects that are directly linked to default database files, for instance, to have any changes in default (kernel) database immediately affect calculations within the project. It is less simple to create such a new "linked" project because in this case, you cannot use the Elements Dialog, and you must know what is contained in various kernel database files in order to make a proper selection. It is recommended to proceed in the following steps creating such a new project.

  1. Define the list of independent components (IC) necessary for describing stoichiometry of the system. Find out which IComp database files contain records for all required independent components. Define which phase types (aqueous, gas, solid (solutions), sorption) must be included. Check which GEMS default thermodynamic database files are to be linked to your new project.

  2. Start GEM-Selektor and go into "Computation of Equilibria" mode. In the Modelling Project dialog, activate the option "Make a new project by linking files from the default database", then click on the "New Project..." button to create a new linked project.

  3. In the appearing project record key dialog, enter the new project name and comment, and press "Ok". Now, a "Database File Configuration for Modelling Project" dialog appears (more...). Expand the "Database" entry and then expand the "Kernel" entry. There, expand one-by-one the "sdref", "icomp", "dcomp", "compos", "reacdc" and "phase" entries and in each of them, click on the files to link/unlink them to/from the project. Press "Ok" when finished, and confirm your file selection by clicking on the "Yes/Proceed" button. 

  4. In the appearing "Select an aqueous activity model" dialog, select the desired aqueous model; next, do so in the appearing "gas mixture model" dialog. You should see now the "Single Equilibrium System" dialog; from now on, the project is operational, and you can create and model chemical systems equilibria and store them in SysEq records. 

Note that any "linked" project will also contain a full set of project database files in the project folder, but, unlike those in an independent project, they will be empty. However, any record created in the process of work within the project will never be stored in the default database, but only in the project database files.

 

6.3. AutoPhaseSetup Dialog and Configuration of Aqueous and Gas/Fluid models

This wizard appears when a new modeling project is created, or an existing project is opened after deactivating the "Retain setup of aqueous (and gas/fluid) phases" option. The dialog facilitates definition of the aqueous electrolyte phase and/or the gas/fluid mixture phase that are automatically assembled upon opening  a GEMS modelling project. Alternatively, a Phase record already available in the project database can be selected, or the aqueous phase (or gas/fluid mixture) can be excluded from the template chemical system definition in this modeling project. More about this can be found here.

6.6. Cloning a New (Expanded) Project from the Existing One

It is not possible to duplicate a modeling project simply by making a copy of its folder under a different name. A special mode of GEM-Selektor can be used for this purpose. In this mode, either an exact copy of the project with a different name can be obtained, or a 'cloned' project can be expanded by adding one or more Independent Components (respectively, more chemical species, phases, and compositions) from the default database supplied together with the program package.

To expand (clone) a project into a new one, go into "Equilibria Mode" and in the Modeling Projects dialog, activate the option "Create a new project using the selected one..."; select the project in the list; and click the "New Project" button. In the appearing record key dialog, enter a new project name and a comment, and click "Ok".

You are how in the Elements Dialog, the same as that appearing when a new project is created (more...). The difference is that the Independent Components selected in the original project are now visible as pressed buttons that cannot be deselected. You can simply do nothing and click "Ok" to produce a copy of the original project with a different name - this makes sense, for instance, to "freeze" the results at certain stage e.g. for reporting, or to produce a GEMS test project case (by deleting some system or process records, adding comments and descriptions, and so on).

However, a more typical case would be to add some elements to the new project while retaining setups and results that have already been done. Many users tend to begin with relatively simple systems and extend them at later stages, perhaps, using data from a 'specific' default database (this needs first a re-configuration of database access as described here). Select additional elements and click "Ok" to proceed. This will get you through the "Setup of aqueous and gas phases..." dialog, then the program will ask you for confirming insertion of new IComp, DComp, ReacDC, Phase and Compos records into the project database. You can accept all insertions to be done automatically (recommended option) or review each action one-by-one (option for experts). The list of automatically or manually rejected records can be found in the file "DiscardedRecords.txt" in the same folder where GEMS3 executable file is located. As the last step, the program will update internal indexation in all existing SysEq records in the project in order to make the newly inserted components and phases accessible in the system definitions; this can take time if there are many SysEq records to update. 

When the cloning/expansion process is finished, the "Single System" dialog appears (more...), where you can check, re-calculate, and save the existing systems (SysEq records), or clone/create new ones. Note that, after the project has been expanded, existing SysEq records must be re-checked and re-calculated before the equilibrium speciation and dual solution can be viewed again.

As an alternative, it is always possible to extend an existing project with more Independent Components, Dependent Components, and Phases without cloning it (more...).


6.7. Other Issues Related to GEMS Modelling Projects

To take a look, switch to your favorite file manager and find the location where the GEM-Selektor is installed (for instance, "C:\MyGems"). Depending on the operation system, your projects are located typically in the folder "/Library/gems3/projects" (in Windows, "C:\MyGems\Library\gems3\projects"; in Linux and Mac OS, "~/Library/gems3/projects"). Let us assume that the project "Carbonat" just has been created. Any binary GEMS database file, as such, has a .pdb file name extension; the file with the same name but a .ndx extension contains indexing information (record keys with references to record locations in the .pdb file) to speed up search in the database. The second element of the file name is always the same as the project (folder) name ("Carbonat"). The first element of the file name is a GEM-Selektor module name that can process the records stored in this database file. In the left column, all files keeping the project input thermodynamic and composition data are shown. In the right column, the file projec.Carbonat.pdb keeps the project system structure lists and configuration (there is just one record "Carbonat:My1stProject" ); the file syseq.Carbonat.pdb will collect all the records with calculated equilibrium states (here still empty).

Files proces.Carbonat.pdb and gtdemo.Carbonat.pdb  will contain your Project Simulator and Graphic/table Demo Sampler records, respectively, and sdref.Carbonat.pdb file will keep your SDref records with bibliographic data or customized scripts. These files are also empty at the beginning.

Note that all your work done on this modelling project will be stored exclusively into .pdb files in the project directory, and nowhere else. Thus, you can backup your work any time simply by zipping the whole project directory; this zip file (e.g. Carbonat.zip) can be sent to your colleague with whom you would like to share your models. You are welcome to send it  also to the developers of GEM-Selektor in the case of bug reporting or if you wish to contribute to the GEMS Kernel database, test suite, or tutorials.

As seen in the left column of the above table, icomp.*, dcomp.*, reacdc.*, phase.* and compos.* files in the project directory are not empty. During creation of the new project, GEM-Selektor used the list of selected independent components (Ca, Mn, Fe, C, Na, Cl, O, H, Zz)  first to copy the respective IComp records to icomp.Carbonat.pdb file. Then the program searched the whole accessible part of the default database for such Dependent Components and compositions that would completely fit into this stoichiometry basis; the conforming DComp, ReacDC, and Compos records were also copied from the default- to the project database files. Next, the program checked all the Phase definitions available in the default database against the lists of copied DComp and ReacDC records and discarded all phases that contain at least one species not conformant with the project system stoichiometry basis. The record keys of discarded multi-component phases have been written to the diagnostic DiscardedRecords.txt file (located in the same folder where the GEMS executable file resides). This file can be used for checking, which phase definitions can be copied later on manually to the project and rebuilt there (instead of creating them from scratch).

Finally, the program has assembled the definitions of aqueous and gas phases from the DComp and ReacDC records previously copied into the project database, and attached all Phase record keys and all the DComp/ReacDC record keys referenced from the phase definitions to the project system structure. Note that those records for Dependent Components that were not referenced in any Phase definition copied into the project, were not included in the project system. To activate such Dependent Components (e.g., minerals), you will have to create Phase records for them in the "Thermodynamic Database" mode and then open the project, which will be extended automatically.    

In your new project folder, the program has created several empty files (for rtparm, sdref, proces, gtdemo and syseq modules) to store the records that you are going to create when working with the modelling project. Although one database file per module is present in the project directory, you always can create new files (as described in the Database Management manual). In doing so, we would recommend to add another part to the file name, for example, reacdc.Carbonat.MySpecies.pdb or reacdc.Carbonat.MyCompos.pdb. Note that records in these additional files will not be visible to the project system definition until you reconfigure the project. It would be wise to have some additional thermodynamic database files never linked to the project system definition, just kept for some supplementary data or calculations; anyway, the files reside in the project directory and will not be lost during backups. 

In principle, it is possible to have any other file types in the project folder, such as saved images, spreadsheets, texts and so on, where the user keeps reporting or external calculations related to this project. GEMS only looks there for .ndx and .pdb files; as long as they are intact, it should work fine. We recommend making backups of the project folder that you currently work with at least twice per work day in order to minimize possible loss of time and data due to seldom cases of the project definition record corruption. 

Manual Extension of the Project Thermodynamic Database

It is possible to extend an existing modeling project without cloning it under a different name (more...). In order to do it, you have to add chemical species and phases to your project database extension files in the "Thermodynamic Database" mode of GEM-Selektor. Please, keep in mind the following recommendations.

  • It is always easier to copy existing records from the default database or use them as templates for creating new records with "Save As..." or "New/Clone" menu commands than create new records 'from scratch'. 

  • To temporarily link default database files, use "Database Files" "Configure" menu command. To copy records, use "Record List" "Copy..." menu command, and change at least one character in the last record key field for each copied record (it is not possible to have two records with the same key).

  • Add records to your project database extension in the following order: IComp; DComp; ReacDC; Phase; Compos.

  • After remaking DComp,  ReacDC, and Compos records, it's advisable to re-calculate them.

  • Backup your newly created or modified database records by exporting them into text files.

  • If you need to copy many records from default database to your database extension file, export them first to a text file. Then edit this text file for changing the last field of record keys to avoid possible "duplicate record key" errors. Finally, link your project extension file only and import into it the records from the edited text file.

7. Project File Configuration Dialog

This dialog appears if the option "Change file configuration of the selected project" has been activated before opening or creating a modeling project (more...). In a tree-like view, it shows two sets of database files linked to the project: "Kernel" and <ProjectName>, the former corresponds to location "/Resources/DB.default" that comes with the program, and the latter - to the location "/Library/gems3/projects/<ProjectName>". Each entry expands into sub-entries corresponding to database chains (e.g. IComp, ReacDC, ...). Expanding an entry shows files available for this data type; currently linked files are highlighted. Each file entry consists of a 'nickname' and a full name of the file. 'Nicknames' of linked files are saved in the Project record (in this way, the program knows which files to open when the project is used next time). Changing file configuration of the project means that some files are un-linked, or some others are linked in addition, both actions done with a mouse click.

Typically, this dialog is used in two situations.

One is when a new database file has been created in the project folder (e.g. to keep some ReacDC records added by the user in an independent project). These records become available in the Equilibria Calculation mode only after the new file is linked to the project through this dialog.

Another situation is when a new "linked" project is created; then the user must select consistent parts of the default database to be accessible from the project in Equilibria Calculation mode.
