3.MTParm Submodule Data



TPun[0]  - Code of units of measurement of molar energy values { j }
 
This code defines in which units of measurement of energy the values in gTP and other MTparm data objects are dispalyed. 
Attention! Only "j" (J/mol) code can be used in this version of GEM-Selektor.
 
  
TPun[1]  - Code of units of measurement of molar volumes { j }
  
 This code defines in which units of measurement of volume the values in the VTP data object are dispalyed.
Attention! Only "j" (J/bar) code can be used in this version of GEM-Selektor.
 
 
TPun[2]  - Code of units of measurement of pressure { b }
 
This code defines in which units of measurement of pressure the values in the MTparm objects are dispalyed.
Attention! Only "b" (bars) code can be used in this version of GEM-Selektor.
   
 
TPun[3] - Code of units of measurement of temperature { C }
 
This code defines in which units of measurement of temperature the values in the MTparm objects are dispalyed.
Attention! Only "C" (Centigrade, deg. Celsius) code can be used in this version of GEM-Selektor.
 
 
T - Current temperature of interest (T)

This cell contains temperature (in units defined by TPun[3] code) for which thermodynamic data in gTP, VTP and other vectors are displayed.These data are automatically computed for all species included in the project system list from the standard properties and temperature corrections provided in thermodynamic database (DComp and ReacDC records) for the temperature value taken from the SysEq record key. When another SysEq record key is created with different temperature, all the data in MTparm module will be automatically re-computed. 
Default value is Tr = 25 oC (298.15 K). 
 
  
P - Current pressure of interest (P)
 
This cell contains pressture (in units defined by TPun[2] code) for which thermodynamic data in gTP, VTP and other vectors are displayed.These data are automatically computed for all species included in the project system list from the standard properties and pressure corrections provided in thermodynamic database (DComp and ReacDC records) for the pressure value taken from the SysEq record key. If 0 is specified there ("Saturated Vapor Curve") then the current pressure P is first calculated from temperature using a SUPCRT92 subroutine.  When another SysEq record key is created for a different pressure, all the data in MTparm module will be automatically re-computed. 
Default value is Pr = 1 bar (1e5 Pa). 
 
    
RTf - Current RT factor (J/mol) 
 
This cell displays current value of RT product, where R=8.31451 J/mole/K is the universal gas constant and T is converted to Kelvins. 
Default value (for 25 C): 2478.97 J/mol.
 
   
RoW - Current densities of liquid water and steam (in g/cm3)
 
Densities are calculated for current T and P values using SUPCRT92 subroutines, if aqueous phase is present in the project system definition. These properties are used in HKF EoS calculations of VTP and gTP values for aqueous species. 
 
 
EpsW - Current dielectric constants of liquid water and steam 
 
Dielectric constants are calculated for current T and P values using SUPCRT92 subroutines, if aqueous phase is present in the project system definition. These properties are used in HKF EoS calculations of VTP and gTP values for aqueous species. 
 
  
VisW - Current dynamic viscosities of liquid water and steam (in kg/m*sec)
 
Dynamic viscosities are calculated for current T and P values using SUPCRT92 subroutines, if aqueous phase is present in the project system definition. These properties are used in HKF EoS calculations of VTP and gTP values for aqueous species. 
 
 
TPmark - Quality codes for temperature and pressure corrections { + - e }
  
Codes in this column show for each Dependent Component, whether the corrections to the current temperature and pressure have been successful. In most techniques, for instance Cp=f(T) integration, temperature corrections are only valid within the temperature interval where the coefficients were originally fitted (these intervals are given in DComp or ReacDC records). Beyond this interval, the extrapolated corrections are usually not reliable and may contain large errors, and even be not applicable at all. The program sets the quality code to inform you about possible problems related to this. 
 
Possible values (separate for each Dependent Component): : 
+  Corrections for current T,P were within the applicability intervals.   
e   Correction for T was extrapolated beyond the applicability interval for this species. 
-    Correction at these T,P cannot be done and the species has to be excluded from CSD. 
 
   
gTP - Molar (molal) standard Gibbs energy functions of Dependent Components at T,P of interest 

This vector contains the main input data for the GEM IPM problem - the values of molar (partial molal for aqueous and surface species) Gibbs energy function corrected to temperature T and pressure P of interest separately for each Dependent Component, as prescribed in the respective source data records (DComp or ReacDC). 
 
The units of measurement are defined by the TPun[0] code. If the corrected gTP cell value could not be calculated, this cell will contain 7777777 (J/mol) and the respective species will be effectively eliminated from equilbrium speciation. 
 
   
gUns - work vector of uncertainty space increments to tp->G + sy->GEX(J/mol)
 
 
gTPuns - Uncertainties of gTP values (reserved for future versions)

This vector will contain (automatically propagated) uncertainties of g(TP) values, to be used in the "Uncertainty Space" algorithm calculations, to be implemented in Version 3 of GEMS-PSI code.
 
   
hTP - Molar (molal) standard enthalpy functions of Dependent Components at T,P of interest  

This vector contains molar (partial molal for aqueous and surface species) enthalpy functions h(TP) (in J/mol), corrected to temperature T and pressure P of interest separately for each Dependent Component, as prescribed in the respective source data records (DComp or ReacDC). 
  
 To be used in future versions of GEMS code, e.g. for minimization of other than G(x) thermodynamic potentials [2002KAR/CHU]. 
 
 
hTPuns - Uncertainties of hTP values (reserved for future versions)
 
 
STP - Molar (molal) entropies of Dependent Components at T,P of interest 

This vector contains molar (partial molal for aqueous and surface species) entropies S(TP) ( in J/K/mol ), corrected to temperature T and pressure P of interest separately for each Dependent Component, as prescribed in the respective source data records (DComp or ReacDC). 
  
 To be used in future versions of GEMS code, e.g. for minimization of other than G(x) thermodynamic potentials [2002KAR/CHU]. 

 
STPuns - Uncertainties of STP values (reserved for future versions)
 
 
CpTP - Molar (molal) standard entropies of Dependent Components at T, P of interest 
 
This vector contains molar (partial molal for aqueous and surface species) heat capacities Cp(TP) at constant pressure (in J/K/mol), corrected to temperature T and pressure P of interest separately for each Dependent Component, as prescribed in the respective source data records (DComp or ReacDC). 

To be used in future versions of GEMS code.
 
 
CTPuns - Uncertainties of CpTP values (reserved for future versions)
 
 
VTP - Molar (molal) volumes of Dependent Components at T,P of interest  
 
This vector contains molar (partial molal for aqueous and surface species) volumes V(TP) ( in J/bar ), corrected to temperature T and pressure P of interest separately for each Dependent Component, as prescribed in the respective source data records (DComp or ReacDC).  
 
The molar volumes of species are used in GEM calculations to determine the volume of phases and the whole chemical system at equilibrium state, and can be also copied into the stoichiometry martix a if the "Vol"  Independent Component (volume balance constraint) is included in the system stoichiometry basis. 
  
The molar volumes will also be used in future versions of GEMS code, e.g. for minimization of other than G(x) thermodynamic potentials [2002KAR/CHU]. 
 
   
vUns - work vector of uncertainty space increments to tp->Vm (J/mol)
 
  
VTPuns - Uncertainties of VTP values (reserved for future versions)

3.1. Data objects accessible on Project Remake only 

 
TPsv[0] - Flag for calculation of H2O properties using EoS subroutines from SUPCRT92 { + - }
 
This flag (if turned on, +) activates SUPCRT92 subroutines to calculate thermodynamic properties of H2O at T,P of interest. 
 
Default values: + if aqueous electrolyte is present in the project system, - otherwise. 
 
 
TPsv[1] - Flag for calculation of thermodynamic properties of aqueous species using HKF EoS { + - }
 
This flag (if turned on, +) activates SUPCRT92 subroutines to calculate thermodynamic properties of aqueous species at T,P of interest using HKF EoS coefficients and standard state data from DComp records.  
 
Default values: + if aqueous electrolyte is present in the project system, - otherwise. 

 
TPsv[2] - Flag for calculation of properties of gases at high T,P using the FGL module (reserved)
 
Reserved for future versions. Default value: -  (not accessible in this version). 
 
   
TPsv[3] - Allocation flag for the TPmark vector { + - }
 
Default value: +
