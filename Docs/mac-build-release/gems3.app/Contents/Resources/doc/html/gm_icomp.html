1. Independent Components (Stoichiometry Units)

The IComp database chain contains records defining the Independent Components (IC), or stoichiometry units, forming the stoichiometry basis in which composition of chemical systems and species can be expressed using stoichiometry coefficients.

The IComp symbols (mainly chemical elements) are used in writing stoichiometric chemical formulae  of  thermodynamic  species  (Dependent Components, DC, stored in  DComp  or  ReacDC  database records), as well as in the Predefined Composition Object (PCO) definitions (stored in  Compos  database records).

The IC symbols, together with DC chemical formulae and PCO bulk vectors, can be used in many ways to specify the bulk composition of the chemical system (vector  b  ) - the main input data for calculation of equilibrium states using the GEM IPM algorithm.  In all cases, chemical composition of the system will be reduced to the vector b  consisting of  n (N )  elements - total numbers of moles of N  Independent Components involved.

Chemical formulae of species are automatically parsed and assembled into the stoichiometry matrix  A  before calculation of  chemical equilibria. This matrix consists of  n (L )  lines, each corresponding to a Dependent Component (DC, species), with  n (N ) columns containing numbers of moles of  IC. In other words, any line of the A  matrix contains an expanded chemical formula of a species.

The main use of the A  matrix consists in setting up the mass-balance constraints for the whole chemical system: numbers of moles of species at equilibrium state, vector  x , must conform to  a condition  A x  =  b .  Lines of the A  matrix can also be used for calculation of molar masses of species. Expanded (parsed) chemical formulae are also checked for charge balance and used in consistency recalculations of thermodynamic data in DComp  and ReacDC modules. In all cases, some of the numbers stored in IComp records are exploited.

IC definitions are kept in the IComp  database module using a three-field  record key  format. All available IComp database records are loaded into GEMS program memory upon opening or constructing the System Profile and also - on creating/remaking the Compos, ReacDC and DComp records.
 
 

1.1. Creating/Remaking  IComp records


As IComp record format contains no dynamic data fields, no Remake wizard is necessary. All you have to do is simply to edit (your own) IComp data fields on the screen form and then save the record to database. When creating a new IComp record, take care that the new record key  is unique relative to the whole database, and the IC symbol is unique within your current System Profile.

Please, remember that deletion of IC record(s) will make unusable all the DComp, ReacDC, and Compos records which refer to it via chemical formulae (GEMS does no automatic checks for this kind of consistency except when a new System Profile is created using Elements Dialog).

3. IComp Input Data Fields

ICname - Name of this Independent Component
 
This comment string up to 40 characters long serves for identification of IC definition. It is  not analyzed by the program and may be used also for references to data source.
 
 
M0i - Atomic (molar) mass of Independent Component
 
Units of measurement: g/atom (g/mol).
This value must always be entered at creation of any new IComp record!
It must contain 0 for volume (Vol) and electrical charge (Zz).
 
The  M0i  number is heavily used for calculation of gram-formula masses of species, as well as masses (concentrations) of the system and its phases.
 
 
S0i - Standard-state absolute entropy  S0  per g/atom
 
Units of measurement:  J/K/mol.
Value of S0  (at Pr  = 1 bar, Tr  = 25 C) must be entered for chemical elements and electrical charge. For di-atomic gases at standard state (e.g., H2, O2), S0  must be equal to 1/2 of S0 (gas) at Pr , Tr .  For electrical charge (Zz),  S0  must equal -0.5 S0  of  hydrogen gas H2.
 
S0i value is used by the program for consistent recalculations between G0,  H0  and S0  values of dependent components at Pr, Tr   (see DComp module).
 
 
Valen - Default valence number (formula charge) of IC in compounds
 
Valence number must be always entered by the user into this field when a new IComp record is created. For chemical elements, this is the most common valence number in compounds considered in aquatic systems.
 
The Valen data object is used by GEMS program when processing chemical formulae  for any IC entries where the valence numer is not given explicitly.
The valence numbers are then used together with stoichiometry coefficients for calculation of the formula charge balance. For charge (Zz), Valen must be +1.
 
Example: Default valence is 2 for IComp Fe (iron).
Chemical formulae:
 FeCO3          (siderite, uses default valences 2 for Fe, -2 for O, +4 for C);
 Fe|3|2O3     (hematite, an explicit valence 3 used for Fe).
 
 
IXi - Relative isotope abundance (1 for chemical elements)
 
The fractional isotope abundance (in natural mixture of isotopes) must be entered by the user when a new IComp record for isotope is created.
 
The IXi number is used by the program (RCthermo() submodule) for calculation of thermodynamic data for isotopic forms of compounds using Grichuk's method ([1987GRI]; [1998SHI/KUL]).
 
 
ICform - Chemical formula of this Independent Component
 
This string up to 40 characters long  may contain an elemental chemical formula of an IC such as petrogenic oxide Al2O3 which itself is a chemical compound.
 
ICform field is not processed in this version of GEM-Selektor. However, for future compatibility, we recommend that for chemical elements, this field contains the same symbol as in the IC_symbol field of IComp record key, or "Empty" symbol if the formula cannot be written (e.g., for humate ligand).


indMT - Index in Periodical (Mendeleev's) Table of chemical elements
 
This data item must contain atomic index for all chemical elements or isotopes (e.g. 1 for hydrogen and its isotopes, 92 for uranium). In GEMS version 2, this index is used only in the Elements Dialog when a new System Profile is created. Special index 0 must be given for electric charge (Zz), 99 for Nit (atmospheric nitrogen) and 100 for Vol (volume) Independent Components. All additional IC are assigned consecutive indices 101, 102, ... up to 112.   

Other usages of this index may be implemented in future versions of the GEM-Selektor package.

 

Data Items Reserved for Future Versions
 
 
StdSt - Code of standard aggregate state (reserved)
 
Identifies the standard phase state in which a pure chemical element (IC) exists at reference Tr = 25 C and P = 1 bar. To be used in calculations in future versions.
Possible values:
 
g     gaseous;
s     crystalline solid;
l      liquid;
u     indefinite (e.g., for organic ligands).
 
 
 IC_DC - Reference DComp record key (reserved)
 
For chemical elements, this field may contain a packed DComp record key pointing to a full set of thermochemical data for this element at standard state.
 
To be used in future versions of GEM-Selektor, for calculation of standard-state Gibbs energy of formation values at elevated temperatures and pressures.  "Empty"  value is default.

 
Cp0i - Standard-state absolute heat capacity  Cp0  per g/atom (reserved)
 
Units of measurement: J/K/mol. Not used in this version of GEM-Selektor.
This field may (optionally) be entered for future prediction of heat capacities of dependent components. Otherwise, put "Empty" value there.
 
 
V0i - Standard state volume V0  per g/atom (reserved)
 
Units of measurement: cm3/mol. Not used in this version of GEM-Selektor.
This field may (optionally) be entered for future prediction of molar volumes of dependent components. Otherwise, put "Empty" value there.
 
 
Ri - Ionic radius, Angstrom (reserved)
 
This data item is reserved for development of prediction algorithms in future versions of GEM-Selektor package.  "Empty"  value is default.
 
 
Zi - Default coordination number in compounds (reserved)
 
This data item is reserved for development of prediction algorithms in future versions of GEM-Selektor package.  "Empty"  value is default.
