1. How to Use GEM-Selektor Help System

The GEM-Selektor v3 code is equipped with an up-to-date HelpViewer based on Qt4 Toolkit functionality. The HelpViewer window can be kept open during the whole GEMS3 work session; it is especially convenient to put it on a second monitor (if available). Hovering the mouse cursor over the control elements shows tooltips about them.

The HelpViewer can be called from the main GEMS3 frame:

  • by clicking on the Help '?' toolbar button;
  • by selecting entries in the 'Help' menu;
  • by clicking on a data object field on any module window page and pressing F1 key, or selecting 'Help' from the right-click pop-up menu;
  • by clicking the 'Learn more' button on (almost) any wizard or pop-up dialog;

Clicking the 'Help on help' '?' toolbar button in the Help Viewer  window will opens this help page.

The HelpViewer allows the user to browse help pages and follow hypertext links in them, as well as navigate using 'Back', 'Forward' and 'Home' toolbar buttons (or the respective items in the 'Go' menu).

The displayed content can be zoomed in or zoomed out using the 'View' menu. The content can be selected by mouse-dragging and printed using the 'File' 'Print' menu command (further choices in the appearing Print Dialog).

The help contents can also be accessed via tree-like headings in the 'Contents' tab.

The 'Index' tab allows the user to jump to a description (if available) of any data object by clicking on the respective label in the scrollable list. To reduce the size of the list, some characters can be entered into 'TextLabel' field.

The 'Search' tab allows to find all help pages containing a given keyword. The list of pages (files) appears in the list below; click on a file link opens the respective help file. The keyword will automatically appear in the  'Find' toolbar field;  exact locations within the help file can be followed by clicking 'Find next' or 'Find previous' toolbar buttons.

The 'Find' function (within currently open help file) can be used separately, also using the 'Find' menu commands.

Executing the 'Help' 'Version' menu command in HelpViewer window opens a small window with information about revision numbers and OS platform of the currently installed GEM-Selektor code.

Taken together, this functionality makes the GEMS3 HelpViewer a rewarding experience.



2. The Phase Information widget

In GEM-Selektor chemical system definitions, as displayed in the System, EqDemo, Multi window pages, there are separate lists of phases (PHvp, PHnam) and of dependent components (DCvp, DCnam), respectively. These lists are connected via the L1_ and L1 vectors, respectively, showing how many DCs each phase contains, under a convention that these blocks of DC names appear in the same order as the phases.

These relationships are clearly visible in both Input and Results tabs of the Single System dialog, where each phase can be expanded to see its DCs. However, in System and EqDemo windows, the lists of names of phases and DCs are displayed in different pages, and sometimes it is difficult for the user to see, to which phase a chosen DC belongs, or which DCs are included into a chosen phase.

To provide such information easly, the Phase Information widget was implemented. It can be invoked by putting cursor on any entry in DCvp, DCnam,  PHvp, PHnam name lists and pressing F5 hot key, or selecting "Show Phase    F5" from the right-click menu.

If invoked from DCvp or PHvp name lists, the Phase Information widget shows Phase record key of the phase, with the number and a scrollable list of indexes and record keys of all its components, with the chosen DC index (in modeling project) highlighted.

If invoked from DCnam or PHnam name lists, the Phase Information widget shows the phase name, with the number and a scrollable list of indexes and names of all its components, with the chosen DC index (in GEM IPM chemical system) highlighted.

To facilitate writing Process, GtDemo and GEM2MT scripts, any phase or component name can be copy-pasted from this widget. To take it to the clipboard, right-click on the desired name in Phase information widget, and select "Copy". 

Click "Ok" to close the widget. "Help" button displays this help page.

 7. GEMS3 Database Mode menu commands and functions

In the Database Mode of GEM-Selektor, any module window (IComp, DComp, ...) can be opened by clicking the respective button in the left pane, or using the 'Modules' menu. Regardless of which module window is open, the main GEMS3 frame will show the same menu bar and toolbar buttons, followed by the record key field. Toolbar buttons largely duplicate the most frequently used menu commands.

The "Record" menu provides commands for processing data in a single record using commands that are briefly described below.

Menu command
 Hotkey
 Explanation
Create(New)
Creates a new database record from scratch. When invoked, it first displays a "Please, set a new record key" dialog with a disassembled key of the currently loaded record (or default key fields if no record yet exists in the module database chain). Enter at least one of the fields and press "Ok" (remember that the record key must be unique within the database chain). At the next step, the respective Remake Wizard will appear.
New(Clone)
Creates a new database record by "cloning" the currently loaded record. When invoked, it first displays a "Please, set a new record key" dialog with a disassembled key of the currently loaded record. Change at list one character in one of the fields and press "Ok"(remember that the record key must be unique within the database chain). At the next step, the respective Remake Wizard will appear.
Display F6
 Permits to select a single record, load it from database, and display it in the module window. When this command is executed, a "Select data record key" dialog appears with a scroll list of record keys (in alphabetic order). Select the one you need by mouse-click and press the "Ok" button. The lists in DComp or ReacDC chains may be quite long; you can display only part of record keys by using the "Set Filter..." button to set the filter template.  If you click on the "Set Filter" button, a small "Template ..." dialog appears with record key fields usually containing asterisks (meaning that any string fits to the template). You can edit some fields by putting letters in them; for instance (if you work with DComp records), putting "a" in the first field and "Fe*" in the second one creates a template, which will display only aqueous species of iron. Press Ok when finished; the template is retained until another template is specified; "Reset" button restores the default template (fits with any record key).
Alternatively, in the GEMS3 main frame, any record of selected type (e.g. DComp) can be loaded and dispalyed simply by clicking on its record key in the list visible on the left side. Above this list, a filter template is visible; it can be edited directly or field-by-field after clicking on the "Filter" button to the left of template.
Remake...
Re-configures and rebuilds the currently selected record. This function is highly specific for record types and is the most sophisticated tool for handling data in GEMS3. In general, after the "Remake" button or menu comand is activated, a Remake Wizard appears for the respective record type. After the remake wizard is finished, the record will not be saved automatically; to save the re-configured record, use "Save" or "Save as" command. 
Calculate
 F9
 Many record types (Compos, DComp, ReacDC, RTparm, Phase) are tied with various subroutines which perform cross-checking of the data, restore missing values, or do other calculations. These subroutines of GEMS3 are executed upon "Calculate" command. They can issue a variety of warnings depending on the data and the record type. The calculated data are not saved automatically to the database; this has to be done using "Save" or "Save as" commands.
Save
Saves current record to the database file under the same key and from where it has been loaded.
Save as ...
Permits to save contents of the data record under another record key (works as "Create" command except that no re-organization of the data structure is performed). If more than one file is linked to the database chain, this command will display a dialog with all files listed and the user will have to select a file where to put the record. Please, remember that the default GEMS database files are write-protected, so normally you should choose only between files existing in your own system profile directories.
Delete
Deletes currently displayed record from the database chain. The contents of the record remain loaded into memory and in principle can be saved using "Save As" or "Create" commands. This operation will not work on write-protected files from the default GEMS database.
Plot data
This function works only in RTPARM and GTDEMO screen forms. After calculation, it will display four graphs (one in GTDEMO) to examine temperature and / or pressure change of selected thermodynamic properties.
Print
This function is still under construction. For some data types (e.g., RTPARM or PHASE), it can print tabulated data or download them as columns of numbers into a text file. A typical "File Selection" dialog appears to let you choose the file name and path.


The "Record List" menu provides commands for processing multiple records; a brief description of its its commands follows.

Browse
This command permits to scan one-by-one all or some records in the database chain and load one for your choice.  First, it displays a "Get Key Template" dialog  which works similar to that described in "Show" command. When the record key template is constructed and "OK" button is pressed, you will see the screen form with contents of the first record and (outside of it) a small "Select this record" window with record key shown in the status line. The "Next" and "Previous" buttons let you walk through the sequence of selected records; "Select" button will change the program to "Show record" mode.

Move
Moves one or several records (with record keys that fit the template) into another database file which has to be linked before executing this command. First, this command shows "Get Key Template" dialog and then, on each record, displays the "File Selection Dialog". This command can be applied only to records kept in the users's extension files, but not in the default database files.

Delete
Deletes several records from the (user extension) database files. First, it shows the "Get Key Template" dialog (multiple selection of record keys is allowed) and then requests separate confirmation for deletion of each record. Remember that deleted records can not be restored in GEMS database! The records are not physically erased - only their keys are eliminated from the index files. The deleted record will occupy place inside of the .pdb file and a reference to this space will be created in the header of the .pdb file. When the next record is inserted it can overwrite part of this space. Since the size of PDB file header is limited, it may happen that stack of deleted records will overflow, with the respective warning message. In this case, compression of database file should be performed. A good rule of thumb is not to delete more than 20-30 records at once.

List
Creates a text file with alphabetic list of record keys in the database chain. First, this command shows "Get Key Template" dialog to  do a (multiple) selection of record keys, and then displays the "File Selection Dialog".

Export...
Unloads records from database chain into GEMS backup format text files. First, this command shows "Get Key Template" dialog (multiple selection of record keys is possible) and then the "File Selection Dialog". The records can be exported all at once or viewed with confirmation for exporting  each record.  This is a good tool to backup your database extension files or copy large parts of the default thermodynamic database into your profile database files. The unloaded text files can be edited with any text editor and imported back into GEMS database extension files.

Import...
Loads records from a (previously exported) GEMS backup format text file to the user's extension database file(s) which must be linked in advance. First, this command displays the "File Selection Dialog" to find the input text file, and then will require to choose a database file where to put each record if more than one file is linked. It is possible to display contents of each imported record before deciding whether to put it into the database file.
 

The "Files"  menu  contains commands for manipulation with GEMS database files. Each database chain for a given data type is physically represented by several pairs of files with identical file names but different extensions. For a given <name>, there are two physical disk files: <name>.pdb (data record file) and <name>.ndx (record index file). Note that all information, including record keys and time of last modification, is recorded in the <name>.pdb file. The corresponding <name>.ndx file, if corrupt, can be restored by execution of  "Rebuild Index" command (this may take a few minutes).

Actualize Files
Permits the user to alter the current configuration of files linked to the database chain. First, it shows a list of all files available in both default and user's extension database, with the previously linked files already selected. Use mouse clicks to select/dismiss the files. Press OK when finished. All linked files will open automatically and .ndx indices will merge into the chain record key index in the program memory.

Link File
Links a file (not yet selected) to the database chain; previously linked files remain linked too.  Works similar to "Actualize" command.

Add Files
Adds  GEMS database files to file configuration or creates empty database files. Proceeds from the "File Selection Dialog".  For developers or experienced user's only!

Rebuild Index
This database support function compresses a selected database file to eliminate the "deleted" spaces in .pdb binary file, and rebuilds its index .ndx file. Choose one or several files from the list and press OK. For large files, this may take several minutes or more.
Warning: compression is a radical operation which, in case of failure or interruption, can make the database file unusable and cause large loss of data. We advice always making a backup copy or unload the file to text format before compression. If an error has occurred during compression (with error message), we recommend to quit GEMS, replace the file with its its backup copy (or
delete it to create a new empty file and load the records from previously unloaded text format); then start GEMS and resume operation.

3.3.10. Advanced Database Maintenance Dialog

What to do if you decided to type in some bibliographic references, or delete some SysEq records because you do not need them anymore, or compress your extension database files with SysEq, Process or GTdemo records? This can be done in the same way as described above for thermodynamic database records if you open a "Database Management (Advanced)" dialog. To do it, first go from the Main Dialog of GEMS  into the "Thermodynamic Database"  dialog, and then press the "Advanced" button. The appearing dialog is pretty similar to the "Thermodynamic Database"  dialog; select one of the necessary data types (e.g., Thermodynamic Systems)  and you will get in a screen form of the same type and with the same set of pop-up menus as described previously.  Use the appropriate menu commands to do the work (however, don't try to delete Profile records or Create/Remake/Edit  SysEq, Proces or GTDEMO records as it could be done only in Calculation of Equilibria mode!). "Uncertainty Space", "Dual Thermodynamics" and "Mass Transport" data types and algorithms are under construction and not available in this version of GEMS. Press "Back" or "Basic" buttons to get back to "Thermodynamic Database" dialog.
