1. GEM2MT Module Operation

The GEM2MT module is a tool for automation of one-dimensional (1-D) reactive mass transport simulations coupled with GEM calculation of equilibrium states in spatially distributed nodes (boxes, volumes) over time steps. GEM2MT can be used to simulate the open system evolution and fluid-rock reactions occurring along a fluid flow path (e.g. vein, fracture, porous media, etc.). This type of simulation solves simple reactive mass transport problems, which requires the definition of fluxes between boxes of certain fluid/rock mass or volume ratios (or alternatively porosity and permeability) where fluid is moved sequentially or simultaneously from one box to the other.

The GEM2MT module also permits testing simple 1-D reactive transport problems with complex chemical reactions, that may be later used as a benchmark for the coupling of GEMS3K with more complex reactive transport models. One-dimensional advection-diffusion numerical reactive transport simulations can be performed, as well as the evolution of various sequential and flow-trough reactor sequences or generic box-flux (reservoir) dynamics can be modeled. In addition, I/O files for standalone GEMS3K calculations can be exported for multiple initial node recipes (in the T mode). Changes of species concentrations or amounts of phases through the node profile with time can be graphically monitored at each time step (wave) or logged to files in VTK format.

The GEM2MT module can simulate reactive transport in three main modes:

  1. Sequential reactor chain and waves ('S');
  2. Flow-through and box-flux sequence ('F', 'B');
  3. One-dimensional reactive transport with advection/dispersion/diffusion ('A', 'C', 'W', 'D')

A typical step by step workflow for the S and F modes is shown in the two examples below. The 'S' and 'F' modes are normally used for simulating fluid-rock interaction by infiltration of fluid into the rock, which leads to a development of alteration zones along a fluid flow path. What actually moves is called an MGP (Mobile Group of Phases), which can be aqueous phase, gas phase, aqueous and gas phases together, or aqueous phase with some fractions of some solid phases (i.e. water with suspended solid particles). A simulation setup consists typically of one node of source fluid and about 50 to 200 nodes of rock that will be interacted with fluid, all to be set first using the GEM2MT wizard when creating or re-making a GEM2MT object.

To prepare for such simulations, two Compos (PCO) records are usually created in thermodynamic database mode, namely one for fresh fluid such as seawater (let us call it "Fluid") and another for the fresh rock (let us call it "Rock"). Ideally, both rock and fluid, cover the same list of chemical elements (if data are not available for some elements present in the other initial compositions then simply put 1e-9 mol for that element). These two system compositions can then be copy/pasted and added later to GEM2MT, either using individual elements (Independent Components) and/or user-defined formulae units.

Next, a parent system record containing the fluid and the rock has to be created in SysEq at the temperature and pressure of interest. This record will have to be selected when creating a new GEM2MT record (object) containing all the phases and elements relevant to the fluid-rock system of interest.

Example 1: "Sequential reactor chain" 'S' mode

In this mode, we can simulate a "wave" of a fluid that will pass through each of the rock nodes once per step (i.e., a wave consists of ca. 100 substeps). Many "waves" are possible. 

  1. Under GEM2MT, create a new record and select the parent system containing the rock and the fluid created in SysEq (e.g. "FRgem2mt").
  2. In Step 1 of the wizard, select the 'S' mode.
  3. In Step 2 of the wizard, set the number of nodes (nC= 101) and the maximum number of steps (in this case, "waves", i.e. 1000). Select 2 for the number of initial chemical system recipes, which will allow us to define the initial compositions and masses of the fluid and rock initial systems. Optionally, we can set the number of user-defined formula units that will be used for augmenting initial bulk elemental compositions of the  fluid and the rock.
  4. In Step 3, we select "Move aqueous fluid".
  5. In Step 4, we select P, T, V, Initial system, Node type, MGP flux, and Initial mass.
  6. In Step 5, we select the variables you want to plot in each node after each "wave" (i.e. n1vPH for volumes of minerals).

After completing the wizard, we get to the GEM2MT module window "Initial" page.  There we need to enter the names of rock and fluid, temperature, pressure, mass of fluid and rock, and the initial composition of the fluid and the rock. This can be done by copy-pasting (transposed) the bulk elemental composition vectors from the test systems (SysEq records) reproducing the fluid and the rock, respectively, or from Compos records "Fluid" and "Rock". It is important to set realistic fluid/rock mass ratio, usually somewhere between 10 and 1000. As an example:

In the first row (index 0), enter the name of system "Fluid" or similar, and add a mass of 0.9 kg.

In the second row (index 1), enter the name of system "Rock" or similar, and add a mass of 0.1 kg.

In both cases, set temperature (e.g. 250 oC) and pressure (e.g. 500 bar) for equilibrium conditions.

Next in the mICb table below of quantity/concentration of IC in initial systems, enter (copy-paste) the concentration and units for each element to define the compositions of rock and fluid in the corresponding rows.

Optionally, type user-defined formulae (e.g. "SiO2", "CO2", etc.) and enter their units and quantities, which will be added (after unit conversions) to the bulk elemental compositions of initial systems for fluid and rock.

The initialization script (visible in the lower part of the "Initial" page) should have been created by the wizard and normally does not need editing in the 'S' mode. Click the "Calculate" toolbar button and (at the very first time) do not plot the nodes. Stop calculation after a few "waves" and start it again with graph plotting to inspect the results. 

Example 2: Flow-through reactor chain ('F') mode

In this mode, we can simulate the simultaneous transport of (reacted) fluid in one direction through all nodes from one node to the next. This mode requires the definition of equal zero-order or first-order fluxes of fluid connecting each node with the next one; equal rates of the fluid flux should be specified (zero-order: in mass/time (step) units; first-order: in 1/time units). The initial setup for the 'F' mode is similar to that for 'S' mode, with an important difference that the same mass of fluid (corresponding to the effective fluid/rock mass ratio) should be present in all nodes containing the rock at the initial step. For example, in the 'F' mode, we may wish to transport 50 g of fluid per step simultaneously through all nodes, which means flux of 50g/step rate, applied to nodes containing 50 g of rock and 500 g of fluid. In other words, at each (time) step, about one tenth of fluid in each box is transported to the next box. The very first box in the chain is declared a "constant-flux source" and it contains only "fresh" fluid but no rock; the very last box is declared a "constant-flux sink". 

  1. Under GEM2MT, we create a new record selecting the parent system containing the rock and fluid created in SysEq (e.g. with a name "FRgem2mt").
  2. In Step 1 of the wizard, we select the 'F' mode.
  3. In Step 2 of the wizard, we set the number of nodes (e.g. nC= 101) and a maximum number of (time) steps (e.g. 1000); note that equilibria in all boxes in the chain are re-calculated (and can be plotted) in one time step. Enter 1 for the number of user-defined formula units (to add at least H2O to the rock initial composition).
  4. In Step 4, we set P, T, V, Initial system, Node type, MGP composition, flux rates, and Initial mass.
  5. In Step 5, we select the output variables you want to plot in each nodes (i.e. n1vPH for volumes of minerals).

After completing the wizard, we get to the GEM2MT module window "Initial" page.  There we need to enter the names of rock and fluid initial systems, their temperature, pressure, masses, and the initial compositions of the fluid and the rock. This can be done by copy-pasting (transposed) the bulk elemental composition vectors from the test systems (SysEq records) reproducing the fluid and the rock, respectively, or from Compos records "Fluid" and "Rock". It is important to set realistic fluid/rock mass ratio, usually somewhere between 10 and 1000. As an example:

In the first row (index 0), enter the name of system "Fluid" or similar, and add a mass of 1 kg.

In the second row (index 1), enter the name of system "Rock" or similar, and add a mass of 0.1 kg.

In both cases, set temperature (e.g. 250 C) and pressure (e.g. 500 bar) for equilibrium conditions.

Next in the mICb table of quantity/concentration of IC in initial systems, enter (copy-paste) the concentration and units for each element to define the compositions of rock and fluid in the corresponding rows.

Optionally, type user-defined formulae (e.g. "H2O", "CO2", etc.) and enter their units and quantities, which will be added (after unit conversions) to the bulk elemental compositions of initial systems for fluid and rock. Check that the composition of the rock also contains enough "H2O" to have fluid after equilibration in desired fluid/rock mass ratio (e.g. if normalized to 1 kg of rock, add 10 kg of "H2O" for a fluid/rock of 10).

The initialization script (visible in the lower part of the "Initial" page) should have been created by the wizard. In this script, we check that the Flux Rate (FDLf) is set to the desired value (e.g. 0.1). This script will then assign the same flux in all the nodes/boxes.

Under the tab "Fluxes", we need to set the amount of phase(s) that are transported in an MGP in the flux (i.e., the composition of the flux in terms of mass(es) of phase(s) in its outgoing box). As an example, for defining a unit flux as composed of 50 g of aqueous solution, enter 0.05 G of aq_gen phase for quantities of phases in MGP column (the wizard already suggests "aq" as the name of this flux composition or "mobile phase group" MGP). Alternatively, you can set for MGP the mole fraction 'n' of 1 for aq_gen phase, meaning that the MGP composition will be that of the whole aqueous phase in source box, and define zero-order flux rates as 0.05 kg/s, or first-order flux rate constants as factors that will result in 0.05 kg/s after multiplication with aq_gen phase mass in the source box.

Click the "Calculate" toolbar button and (at the very first time) do not plot the nodes. Stop calculation after a few "steps", then start it again with plotting. 

1.1. Sequential reactor chains and waves ('S' mode)

Such simulations are useful for predicting the development of rock alteration zones upon progressive infiltration of a fluid supplied from a constant source (e.g. a large fracture). This model is a good approximation for steady-state irreversible processes such as weathering, metasomatism, evaporation of seawater etc.

The simplest case is when there is only one "reactor" (box) at given temperature T and pressure P, in which initially only rock of known bulk composition is present at a prescribed initial mass. At step 0, the fluid of defined composition is added to the "rock" at a given fluid-rock mass ratio and left to equilibrate, then the aqueous composition/ speciation and the phase assemblage and composition of the reacted rock is explored (sampled). At step 1, the reacted fluid in the box is replaced by the "fresh" fluid at the same fluid-rock mass ratio, and equilibrated again. The loop is repeated many times, allowing the user to follow how the phase composition of the altered rock changes along the way. Note that for a single box, this type of simulation can be done in the Process module of GEM-Selektor as the leaching-type "flow-through" reactor 'R' mode. 

The GEM2MT module permits to set up and run a more natural and flexible sequential scheme consisting of many "reactors" (nodes, boxes) arranged in a 1-D chain, where the fluid (water) moves from the previous to the next box. The boxes may, in principle, have different rock compositions, temperatures and pressures, if desired. "Fresh" fluid with a constant composition enters the first box at a prescribed initial fluid-rock mass ratio and equilibrates with the rock. The equilibrated fluid is then moved to the next box (at prescribed fluid-rock ratio) and reacts there, and so on until the last box is reacted. This completes a so-called "wave".

Any number of sequential reactor "waves" (which correspond to steps) can be performed. At the beginning of the second wave (step 2), the "fresh" fluid is put into the first box with the reacted rock composition remaining after the first "wave". After equilibration, the fluid is taken into the second box to react with the rock already reacted with the the first "wave", and so on. After many "waves", the formation and propagation of rock alteration zones can be followed by plotting amounts of phases in all boxes after each "wave".

Note that, regardless of units in which the MGP composition is set, the 'S' mode simulation moves the whole mass of aqueous phase per step in each box. So, setting the MGP as a mole fraction 'n' of 1.0 for the aq_gen phase would be absolutely correct.

1.2. Flow-through box-flux models (F, B modes)

In the 'F' mode, the 1-D reactive transport column is represented by many equal-size boxes that may have different rock compositions, temperatures and pressures, if desired. The first box must be coded as a constant-flux (Cauchy) source (3), and the last box - as a constant-flux sink (-3); all the boxes in-between are normal local-equilibrium systems (1). Equal zero-order fluxes of aqueous fluid "aq" (not too high!) should be set between boxes 0 and 1, 1 and 2, 2 and 3, ... until nC-2 and nC-1 (sink). These values are initialized automatically and can be modified from the math script in the "Initial" page of the GEM2MT window and found in the "Nodes" page.

In the "Fluxes" page of the GEM2MT window, the quantity of the "aq" (aqueous fluid) mobile group of phases (MGP) should be defined in the PGT table and units in UMGP column. This corresponds to the mass or concentration of fluid moved at each time step. A column in the PGT table defines quantities of phases moved from the source box in one mole of the flux. If there is another MGP defined, e.g. "gas", then two-phase transport can be arranged between boxes. 

At each unit time step, a given mass of reacted aqueous fluid is moved into the next box replacing there the equivalent mass of fluid that moves simultaneously to the next box and so on (fresh fluid from the constant-flux source moves from box 0 into box 1, and the same mass of reacted fluid moves away from the last box). In the 'F' mode, the transport is performed by direct iteration; in the 'B' mode, this simultaneous transport is conducted using the built-in integrator of the ODE system (the Bulirsch-Stoer algorithm) of mass balance equations for all elements in all boxes. Hence, in the 'B' mode, the simulated megasystem of boxes connected with first-order fluxes will first undergo a transient state and then come to a dynamic stationary state.

The mass of fluid moved by the flux between boxes q-1 and q is defined by the quantities of phases in MGP (or flux composition), the flux rate, and the flux order. The simplest zero-order (constant) flux moves a given mass of its composition times as set by the flux rate constant (set in the FDLf table on "Fluxes" page). The rate 0.1 means that 0.1 kg/s of the flux composition moves per unit time. For the 1st order flux, the rate constant v = 0.1 means that 0.1 of flux composition times the mass of its source MGP (in kg) is moved per unit time. Such a flux will automatically decrease if the mass of its source phase(s) in the source box decreases, and vice versa, which helps the megasystem to reach the stationary state. 

After the transport step, the equilibrium speciation is computed in all nodes by the GEM algorithm, and the next iteration of transport occurs followed by the next equilibration in all boxes, and so on, until the maximum number of steps or the total time of simulation is reached. After each equilibration, the masses, volumes or concentrations of interest (both in aqueous and solid parts) are sampled and plotted.

The 'F' mode simulation with equal zero-order fluxes is, in fact, somehow analogous to 1-D pure advection transport, except that the time step is set to 1, the tracer dispersion is ignored, and the hydraulic consistency criteria are not applied.

The B mode operates in a similar way except the combination of boxes and fluxes may be arbitrary (not only chain-like), the boxes can have different size, and the fluxes may be of different order and rate, and some fluxes can be defined as elemental production or sink in a given box. This setup is suitable e.g. for simulating geochemical cycles or redox-sedimentation dynamics in water columns. The transport step is performed by using the ODE integrator of the system of mass balance equations for all elements in all boxes.The time reduction factor can be set by the user (default value of 1) if the desired time stepping is much larger than 1 or 2 seconds. For example, if the time reduction factor is set at 1000 and time step at 1000 s then each internal integration step will be performed at time step duration of 1 s but the user will see it as a 1000 s step. The flux rate is still defined in "real" time, e.g. in kg/s.

Some theoretical background of the mass-balance integration in a reservoir box-flux "megasystem"can be found e.g. in

[Lerman A., Mackenzie F.T., Garrels R.M. (1975), Modeling of geochemical cycles: Phosphorus as an example, Geol. Soc. Amer. Memoir 142, p. 205-218].

[Kulik D.A., Chudnenko K.V. and Karpov I.K. (1993), An algorithm for simulating the evolution of a system of local-equilibrium reservoirs connected by mobile phase-group flows. Geochemistry International 30 (1), p. 90-102.]

[Lerman A., Wu Lingling (2008), Kinetics of Global Geochemical Systems, Chapter 13 in book: Kinetics of Water-Rock Interaction. Publisher: Springer. Editors: Susan Brantley, James Kubicki, Art White, p. 655-736]. 

1.3. One-dimensional reactive transport ('A', 'C', 'W', 'D' modes)

The GEM2MT module allows the user to set up and run simplified  simulations of 1-D reactive transport in porous media using the finite-difference advection-dispersion algorithm (classical 'A' mode and implicit Crank-Nicolson 'C' mode) for advection-dominated cases. For the diffusion-dominated transport, the random-walk algorithm ('W' mode, also accounts for advection) can be used; the pure diffusion transport method ('D' mode TBD) is under construction in this version of GEM-Selektor. Theoretical background for all these transport modelling methods can be found in numerous textbooks.

The non-iterative coupling of transport and chemistry is implemented in all cases (so-called "operator splitting" approach): the mass transport model integration over all nodes alternates with GEM calculation of equilibrium states in all nodes after each time step, until the maximum time or number of time steps is reached. The profiles of amounts of phases or aqueous species of interest can be plotted after each time step (or exported to VTK files).

This simple implementation of 1-D reactive transport is by no means designed to compete with popular coupled codes such as PFLOTRAN, OpenGeoSys or CRUNCH. The purpose of 'A', 'C', 'W' modes in GEM2MT is rather in the education and testing of reactive system behavior before performing heavy computations of 2-D or 3-D cases using coupled RT codes on parallel high-performance computer hardware.

Of particular interest is the inspection of the evolution of megasystems. This involves the comparison between 1-D box-flux (mode 'B') models with ODE integration and explicit definition of fluxes, with the more rigorous models that consider hydraulic advection-dispersion (diffusion). The latter include concentration gradients (of total dissolved elements or aqueous species) solved by PDE intergration ('A', 'C' modes) or a stochastic simulation of Brownian movement of particles ('W' mode).

The main difference between the box-flux models and the hydraulic advection-dispersion-diffusion models is the choice of transport parameters (defined initial state for all boxes or nodes is required in all cases). While in 'B' or 'F' modes, the zero- or first order fluxes must be set by the MGP composition, flux rate or rate constant, the initial transport parameters must be defined in all nodes in 'A', 'C', 'W' modes. These include porosity, permeability, advection velocity, dispersivity, tortuosity and diffusivity (common or individual diffusion coefficients).

These parameters can be conveniently set upon initialization of the RT problem using the initialization math script in the "Initial" page of the GEM2MT window. If needed, some parameters in some nodes may change in the process (e.g. due to mineral dissolution or precipitation, porosity and permeability may change), which has to be programmed in the last part of the initialization script.

2. GEM2MT Create/Remake Wizard

The GEM2MT simulator is a tool for automation of simple reactive mass transport models coupled with GEM calculation of equilibrium states in spatially distributed nodes (boxes, volumes) over time steps.

It is possible to perform 1D advection-diffusion numerical finite-difference modeling, as well as the box-flux (reservoir) dynamics.

Changes of species concentrations or amounts of phases through the node profile with time can be graphically monitored and logged to disk files.

The GEM2MT simulator can be configured by choosing its mode of operation and setting up the number of nodes (boxes, volumes).

You may also have to type math scripts that define how to distribute initial system compositions over nodes, and which data to sample from nodes for graphic presentation or export to text files.

This wizard helps you configure and set-up a new RT simulation step-by-step.

2.1. GEM2MT Wizard Step 1:  Select the mode of simulation

Possible GEM2MT tasks fall into three groups:
  • Box-flux ('S', 'F', 'B') simulations, where the transport is defined in terms of fluxes of mobile phase groups (MGP) between boxes or reactors.
  • 1-D reactive transport simulations using numerical advection-dispersion finite-differences ('A'), Crank-Nicholson ('C'), advection-diffusion finite-volumes ('D' TBD) or advection-diffusion random-walk ('W') algorithms.
  • Preparation ('T') of initial chemical systems and exporting them into GEMS3K input files for external coupled RMT codes.

'S'     Sequential reactors chain with waves (moving fluids or solids between reactors one-by-one)

This flexible scheme consists of many "reactors" (boxes) arranged into a 1-D chain, where at each step, the whole fluid phase is moved (copied) from the previous into the next box. The constant composition "fresh" fluid enters the very first box with a prescribed fluid-rock mass ratio and reacts there. The equilibrated fluid is taken to the next box (at prescribed fluid-rock ratio, replacing the fluid left from the previous "wave") and reacts there, and so on until the last box is reacted, thus completing the so-called "wave". Any desired number of "waves" can be performed. At the beginning of second wave, the "fresh" fluid replaces the old fluid phase in the first box. After equilibration, the fluid is taken into the second box to react, and so on. From "wave" to "wave", the propagation of rock alteration zones can be followed by plotting amounts of phases in all boxes. A variant of 'S' mode allows to move all solids instead of the fluid phase from box to box, like in a dialysis bag (with an idea to follow the fluid evolution).

'F'     Flow-through reactors chain with fluxes (simultaneous transport then equilibration)

This scheme also consists of many "reactors" (boxes) arranged into a 1-D chain connected by 1st order fluxes. At each (time) step, part of the fluid phase is moved from the previous into the next box simultaneously for all boxes. The very first box is usually defined as a constant source of "fresh" fluid, and the last box in the chain - as having the infinite sink for the fluid. 1st order flux means that the mass (of fluid) transported per (time) step is proportional to the mass of fluid phase in the outgoing box. So, the flux rate constant (typically about 0.1 to 0.3) actually tells what fraction of the fluid mass is transported per (time) step. The effect of this feedback is that, even if the boxes initially contained different masses of fluid, with time the masses of fluid would become equal. This scheme simulates the propagation of rock alteration zones, which can be followed by plotting amounts of phases in all boxes. The ODE integration (as in B mode) is not performed. Compared with S mode, the F mode produces results more similar to those of classical advection RT algorithms.

'B'     Box-flux megasystem (with time-step integration of complex arbitrary MGP fluxes)

This scheme is similar to the F scheme, but allows the most general setup of the box-flux dynamic transport scheme, not only a unidirectionally connected chain of boxes. In the B scheme, shortcut fluxes and loops are allowed, as well as zero-, first- and second-order fluxes, box (point) sinks and box (point) sources of given elemental composition. The transport step is performed by the built-in integrator of a large  ODE system of time derivatives of masses of each chemical element in each box (reservoir). After the transport step, chemical equilibria are re-calculated in all boxes. Normally, with time (number of steps), such complex dynamic systems evolve to a steady state when masses and compositions of reservoirs do not change anymore. This scheme may be good for simulating water-sediment columns or geochemical cycles.  

'A'     1-D (advection-dispersion) reactive transport using classical finite-difference method

In this mode, the GEM2MT module performs a simple "textbook" finite-difference simulation of 1-D column reactive transport, in which not "tracer" but the whole composition of aqueous phase including water is moved between nodes during the "transport loop", followed by a "chemical equilibration loop" for all nodes at each time step (the so-called operator-splitting sequential non-iterative coupling of reaction and transport).

The column of a given length l is thought to be divided into n = 100 to 1000 nodes of equal size (l/n) with implicit diameter that defines the same characteristic volume (e.g. 1 cm3) of the node. Each node is characterized by several transport properties (porosity, permeability, dispersivity, fluid advection velocity, temperature, pressure).

The "fresh" fluid is fed from the left into node 0 and 1 that are declared as infinite flux sources (type 3) and only contain fluid. Other nodes are filled with "rock+fluid" of a given initial composition and porosity that may or may not correspond to real fluid/rock volume ratio. The very last rock-fluid node is marked as an infinite constant-flux sink of aqueous fluid (type -3).

The time step duration is determined from the Courant number, and can be augmented by dividing by a prescribed time step factor (>1). The advection velocity is input instead of the hydraulic head to simplify handling of chemical systems (that can be formally processed at constant pressure, even though this is not quite physically correct).

'C'     1-D (advection-dispersion) reactive transport using implicit Crank-Nicolson scheme

In this mode, the problem setup is the same as in 'A' mode, with the only difference that the simulation is performed using the Crank-Nicholson scheme, believed to be more robust regarding to numerical dispersion and/or error accumulation. Both 'A' and 'C' methods are only applicable to advection-dominated cases.

'W'    1-D (advection-diffusion) reactive transport using the RW (random-walk) algorithm

This method is suitable for both advection- and diffusion-dominated cases of reactive transport. Geometrical setup of 1-D column and its discretization into nodes is, in principle, the same as in the 'A' or 'C' modes (with the difference that only one fluid source node is needed instead of two). Another difference is that solute diffusivity (diffusion coefficient) Dif must be provided at input for each node. In the simplest case, the same Dif value is applied to all aqueous solutes and water-solvent; as solutes, total dissolved elements are treated as default.

The random-walk method effectively simulates the Brownian movement of "particles" of different types (about 500 to 1500 particles per node). At each time step, after the chemical equilibration loop, to one particle type, the elemental composition is assigned that corresponds to the current composition of the aqueous fluid in the node in which this particle currently resides. Other particle types can be assigned for different cations and anions, if their diffusion is to be considered individually, or to solid phase compositions that are supposed to migrate as nanoparticles.

At the following transport loop, the velocity and the direction of each particle is determined by (pseudo)random numbers obtained from a random number generator subroutine (Brownian movement) and augmented by a flow component in one direction (if advection also takes place) or even by the electrical potentials on solids in nodes. The path (velocity times time step) of each particle may move it within a given node, or from this to the next or to the previous node. The bookkeeping of particles that have crossed the boundaries of each node from all the adjacent nodes results in re-calculation of augmented bulk composition of nodes, and thus makes them ready for the next chemical equilibration loop. The time stepping should be selected such that no particle can cross more than one node boundary in one time step.

'V'    1-D reactive transport using finite-volume particle tracking algorithm (TBD)

This mode runs a similar setup of the advection-diffusion problem as the random-walk algorithm, but with better account of mass balance in nodes and of volume effects on porosity related to dissolution/precipitation of minerals.

'D'     1-D (diffusion) reactive transport using finite-volume method (TBD)

'T'      Export of initial-state GEMS3K input files set for external coupled RT codes

To facilitate initialization of large reactive transport simulations with transport codes coupled with GEMS3K, this mode lets the user to set up the necessary number of initial chemical system variants (e.g. fresh fluid, various rock types) as bulk elemental and/or formula compositions, as well as to define temperature and pressure intervals and steps for exporting a grid array of thermodynamic data for subsequent interpolation in case of reactive transport simulations with variable temperature and pressure. The export of all necessary GEMS3K input files is then performed when the "Calculate" toolbar button is activated.

2.2. GEM2MT Wizard Step 2: Dimensions for the RT model and initial chemical systems

Set here the number of local-equilibrium nodes (boxes, volumes) for the transport model (except 'T' mode), and the number of system definitions for initialization of nodes (all but 'T' modes) or for export to GEMS3K input files ('T' mode). You can set also some options related to usage of math scripts and file output of simulation results.

nC defines how many spatially distributed nodes (boxes, volumes) will be considered in the mass transport model. Typically, 10 to 1000 nodes can be used. Exception: in  'T' mode, nC = nIV must be set.

Tau defines the physical time span for 'A', 'D', 'W', 'F',  'B' type RT models. The start value and the step value can be set. The unit of measurement is arbitrary (i.e. the user must keep it in mind), although all the transport parameter values (fluxes, velocities) must be consistent with the chosen time unit (e.g. second).  

ntM  is the maximum number of (time) steps (at least one; default 1000, negative- unlimited). GEM2MT simulation will stop when this number is reached or upon the user's cancel command. The "real" time scale can be defined later in the GEM2MT window.

nIV is the number of initial chemical system definitions (e.g. rock and fluid) that are needed for the initial distribution into all nodes to define the time-zero state of the reactive mass transport problem. This distribution must be set in the "DiCp" vector, which can be filled out upon an execution of the math script for node initialization. In many cases, this script can be created by this wizard, then perhaps modified by the user before the GEM2MT run.

Lb is the number of optional user-defined formula units for setting composition recipes of initial chemical system definitions (on Page 1 of GEM2MT window). If Lb is set to 0 then the tick below should be set on to use the quantities of Independent Components instead (both ways can be used together, in which case they will add up into the bulk composition vectors of initial systems). Typical formula units are oxides ("H2O", "CO2", ... ) or other compounds, or more complex stoichiometries like bulk rock compositions. In GEM2MT module, the length of formula unit is limited to 40 characters.

Smart initial approximation for GEM calculations for each node at every time step can be activated to save on the computational time. Deactivate this mode if error messages appear or to check whether the same results are obtained with the automatic initial approximation (takes longer).

If options for math scripts are not selected then the respective wizard pages will not be accessible, and the text editor fields on GEM2MT window pages will not be visible. However, in some simulation modes, the scripts are mandatory and will be created irrespective of the user's selection.

2.3. GEM2MT Wizard Step 3: Settings for mass fluxes in reactor/box models ('S',  'F',  'B' modes)

This wizard page appears in 'S', 'F' and 'B' modes only and completes the setup of RT simulations based upon reactors (boxes) connected with defined fluxes of aqueous fluid ('S' mode) or MGP (mobile phase groups in 'F', 'B' modes). The number of boxes nC should have already been defined at Step 2. nC tells how many equilibrating reactors (boxes) will be considered in the megasystem.

At each time step, the transport of matter between boxes is performed by fluxes of mobile groups of phases (MGP). There can be MGP fluxes of three types: normal, source, and sink.

A normal flux is defined by the index of its source box, where the flux begins; the index of its destination box, where the flux ends; the index of MGP definition; the flux order (0, 1, 2, 3), and the flux rate constant. The flux order default is 0 (flux rate constant in mass/time units). Possible flux orders:

0 - Zero-order (absolute) flux j0 = v0, with a rate (constant) v0 in mass/time (kg/s) units; 

1 - First-order flux j1 = v1mout,f  relative to its MPG mass in outgoing box, with a rate constant v1 in 1/time (1/kg) units;

2 - Second-order flux j2 = v2(mout+min), relative to the sum of source and sink MGP masses flux: rate constant v2 in 1/time (1/s) units;

3 - First-order flux j3 = v3min, relative to the MGP mass in the receiving box, with a rate constant v3 in 1/time (1/s)  units. 

A point source flux is defined by its bulk composition (amounts of independent components), which can be entered in the GEM2MT window if nSFD > 0 is set below on this wizard page; the index of its destination box; the flux order (0, 1); and the positive flux rate constant vq,f > 0. The first order point-source flux is defined relative to the mass of q-th box.

A sink point flux is a point source flux with negative flux rate constant vq,f < 0.

A sink flux of MGP is an outgoing MGP flux (with negative flux rate constant) from a box q declared as a Cauchy sink (code -3).

All nFD fluxes should be set in the list accessible on the Fluxes page of GEM2MT module window. nSFD bulk compositions of point source fluxes (in moles of IComps) should be provided in the table for source fluxes at the same page. All this has to be done by the user in the 'B' mode. However, in 'S' and 'F' modes, nearly the whole setup of fluxes will be done automatically. The user must only select which phases to include in a single MGP by assigning some amounts or mole fractions to them.

Optionally, the user can set nSFD to 1 which will define a source flux into the first box defined as a normal node (and enter its bulk composition later on), and then proceed to the next wizard step.

The settings are  organized in two groups.

(1) Dimensions of MGP fluxes:

nMGP - Number of mobile phase groups (MGP): set it only for the 'B' mode (will be set automatically to 1 in S and F modes)

nFD - Total number of MGP flux definitions: set it only for the 'B' mode (will be set automatically in S and F modes). The minimum value of nFD is nC + nSFD (i.e. the number of nodes/boxes + number of elemental source compositions).

nSFD - Number of elemental source composition definitions: optional to set in 'B', 'S' and 'F' modes. In 'S' and 'F' modes, can be 0 or 1; if set to 0 (default) then the first box (index 0) will be treated as a constant source of MGP transported into the second box; otherwise, the composition of the source flux at a given rate can be added to bulk composition ofbox 0-th  at every time step (this option requires editing the initialization script).

(2) Only 'S' and 'F' mode (sequential and flow-through reactors):

Move aqueous (aq) phase: this is default for 'F' and 'S' - mode setups, the moving MGP will consist of aqueous phase only.

Move gas/fluid/plasma phase(s): Composition of gaseous phase can be moved from one reactor to the next, also as an addition to the aqueous phase composition (when the previous option is selected).

Move all solids together ('dialysis bag'): 'S' mode only. Alternatively to the two above options, the bulk composition of all solid phases will be moved from reactor to reactor (then neither aq nor gas phase can be moved).

In 'S' and 'F' modes, the outgoing MGP flux from the last box is automatically defined as a sink flux.

2.7. GEM2MT Wizard Step 7: Settings for the GEMS3K data exchange interface

These settings are important for the additional 'T' mode for preparing the GEMS3K .dch, .ipm and .dbr input files for external coupled codes, and may also be useful for certain built-in reactive transport modes. Settings are arranged in three groups.

(1) GEMS3K memory and file data exchange

Select ICs, DCs and phases to be exchanged via DATABR (take all, if unchecked): Having this option unchecked is recommended, which will take all components and phases into RT data exchange bridge.

Write IPM, DCH and DBR files in binary mode (in text mode, if unchecked): Text mode (unchecked) is default and recommended for keeping the files human-readable.

Do not write data items that contain only default values (text mode only): If this option is checked, only the data entries that contain non-default values will be written into data exchange files, while those containing only default values will be skipped. This produces compact files that are easy to read and modify in text editor (especially DBR files), but may potentially become incompatible with future GEMS3K versions if some default values will change. Leave unchecked if in doubt.

Write .dch, .ipm, .dbr files with comments for all data entries (text mode only): This is the "verbose" mode for education purposes. If this option is checked, a comment line will appear before each data entry, explaining what data it contains (similar to GEM-Selektor tooltips on data objects). May be safely unchecked by experienced users.

(2) Automatic setup of GEMS3K lookup arrays with thermodynamic data

This setup is used for creating lookup arrays for thermodynamic properties of DCs (dependent components) with regular T (temperature) and P (pressure) grid. This applies to standard Gibbs energies go, molar volumes Vo, optionally ehthalpies ho, entropies So and heat capacities Cpo (the optional arrays will be created if the respective properties are present in the parent SysEq records).

Default is one pressure value (1 bar) and one temperature value (25 oC), in which case no interpolation of thermodynamic data can be performed, and all RT calculations will be limited to 1 bar 25 oC only. Another single value of T or P can be set by entering equal number in "Minimal" and "Maximal" fields, while leaving 0 in "Step" field. RT calculations will be performed if the actual T and P values in nodes deviate from the grid T, P values plus-minus the respective "Tolerance".

To perform RT simulations with variable T and/or P, the lookup arrays must be created by setting "Maximal" T or P greater than "Minimal", and setting "Step" to 5 - 10 C for temperature and 10 - 50 bar for pressure. Usually, good interpolation results require 30-40 grid points in a large temperature range and 10-20 points in a large pressure range. Leaving one of "Step" values zero means that simulations will be performed at constant value of this parameter and variable values of another. For a given node, the interpolation of thermodynamic data in the built-in GEMS3K code will be done if the actual T, P values for the node differ from the closest grid point T, P pair by more than "Tolerance".

(3) T and P grid values for lookup arrays

Enter pressure and temperature grid values instead of automatic setup. If this option is activated, the automatic setup of T, P grid is disabled. Set the required numbers of T and P grid values in spin boxes, and enter manually or copy-paste irregular T and P sequences for the lookup grid into the table appearing in the next Step 3b page of this wizard.

Disable data interpolation between grid T values (or P values), use data for TP pairs instead. Activation of this option disables the data interpolation over the lookup T, P grid array. In this case, equal number of T and P points must be entered, and the T, P values must be entered pairwise into the respective columns in the next Step 3b page of this wizard. RT calculations can then only be performed if T, P values for a given node correspond to one of the given T, P pairs (plus-minus "Tolerance").

3.2. GEM2MT Window Page "Nodes" (Array of Nodes)

Here, the properties of each node (box, reactor) have to be defined. Normally, this is done automatically upon execution of the "mtExpr" initialization script (visible in the "Initial" page). However, the script can be disabled or assign only part of the data. Here, we describe what should be in which data object or column, to also be able to set things up by editing.

Below the title line, a "mtSzLc" object having three cells is seen, which keeps spatial dimensions (m) of the RT problem along x, y, z axis. In the present version of the GEM-Selektor code, only the x dimension (the first cell) needs to be set for 1-D RT simulations ('A', 'C', 'D', 'W' type). This value is used in calculation of temperature and pressure gradients and in determining the maximum possible time step duration from the Courant criterion (enhanced in 'D' and 'W' cases by accounting for diffusion).    

The next, "ADpar" row contains the time step reduction factor (default 1) in ADpar[0] cell, followed by the initial values of node transport properties that can be distributed to all nodes by the mtExpr initialization script.

The time step reduction factor (ADpar[0]) in 'A', 'C', 'D', 'W' modes is multiplied by the maximum time step dtmax determined from the Courant criterion to yield the time step dt actually used in calculation. The rest of ADpar array contains numerical cutoffs: one (ADpar[8]) for IC amount differences in the node between time steps (mol), default 1e-9 mol; another (ADpar[9]) for minimal amounts of IC in node bulk compositions (mol), usually 1e-12 mol.

Below the ADpar row, a large table describing all nodes (in their 1-D sequence) is located.  The first column DiCp[0] contains the index of initial system bulk composition (index of row in "mBn" table in the "Initial" page) for each node. The second column DiCp[1] contains the node type for each node:

0:   normal node;
   Boundary condition nodes:
1:   Dirichlet source (constant composition source);
-1:  Dirichlet sink;
2:   Neumann source (constant gradient source);
-2:  Neumann sink;
3:   Cauchy source (constant flux source);
-3:  Cauchy sink;
4:   Input time-depended function (TBD).  

In a typical 1-D setup, all the nodes except the first and the last one should have type 0 (normal), the first node (with index 0) has type 3 (constant flux source of fluid), and the last node has type -3 (sink of fluid). Sources can also be declared in some nodes in the middle of the 1-D setup.

The next four columns contain: StaP[0] pressure (bar) in each node; StaP[1] temperature (C) in each node; StaP[2] transport volume (dm3) of each node (usually zeros); StaP[4] mass (kg) of each node. The following 7 columns to the right belong to the HydP table of transport properties of nodes, used in A,C,D,W modes only and initially filled out by the mtExpr initialization script by copying from the ADpar array using the following assignments:

HydP[qc][0] =: ADpar[1];   $ Vt:  total node volume (m3)
HydP[qc][1] =: ADpar[2];   $ vp:  fluid advection velocity (m/s)
HydP[qc][2] =: ADpar[3];   $ eps: effective porosity, 0<eps<=1
HydP[qc][3] =: ADpar[4];   $ Km:  effective permeability (m2)
HydP[qc][4] =: ADpar[5];   $ al:  specific longitudinal dispersivity (m)
HydP[qc][5] =: ADpar[6];   $ Dif: general aqueous medium diffusivity (m2/s)
HydP[qc][6] =: ADpar[7];   $ nto: tortuosity factor, usually 1<=nto<10    

Upon the RT simulation, transport properties, temperatures and masses of nodes may in principle change, if this is required by RT algorithm, and/or programmed in the third part of the mtExpr script.

3.3. GEM2MT Window Page "Fluxes" (MGP Compositions, Array of Fluxes)

The data in this page is not used in 'A', 'C', 'D', 'W' modes of GEM2MT, but is essential for MGP box-flux transport models (modes 'S', 'F', 'B').

The first table (below the comment line) describes compositions of mobile phase groups MGP (one per MGP column) in terms of quantities of phases from the outgoing box (node). Each MGP must have an ID (short name) and must contain non-zero quantity of at least one phase. Usually, there is one MGP "aq" that contains aqueous electrolyte phase "aq_gen". In 'B' mode, other MGPs can be defined too, e.g. one for moving the gas phase, or one for moving aqueous and gas phases together, or one for moving aqueous and some minor (suspended) solid phases, and so on.

The next (optional) BSF table contains elemental compositions of point sources/sinks used in 'B' mode for special fluxes performing a production or a removal of matter in certain boxes. Each such source, in fact, is also an MGP with index of row in source composition table used as the name of this MGP (i.e. "0", "1". ...).

The next table defines all the fluxes between, to and from boxes (nodes). The first two columns must contain the indexes of outgoing (FDLi[0]) and incoming (FDLi[1]) boxes. The index -1 in FDLi[1][qf] means that a sink (or production or removal) flux with index qf is defined in the box with the index given in FDLi[0][qf].

The next column FDLmp identifies as MGP moved by this flux by MGP ID name. This can be either a real name such as "aq" or "aqueous" available in the MGP composition table above, or "0" or "1" ... which is an index of production or removal bulk composition - one of defined in the BSF table above.

The next two FDLf[0] and FDLf[1] columns define the order and the rate (constant) for each flux. Possible flux orders:

0:  Zero-order (absolute) flux j0 = v0, with rate v0 in mass/time (kg/s) units (default); 
1:  First-order to the outgoing box mass flux j1 = moutv1, with a rate constant v1 in 1/time (1/s) units;
2:  Second-order to outgoing and incoming MGP mass flux j2 = (mout+min)v2, with a rate constant v in 1/time units, proportional to the sum of masses in both boxes;
3:  First-order flux j3 = minv3, to MGP mass in receiving box, with a rate constant v3 in 1/time units.

The columns FDLf[2] and FDLf[3] are reserved for future development. The following column FDLid may contain different IDs for individual fluxes, if desired (not used in calculations). The last FDLop column is reserved for future development.   

Scrolling much below the mentioned tables, you will find work matrices MB (current masses of chemical elements in all boxes) and below that, a dMB matrix with current derivatives of masses of elements in all boxes. These matrices are updated at every time integration step and can be used for monitoring/debugging purposes.

ADpar[0] Advection, diffusion and box-flux transport: time step reduction factor tf (usually 1; 0 resets to 1)

This applies to 'A', 'C', 'D', 'W' and 'B' modes.

In the 'B' mode, ODE integrator implicitly assumes time in seconds and sets the maximum time step duration dt to 1.7 s. In the case of slow fluxes, the factor tf can be applied; for example, if tf = 60 is set then the whole time iterator mtTau should define maximum time and time step in seconds multiplied by 60, i.e. effectively each time step will be one minute (or maximum 1.7 min or 102 s). if tf = 3600 then the time step will be effectively 1 to 1.7 h, and so on. The flux rate constants in FDLf table must be scaled up accordingly (just as dt, they will be internally divided by the tf factor for the ODE integrator). 

In 'A', 'C', 'D', 'W' modes, the time step duration dt is determined from the CFL (Courant-Friedrichs-Levy) condition. For 1-D transport case, the CFL condition has the form:

C = vp * dt / dx <= Cmax, where C is the Courant number, and dx = x/nC is the node size.  

The Cmax value depends on the method. For explicit finite differences (A mode) or diffusion-dominated transport (W, D modes), Cmax = 0.5; for Crank-Nicolson implicit method (C), Cmax = 1.0. For diffusion-controlled transport with advection term (W, D modes),

C = min( (vp * dt / dx), (dt / dx2) ) <= Cmax.

From the above expression, the maximum allowed time step duration dt can be evaluated and used in RT simulations that account for advection, dispersion and diffusion:

dtmax = min( (Cmax * dx / vp), (Cmax * dx2 / (al * vp + Dif) ) )  

where al is the dispersivity, and Dif is the diffusivity in aqueous medium.

The actually used time step in GEM2MT is: dtactual = dtmax / tf.
The higher values of time step reduction factor tf > 1 may be applied in case of problems with the numerical stability of RT simulations.
     

ADpar[1] Vt - initial total node volume (m3)

This value of node volume is only used in the context of porosity - permeability relationships (i.e. it is not the same as the actual volume of the node system as computed by GEM). Will be copied for all nodes by the RT problem initialization script.


ADpar[2] vp - fluid advection velocity (m/s), usually 1e-10 < vp < 1e-3

This inital value of advection velocity will be copied into all nodes by the RT problem initialization script. This value should be not less than 1e-10 m/s and usually is about 1e-6 or 1e-7 m/s for groundwater flow in the aquifer.

The fluid advection velocity depends on the value of Darcy flux q and porosity eps as vp=q/eps.
The Darcy flux q or discharge per unit area (in m/s) depends on the permeability Km, pressure gradient (hydraulic head) dP (in Pa/m), and fluid viscosity mu as q = -Km/mu * dP. The minus sign shows that the flow occurs against the pressure gradient (i.e. from high to low pressure).
 

ADpar[3] eps - initial effective porosity, usually 0 < eps < 1 (default 1)

The eps value will be copied for all nodes by the RT problem initialization script.

Typical porosity values are between 0.2 and 0.6 for different rock types, can be effectively less if a fractured reservoir rock is approximated as a porous medium.

Porosity is a factor that connects the advection velocity vp with the Darcy flux q as vp=q/eps.

The advective flux of aqueous fluid through area A (m2) cross section (in kg/s) is  J = den * A * eps * vp, where J is the density (of aqueous fluid in kg/m3).


ADpar[4] Km - initial effective permeability, m2, usually 1 darcy = 1e-12 m2

Permeability is the property of porous medium (rock) showing the ability of the fluid to flow through this medium.

Permeability is related to the advection velocity vp as vp = -Km/mu * dP/eps, where dP is pressure gradient (Pa/m), eps is the porosity, and mu is the fluid viscosity (mu = 1e-3 Pa*s for water at room conditions, less at higher temperatures).

Typical values of intrinsic permeability (in millidarcy = 1e-15 m2) for common types of rocks: 1e8 to 1e-4 for gravel, sand, highly fractured rock;1e4 to1 for fine sand, silt, loess, loam, peat, layered clay, oil reservoir rocks, sandstone; 0.1 to 0.0001 for unweathered clay, limestone, dolomite, fresh granite, basalt.

The Km value will be copied to all nodes by the RT problem initialization script. 
    

ADpar[5] al - initial value of specific longitudinal dispersivity (m), usually 0.001 to 1 m

Mechanical dispersion (hydrodynamic dispersion) results from water movement at rates both greater and less than the average linear velocity. This is because fluids move faster through the center of the pores than along the edges, or travel shorter pathways and/or splitting or branching to the sides, or move faster through larger pores than through smaller pores. All this results in mixing along the flow paths, which is called mechanical dispersion. Mixing that occurs in the direction of flow is called longitudinal dispersion; that occurring normal to the direction of flow is called transversal dispersion.

In 1-D advection models, by analogy with diffusion, the dispersion is accounted for as a mechanical dispersion coefficient Dm = al * vp, where al is the dispersivity, and vp is the (average) advection velocity. Dm (in m2/s) can be combined with effective diffusion coefficient De into the hydrodynamic dispersion coefficient Dh to be used in the Fick's First Law term describing the dispersion flux.

The values of longitudinal dispersivity may vary from 0.001 to 10 m for column length up to 10 m, depending on the soil/rock type (about 1 in homogeneous sandy aquifer, less in clayey rocks).

The al value will be copied into all nodes by the RT problem initialization script.
    

ADpar[6] Dif  - initial general aqueous medium diffusivity (m2/s), usually 2e-9

Molecular diffusion is the process where the thermal motion of individual molecules or particles causes a flux of solutes from areas of higher concentration (i.e., chemical activity) to areas of lower concentration.

The diffusion coefficient Dif is a proportionality constant between the molar flux due to molecular diffusion and the concentration gradient, and is a function of the temperature and molecular weight. In locations where the advective flux is low or negligible (clayey aquitards and dense rocks), diffusion may be the dominant transport mechanism.

The effective diffusion coefficient De as used in the Fick's First Law to describe diffusion transport of water and solutes in porous medium, is connected to the aqueous medium diffusion coefficient Dif as De = Dif * eps, where eps is the porosity.

The value of self-diffusion coefficient of water at 20 oC is 2e-9 m2/s, it increases to almost 8.6e-9 m2/s at 100 C. Diffusion coefficients of aqueous solutes are in the range from 1e-9 to 3e-9 m2/s at room temperature. For this reason, a common Dif value of 2e-9 m2/s is frequently used for water and all solutes.
 

ADpar[7] nto - initial tortuosity factor, usually 1<=nto<10

In porous medium, diffusion pathways of molecules is complicated by pore structure and arrangement relative to simple Brownian movement in aqueous medium. Therefore, the diffusivity of solutes Dpm measured in porous medium is different from that in aqueous medium Dif at the same temperature. Both can be related using porosity eps and the tortuosity factor nto as follows:

nto = eps * Dif / Dpm  

There are complex relationships between porosity and tortuosity because of pore size, geometry and distribution.

The tortuosity factor is used in D and W modes only.

HydP[1] vp: fluid advection velocity in nodes (m/s)
 
Fluid advection velocity vp (initially taken from ADpar[2]) can be usually set to 1e-6 m/s.
This value should be not less than 1e-10 m/s and usually is about 1e-6 or 1e-7 m/s for groundwater flow in the aquifer.

The fluid advection velocity depends on the value of Darcy flux q and porosity eps as vp=q/eps. The Darcy flux q or discharge per unit area (in m/s) depends on the permeability Km, pressure gradient (hydraulic head) dP (in Pa/m), and fluid viscosity mu as q = -Km/mu * dP. The minus sign shows that the flow occurs against the pressure gradient (i.e. from high to low pressure).


HydP[2] eps: effective (actual) porosity in nodes (0 < eps < 1)

The porosity value (initially taken from ADpar[3]) will be copied for all nodes by the RT problem initialization script.

Typical porosity values are between 0.2 and 0.6 for different rock types, can be effectively less if a fractured reservoir rock is approximated as a porous medium. Porosity is a factor that connects the advection velocity vp with the Darcy flux q as vp=q/eps. The advective flux of aqueous fluid through area A (m2) cross section (in kg/s) is  J = den * A * eps * vp, where J is the density of the fluid (of aqueous fluid in kg/m3).


HydP[3] Km: effective (actual) actual permeability in nodes, m2

The permeability value (initially taken from ADpar[4]) will be copied for all nodes by the RT problem initialization script.

Permeability is the property of porous medium (rock) showing the ability of the fluid to flow through this medium. Permeability is related to the advection velocity vp as vp = -Km/mu * dP/eps, where dP is pressure gradient (Pa/m), eps is the porosity, and mu is the fluid viscosity (mu = 1e-3 Pa*s for water at room conditions, less at higher temperatures).

Typical values of intrinsic permeability (in millidarcy = 1e-15 m2) for common types of rocks: 1e8 to 1e-4 for gravel, sand, highly fractured rock;1e4 to1 for fine sand, silt, loess, loam, peat, layered clay, oil reservoir rocks, sandstone; 0.1 to 0.0001 for unweathered clay, limestone, dolomite, fresh granite, basalt.


HydP[4] al: specific longitudinal dispersivity of porous medium in nodes, m

The dispersivity value (initially taken from ADpar[5]) will be copied for all nodes by the RT problem initialization script.

Mechanical dispersion (hydrodynamic dispersion) occurs because fluids move faster through the center of the pores than along the edges, or travel shorter pathways and/or splitting or branching to the sides, or move faster through larger pores than through smaller pores. All this results in mixing along the flow paths, which is called mechanical dispersion.

In 1-D advection models, by analogy with diffusion, the dispersion is accounted for as a mechanical dispersion coefficient Dm = al * vp, where al is the dispersivity, and vp is the (average) advection velocity. Dm (in m2/s) can be combined with effective diffusion coefficient De into the hydrodynamic dispersion coefficient Dh to be used in the Fick's First Law term describing the dispersion flux. The values of longitudinal dispersivity may vary from 0.001 to 10 m for column length up to 10 m, depending on the soil/rock type (about 1 in homogeneous sandy aquifer, less in clayey rocks).


HydP[5] Dif: general diffusivity of disolved matter in nodes, m2/s

The diffusivity value (initially taken from ADpar[6]) will be copied for all nodes by the RT problem initialization script.

The diffusion coefficient Dif is a proportionality constant between the molar flux due to molecular diffusion and the concentration gradient, and is a function of the temperature and molecular weight. In locations where the advective flux is low or negligible (clayey aquitards and dense rocks), diffusion may be the dominant transport mechanism. The effective diffusion coefficient De, as used in the Fick's First Law to describe diffusion transport of water and solutes in porous medium, is connected to the aqueous medium diffusion coefficient Dif as De = Dif * eps, where eps is the porosity.

The value of self-diffusion coefficient of water at 20 oC is 2e-9 m2/s, it increases to almost 8.6e-9 m2/s at 100 C. Diffusion coefficients of aqueous solutes are in the range from 1e-9 to 3e-9 m2/s at room temperature. For this reason, a common Dif value of 2e-9 m2/s is frequently used for water and all solutes.


HydP[6] nto - tortuosity factor in nodes (1 to 10) 

The tortuosity factor is used in D and W modes only. Its value (initially taken from ADpar[6]) will be copied for all nodes by the RT problem initialization script.

In porous medium, diffusion pathways of molecules are complicated by pore structure and arrangement relative to simple Brownian movement in the aqueous medium. Therefore, the diffusivity of solutes Dpm measured in porous medium is different from that in aqueous medium Dif at the same temperature. Both can be related using porosity eps and the tortuosity factor nto: nto = eps * Dif / Dpm. There are complex relationships between porosity and tortuosity because of pore size, geometry and distribution.

In this column ('B', 'F', 'S' modes), the flux order has to be defined. Default is 0 (constant flux in units mass/timestep). Possible values:

1 - First-order to source box mass flux, v in 1/time/kg units;

4.1. Data objects only used in GEM2MT initialization and sampling  scripts

The data objects listed below are only used in GEM2MT scripts. Some of them can be seen in GEM2MT "Settings" page, these are current values of indices ("kv", "qc", "qf", ... ) and iterators ( "cTt", "cPt", ...). Others are only visible in the GEM2MT wizard in the lists of objects for setting up the automatic sampling script to show how properties of all nodes change with simulation (time) steps.

These data objects are actually mapped to two arrays of data representing the nodes (boxes) in two states: state 0 and state 1.

(0) After each equilibration loop, new speciation is calculated by GEMS3K kernel in all state 1 nodes. Hence, we recommend to use the node state 1 objects in the sampling script to collect and plot chemical properties of nodes after the equilibration.

(1) Before the next transport model integration loop at the next time step, the data from state 1 nodes are copied into state 0 nodes, which thus keep the composition and transport properties from the previous time step.

(2) Next, the mtExpr script is executed (to optionally modify tables with transport properties of nodes and possibly rates of fluxes), and the transport loop starts acting on state 1 nodes. It may use (modified) tables of transport properties of nodes and fluxes to augment those of nodes, then performs the chosen mass transport algorithm that moves the chemical elements between nodes. Most transport methods also use the previous state 0 properties of nodes.

(3) After the transport loop is finished on state 1 nodes, the equilibration loop starts on those nodes - by executing the GEMS3K solver node-by-node. In the automatic initial approximation (AIA) mode, the solver just takes the modified bulk chemical composition of the node and writes back the updated speciation in phases. In smart initial approximation (SIA) mode, the GEMS3K solver takes the speciation saved in this node state 0 (i.e. from GEM calculation at previous time step). In most cases, this leads to factor 5-10 faster simulations.

When equilibria in all nodes are computed, the control goes to step (0) to start the next trasport loop, unless the maximum number of steps is reached or the user cancelled the simulation.

We provide the access also to state 0 node contents because this is the only way to follow in a simple manner how big are changes in properties of nodes between the current and the previous time step.
