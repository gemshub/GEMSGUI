 1. The GEM Task ("Multi") Data Structure

1.1. Introduction  

The GEM Task ("Multi") data structure is dynamically allocated in computer memory and keeps all scalars and arrays containing input, work and output data of the GEM IPM-3 algorithm (more...) used in calculation of an equilibrium state. In the compressed form, this data can be saved into the SysEq record, from where it can be retrieved anytime. Contents of the Multi structure can also be exported into GEMS3K input files and used in standalone coupled code calculations.

In the GEM approach, no "master" and "product" species are distinguished. The stoichiometry of all gaseous, aqueous, liquid, solid or surface species is written as chemical formulae in terms of  Independent Components (IC). The term "independent component" must be understood as an independent extensive thermodynamic variable representing the total molar quantity in the system (typically number of moles of chemical element in the total bulk composition).

All chemical species which can exist in the system are then defined as "Dependent Components" (DC) as they are composed of fixed  quantities of IC.  The DC stoichiometries are represented in the Multi structure as a stoichiometry matrix A; a row of this matrix, multiplied by the number of moles of DC (or species) xj, comprises a contribution of this DC to the overall mass balance relative to the input total number of moles of  IC bi  in the system (bulk composition).

The stability of any species (DC) at   T, P of interest is defined by its standard molar (or molal in case of aqueous and surface species) Gibbs energy  function Goj . The Gibbs energies, together with the activity terms in the respective phases, compose primal chemical potentials, then multiplied by xj  and summed up into the total Gibbs energy G (x)  of the whole system. The GEM algorithms seek for values of  xj  such that, for the imposed material balance and optional metastability constraints,  bring the G (x ) scalar to minimum. These xj  values in equilibrium comprise the so-called primal solution of the GEM task; many other quantities such as amounts of phases, activity coefficients of species, concentrations in various scales etc. can be directly calculated from the primal solution.

Simultaneously with primal solution x  data vector, the dual solution u data vector is calculated, which contains chemical potentials of Independent Components in this equilibrium state. Using the dual solution, it is convenient to compute general activities of chemical species, even those present in zero amounts, or kept by non-trivial metastability constraints. This includes fugacities of gases, as well as pH, pe, Eh of aqueous electrolyte, and stability indexes of single- and multi- component phases.   

1.2. Input data

To set up a single GEM chemical equilibrium problem, the following input items must be provided:

  • list of names of Independent Components;
  • bulk chemical composition of the whole system, i.e. mole amounts of Independent Components (vector b);
  • list of names of Phases that may appear in the equilibrium state;
  • list of names of chemical species (Dependent Components) arranged according to the order of Phases;
  • standard molar/molal Gibbs energy function values gTP at T, P  of interest for all Dependent Components;
  • interaction parameters for calculation of activity coefficients of species in non-ideal multi-component phases;
  • optionally, specific surface areas of phases; metastability constraints for some components in some phases;
  • stoichiometry matrix A made of elements  aij - mole amount of each Independent Component in one mole of each Dependent Component. 

1.3. Output data

GEM task calculation produces the following main results:

  • total scalar Gibbs energy of the system G(x) = G(u,b) in equilibrium state;
  • dual solution of GEM task: vector u - chemical potentials of Independent Components in equilibrium state;
  • primal solution of GEM task: vector x - mole amounts of Dependent Components in equilibrium state;
  • concentrations, activities, activity coefficients of Dependent Components in respective Phases;
  • activity-based properties of aqueous or gaseous species (pH, pe, Eh, gas fugacity etc.);
  • amounts, volumes, masses, stability indices, and other integral properties of Phases;
  • mass balance residuals, convergence criteria, other information about quality of primal and dual solutions.

1.4. Relationships with the project system data structure

The GEM task input data in the Multi structure (and GEMS3K input files) are generated from the Chemical System Definition (CSD) and Recipe for Bulk Chemical Composition (BCC)  assembled in the project system framework (more ...) and condensed by eliminating names and data of all ICs, DCs and Phases that were turned off in the CSD.

This means that in GEM-Selektor, two indexation systems for components and phases exists at the same time: one related to modeling project lists and another- to internal Multi component- and phase- lists. These indexations are the same only if no components and phases were turned off in the particular CSD. Both indexations are saved (in compact form) in SysEq records. The program knows for each data object (list, vector, table) which indexation should be applied to it, and can convert indexes reciprocally between two indexation systems. This gives an unmatched freedom in creating CSDs of different complexity within the same modeling project.

In standalone kernel GEMS3K, only "internal" (Multi) indexation is used, and no components or phases can be turned off (they can only appear in zero amounts in mass balance and in primal solution, if unstable).


2. How to Work with GEM Task Data for a Chemical System

The setup of Chemical System Definition (CSD) and Recipe up to Bulk Chemical Composition (BCC) is described here.

After checking the BCC for consistency with the CSD, both will be condensed into a GEM task by eliminating all Phases and DCs that were turned off in the CSD, and calculation of the equilibrium state begins. This process is activated by clicking the "Calculate Equilibrium with GEM" upper toolbar button or by "Calculate" "GEM IPM" menu command. During the calculation, the Progress Dialog appears (more...) that shows some main information about the equilibrium state. When done, the Progress Dialog will show the "Accept" button, clicking on which saves the SysEq record with CSD, BCC and GEM task results into the project database.

The results can be examined in the "Results:" tab of the "Single System" dialog (more...) and in detail - in the EqDemo window, which can be opened by clicking the "Open EqDemo window" toolbar button or using the "View" "GEM IPM dual solution etc." menu command. Even more detail can be seen in the "Multi" window pages accessible using the "View" "Details for experts" menu command.

Furthermore, CSD, BCC and GEM calculation results can be printed (into ASCII text files) using the "Print" menu commands:

In each case, the new text file name and path must be entered or selected in the "File open/save" dialog that appears afterwards. 

After using the AIA (automatic initial approximation, default) mode, it could be of interest to check whether the GEM task results will change or not if the SIA (smart initial approximation) mode of GEM IPM has been used. To do so, open the "Calculate" and toggle off the "AIA (simplex)" entry, then repeat the GEM IPM calculation. Normally, it does about one iteration on the same system, and the GEM results do not change. If they do change or GEM IPM does many iterations, then the numerical settings and/or CSD and BCC must be revised (more...). Note that any next GEM calculation, also for any new (cloned) system, will be done in the SIA mode without the simplex LP step, until the "AIA (simplex)" menu entry will be toggled back, or the project session will be closed.

When the GEM IPM algorithm cannot converge and error messages appear over the Progress Dialog, it may be possible to locate the reason using the "Stepwise" mode of GEM IPM calculations. To do so, open the "Calculate" menu and check there the "Stepwise" entry. Then open the EqDemo window and start GEM calculation. You will be able to click on "Step" button in the Progress dialog (more...) to make a next iteration and examine the data in EqDemo and Multi window pages. To leave the stepwise mode, click "Resume" button ("Stop" goes back into "Stepwise" mode).


2.1. GEM Task Calculation and Progress Dialog

This dialog appears only in Single System Equilibrium Calculation mode after clicking the "Calculate equilibrium with GEM" toolbar button (or "Calculate" "GEM IPM" menu command). Its purpose is to show the progress of GEM IPM algorithm, the extent of convergence, and how some main properties of the system change over the iterations, especially for large systems (then the dialog updates ca. once per second). Here is the typical appearance of this dialog after the GEM calculation has converged.




In the upper part, a progress bar is visible; 100% means that the Dikin criterion of convergence (CD) has reached the prescribed tolerance Pa_DK (more...); less than 100% means that only part of the maximum 'distance' between CD and DK has been worked through. Above the progress bar, the currently active stage of IPM algorithm is indicated (e.g. Simplex initial approximation; Mass balance refinement; Main IPM descent iterations, etc.).

Below, under "System:", the current SysEq record key is shown, where the CSD and results will be saved afterwards.

The "Iter" field shows three numbers, the first is the number of Phase Selection loops done, the second shows the total number of mass balance refinement iterations performed, and the third - the total number of main IPM descent iterations performed. These numbers help to see if some unexpected behavior occured, or some numerical settings were not optimal.

The next four boxes below show the total masses of phases (grams) in four main aggregate states: Gaseous, Aqueous, Liquid and Solid. This information is also shown on the pictogram to the left (in white, blue, green and black, respectively).

For aquatic systems, also the values of pH, pe, and IS (ionic strength) in aqueous solution are shown.

The Accept button closes the dialog while saving the GEM task results into the SysEq record with the indicated key, while the Dismiss button (default) closes the dialog without saving record contents.

In Stepwise mode, the Progress dialog looks like shown below:




The Step button performs one iteration of the current IPM stage (visible on the iteration counter). It is possible to go to other windows and to the Single System dialog to explore the GEM task data.

The Resume button restores normal (non-stepwise) run, the button Stop appears instead of Step. Clicking on Stop restores the Stepwise mode.

The Cancel button breaks the calculation process and cancels the GEM IPM algorithm. The GEM task data remain, but may be indefinite. The SysEq record is not saved.


2.2. 'Results' Tab of the Single System Dialog

This dialog, used in the "Equilibria Calculation" mode of GEMS, consists of two tabs: one called "Input: System Definition" described here...  that provides a compact list view on the setup of chemical system, and another that shows a compacted GEMS task with results of calculation of the equilibrium state. The latter dialog  "Results: Equilibrium State" is described below. Detailed data related to GEMS task and GEM primal and dual calculation results can be accessed in the EqDemo window pages. Hover the cursor over the data field in the tab ca. 1 s to see a tooltip.

The "Results:" tab contains a hierarchical list of phases available in the modeling project system, with names visible in the "Phases/species" column (more help on any column can be obtained by selecting any data element and pressing F1) . By clicking on a small square with '+' to the left of the Phase name expands the list showing subordinate entries for phase components (species, Dependent Components). The output data in some columns is sometimes different for a phase or for a Dependent Component (see below).

Attention! In comparison with the "Input:" tab, the "Results:" tab does not show any phase or chemical species that have been turned off in the chemical system definition; here in the "Results:" tab, only the data in a 'condensed' GEM task that were actually involved in this calculation of equilibrium state is visible.

The "Results:" dialog tab consists of several columns. 

The "L" column shows the number of Dependent Components in each Phase. Pure phase contains 1 component. Solution phases (more than 1 component) are shown on top, the topmost (if present) is the aqueous phase. These numbers are also visible in the L1 column in EqPh page of EqDemo window.

The "Type" column (PHC data object) shows codes of Phase aggregate state ('a' aqueous, 'g' gaseous, 's' solid etc.).

The "Amount (mol)" column for a Phase contains its total mole amount in equilibrium state (see also the Xa column in EqPh page of EqDemo window). For a chemical species (Dependent Component in a phase), this column displays the mole amount of this DC in equilibrium state (mapped from the x vector in EqDC page of EqDemo window). These numbers comprise the GEM primal solution (i.e. chemical speciation in this equilibrium state). 

The "LogSI/Activity" column for a Phase shows its stability index in equilibrium state (lgSI, in decimal logarithmic scale, see also Fa in EqPh page of EqDemo window).  For any phase present in a positive amount in equilibrium under trivial metastability constraints, lgSI is close to 0 (in fact, its value should be between Pa_DF[0] and -Pa_DF[1]); for any unstable phase, lgSI < -Pa_DF[1] has a negative value (the phase may still be present in positive amount if it includes a component with a non-trivial metastability constraint from below, dll). Positive value lgSI > Pa_DF[0] usually means that the phase contains a component with a non-trivial metastability constraint from above, dul, i.e. the system is over-saturated with respect to this phase.  

The "LogSI/Activity" column for a Dependent Component shows its activity aj in the respective phase (in activity scale; for gases, the fugacity fj =  aj P  is shown), computed out of the GEM dual solution vector u (see EqIC page of EqDemo window) and DC standard (molar) Gibbs energy function at T,P of interest (more ...). For any stable DC (under trivial metastability constraints), this dual-thermodynamic activity must be (almost) the same as the primal activity calculated as a product of concentration and activity coefficient (see below). For any pure phase, the decimal logarithm of its DC activity equals the lgSI (Fa) value for the phase.   

The "Concentration" column (for Dependent Components only) contains the DC concentration Cj in the respective phase (for pure phase, always 1) in the appropriate concentration scale: for aqueous species, the molality (see also the my vector in EqDC page); for gas mixture and non-electrolyte liquid and solid solutions, the mole fraction (see Wxx vector in EqDC page).

The "Activity coeff." column (for Dependent Components only) contains the DC composite activity coefficient, defined as gammaj = aj / Cj , i.e. a ratio of primal activity to concentration in the phase. For gases,  gammaj = fj / P / Cj . "Composite" means that gammaj may contain various contributions also related to partial complex configurational entropy, non-logarithmic asymmetry terms, electrostatics, etc. (see details in "Activity-coeffs.pdf" file in the "/Resources/doc/pdf" folder). For Cj = 0, gammaj is indefinite and set to 1 by the program. Primal activity coefficients are computed by the built-in functions belonging to the TSolMod class library (Wagner et al., 2012), or from Phase scripts, and initially collected in the lnGam data object at each GEM IPM iteration.

Note that the product gammaj Cj  (only when Cj > 0) is never exactly equal to the dual-thermodynamic activity aj (shown in the logSI/Activity column) because the absolute convergence of GEM algorithm is not possible (more ...). However, usually the deviation is less than in the fifth digit after the decimal point. For DCs present in zero amount, only dual thermodynamic activity can be returned because calculation of primal activity, concentration, and activity coefficient is not possible.

The information shown in the "Results:" tab can be written into a text file using the "Print" "Results: Equilibrium State" menu command. It can also be copy-pasted into Microsoft Excel or another electronic spreadsheet program using the right-click menu (below) that appears upon right-clicking with cursor over any data entry:


 

In this menu, the "Help" command (or F1 hot key) shows the help window with specific information for this data item. The "Select ..." commands highlight respective parts of the dialog tab. The "Copy" command takes selected data into clipboard, and "Copy with names" takes data with column headings into the clipboard. After that, the data can be pasted into a third-party spreadsheet or into a text editor window.

Detailed GEM Task input and output can be printed into text files as formatted table-like text using the first three commands in the "Print" menu that uses print scripts supplied within GEMS installer (these scripts are copied into the modeling project as SDRef records and can be modified by the user).  The "GEM Results..." command can also be invoked directly by clicking the "Print data to text file" upper toolbar button. 


3. EqDemo Submodule Window

This window shows in detail the most relevant contents of the GEM Task data structure ("Multi"). It appears upon clicking the "Open EqDemo window" toolbar button or upon the "View" "GEM IPM task results..." menu command. The window consists of five pages: EqIC, EqPh, EqDC, EqSurf and EqGen. Any data field or table on every page can be copied into clipboard (and pasted into third-party spreadsheet or text editor) by right-clicking on any data field and selecting from the appearing menu.


3.1. EqDemo Window EqIC Page

In this page, the tname and tnote comment lines are shown for identification of this CSD and GEM task saved in the SysEq record with key visible to the right of the upper toolbar.  Below, a table appears, combined of several data columns related to Independent Components used in this GEM task as the stoichiometry basis.

The ICnam column contains symbols of Independent Components. If necessary, click on an IC symbol and press F7 on the keyboard (or right-click and select "Show record" menu item) to see contents of this record appearing in the IComp window.

The b column contains the (compressed) input BCC (bulk chemical composition, in moles) of the system. This is essentially the same numbers as in the B_ vector (on IComp page of the System window) for all ICs that are turned on in the CSD.

The Cb column contains (after the last GEM IPM calculation) the mass balance residuals (moles) that show how precisely the input BCC was split by GEM into the amounts of DCs in the primal solution (speciation) vector x. The mass balance is very good if the absolute values of residuals do not exceed 10-12 mol, or 10-13 fraction of any respective total amount in the b vector. The relative mass balance tolerance (10-13 by default) can be changed as Pa_DHB in the Numerical Settings (more...).

The u column contains the GEM dual solution, i.e. chemical potentials of ICs in this equilibrium state (shown in the normalized ln a scale). These numbers are used in GEMS for calculation of dual-thermodynamic activities (fugacities) of chemical species, as well as functions such as pH, pe, Eh. Summing up the products bjuj results in a total Gibbs energy of the system (in moles) in this equilibrium state.

The lgm_t and m_t columns appear only if the aqueous phase is present in the GEM task. The latter contains total dissolved molalities of ICs (for H and O without the water-solvent), and the former - decimal logarithms of the latter. This data, calculated automatically out of the primal solution (speciation) vector x, provide a good overview on solubility of each chemical element (IC) in the aqueous phase.  


3.2. EqDemo Window EqPh Page

This page displays three tables showing relevant output for Phases taken into this GEM task.

The upper table shows the list of phase names PHnam extracted from Phase record keys. It is possible to click on a phase name and press F7 on the keyboard (or right-click and select "Show record" menu item) to see contents of the respective record appearing in the Phase window. It is possible to edit and save Phase records; however for changes to take effect, it may be necessary to re-open the project or re-load the SysEq record.

The Xa column shows total mole amounts of phases in equilibrium state, computed from the primal solution (speciation) vector x using the number of DCs in each phase from the L1 column. The Fa column shows the (logarithmic) phase stability criteria; for a pure phase, Fa is the same as the logarithmic saturation index. The phM column contains total masses of phases in grams), and phVol column - total volumes of phases in cm3. 

The table below shows the total bulk compositions of all solid phases (bXs, the first row, moles) and bulk compositions of multi-component phases (bXa, other rows, moles) that are both frequently used in Process simulations and reactive transport runs (more...). The columns correspond to Independent Components, the row indexes are the same as those of phases in the upper table. The pmXs cell contains (automatically computed) total mass of all solid phases in grams. 

The lowermost table shows an overview of specific surface areas (Aalp column, in m2/g), specific free energies of phase-aqueous (Sigw) or phase-gas (Sigg) interfaces, and of the specific phase stability shift Yof. These properties are mainly used in systems containing dispersed or sorption phases. 


3.3. EqDemo Window EqDC Page

This page displays two scrollable tables showing the output for Dependent Components involved in Phases in this GEM task.

The DCnam columns contain symbols of Dependent Components. If necessary, click on an DC symbol and press F7 on the keyboard (or right-click and select "Show record" menu item) to see contents of this record appearing in the DComp or ReacDC window. There, records can be edited and saved in the project database (e.g. for bugfixing), but the changes will not be operational until the temperature or pressure has changed in the SysEq record key. 

The x column - the primal solution of GEM problem (also called the speciation) - contains mole amounts of DCs in this equilibrium state; these numbers are also visible in the "Results:" tab of the Single System dialog. The lga column contains dual-thermodynamic activities (for gases - fugacities) of DCs in respective phases (in decimal logarithmic scale). These numbers (converted into thermodynamic concentration scale) are also visible in the "Results:" tab, as well as the DC activity coefficients given in the gamma column. Molalities of aqueous species are shown in the my column (and also in the in the "Results:" tab).  

In the lower table, the c column contains the normalized Gibbs energy functions of DCs converted to common mole fraction basis. The fDQF column contains optional pure-state fugacities used in some fluid models, or Darken constants used in some solid solution models (more...). The Wxx column contains mole fractions of DCs in respective phases (part of this data is also visible in the "Results:" tab). The v column contains DC primal chemical potentials (normalized), and the fx column contains dual-thermodynamic DC chemical potentials (in J/mol).  The DCC column shows the class codes of DCs used internally to select the appropriate expressions for primal chemical potentials.

4. Input Data for a GEM Task and IPM Run

The data objects listed below are automatically filled out with the GEM IPM input data, as specified in the CSD (see gems_bcc.html  manual). In this process, necessary conversions are performed, e.g. values of the molar Gibbs energy function g(TP) of Dependent Components are scaled to dimensionless units in a uniform rational concentration scale (see  theory_gem.html  manual for details). 

For the "Multi" data objects used in surface complexation models, see gems_scm.html manual.
  

Scalars

_nnr[0] - Number of Independent Components in the IPM problem n(N)
 
L[0] - Number of Dependent Components in the IPM problem n(L)
 
L[1] - Number of Dependent Components included into  multicomponent phases n(Ls)
 
L[2] - Index of water-solvent LO, equals number of aqueous species n(Laq)
 
This index is kept separately in order to locate quickly the water-solvent H2O, which must be the last species in the aqueous phase. If the aqueous phase is not included then this index contains -1. 
  
L[3] - Number of Dependent Components included in the gas phase PG = n(Lg)
 
L[4] - Number of Dependent Components in the liquid hydrocarbon phase PSOL (reserved)
   
L[5] - Number of Dependent Components included in sorption phases n(Lads)
 
Fi[0] - Number of Phases included in the IPM problem n(Fi)
   
Fi[1]  - Number of multi-component phases n(Fis)

Fi[2]  - Number of sorption multi-component phases n(Fia)
   
TC - Temperature (C) of interest (min-max interval for parametric problems) 
 
In a usual GEM IPM run, both cells contain the same value of temperature of interest. 
     
TK - Temperature (K) of interest T (min.-max. interval for parametric problems) 
 
  In a usual GEM calculation run, the second cell should contain the temperature of interest.
 
PG - General pressure (bars) of interest P (min.-max. interval for parametric problems)
 
In a usual GEM IPM run, the second cell should contain the pressure of interest. 
  
Vx -  Initial V and calculated V(x) volume of the system (cm3)
 
Gx - Gibbs potential of the system in J (min.-max. for parametric problems) 
  
 Reserved for usage in future versions.
 
Mbx - Total mass of the system, kg
 
Caclulated automatically from the bulk composition (vector b).

Vectors and matrices

L1 - Number of Dependent Components included into each phase (l-alpha vector).
  
This input vector of n(Fi) length contains numbers of DC (species) included into each phase. Multi-component phases always stay in the beginning of the phase list PHnam. Aqueous phase, if present, is always the first phase. Gas phase is always the next to the aqueous phase.
See also comments to the L1_ data object.  
 
Lsmod - Dimensions of IPxPM and pmc arrays - three per multicomponent phase
 
This input array has n(Fis) rows and three columns. In each row:
The first number is the number of parameters of non-ideal mixing model per this multicomponent phase (i.e. number of  rows in the respective blocks of IPxPM and pmc arrays).
The second number is the number of columns in the respective block of the IPxPM array, i.e. the maximum order of interaction parameter.
The third number is the number of  columns in the respective block of the pmc array, i.e. the number of  P,T coefficients per interaction parameter.
  
Lsmdc - Dimensions of dmc and MoiSN tables - three per multicomponent phase

The Lsmdc array has n(Fis)  rows and 3 colums. In each row:
The first number is the number of parameters/coefficients per component in the respective block of the dmc array, which usually depends on the mixing model code in the smod row.
The second and third numbers for simple mixing are zeros or, for multi-site mixing phase models: the second number is the number of sublattices (site types); the third number is the total number of different moieties acting on all sublattice sites (moieties having the same formula, but acting on different sites, are considered as different moieties!).
 
IPxPM - Collected array of indexation tables for interaction parameters of non-ideal solutions

Number of interaction parameters used by k-th phase model is given in Lsmod[k][0], and their maximum order (number of  components or moieties involved in the interaction) is given in Lsmod[k][1]. Elements contain indexes (within each phase) of DCs that are involved in each interaction. If the order of the interaction parameter is less than Lsmod[k][1], the remaining elements in the table row should be filled with -1 (except the Pitzer model where the last element in the row indicates the type of interaction coefficient). The respective values of interaction parameter coefficients should be provided in the pmc array k-th block. The indexation table for each phase will be transferred to a TSolMod library function that calculates activity coefficients of phase species (more...).
 
pmc - Collected array of interaction parameter coefficients for multi-component phases
 
This input vector has total length of sum(of Lsmod[k][0]*Lsmod[k][2] elements) over n(Fis). The coefficients are automatically collected from the Phase records and stored in the same order as phases in the PHnam list, Lsmod[k][2] coefficients per k-th phase. These coefficients are are transferred to a phase script or a TSolMod library function that calculates activity coefficients of species (more...). 
 
dmc - Vector of non-ideality coefficients for species in multi-component phases
 
This input vector has total length of sum(of Lsmdc[k][0]*L1[k] elements) over n(Fis). The coefficients are automatically collected from Phase records and stored in the same order as phases in the PHnam list, Lsmdc[k][0]*L1[k] coefficients per k-th phase, in the same order as species in DCnam list, Lsmdc[k][0] coefficients per species. These coefficients are transferred to a phase script or a TSolMod library function that calculates activity coefficients of phase species (more...).
 
MoiSN - Collected array of end member moiety- site multiplicity numbers in multi-site SS models
 
This input vector has total length of sum(of L1[k]*Lsmdc[k][1]*Lsmdc[k][2] elements) over n(Fis). The coefficients are automatically collected from Phase records and stored in the same order as the phases are listed in the PHnam list, L1[k] tables each with Lsmdc[k][1]*Lsmdc[k][2] occupancy numbers per k-th phase component. Upon GEM IPM run, the sub-array for  each phase is transferred to a TSolMod library function that calculates activity coefficients of phase species (more...).
 
dll - Lower metastability restrictions for DC mole quantities  x  at equilibrium state
 
Input vector (of length n(L)) of lower restrictions (moles) contains metastability restrictions from below to the sought-for mole quantities x(j) of Dependent Components at equilibrium state. The GEM IPM algorithm will keep these quantities greater or equal to the prescribed restrictions. Default (trivial) value is 0 moles.
See also comments to the dll_ data object.
      
dul - Upper metastability restrictions for DC mole quantities  x  at equilibrium state
 
Input vector (of length n(L)) of upper restrictions (moles) contains metastability restrictions from above to the sought-for mole quantities x(j) of Dependent Components at equilibrium state. The GEM IPM algorithm will keep these quantities lower or equal to the prescribed restrictions. Default (trivial) value is 106 (1e6) moles.
See also comments to the dul_ data object.

See also comments to the gEx_ data object.

See also comments to the Yof_ data object.

Contents of this vector represent a (compressed) VTP vector. Molar volumes are used in calculation of volumes of phases as v(k) = sum(vol(j)*x(j) (phVol object) and volume of the whole system V(x) = sum(v(k)) (Vx[1] data object cell) at GEM IPM iterations and equilibrium state.

Input DC molar masses are used in calculation of masses of phases (phM vector) and mass-unit concentrations (wy  and  w_t  vectors). Length of this vector is n(L). 
     
lngmf - Fictive DC activity coefficients to be used only in AIA LP simplex initial approximation (in ln scale) 
 
This vector (length n(L)) contains natural logartihms of fictive DC activity coefficients to be used in the AIA LP simplex initial approximation only, in order to force small amounts of some DCs in the initial approximation.
These corrections are necessary in the systems containing solid solution phases with miscibility gaps, or when the feasible initial approximation cannot be obtained otherwise. 
Default value is zero.
See also comments to the lnGmf_ data object.
  
ez - Formula charges of Dependent Components
 
This input vector (length n(L)) contains negative, positive or zero formula charges of DC, obtained automatically from chemical formula list together with the A stoichiometry matrix. Used in calculation of activity coefficients and Coulombic correction factors for aqueous and surface species. 
  
g0  - Molar (molal) standard Gibbs energy functions of Dependent Components at T,P of interest 

This vector contains the main input data for the GEM IPM task - the values of molar (partial molal for aqueous and surface species) Gibbs energy function go(T,P) corrected to temperature T and pressure P of interest and normalized into mol/mol (lnK) units. 
See also comments to the gTP data object.
 
icM - Input  atomic (molar) masses of Independent Components, g/mol
 
This vector is used in calculation of mass quantities, such as the mass of the whole system Mbx or masses of phases phM, as well as total dissolved mass concentrations w_t.
 
b - Bulk composition of the system (moles of Independent Components) - main input in GEM task

This vector defines stoichiometry of the whole system and sets the mass balance constraint equations Ax = b (see more in theory_gem.html manual). 

A - Input stoichiometry matrix of Dependent Components 
 
Each row of the stoichiometry matrix A corresponds to one Dependent Component (species). Columns of this matrix correspond to Independent Components. The matrix element a(ji) is a mole quantity of i-th Independent Component in one mole of j-th Dependent Component. 
The matrix A is used in numerical GEM IPM algorithms (more ... ), and in calculations of the mass balance residuals Cb. The matrix is automatically created from the list of chemical formulae of Dependent Components kept in the Project data structure.

This matrix contains six type codes  per each multicomponent phase, collected from the respective Phase records (p_solt field). The codes are used in GammaCalc() and other procedures to control calculation of activity coefficients or EDL Coulombic corrections for surface species.
  
RLC - Type codes for DC metastability restrictions given in the  dll  and  dul  vectors
  
This code, for each line of the dll and dul vectors, defines how the sought-for mole quantity x of the respective Dependent Component in the GEM primal solution will be restricted by the IPM algorithm. 
Possible codes:  see comments to the RLC_ data object.

5. Output Data - GEM Results

Scalars

FI[0] - Number of phases present in non-zero amounts in equilibrium state
 
FI[1] - Number of multi-component phases present in non-zero quantities in equilibrium state
 
FI[2] - Number of sorption phases present in non-zero quantities in equilibrium state
 
It - Number of completed GEM IPM algorithm iterations
 
 Current number of completed iterations in MBR and IPM-3 algorithms (cumulative). 
  
GX - Total equilibrium Gibbs energy of the system, in moles
  
This value is updated (as an approximation) on iterations of the IPM algorithm; after the convergence is achieved, GX contains the G(x) potential at equilibrium state. 
 
IS - Effective molal ionic strength of aqueous electrolyte
 
This value is calculated in  GammaCalc() procedure at each MBR and IPM-3 iteration (as an approximation); after the  IPM algorithm has converged, IS is an equilibrium value. If aqueous phase is not included, this cell contains 0 or EMPTY. 
   
pH - pH of aqueous solution (in molal activity scale)
 
pH of aqueous solution (-log a(H+) ) is calculated (as an approximation) from chemical potentials of hydrogen u(H) and charge u(Zz) at each IPM iteration, if the aqueous phase is included. Otherwise, this cell contains 0 or EMPTY.  
     
pe - pe of aqueous solution (in molal activity scale)
 
pe of aqueous solution (-log a(e-) ) is calculated (as an approximation) from dual chemical potential of charge u(Zz) at each IPM iteration, if the aqueous phase is included. Otherwise this cell contains 0 or EMPTY.  
 
Eh - Eh of aqueous solution, Volts
 
This value is calculated from pe on each IPM iteration, if the aqueous phase is included. Otherwise, this cell contains 0 or EMPTY.
   

Vectors

phM - Masses of phases (in grams) in equilibrium
 
This vector (of n(Fi) length) contains masses of phases calculated from their bulk compositions in equilibrium state (bXa matrix).  
  
phVol - Volumes of phases (in cm3)  in equilibrium
 
This vector (of n(Fi) length) contains masses of phases calculated from their speciation at equilibrium state (x vector) and DC molar volumes (vol  vector).  
Attention! Volumes of phases may contain large errors if molar volumes of some species are missing in the project thermodynamic database or are incorrectly extrapolated to P,T of interest. 
   
Xa - Output total mole amounts of phases in equilibrium
 
This vector (of n(Fi) length) contains the total number of moles of species in each phase at equilibrium state, calculated from the "primal solution" vector x using a vector of number of species in phases  L1.  In turn, the Xa vector elements are used for calculation of mole-fraction-based concentrations and activities within each multicomponent phase.
 
Xwa - Output mole amount of the solvent (sorbent) in multi-component phases
 
This vector (of n(Fis) length) contains the mole amount of water-solvent (for aqueous phase) or sorbent (sorption phases). These quantities are computed from the the "primal solution" vector x and are also used in calculation of activities and concentrations in "practical" scales inside of the IPM algorithm.
  
Fa - Values of the phase stability criterion Om(alpha) or f(alpha) in equilibrium
 
This diagnostic vector (of n(Fi) length) contains for each phase the value of logarithmic stability index Om(alpha) at Pa_PC = 2, or old Karpov phase stability criteria f(alpha) at Pa_PC = 1, both based on the Kuhn-Tucker necessary and sufficient conditions of chemical equilibrium (see more in theory_gem.html manual). These criteria are used in Phase Selection procedure for checking the correctness of the computed stable phase assemblage (currently except phases that contain components with non-trivial metastability contraints).
A stable phase present in non-zero quantity (in the Xa vector) normally should have a close-to-zero value of stability criterion (numerically, within the interval set by Pa_DF tolerances). Criterion values outside of this interval point to an inconsistency between phase stability and its amount in the mass balance.
Another case when the stability criterion is outside the Pa_DF interval occurs when some components in the phase are subject to non-trivial metastability restrictions (given in dll and dul vectors). 
Values of the Karpov f(alpha) criterion for phases whose mole amounts (in the Xa vector) have been zeroed off are meaningless and are also zeroed off.  However, the Om(alpha) values for such phases remain valid and show how much is the rest of the system is undersaturated with respect to this phase.
Note that correct stability criterion values are displayed only immediately after the GEM IPM calculation, because they are not stored in the SysEq records. 
 
pmXs -  Total mass of solid phases in the system (g), obtained from the bXs_ vector
   
The total mass of solid phases in the equilibrium state is automatically computed from the contents of bXs vector after the GEM IPM algorithm has converged, as well as by browsing SysEq records in Equilibria Calculation mode.
 
bXs_ -  Total output bulk composition of all solid phases (in moles of Independent Components)
   
This output vector (of n(N) length) is computed from the primal solution vector x and from the table of bulk equilibrium compositions of multi-component phases bXa after the GEM IPM algorithm has converged, as well as upon browsing SysEq records in Equilibria Calculation mode.
Attention! In Process and GEM2MT scripts for sequential reactors, please use the bXs name of this data object instead of  bXs_ .
 
bXa - Output bulk compositions of multi-component phases (in moles of Independent Components)
  
Each row of this table corresponds to one Phase; columns correspond to Independent Components. The matrix element is a mole quantity of i-th Independent Component in the k-th (multi-component) phase in equilibrium state. The size of the matrix is n(Fis) * n(N). It is computed upon convergence of the GEM IPM algorithm or upon browsing SysEq records in Equilibria Calculation mode.
     
x  - Output mole quantities of Dependent Components (species) at equilibrium state ("primal GEM task solution") 
 
This vector (of n(L) length) contains the "primal solution" output of the GEM IPM algorithm - the mole amounts of all chemical species in equilibrium state. The quality of this solution can be characterized by the mass balance residuals (Cb vector) and the phase stability criteria (Fa vector). The x vector is the basis for calculation of quantities of phases and concentrations of species in various scales. 
  
v - Primal chemical potentials of Dependent Components (dimensionless in ln(a) scale)
 
This output vector (of n(L) length) contains chemical potentials of DCs (species), defined in a usual way via the standard molar Gibbs energy function g0, mole amount x (primal IPM solution), phase mole amount Xa or that of the carrier Xwa,  activity coefficient lnGam, and scale factors. For charged surface species and electrostatic SCM, the "primal" potential includes the Coulombic term and is therefore an electrochemical potential. See more in theory_gem.html and theory_scm.html manuals.    
The chemical potentials in the  v  vector can be converted into J/mol units multiplying them by RT .    
  
fx - Dual chemical potentials of Dependent Components (dimensionless in ln(a) scale)
 
This output vector (of n(Ls) length) contains values of chemical potential of DC (species), defined via chemical potentials of Independent Components u (dual IPM solution) and species stoichiometry (respective j-th line of the matrix A). 
For all species (with trivial metastability restrictions) present in non-zero quantity at equilibrium state, the value from the fx vector should be (almost) equal to that from the v vector. For species affected by non-trivial restrictions, the difference v(j)*RT - fx(j) would show a level of induced metastability in ln(a) units.  
  
lnGam - Calculated  ln of internal DC activity coefficients in phases
 
This output vector (of n(L) length) contains natural logarithms of species activity coefficients, calculated in built-in TSolMod class library functions or from Phase scripts upon iterations of the GEM IPM algorithm. The lnGam vector is part of the GEM IPM primal solution (together with the x vector). Non-ideality parameters are provided through the Lsmod, Lsmdc,pmc and dmc input data objects. 
The activity coefficients are called internal because they do not iclude non-logarithmic asymmetry terms e.g. for aqueous species and water-solvent (see /Resources/doc/pdf/Activity-Coeffs.pdf). Activity coefficients are referenced to the same standard concentration scale as the respective g0 values (e.g. molal for aqueous and surface species, mole fraction for solids and gases).
For the species in  multicomponent phases with simple ideal mixing, as well as in single-component phases, the lnGam values are set to 0.  For the ideal multi-site solid solutions, the lnGam values of end members are fictive activity coefficients defined as differences of ln(activity) - ln(mole fraction).
   
gamma - Calculated external activity coefficients of Dependent Components  (species)
 
This output vector (of n(L) length) contains 'external' DC activity coefficients, calculated from the elements of lnGam vector, plus the non-logarithmic terms where appropriate (see /Resources/doc/pdf/Activity-Coeffs.pdf).  
The product gamma * concentration must be (numerically) equal to the generic activity of any DC present in positive amount in equilibrium state under trivial metastability constraints. This is a direct consequence of the Karush-Kuhn-Tucker conditions of equilibrium (more ...). See also Results: tab.
  
vEx - Excess molar Gibbs energy of Dependent Components (dimensionless in ln(a) scale)
   
This diagnostic vector (of n(L) length) is calculated at each IPM iteration from lnGam and (where appropriate) from electrostatic terms. For any simple ideal species (also in single-component condensed phases), the vEx vector cells contain 0.
   
c -  Normalized energy function of Dependent Components (dimensionless in ln(a) scale)
   
This diagnostic vector (of n(L) length) is re-calculated at each IPM iteration from g0 and vEx vectors and used in the main linearization process of the IPM algorithm (see theory_gem.html).
 
lga - Decimal logarithms of activity (fugacity) of Dependent Components at equilibrium state
 
This output vector (of n(L) length) contains decimal logarithms of activity of Dependent Components (in the appropriate standard-state concentration scales). These values are computed directly from dual DC chemical potentials as 0.4343*(fx(j) - g0(j)) and include all possible corrections relative to simple ideal mixing, involving electrostatic and complex configurational entropy terms wherever appropriate . 
For all species present in non-zero quantities at equilibrium, the lga values should be (almost) equal to the logarithms of activities calculated from the primal DC chemical potentials (v vector), or as product of concentration times external activity coefficient.
For species of multi-component phases that are present in too small concentrations and have been zeroed off by the IPM algorithm, the lga values are still valid even if they are very negative (i.e. refer to physically meaningless concentration). 
For single-component condensed phases (e.g. minerals): the lga value is equivalent to decimal logarithm of the saturation index (not necessarily in aqueous solution). At trivial metastability constrants, this number is always negative (undersaturation) if the mineral is present in zero mole quantity, or close to zero (if the mineral is present in non-zero quantity). At non-trivial restrictions in the dll and/or dul vectors, the mineral will be present in non-zero quantity but its lga value will be far from zero (positive in case of oversaturation, negative at undersaturation) and will show, in fact, the level of imposed metastability.
 
wy - Mass concentrations of DC in multi-component phases, % (ppm in aqueous phase)
 
This output vector (of n(Ls) length) contains mass concentrations of Dependent Components related to total mass of the respective multicomponent phase. 
  
Wxx - Mole fractions of Dependent Components  in multi-component phases 
 
This output vector (of n(Ls) length) contains mole fractions of species  in multicomponent phases. It is not used directly in IPM calculations, but may be heavily used in TSolMod class library functions, as well as in Phase, Process or GtDemo math scripts.  
 
my - Molality concentrations of Dependent Components (in mol per kg H2O)
 
This output vector (of n(L) length) contains molalities of all species except water-solvent, calculated if aqueous electrolyte phase is present at equilibrium state; otherwise, this vector contains zeros. 
For water-solvent, this vector contains a constant  H2O_mol_to_kg = 55.50837344   - the number of moles in 1 kg H2O.  
 
u - Chemical potentials of Independent Components in equilibrium: GEM task dual solution (dimensionless in mol/mol)
 
This output vector (of n(N) length) contains the "dual solution" of the GEM IPM algorithm - the chemical potentials of IC (stoichiometry units) in equilibrium state.  These numbers are used for checking the Karush-Kuhn-Tucker necessary and sufficient conditions of equilibrium, calculation of phase stability criteria, dual chemical potentials of species fx, general DC activities lga, and other output parameters such as pH and pe.  
  
uc - Chemical potentials of ICs from two previous IPM iterations (J/mol)

This table (of n(N) rows and 2 columns) contains the "dual solution" vector u (IC chemical potentials) approximations from two previous IPM iterations.
These values are used in detection of divergence of the dual solution approximation (controlled by Pa_LLG).
 
  uefd - Lagrange multipliers for ICs in Mass Balance Refinement (MBR) procedure

This vector shows values used in MBR for correcting the primal solution approximation to obey the mass balance, i.e. to become feasible (controlled by Pa_DHB).
 
ue - Chemical potentials of Independent Components at equilibrium state (J/mol)

This output vector (of n(N) length) contains the "dual solution" (IC chemical potentials) converted from the u vector into (J/mole) units by RT multiplication.
 
Cb - Mass balance residuals for Independent Components (at GEM IPM iterations)
 
This output vector (of n(N) length) contains mass-balance residuals (in moles) that characterize the quality of the primal IPM solution (vector x) as compared to the input bulk chemical composition of the system (vector b). The Cb vector is calculated using the stoichiometry matrix A  as b - ATx  (in vector-matrix form, T stands for transpose operation) and therefore Cb it is not stored in SysEq records (i.e. Cb is recalculated every time when a SysEq record is loaded from the database). 
The Cb vector is calculated several times on each iteration of the IPM algorithm eiher for mass balance refinement or for the main IPM descent. The mass-balance precision (i.e. maximum absolute mass balance residual) of the IPM-3 algorithm is usually better than 1e-13 relative (is set in Pa_DHB).
If the required precision cannot be achieved in about 100 loops of the MBR algorithm, then this indicates serious inconsistencies in the input thermodynamic data, chemical system definition, or equations (parameters) for activity coefficients. An error message will be generated and GEM calculation stops. The maximum number of MBR iterations can be set in Pa_DPV[0] in the Project window Controls page. 
 
m_t - Output total dissolved molalities of Independent Components in equilibrium state
 
This output vector (of n(N) length) is calculated from the bulk composition of aqueous phase and the mole amount of water-solvent, if aqueous phase is present at the equilibrium state. An m_t(i) value is a total mole amount of i-th IC per 1 kg of H2O-solvent; for H and O, the amounts of these elements contained in water-solvent are excluded. 
 
lgm_t - Decimal logarithms of total dissolved molalities of Independent Components in equilibrium state

This output vector (of n(N) length) contains decimal logarithms of total dissolved molalities, calculated from the m_t vector.
 
w_t - Output total dissolved mass concentrations of Independent Components in equilibrium state
 
This output vector (of n(N) length) contains total dissolved mass concentrations of Independent Components in grams per 1 kg of aqueous solution (i.e. thesalinity in per mil). It is calculated if aqueous phase is present at the equilibrium state; the mass of the solvent H2O is not included in calculation of values for H and O. 
  
Gfug - Partial fugacities of gases (in mole fraction scale)
 
Calculated partial fugacities of gases (normalized to 1 bar total pressure). At the state of interest, fugacities can be found from multiplication by the total pressure PG.  
  
lgFug - Decimal logarithms of partial fugacities of gases
  
Calculated logarithms of partial fugacities of gases (normalized to 1 bar total pressure). At the state of interest, log(fugacities) of gases can be found in the respective cells of the lga  vector. The difference between the two values equals  log PG (where PG  is total pressure in bars). 
 
lgPpG - Decimal logaritms of partial pressures of gases
 
Calculated logarithms of partial pressures of gases. For j-th gas, they are related to partial fugacity as:
     lgFug(j) + lg(PG) = lgPpG(j) + 0.4343*lnGam(j)
where lnGam contains logarithms of fugacity (activity) coefficients.

Example: the total mole quantity of i-th IC is b[i] in the Multi structure and B_[mui[i]] in the project system structure.
 
muk - Phase indices in the Project System

This vector of Phase indices connects data for phases (packed into the "Multi" IPM structure) with the same data within the expanded project system data structure. Indices start from 0.
The muk vector  is automatically created within the "Multi" structure because some phases can be excluded from CSD and hence will not be taken into the IPM problem. The muk vector can be used in math scripts. 
Example: the specific surface area of k-th phase is Aalp[k] in the Multi structure and Aalp_[muk[k]] in the project system structure.
  
muj - DC indices in the Project System
 
This vector of DC indices connects data for Dependent Components (packed into the "Multi" IPM structure) with the same data within the expanded project system data structure. Indices start from 0.
The muj vector  is automatically created within the "Multi" structure because some Dependent Components (species) can be excluded from CSD and hence will not be taken into the IPM problem. The muj vector can be used in math scripts. 
Example: the upper mole quantity restriction of j-th DC is dul[j] in the Multi structure and dul_[muj[j]] in the project system structure.
